## 1️⃣ 基礎概念

### 什麼是雜湊表？

**生活中的比喻**：

想象你在圖書館找書：
- **沒有雜湊表**：一本一本找 → O(n)
- **有雜湊表**：根據書號直接定位 → O(1)

**雜湊表就是「快速查詢」的資料結構**

| 資料結構 | 查詢速度 | 適用場景 |
|---------|---------|---------|
| 陣列 | O(1) | 已知道索引 |
| 連結串列 | O(n) | 順序存取 |
| 雜湊表 | O(1) | 根據 key 查詢 |

## 2️⃣ Python 中的雜湊表

### 字典（Dict）- 就是雜湊表！

```python
# 基本操作
scores = {"小明": 95, "大華": 87, "小美": 92}

# 查詢（O(1)）
print(scores["小明"])  # 95

# 新增/修改
scores["小華"] = 88

# 刪除
del scores["大華"]
```

### 集合（Set）- 也是雜湊表！

```python
# 集合：自動去重
fruits = {"apple", "banana", "apple", "orange"}
print(fruits)  # {'apple', 'banana', 'orange'}

# 檢查是否存在（O(1)）
print("apple" in fruits)  # True
```

## 3️⃣ 經典題目示範

### 題目 1：兩數之和（LeetCode #1）

```
問題：找出陣列中兩個數字相加等於 target 的索引
```

**解題思路**：
- 暴力解：兩層迴圈 → O(n²)
- 雜湊表：一層迴圈，一邊存一邊查 → O(n)

```python
def two_sum(nums, target):
    seen = {}  # 雜湊表：值 -> 索引
    
    for i, num in enumerate(nums):
        complement = target - num
        
        if complement in seen:
            return [seen[complement], i]
        
        seen[num] = i
    
    return []
```

**解釋**：
1. 走訪陣列
2. 檢查 `target - num` 是否已經看過
3. 有找到就回傳，沒有就存到雜湊表

### 題目 2：出現次數最多的元素（LeetCode #169）

```
問題：找出陣列中出現次數超過一半的元素
```

```python
from collections import Counter

def majority_element(nums):
    # Counter 底層就是雜湊表
    count = Counter(nums)
    return count.most_common(1)[0][0]
```

## 4️⃣ 雜湊表的重要特性

### 雜湊函數

```
key → 雜湊函數 → 陣列索引
"apple" → hash() → 3
```

**好的雜湊函數特點**：
- 速度快
- 均勻分布
- 衝突少

### 碰撞處理

| 方法 | 說明 |
|------|------|
| 鏈結法 | 同 index 用串列存多個值 |
| 開放定址 | 往後找下一個空位 |

**Python 字典使用鏈結法**
