### 1️⃣ 基礎概念（用生活比喻，越詳細越好）

#### 🎯 什麼是陣列？

想象你有一排 **連續編號的置物櫃**，每個櫃子都有固定的編號（從 0 開始）：
- 櫃子 0、櫃子 1、櫃子 2、櫃子 3、... 一直到櫃子 n-1
- 每個櫃子只能放 **一個東西**
- 你可以根據櫃子編號 **直接取物**，不用一個一個翻找

這就是「陣列」！在電腦科學中：
- 陣列（Array）是一塊**連續的記憶體空間**
- 每個元素佔用相同大小（假設是 4 bytes）
- 可以用**索引（Index）**直接訪問任意位置

#### 🎯 為什麼索引從 0 開始？

這是歷史習慣！想象指標運算：
- 第 i 個元素的地址 = 起始地址 + (i × 每個元素大小)
- 如果從 0 開始，第 0 個元素就是 起始地址 + 0，不用減 1
- 這讓指標運算更直觀

### 2️⃣ 核心知識（演算法解釋、時間/空間複雜度分析）

#### 📌 陣列的基本操作

| 操作 | 時間複雜度 | 空間複雜度 | 說明 |
|------|-----------|-----------|------|
| 讀取（Access） | O(1) | - | 直接用索引訪問，就像開指定編號的置物櫃 |
| 搜尋（Search） | O(n) | - | 從頭遍歷找目標，像問每個人「是你嗎？」 |
| 插入（Insert） | O(n) | O(1) | 要把插入點後面的元素全部往後移 |
| 刪除（Delete） | O(n) | O(1) | 刪除後要把後面的元素全部往前補 |
| 新增（Append） | O(1)* | O(1) | * 如果陣列已滿，要擴容就是 O(n) |

#### 📌 時間複雜度圖解

```
Access（讀取）: O(1)     🔥🔥🔥 最快，直接命中
Search（搜尋）: O(n)     🔥      最慢，需要一個一個檢查
Insert（插入）: O(n)     🔥      需要搬遷後面的元素
Delete（刪除）: O(n)     🔥      需要填補空位
```

#### 📌 大 O 記號詳解

- **O(1)** = 常數時間 = 不管資料多少，時間都一樣
- **O(n)** = 線性時間 = 資料量加倍，時間也加倍
- **O(n²)** = 平方時間 = 資料量加倍，時間變 4 倍（嵌套迴圈）

### 3️⃣ 經典題目（LeetCode 實際題目，Python 程式碼，每行都要有註解）

#### 🏋️ 題目 1：最大子陣列之和（Maximum Subarray）

**LeetCode 編號**：53
**難度**：Easy
**標籤**：Array, Dynamic Programming, Divide and Conquer

**題目描述**：
給定一個整數陣列，找出具有最大和的連續子陣列（子陣列必須至少包含一個元素）。

**範例**：
```
輸入：nums = [-2,1,-3,4,-1,2,1,-5,4]
輸出：6
解釋：連續子陣列 [4,-1,2,1] 有最大和 6
```

**完整程式碼（含中文註解）**：

```python
def maxSubArray(nums):
    """
    最大子陣列和 - Kadane 演算法
    
    核心想法：
    - 如果前面的總和是負的，那還不如從頭開始
    - 每次都選擇「到目前為止最好的選擇」
    
    時間複雜度：O(n) - 只用一次遍歷
    空間複雜度：O(1) - 只用常數個變數
    """
    
    # 初始化：假設第一個元素是目前最大的和
    max_sum = nums[0]          # 紀錄目前找到的最大和，初始為第一個元素
    current_sum = nums[0]      # 紀錄以當前位置結尾的最大子陣列和
    
    # 從第二個元素開始遍歷（索引 1 到 len(nums)-1）
    for i in range(1, len(nums)):
        # 選擇：繼續延續之前的子陣列，還是從現在重新開始
        # 如果 current_sum 是負的，加上只會讓和變小，不如重開
        # max(繼續, 重新開始) 讓我們選擇較大的那個
        current_sum = max(nums[i], current_sum + nums[i])
        
        # 更新最大和：如果 current_sum 比 max_sum 大，就更新
        max_sum = max(max_sum, current_sum)
    
    # 回傳找到的最大和
    return max_sum

# ============ 測試範例 ============
print(maxSubArray([-2,1,-3,4,-1,2,1,-5,4]))  # 輸出: 6
print(maxSubArray([1]))                       # 輸出: 1
print(maxSubArray([5,4,-1,7,8]))              # 輸出: 23
```

**程式碼解析**：

```
範例演練：nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]

i=0: max_sum=-2, current_sum=-2
i=1: current_sum = max(1, -2+1) = max(1, -1) = 1
     max_sum = max(-2, 1) = 1
i=2: current_sum = max(-3, 1-3) = max(-3, -2) = -2
     max_sum = max(1, -2) = 1
i=3: current_sum = max(4, -2+4) = max(4, 2) = 4
     max_sum = max(1, 4) = 4
i=4: current_sum = max(-1, 4-1) = max(-1, 3) = 3
     max_sum = max(4, 3) = 4
i=5: current_sum = max(2, 3+2) = max(2, 5) = 5
     max_sum = max(4, 5) = 5
i=6: current_sum = max(1, 5+1) = max(1, 6) = 6
     max_sum = max(5, 6) = 6
i=7: current_sum = max(-5, 6-5) = max(-5, 1) = 1
     max_sum = max(6, 1) = 6
i=8: current_sum = max(4, 1+4) = max(4, 5) = 5
     max_sum = max(6, 5) = 6

最終答案：6
```

**複雜度分析**：

```
時間複雜度：O(n)
- 我們只遍歷陣列一次
- 每次迴圈內的運算是 O(1)
- 總共 O(n × 1) = O(n)

空間複雜度：O(1)
- 只使用兩個變數：max_sum 和 current_sum
- 不管輸入多大，記憶體用量都是固定的
```

#### 🏋️ 題目 2：移除元素（Remove Element）

**LeetCode 編號**：27
**難度**：Easy
**標籤**：Array, Two Pointers

**題目描述**：
給定一個陣列 nums 和一個值 val，原地移除所有值等於 val 的元素，並回傳新的長度。

**範例**：
```
輸入：nums = [3,2,2,3], val = 3
輸出：2, nums = [2,2,_,_]
解釋：移除所有等於 3 的元素後，剩下 [2,2]
```

**完整程式碼（含中文註解）**：

```python
def removeElement(nums, val):
    """
    原地移除元素 - 雙指標技巧
    
    核心想法：
    - 用一個「慢指標」當作寫入位置
    - 用一個「快指標」遍歷找不等於 val 的元素
    - 把不等的元素搬到慢指標位置
    
    時間複雜度：O(n) - 只遍歷一次
    空間複雜度：O(1) - 只用常數個變數
    """
    
    # slow 代表「下一個要放有效元素的位置」
    # 初始化為 0，表示從頭開始
    slow = 0
    
    # fast 遍歷整個陣列
    for fast in range(len(nums)):
        # 如果目前元素不等於 val，就把它搬到 slow 位置
        if nums[fast] != val:
            # 把 nums[fast] 複製到 nums[slow]
            nums[slow] = nums[fast]
            
            # slow 前進，準備下一個位置
            slow += 1
    
    # slow 就是新陣列的長度
    return slow

# ============ 測試範例 ============
nums1 = [3, 2, 2, 3]
result1 = removeElement(nums1, 3)
print(f"新長度: {result1}, 陣列: {nums1[:result1]}")
# 輸出: 新長度: 2, 陣列: [2, 2]

nums2 = [0,1,2,2,3,0,4,2]
result2 = removeElement(nums2, 2)
print(f"新長度: {result2}, 陣列: {nums2[:result2]}")
# 輸出: 新長度: 5, 陣列: [0, 1, 3, 0, 4]
```

**程式碼解析**：

```
範例演練：nums = [3, 2, 2, 3], val = 3

初始：slow = 0, fast = 0
nums[fast] = 3 == val → 跳過，fast++
fast = 1: nums[fast] = 2 != val
  → nums[slow] = nums[1] = 2，slow = 1
fast = 2: nums[fast] = 2 != val
  → nums[slow] = nums[2] = 2，slow = 2
fast = 3: nums[fast] = 3 == val → 跳過

完成：slow = 2
新陣列 = nums[0:2] = [2, 2] ✓
```

**複雜度分析**：

```
時間複雜度：O(n)
- fast 指標走完整個陣列一次
- 最壞情況（所有元素都要移除）也是 O(n)

空間複雜度：O(1)
- 只使用 slow 和 fast 兩個指標
- 沒有額外申請記憶體
```
