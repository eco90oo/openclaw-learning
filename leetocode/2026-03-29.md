### 1️⃣ 基礎概念（用生活比喻，越詳細越好）

#### 🔸 什麼是陣列（Array）？

**比喻**：想像你家裡的衣櫃！
- 衣櫃有一排固定的「掛鉤」，每個掛鉤有一個固定的編號（索引）
- 第 1 個掛鉤、 第 2 個掛鉤、 第 3 個掛鉤...依此類推
- 你知道每件衣服放在哪個編號，就能馬上找到它

**正式定義**：陣列是一種**連續記憶體**的資料結構，用來存放**相同類型**的元素。每個元素透過索引（Index）來存取。

```
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│  索引 0 │  索引 1 │  索引 2 │  索引 3 │  索引 4 │
├─────────┼─────────┼─────────┼─────────┼─────────┤
│   10    │   20    │   30    │   40    │   50    │
└─────────┴─────────┴─────────┴─────────┴─────────┘
```

#### 🔸 為什麼面試常考陣列？

| 原因 | 說明 |
|------|------|
| ✅ 基礎中的基礎 | 所有複雜資料結構的根基 |
| ✅ 面試必考 | 90% 的技術面試都會有陣列題目 |
| ✅ 考驗最佳化思維 | 能看出你對複雜度的理解 |
| ✅ 練習程式碼品質 | 變數命名、邏輯清晰度 |

### 2️⃣ 核心知識（演算法解釋、時間/空間複雜度分析）

#### 🔸 陣列基本操作效能

| 操作 | 時間複雜度 | Python 語法 | 說明 |
|------|------------|-------------|------|
| **讀取** | O(1) | `arr[i]` | 直接算位移，就像知道衣櫃編號直接拿 |
| **寫入** | O(1) | `arr[i] = x` | 直接覆寫，不用移動其他衣服 |
| **搜尋** | O(n) | `x in arr` | 要一件一件檢查（最壞情況在最後） |
| **新增到末尾** | O(1) | `arr.append(x)` | 衣櫃最下面新增一個掛鉤 |
| **新增到中間** | O(n) | `arr.insert(i, x)` | 要把後面所有衣服往後挪一件 |
| **刪除** | O(n) | `arr.remove(x)` / `del arr[i]` | 要把後面所有衣服往前補 |

**記憶口決**：「讀寫是 O(1)，插入刪除要搬家」

#### 🔸 什麼是 Big O Notation？

**比喻**：就像描述「從你家到公司的交通時間」

```
- O(1)      : 固定 10 分鐘（不管有多少人，永遠一樣快）
- O(log n)  : 二分搜尋，像上班坐電梯每層樓停留一次
- O(n)      : 線性時間，走路去上班，人越多越慢
- O(n²)     : 雙重迴圈，像每個人都要和每個人打招呼
- O(2ⁿ)     : 指數時間，像細菌繁殖，越長越多（千萬別在面試出現！）
```

#### 🔸 空間複雜度分析

**定義**：演算法額外使用的記憶體空間

| 類型 | 說明 | 範例 |
|------|------|------|
| O(1) | 固定額外空間 | 只用幾個變數 |
| O(n) | 線性額外空間 | 建立新陣列存放結果 |

### 3️⃣ 經典題目（LeetCode 實際題目，Python 程式碼）

#### 🔸 題目 1：最大子陣列和（LeetCode 53 - Maximum Subarray）

**題目翻譯**：找到連續的元素總和最大的子陣列

**比喻**：就像看股票，要找到連續幾天賺最多的區間

```python
def maxSubArray(nums):
    """
    使用「貪心策略 Kadane's Algorithm」
    核心想法：從左到右掃描，如果目前的總和變成負數，
            就重新開始（因為負數會拉低後面的總和）
    """
    
    # 初始化：目前最大的總和 = 第一個元素
    # 就像先記住第一天的��價
    max_sum = nums[0]
    
    # 目前的總和從第一個元素開始
    current_sum = nums[0]
    
    # 從第二個元素開始扫描（索引 1）
    for i in range(1, len(nums)):
        # 如果 current_sum 是負數，就放棄之前的，重新開始
        # 這就像如果前面是赔錢的，還不如從今天重新開始
        if current_sum < 0:
            current_sum = nums[i]
        else:
            # 否則把今天的股價加進去
            current_sum += nums[i]
        
        # 更新最大總和
        if current_sum > max_sum:
            max_sum = current_sum
    
    return max_sum


# 測試範例
test_nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]
result = maxSubArray(test_nums)
print(f"最大子陣列和: {result}")  # 輸出: 6（子陣列 [4, -1, 2, 1]）
```

**複雜度分析**：
- ⏱️ 時間複雜度：O(n) - 只需扫描一次
- 💾 空間複雜度：O(1) - 只用了兩個變數

#### 🔸 題目 2：搬運元素（LeetCode 27 - Remove Element）

**題目翻譯**：把陣列中所有等於目標值的元素移除，回傳新長度

**比喻**：就像搬家時把不要的東西丟掉，剩下的往前擠

```python
def removeElement(nums, val):
    """
    使用「雙指標」技巧：
    - write_idx: 下一個要放好元素的位置
    - read_idx: 目前檢查到的位置
    """
    
    # write_idx 是「寫入位置」，一開始放在陣列最前面（索引 0）
    write_idx = 0
    
    # 用 read_idx 檢查每個元素
    for read_idx in range(len(nums)):
        # 如果這個元素不是要移除的值
        if nums[read_idx] != val:
            # 就把它搬到 write_idx 的位置
            nums[write_idx] = nums[read_idx]
            # write_idx 往後挪一個位置，準備放下一個
            write_idx += 1
    
    # 回傳新陣列的長度（就是 write_idx）
    return write_idx


# 測試範例
test_nums = [3, 2, 2, 3]
val = 3
new_len = removeElement(test_nums, val)
print(f"新長度: {new_len}")  # 輸出: 2
print(f"陣列內容: {test_nums[:new_len]}")  # 輸出: [2, 2]
```

**複雜度分析**：
- ⏱️ 時間複雜度：O(n) - 最壞情況要檢查每個元素
- 💾 空間複雜度：O(1) - 原地修改

#### 🔸 題目 3：旋轉陣列（LeetCode 189 - Rotate Array）

**題目翻譯**：把陣列向右旋轉 k 個位置

**比喻**：就像把衣櫃裡的衣服向右平移

```python
def rotate(nums, k):
    """
    使用「三次反轉法」：
    1. 先全部反轉
    2. 前 k 個反轉
    3. 後面 n-k 個反轉
    """
    n = len(nums)
    k = k % n  # 防止 k 大於陣列長度
    
    # 定義反轉函數（幫助我們重複使用）
    def reverse(start, end):
        # 從 start 到 end-1，兩兩交換
        while start < end:
            # 交換兩個位置的值
            nums[start], nums[end] = nums[end], nums[start]
            start += 1
            end -= 1
    
    # 步驟 1：全部反轉
    # [7, 6, 5, 4, 3, 2, 1] → 反轉後 → [1, 2, 3, 4, 5, 6, 7]
    reverse(0, n - 1)
    
    # 步驟 2：前 k 個反轉
    # [1, 2, 3] → 反轉後 → [3, 2, 1]
    reverse(0, k - 1)
    
    # 步驟 3：後面 n-k 個反轉
    # [4, 5, 6, 7] → 反轉後 → [7, 6, 5, 4]
    reverse(k, n - 1)


# 測試範例
test_nums = [1, 2, 3, 4, 5, 6, 7]
rotate(test_nums, 3)
print(f"旋轉後: {test_nums}")  # 輸出: [5, 6, 7, 1, 2, 3, 4]
```

**複雜度分析**：
- ⏱️ 時間複雜度：O(n) - 每個元素最多被反轉兩次
- 💾 空間複雜度：O(1) - 原地修改
