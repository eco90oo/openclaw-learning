# 📚 【Day 3: 雙指標技巧（Two Pointers）- 經典題目】

## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是雙指標？

**比喻**：想象你在玩「終極密碼」猜數字遊戲：
- 老師說範圍是 1-100
- 你猜 50，老師說「太小了」
- 你猜 75，老師說「太大了」
- 你持續縮小範圍，最後找到答案

這就是**雙指標**的核心概念：**用兩個位置（指標）來追蹤範圍，逐步縮小搜尋空間**！

### 📊 雙指標的兩大類型

| 類型 | 比喻 | 應用場景 |
|------|------|----------|
| **相向指標** | 兩個人從操場兩端走向中間 | 排序陣列中找兩數之和、反轉陣列 |
| **同向指標（快慢指標）** | 兩個人賽跑，速度不同 | 環形檢測、移除重複、分割陣列 |

### 🔍 雙指標為什麼厲害？

**暴力法的悲劇**：
```
兩數之和：O(n²) - 嘗試所有配對
雙指標：  O(n)  - 充分利用排序特性！
```

## 2️⃣ 核心知識（時間/空間複雜度分析）

### ⏰ 雙指標複雜度分析

| 情境 | 時間複雜度 | 空間複雜度 | 說明 |
|------|------------|------------|------|
| **相向指標** | O(n) | O(1) | 兩指標最多移動 n 次 |
| **快慢指標** | O(n) | O(1) | 快指標走 2n 步，慢指標走 n 步 |
| **三指標** | O(n) | O(1) | 分割陣列 |

### 🎯 雙指標的核心思維

1. **有序陣列**：先排序，然後用雙指標夾擊
2. **原地操作**：不需要額外空間，直接在原陣列上修改
3. **一次遍歷**：每個元素最多被訪問一次

### 📌 LeetCode 167 - 兩數之和 II（排序陣列）

**題目**：給定已經排序的陣列，找出兩個數使它們的和為目標值。返回它們的索引。

**範例輸入**：`numbers = [2,7,11,15], target = 9`
**範例輸出**：`[1, 2]`（因為 2+7=9）

### 💻 Python 程式碼（詳細註解）

def twoSum(numbers, target):
    """
    兩數之和 II - 使用雙指標（相向指標）
    
    比喻：就像在水桶的兩端放兩個木塊，慢慢往中間移動
    找到和為目標值的組合
    
    參數:
        numbers: List[int] - 已經排序的整數陣列
        target: int - 目標和
    返回:
        List[int] - 兩個索引（1-indexed）
    
    時間複雜度: O(n) - 每個元素最多被訪問一次
    空間複雜度: O(1) - 只用了兩個指標
    """
    
    # left 指標從陣列左邊開始
    # right 指標從陣列右邊開始
    left = 0             # 初始化左指標為 0（第一個元素）
    right = len(numbers) - 1  # 初始化右指標為最後一個索引
    
    # 當左指標還沒超過右指標時，繼續搜尋
    while left < right:
        # 計算目前兩個位置的數字之和
        current_sum = numbers[left] + numbers[right]
        
        # 如果和正好等於目標值，找到答案了！
        if current_sum == target:
            # 返回索引（題目要求 1-indexed，所以 +1）
            return [left + 1, right + 1]
        
        # 如果和小於目標值，需要讓和變大
        # 由於陣列是升序排列，左邊的數比較小
        # 所以讓 left 向右移動，嘗試更大的數
        elif current_sum < target:
            left += 1  # 左指標向右移動
        
        # 如果和大於目標值，需要讓和變小
        # 右邊的數比較大，所以讓 right 向左移動
        else:
            right -= 1  # 右指標向左移動
    
    # 理論上一定會找到（題目保證有解），如果沒找到返回空
    return []


# 🧪 測試
print(twoSum([2,7,11,15], 9))     # 輸出: [1, 2]
print(twoSum([2,3,4], 6))         # 輸出: [1, 3]
print(twoSum([-1,0], -1))         # 輸出: [1, 2]

### 📌 LeetCode 344 - 反轉字串

**題目**：編寫一個函數，反轉字串。必須原地反轉。

**範例輸入**：`s = ["h","e","l","l","o"]`
**範例輸出**：`["o","l","l","e","h"]`

### 💻 Python 程式碼（詳細註解）

def reverseString(s):
    """
    反轉字串 - 使用雙指標（相向指標）
    
    比喻：就像把繩子對折，兩端互換位置
    
    參數:
        s: List[str] - 字元陣列（原地修改）
    返回:
        None - 直接修改輸入陣列
    
    時間複雜度: O(n) - 需要交換 n/2 次
    空間複雜度: O(1) - 只用了兩個指標，沒有額外空間
    """
    
    # left 從開頭開始
    # right 從結尾開始
    left = 0                  # 左指標初始化為 0
    right = len(s) - 1       # 右指標初始化為最後一個索引
    
    # 當左指標還沒超過右指標時，繼續交換
    while left < right:
        # 交換左右兩個位置的元素
        # Python 的 tuple 交换技巧：一行代码交换两个变量
        s[left], s[right] = s[right], s[left]
        
        # 左指標向右移動
        left += 1
        # 右指標向左移動
        right -= 1
    
    # 返回字串（可選），題目要求原地修改，所以返回 None 也可以
    return s


# 🧪 測試
print(reverseString(["h","e","l","l","o"]))  # 輸出: ["o","l","l","e","h"]
print(reverseString(["H","a","n","n","a","h"]))  # 輸出: ["h","a","n","n","a","H"]

### 📌 LeetCode 876 - 鏈表的中間節點

**題目**：給定單向鏈表，返回中間節點。如果有兩個中間節點，返回第二個。

**範例輸入**：`[1,2,3,4,5]`
**範例輸出**：`[3,4,5]`

### 💻 Python 程式碼（詳細註解）

def middleNode(head):
    """
    找鏈表中間節點 - 使用快慢指標（雙指標進階）
    
    比喻：就像兩個人跑步，一個人跑兩步，另一個人跑一步
    當快的人到終點時，慢的人剛好在中間
    
    參數:
        head: ListNode - 鏈表頭節點
    返回:
        ListNode - 中間節點
    
    時間複雜度: O(n) - 快指標走完整個鏈表
    空間複雜度: O(1) - 只用了兩個指標
    """
    
    # slow 慢指標：每次走一步
    # fast 快指標：每次走兩步
    slow = head   # 初始化慢指標
    fast = head   # 初始化快指標
    
    # 當 fast 和 fast.next 都不為空時，繼續前進
    # 注意：要用 and，不能用 or，因為我们要检查 fast.next 是否存在
    while fast and fast.next:
        # slow 走一步
        slow = slow.next
        # fast 走兩步
        fast = fast.next.next
    
    # 當循環結束時，slow 正好在中間位置
    # 如果是奇數個節點：slow 是中間
    # 如果是偶數個節點：slow 是第二個中間（符合題目要求）
    return slow


# 定義鏈表節點類別
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val      # 節點的值
        self.next = next    # 指向下一個節點的指標


# 🧪 測試輔助函數：建立鏈表
def create_linked_list(arr):
    """從列表建立鏈表"""
    if not arr:
        return None
    
    # 建立頭節點
    head = ListNode(arr[0])
    current = head
    
    # 依序建立後續節點
    for val in arr[1:]:
        current.next = ListNode(val)  # 建立新節點並連接
        current = current.next        # 移動到新節點
    
    return head


def linked_list_to_list(head):
    """將鏈表轉換為列表（便於顯示）"""
    result = []
    while head:
        result.append(head.val)
        head = head.next
    return result


# 測試
head1 = create_linked_list([1,2,3,4,5])
result1 = middleNode(head1)
print(linked_list_to_list(result1))  # 輸出: [3, 4, 5]

head2 = create_linked_list([1,2,3,4,5,6])
result2 = middleNode(head2)
print(linked_list_to_list(result2))  # 輸出: [4, 5, 6]

### 📌 LeetCode 141 - 環形鏈表

**題目**：判斷鏈表是否有環。

**範例輸入**：`head = [3,2,0,-4], pos = 1`（表示尾部連接到索引 1）
**範例輸出**：`true`（存在環）

### 💻 Python 程式碼（詳細註解）

def hasCycle(head):
    """
    判斷鏈表是否有環 - 使用快慢指標
    
    比喻：
    想象兩個人在操場跑步，一個人速度快（fast），一個人速度慢（slow）
    如果操場是圓形的（帶環），fast 最終會追上 slow
    如果是直線（無環），fast 會跑到終點
    
    參數:
        head: ListNode - 鏈表頭節點
    返回:
        bool - 是否有環
    
    時間複雜度: O(n) - 最壞情況繞一圈
    空間複雜度: O(1) - 只用了兩個指標
    """
    
    # 邊界情況：空鏈表或只有一個節點，不可能有環
    if not head or not head.next:
        return False
    
    # 初始化快慢指標
    # slow 一次走一步
    # fast 一次走兩步
    slow = head           # 慢指標從頭開始
    fast = head.next      # 快指標從第二個節點開始
    
    # 當 fast 和 fast.next 都不為空時，繼續前進
    # 為什麼這樣判斷？
    # 因為 fast 走兩步，需要確保 fast.next 存在
    while fast and fast.next:
        # 如果快慢指標相遇，表示有環！
        # 這是因為 fast 每次比 slow 多走一步
        # 在環中，fast 最終會繞回來追上 slow
        if slow == fast:
            return True
        
        # slow 走一步
        slow = slow.next
        # fast 走兩步
        fast = fast.next.next
    
    # fast 到達終點（None），說明沒有環
    return False

### 📌 LeetCode 27 - 移除元素

**題目**：給定陣列 nums 和值 val，原地移除所有等於 val 的元素。

**範例輸入**：`nums = [3,2,2,3], val = 3`
**範例輸出**：`2`（陣列變為 [2,2]）

### 💻 Python 程式碼（詳細註解）

def removeElement(nums, val):
    """
    移除元素 - 使用雙指標（快慢指標）
    
    比喻：
    - fast 指標像「偵探」，到處跑找到不是 val 的元素
    - slow 指標像「搬運工」，把非 val 的元素放到前面
    
    參數:
        nums: List[int] - 整數陣列（原地修改）
        val: int - 要移除的元素值
    返回:
        int - 不包含 val 的元素個數
    
    時間複雜度: O(n) - fast 遍歷整個陣列一次
    空間複雜度: O(1) - 只用了兩個指標
    """
    
    # slow 代表「下一個應該放置非 val 值的位置」
    # 也可以叫 write_index（寫入位置）
    slow = 0
    
    # fast 用來遍歷整個陣列
    for fast in range(len(nums)):
        # 如果當前元素不是要移除的值
        if nums[fast] != val:
            # 把這個元素複製到 slow 位置
            nums[slow] = nums[fast]
            # slow 向後移動，準備放下一個非 val 元素
            slow += 1
    
    # slow 就是新陣列的長度
    return slow


# 🧪 測試
nums1 = [3,2,2,3]
print(removeElement(nums1, 3))  # 輸出: 2
print(nums1[:2])                # 輸出: [2, 2]

nums2 = [0,1,2,2,3,0,4,2]
print(removeElement(nums2, 2)) # 輸出: 5
print(nums2[:5])                # 輸出: [0, 1, 3, 0, 4]
