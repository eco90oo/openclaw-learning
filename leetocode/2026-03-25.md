# 📚 【Day 2: 陣列進階（二分搜尋、旋轉陣列）】

## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是二分搜尋？

**比喻**：想象你在翻字典找一個字（比如「技術」的「技」）。你不會從第一頁開始一頁一頁翻，而是：
1. 先翻到中間，看看這個字在前面還是後面
2. 如果在後面，就往後翻到 1/4 的位置
3. 繼續這樣折半搜尋

這就是**二分搜尋（Binary Search）**的核心概念：**每次把搜尋範圍減半**！

```
傳統搜尋（暴力法）:  O(n) - 一個一個找
二分搜尋:            O(log n) - 每次砍半！
```

### 📊 旋轉陣列是什麼？

**比喻**：想象一個已經排序好的陣列 `[1, 2, 3, 4, 5, 6, 7]`，我們把它「旋轉」：

```
原始陣列: [1, 2, 3, 4, 5, 6, 7]
旋轉 3 格: [4, 5, 6, 7, 1, 2, 3]
                  ↑ 這裡是旋轉點
```

旋轉點就是原本陣列的「斷點」，旋轉後陣列變成兩段各自有序的陣列。

## 2️⃣ 核心知識（時間/空間複雜度分析）

### ⏰ 二分搜尋複雜度

| 情況 | 時間複雜度 | 說明 |
|------|------------|------|
| **一般情況** | O(log n) | 每次把範圍減半 |
| **最佳情況** | O(1) | 第一次就找到！ |
| **最差情況** | O(log n) | 需要 log₂(n) 次 |

### 🔍 二分搜尋的四種變體

1. **標準二分**：找目標值
2. **找左邊界**：找第一個 >= 目標的位置
3. **找右邊界**：找最後一個 <= 目標的位置
4. **旋轉陣列**：在旋轉排序陣列中搜尋

## 3️⃣ 經典題目（LeetCode 實際題目）

### 📌 LeetCode 704 - 二分搜尋

**題目**：經典二分搜尋，在排序陣列中找目標值。

**範例輸入**：`nums = [-1,0,3,5,9,12], target = 9`
**範例輸出**：`4`（9 在索引 4）

### 💻 Python 程式碼（詳細註解）

```python
def search(nums, target):
    """
    二分搜尋：找目標值
    
    參數:
        nums: List[int] - 已經排序的整數陣列
        target: int - 要搜尋的目標值
    返回:
        int - 目標值的索引，如果不存在返回 -1
    
    時間複雜度: O(log n) - 每次把搜尋範圍減半
    空間複雜度: O(1) - 只用了几個變數
    """
    
    # left 是搜尋範圍的左邊界
    # right 是搜尋範圍的右邊界
    left = 0                      # 初始化左指標為 0
    right = len(nums) - 1        # 初始化右指標為最後一個索引
    
    # 當左邊界 <= 右邊界時，繼續搜尋
    # 注意：這裡要用 <= 而不是 <，因為要考虑 left == right 的情況
    while left <= right:
        # 計算中間位置
        # 防止溢位：使用 left + (right - left) // 2 而不是 (left + right) // 2
        mid = left + (right - left) // 2
        
        # 如果找到目標，直接返回索引
        if nums[mid] == target:
            return mid            # 返回中間索引
        
        # 如果目標在左半邊，調整右邊界
        elif nums[mid] > target:
            right = mid - 1       # 目標在左邊，搜尋範圍縮小到 [left, mid-1]
        
        # 如果目標在右半邊，調整左邊界
        else:
            left = mid + 1        # 目標在右邊，搜尋範圍縮小到 [mid+1, right]
    
    # 搜尋完都沒找到，返回 -1
    return -1


# 🧪 測試
print(search([-1,0,3,5,9,12], 9))   # 輸出: 4
print(search([-1,0,3,5,9,12], 2))   # 輸出: -1
print(search([5], 5))               # 輸出: 0
```

### 📌 LeetCode 33 - 搜尋旋轉排序陣列

**題目**：陣列原本是排序的，後來被旋轉過。在旋轉陣列中找目標值。

**範例輸入**：`nums = [4,5,6,7,0,1,2], target = 0`
**範例輸出**：`4`

### 💻 Python 程式碼（詳細註解）

```python
def search(nums, target):
    """
    搜尋旋轉排序陣列
    
    思路：
    1. 先找出旋轉點（數組從哪裡斷開）
    2. 根據 target 在哪一段來決定搜尋範圍
    
    時間複雜度: O(log n) - 二分搜尋
    空間複雜度: O(1)
    """
    
    # 邊界情況：空陣列
    if not nums:
        return -1
    
    left = 0                      # 左指標
    right = len(nums) - 1        # 右指標
    
    # 標準二分搜尋，但加入旋轉陣列的判斷
    while left <= right:
        # 計算中間位置
        mid = left + (right - left) // 2
        
        # 如果找到目標，直接返回
        if nums[mid] == target:
            return mid
        
        # 判斷左半邊是否有序（沒有被旋轉）
        # 這是最關鍵的判斷！
        if nums[left] <= nums[mid]:
            # 左半邊是有序的
            # 判斷 target 是否在左半邊
            if nums[left] <= target < nums[mid]:
                # target 在左半邊，搜尋左半邊
                right = mid - 1
            else:
                # target 不在左半邊，搜尋右半邊
                left = mid + 1
        
        # 否則，右半邊是有序的
        else:
            # 右半邊是有序的
            # 判斷 target 是否在右半邊
            if nums[mid] < target <= nums[right]:
                # target 在右半邊，搜尋右半邊
                left = mid + 1
            else:
                # target 不在右半邊，搜尋左半邊
                right = mid - 1
    
    # 沒找到
    return -1


# 🧪 測試
print(search([4,5,6,7,0,1,2], 0))   # 輸出: 4
print(search([4,5,6,7,0,1,2], 3))   # 輸出: -1
print(search([1], 0))               # 輸出: -1
```

### 📌 LeetCode 153 - 尋找旋轉排序陣列中的最小值

**題目**：找到旋轉陣列中的最小值（旋轉點）。

**範例輸入**：`nums = [3,4,5,1,2]`
**範例輸出**：`1`

### 💻 Python 程式碼（詳細註解）

```python
def findMin(nums):
    """
    找旋轉排序陣列的最小值（旋轉點）
    
    比喻：想象一個轉盤，最小值就是轉盤「斷掉」的地方
    
    時間複雜度: O(log n) - 二分搜尋
    空間複雜度: O(1)
    """
    
    # 如果陣列只有一個元素，直接返回
    if len(nums) == 1:
        return nums[0]
    
    left = 0
    right = len(nums) - 1
    
    # 注意：如果陣列沒有旋轉（原陣列），第一個元素會小於最後一個
    # 這種情況直接返回第一個元素
    if nums[right] > nums[0]:
        return nums[0]
    
    # 二分搜尋找到旋轉點
    while left <= right:
        mid = left + (right - left) // 2
        
        # 判斷是否是最小值（旋轉點）
        # 如果 mid 的右邊比 mid 大，那麼 mid 就是最小值
        if nums[mid] > nums[mid + 1]:
            return nums[mid + 1]
        
        # 如果 mid 的左邊比 mid 大，那麼 mid-1 就是最小值
        if nums[mid - 1] > nums[mid]:
            return nums[mid]
        
        # 根據中間值和左邊值的關係，決定搜尋方向
        # 如果 nums[mid] > nums[0]，說明最小值在右半邊
        if nums[mid] > nums[0]:
            left = mid + 1
        else:
            # 否則，最小值在左半邊
            right = mid - 1
    
    # 理論上不會走到這裡
    return nums[0]


# 🧪 測試
print(findMin([3,4,5,1,2]))    # 輸出: 1
print(findMin([4,5,6,7,0,1,2])) # 輸出: 0
print(findMin([2,3,4,5,6,7,8,0,1])) # 輸出: 0
```
