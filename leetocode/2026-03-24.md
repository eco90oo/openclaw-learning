# 📚 【Day 1: 陣列基礎與常見操作 - 詳細時間複雜度分析】

## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是陣列？

**比喻**：想象一條很長的停車場，每個停車位都有一個固定的編號（索引），從 0 開始。你可以根據編號快速找到任何停車位上的車。

```
停車場（陣列）：
[車A] [車B] [車C] [車D] [車E] ... 
  0     1     2     3     4    ← 索引（位置）
```

### 📊 陣列的特性

| 特性 | 說明 | 比喻 |
|------|------|------|
| **連續記憶體** | 陣列的元素在記憶體中是連續排列的 | 停車場的停車位是連成一排的 |
| **固定大小** | 建立時就要決定大小（靜態陣列） | 停車場建好後車位數就固定了 |
| **索引訪問** | 用 O(1) 時間存取任意位置 | 知道編號就能直接走到該車位 |
| **同質資料** | 通常存放相同類型的資料 | 每個停車位只能停一種車 |

### 🐍 Python 中的列表 vs 陣列

Python 的 list 比傳統陣列更彈性，可以動態調整大小（但會有效能開銷）：

```python
# Python 的 list 可以動態增減
my_list = [1, 2, 3]    # 建立列表
my_list.append(4)      # 尾部加入 - O(1) 平均
my_list.insert(0, 0)  # 頭部插入 - O(n)
```

## 2️⃣ 核心知識（時間/空間複雜度分析）

### ⏰ 時間複雜度懶人包

| 操作 | 平均情況 | 最差情況 | 說明 |
|------|----------|----------|------|
| **訪問** `arr[i]` | O(1) | O(1) | 直接計算記憶體位置 |
| **搜尋** | O(n) | O(n) | 需要遍歷 |
| **插入尾部** | O(1) | O(n)* | *觸發擴容時 |
| **插入中間** | O(n) | O(n) | 需要移動元素 |
| **刪除尾部** | O(1) | O(1) | 直接移除 |
| **刪除中間** | O(n) | O(n) | 需要移動元素 |

### 🔍 為什麼訪問是 O(1)？

陣列的記憶體地址計算公式：
```
記憶體位置 = 基地址 + (索引 × 每個元素的大小)
```

就像停車場：
- 基地址 = 停車場入口
- 索引 = 要找的車位編號
- 每個元素大小 = 每個車位的寬度

**一步到位**，不需要一個一個找！

### 📈 空間複雜度

- **基本空間**：O(n) - 存放 n 個元素
- **額外空間**：O(1) - 不考慮輸入輸出，只看額外使用的空間

## 3️⃣ 經典題目（LeetCode 實際題目）

### 📌 LeetCode 485 - 最大連續 1 的個數

**題目**：給定一個二進制陣列，找出最長的連續 1 的長度。

**範例輸入**：`[1,1,0,1,1,1]`
**範例輸出**：`3`（最後三個 1 是最長的）

### 🧠 解題思路

**兩種方法**：
1. **暴力法**：遍歷所有位置，以每個位置為起點往右數
2. **一次遍歷**：用計數器，遇到 1 就加 1，遇到 0 就重置

**推薦方法二**，因為只需要一次遍歷！

### 💻 Python 程式碼（詳細註解）

```python
def findMaxConsecutiveOnes(nums):
    """
    找最大連續 1 的個數
    
    參數:
        nums: List[int] - 二進制陣列（只含 0 和 1）
    返回:
        int - 最長連續 1 的數量
    
    時間複雜度: O(n) - 只需要遍歷一次陣列
    空間複雜度: O(1) - 只用了兩個變數
    """
    
    # max_count: 記錄目前為止看到的最長連續 1 的數量
    # current_count: 記錄當前連續 1 的數量
    max_count = 0      # 初始化為 0（還沒看過任何 1）
    current_count = 0  # 初始化為 0（當前連續數為 0）
    
    # 遍歷整個陣列
    for num in nums:
        # 如果當前元素是 1
        if num == 1:
            # current_count 加 1，表示連續 1 的長度增加
            current_count += 1
        else:
            # 如果遇到 0，表示連續中斷了
            # 更新 max_count（保存到目前為止最好的結果）
            max_count = max(max_count, current_count)
            # 重置 current_count，因為連續 1 被打斷了
            current_count = 0
    
    # 循環結束後，可能最後還有一段連續 1
    #（例如 [1,1,1] 結尾，沒有遇到 0 來觸發更新）
    # 所以要再比較一次
    max_count = max(max_count, current_count)
    
    return max_count  # 返回最大連續 1 的數量
```

### 🧪 測試程式

```python
# 測試範例
print(findMaxConsecutiveOnes([1,1,0,1,1,1]))  # 輸出: 3
print(findMaxConsecutiveOnes([1,0,1,0,1,0]))  # 輸出: 1
print(findMaxConsecutiveOnes([1,1,1,1]))       # 輸出: 4
print(findMaxConsecutiveOnes([0,0,0]))        # 輸出: 0
```

### 📌 LeetCode 283 - 移動零

**題目**：將陣列中所有的 0 移到末尾，同時保持非零元素的相對順序。

**範例輸入**：`[0,1,0,3,12]`
**範例輸出**：`[1,3,12,0,0]`

### 💻 Python 程式碼（詳細註解）

```python def moveZeroes(nums):
    """
    將 0 移到陣列末尾
    
    參數:
        nums: List[int] - 整數陣列
    返回:
        None - 直接修改輸入陣列（原地操作）
    
    時間複雜度: O(n) - 只需要遍歷一次陣列
    空間複雜度: O(1) - 只用了一個變數，沒有額外空間
    """
    
    # pointer 用來標記「下一個非零元素應該放的位置」
    # 這個技巧叫做「兩指標法」
    pointer = 0  # 初始化為 0（從陣列開頭開始）
    
    # 第一次遍歷：把所有非零元素往前搬
    for i in range(len(nums)):
        # 如果當前元素不是 0
        if nums[i] != 0:
            # 把這個非零元素放到 pointer 位置
            nums[pointer] = nums[i]
            # pointer 向後移動，準備放下一個非零元素
            pointer += 1
    
    # 第二次：把剩下的位置都填成 0
    # pointer 之後的位置都是空的，要填上 0
    for i in range(pointer, len(nums)):
        nums[i] = 0


# 另一種寫法（用 swap，更直觀）
def moveZeroes_v2(nums):
    """
    使用交换思路的版本
    
    思路：找到非零元素就和後面的 0 交換
    """
    zero_pos = 0  # 紀錄第一個 0 的位置
    
    # 遍歷找非零元素
    for i in range(len(nums)):
        if nums[i] != 0:
            # 把非零和 0 交換位置
            nums[i], nums[zero_pos] = nums[zero_pos], nums[i]
            # zero_pos 向後移動（現在這個位置變成 0 了）
            zero_pos += 1
```

### 🧪 測試程式

```python
# 測試
nums1 = [0,1,0,3,12]
moveZeroes(nums1)
print(nums1)  # 輸出: [1, 3, 12, 0, 0]

nums2 = [0]
moveZeroes(nums2)
print(nums2)  # 輸出: [0]
```
