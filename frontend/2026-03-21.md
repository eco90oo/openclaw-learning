## 1️⃣ 基礎概念（用生活比喻）

### 🎯 本章節目標
讓後端工程師理解：**前端在做什麼？為什麼他們這樣寫？**

### 💡 比喻：餐廳的廚房與前臺

想象一間餐廳：
- **後端 = 廚房**：負責備料、烹飪、管理庫存（資料庫、API邏輯）
- **前端 = 服務生**：負責擺盤、菜單呈現、接收客人點餐（頁面呈現、互動）
- **API = 點菜系統**：服務生把客人要的菜告訴廚房，廚房做好後傳回給服務生

> 🏆 **核心理解**：後端是「做事的」，前端是「給人看的」。兩者透過 API 溝通。

---

### 📦 HTML = 網頁的骨架

**比喻**：HTML 就像蓋房子的鋼筋水泥 — 決定結構。

```html
<!-- 這是一個簡單的 HTML -->
<div class="user-card">
  <h1>使用者名稱</h1>
  <p>這是個人簡介</p>
  <button>按鈕</button>
</div>
```

**後端視角**：你可以把 HTML 想象成後端 Template Engine（如 Jinja2、Thymeleaf）渲染出來的東西，只不過前端用 JavaScript 動態生成。

---

### 🎨 CSS = 網頁的化妝品

**比喻**：CSS 就像房子的裝潢 — 決定外觀。

```css
/* 讓文字變紅色 */
h1 {
  color: red;
}

/* 讓按鈕變圓角 */
button {
  border-radius: 8px;
  padding: 10px 20px;
}
```

**後端視角**：相當於後端的「樣板檔案」加上「樣式」，決定呈現效果。

---

### ⚡ JavaScript = 網頁的行為

**比喻**：JavaScript 就像房子的電器和自動化的東西 — 決定互動。

```javascript
// 點擊按鈕時執行的動作
document.querySelector('button').addEventListener('click', function() {
  alert('你點擊了按鈕！');
});
```

**後端視角**：相當於後端的「Controller + Service」邏輯，只不過在前端瀏覽器執行。

## 2️⃣ 核心知識（專有名詞解釋）

### 🔑 後端工程師必須知道的術語

| 術語 | 白話解釋 | 後端類比 |
|------|----------|----------|
| **DOM** | 瀏覽器裡的「虛擬記憶體樹」，讓 JS 能操控頁面 | 相當於 ORM 的物件圖 |
| **Component（元件）** | 可重複使用的 UI 區塊 | 後端的「Component」或「Widget」 |
| **Props** | 傳給元件的參數 | 函數的「參數」 |
| **State** | 元件內部的狀態 | 服務的「Instance Variable」 |
| **Hooks** | React 提供的特殊函數，用於管理狀態和生命週期 | 後端的「Lifecycle Hook」 |
| **Virtual DOM** | React 的效能優化技術，用 JS 物件模擬 DOM | 資料庫的「View」概念 |
| **Routing（路由）** | 決定 URL 對應哪個頁面 | 後端的「Router / Dispatcher」 |
| **AJAX/Fetch** | 瀏覽器發送網路請求 | 後端的 HTTP Client |
| **CORS** | 瀏覽器的安全機制，限制跨域請求 | API 的「訪問控制」 |
| **Bundle** | 打包後的 JS/CSS 檔案 | 後端的「Build Artifacts」 |

### 🔥 React 核心概念

**React 是什麼？**
- 目前最流行的前端框架
- 核心理念：**「用 JS 描述 UI」**
- 所有東西都是「元件」

```jsx
// React 元件範例 - 看起來像 HTML 但其實是 JavaScript
function UserCard({ name, bio }) {
  return (
    <div className="user-card">
      <h1>{name}</h1>
      <p>{bio}</p>
    </div>
  );
}

// 使用元件
<UserCard name="小明" bio="我是後端工程師" />
```

**⚠️ 後端工程師注意！**
- `className` 而不是 `class`（因為 class 是 JS 的關鍵字）
- `{name}` 是 JS 表達式，用於動態顯示資料

---

### 🟢 Vue 核心概念

**Vue 是什麼？**
- 另一個流行的前端框架
- 核心理念：**「漸進式框架」**，簡單易學
- 特色：雙向綁定

```vue
<!-- Vue 範例 -->
<template>
  <div class="user-card">
    <h1>{{ name }}</h1>
    <p>{{ bio }}</p>
  </div>
</template>

<script>
export default {
  props: ['name', 'bio']  // 類似 React 的 props
}
</script>
```

### 🌐 API 對接（後端視角）

#### 前端如何呼叫你的 API？

```javascript
// 使用 Fetch API（瀏覽器內建）
fetch('https://api.example.com/users/123')
  .then(response => response.json())  // 解析 JSON
  .then(data => {
    console.log(data);  // 使用資料
  });
```

#### 前端如何處理錯誤？

```javascript
fetch('https://api.example.com/users/123')
  .then(response => {
    if (!response.ok) {
      throw new Error('HTTP 錯誤 ' + response.status);
    }
    return response.json();
  })
  .then(data => {
    // 成功處理
  })
  .catch(error => {
    // 錯誤處理
    console.error('請求失敗:', error);
  });
```

#### 現在流行：Async/Await 語法

```javascript
async function getUser() {
  try {
    const response = await fetch('https://api.example.com/users/123');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('錯誤:', error);
  }
}
```

> 💡 **後端視角**：這就像你用 `requests` 或 `axios` 發 HTTP 請求，只不過在前端執行。

## 3️⃣ 實際操作（程式碼範例）

### 🎯 任務：閱讀並理解一個簡單的 React 元件

以下是一位前端工程師寫的程式碼，請試著解讀：

```jsx
import React, { useState, useEffect } from 'react';
import axios from 'axios';

function UserList() {
  const [users, setUsers] = useState([]);      // 狀態：使用者列表
  const [loading, setLoading] = useState(true); // 狀態：載入中
  const [error, setError] = useState(null);     // 狀態：错误

  // 元件載入時執行（類似後端的 __init__）
  useEffect(() => {
    fetchUsers();
  }, []); // 空陣列 = 只執行一次

  async function fetchUsers() {
    try {
      const response = await axios.get('/api/users');
      setUsers(response.data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  }

  if (loading) return <div>載入中...</div>;
  if (error) return <div>錯誤: {error}</div>;

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

### 📝 程式碼解讀

| 行號 | 程式碼 | 意思 |
|------|--------|------|
| 1 | `import React` | 引入 React 庫 |
| 1 | `useState` | 用於管理元件內的狀態（類似變數） |
| 1 | `useEffect` | 用於處理副作用（類似生命週期鉤子） |
| 7 | `useState([])` | 初始化 users 為空陣列 |
| 15 | `useEffect(..., [])` | 元件渲染後執行一次（類似 mounted） |
| 19 | `axios.get('/api/users')` | 發送 GET 請求到你的後端 API |
| 26 | `if (loading) return...` | 條件渲染：載入中顯示什麼 |
| 32 | `{users.map(...)}` | 迴圈渲染列表（類似 Jinja2 的 for） |
| 33 | `key={user.id}` | React 列表渲染必須的 key（效能優化） |

### 🔄 對比：後端視角 vs 前端視角

| 功能 | 後端（Express/Koa） | 前端（React） |
|------|---------------------|---------------|
| 定義路由 | `app.get('/api/users', handler)` | `<Route path="/users" element={<UserList />} />` |
| 處理請求 | `req.body`, `req.params` | API 呼叫（axios/fetch） |
| 回應格式 | `res.json({ data })` | `response.data` |
| 渲染畫面 | Template Engine | JSX |
| 管理狀態 | 變數/資料庫 | `useState`, `useReducer` |
| 副作用 | Middleware | `useEffect` |

## 📚 課後延伸

### 常見問題 Q&A

**Q: 前端說「CORS 錯誤」是什麼？**
> A: 瀏覽器的安全機制。如果你的 API 和前端的網域不同，瀏覽器會阻止請求。解決方法：在後端設定 CORS header（`Access-Control-Allow-Origin`）。

**Q: 前端說「 hydration 」是什麼？**
> A: SSR（Server-Side Rendering）術語。伺服器先渲染 HTML，瀏覽器再接管變成客戶端渲染。

**Q: 前端說「 CSR / SSR / SSG 」是什麼？**
> A:
> - CSR（Client-Side Rendering）：瀏覽器下載 JS 後渲染（傳統 React）
> - SSR（Server-Side Rendering）：伺服器渲染後回傳 HTML（Next.js、Nuxt）
> - SSG（Static Site Generation）：建置時生成靜態 HTML
