## 1️⃣ 基礎概念

### 什麼是狀態管理？

**比喻**：想像你在管理一家餐廳的庫存

- **原始做法**：每次要確認食材，直接去冰箱看
- **問題**：如果厨房同時很多人要查詢，會很混亂
- **解決方案**：用一個「庫存管理系統」，所有人不論何時都查同一份資料

**在前端就是「狀態管理」**：
- 讓元件之間共享資料
- 不需要層層傳遞 props
- 資料改變時，自動更新相關畫面

## 2️⃣ React Context 是什麼？

### 生活中的比喻

```
餐廳會員系統：
- 每個服務生都需要知道客人是不是會員
- 傳統做法：問經理 → 經理查會員資料 → 回覆
- Context 做法：吧台放一個「會員資料庫」，任何服務生都能直接查
```

### 實際程式碼

```jsx
// 1. 建立 Context
const UserContext = createContext();

// 2. 提供者（Provider）
function App() {
  const [user, setUser] = useState({ name: '小明', level: 'VIP' });
  
  return (
    <UserContext.Provider value={{ user, setUser }}>
      <MainPage />
    </UserContext.Provider>
  );
}

// 3. 使用者（Consumer）
function UserProfile() {
  const { user } = useContext(UserContext);
  return <div>歡迎 {user.name}</div>;
}
```

## 3️⃣ Zustand - 簡單強大的狀態管理

### 為什麼用 Zustand？

| 特性 | Redux | Zustand |
|------|-------|---------|
| 程式碼量 | 多 | 少 |
| 學習曲線 | 陡峭 | 平緩 |
| 效能 | 好 | 好 |
| 適用規模 | 大型 | 中小型 |

### 實際程式碼

```javascript
// 建立 Store
import { create } from 'zustand';

const useStore = create((set) => ({
  // 狀態
  count: 0,
  user: null,
  
  // 方法
  increment: () => set((state) => ({ count: state.count + 1 })),
  setUser: (user) => set({ user }),
}));

// 使用元件
function Counter() {
  const { count, increment } = useStore();
  
  return (
    <div>
      <p>計數：{count}</p>
      <button onClick={increment}>+1</button>
    </div>
  );
}
```

## 4️⃣ 什麼時候該用什麼？

| 情況 | 建議 |
|------|------|
| 小型專案，狀態簡單 | useState 就夠了 |
| 多個元件需要共享資料 | React Context |
| 中型專案，需要管理複雜狀態 | Zustand |
| 大型專案，需要時間旅行、最佳化 | Redux Toolkit |

## 5️⃣ 後端工程師需要知道的重點

1. **狀態來源**：後端 API 回傳的資料，前端會存到狀態管理
2. **資料流動**：了解資料從哪裡來、到哪裡去
3. **效能考量**：不是什麼都往 Context 丟，會造成不必要的重新渲染
