## 1️⃣ 基礎概念（用生活比喻）

### 🎯 本章節目標
讓後端工程師理解：**前端如何與後端「安全、高效、優雅」地對接**

### 💡 比喻：餐廳點餐系統進化版

想象昨天的餐廳比喻升級：

**基礎版**：服務生直接去廚房喊單
**進階版**：有制度的點餐系統

| 餐廳環節 | 對應技術 | 說明 |
|----------|----------|------|
| 菜單 | **API Schema** | 明確定義有哪些菜（API 格式） |
| 點菜單 | **Request Body** | 客人要的菜（POST 資料） |
| 廚房回應 | **Response** | 做好了的菜（JSON 回應） |
| 等菜 | **Loading 狀態** | 廚房正在做 |
| 上錯菜 | **Error 處理** | 400/500 錯誤 |
| 會員卡 | **Authentication** | 確認客人身份 |
| 廚房號碼牌 | **Token / Session** | 確認身份的證明 |
| 催單 | **Request Timeout** | 等太久還沒回應 |

## 2️⃣ 核心知識（專有名詞解釋）

### 🔑 API 對接常見術語

| 術語 | 白話解釋 | 後端類比 |
|------|----------|----------|
| **RESTful** | 一種 API 設計規範，用 HTTP Method 區分動作 | 後端 Controller 的 CRUD 映射 |
| **GraphQL** | 另一種 API 設計，可以一次要「剛好」的資料 | SQL 的 SELECT 指定欄位 |
| **JSON** | 前後端溝通的「普通話」 | 後端的 POJO / Dictionary |
| **HTTP Status Code** | 回應的狀態代碼 | 後端的 response code |
| **Authorization** | 確認「你是誰」 | 後端的登入/權限 |
| **Request Header** | 請求的「瑞克號船頭訊息」 | 後端的 Middleware 資料 |
| **Request Body** | 請求的「貨櫃內容」 | 後端的 `@RequestBody` |
| **Query Parameters** | URL 問號後面的參數 | 後端的 `@RequestParam` |
| **Path Variables** | URL 中的參數 | 後端的 `@PathVariable` |

---

### 🔥 HTTP 狀態碼（後端必須讓前端知道）

| 代碼 | 意思 | 场景 |
|------|------|------|
| **200** | 成功 | 查詢到資料 |
| **201** | 已建立 | 新增成功 |
| **400** | 客戶端錯誤 | 參數錯誤、格式不對 |
| **401** | 未認證 | 沒登入、Token 過期 |
| **403** | 無權限 | 登入了但沒權限 |
| **404** | 找不到 | 資源不存在 |
| **500** | 伺服器錯誤 | 後端炸了 |

> 💡 **後端工程師要注意**：
> - 不要すべての錯誤都回傳 200 + error 欄位
> - 正確的 HTTP Status 讓前端好處理

### 📡 前端常見的請求方式

#### 1️⃣ Fetch API（瀏覽器內建）

```javascript
// GET 請求
fetch('/api/users')
  .then(res => {
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    return res.json();
  })
  .then(data => console.log(data))
  .catch(err => console.error(err));
```

#### 2️⃣ Axios（更流行）

```javascript
// GET 請求
axios.get('/api/users')
  .then(res => res.data)
  .catch(err => console.error(err));

// POST 請求（帶 body）
axios.post('/api/users', { name: '小明', email: 'test@test.com' })
  .then(res => res.data);

// 設定 Header（帶 Token）
axios.get('/api/users', {
  headers: { Authorization: 'Bearer eyJhbGciOiJIUzI1Ni...' }
});
```

> 💡 **後端視角**：Axios 就像 Python 的 `requests` 庫，用法幾乎一模一樣！

### 🎯 認證與授權（前端視角）

#### 常見的 Token 傳遞方式

```javascript
// 方式一：Header 攜帶
fetch('/api/private', {
  headers: {
    'Authorization': 'Bearer your-token-here'
  }
});

// 方式二：Cookie（自動攜帶）
fetch('/api/private'); // 瀏覽器自動帶 Cookie

// 方式三：Bearer Token（最常見）
const token = localStorage.getItem('token');
axios.defaults.headers.common['Authorization'] = `Bearer ${token}`;
```

> ⚠️ **後端工程師要注意**：
> - 如果用 Bearer Token，前端需要知道在哪裡拿 token
> - 如果用 Cookie，注意 CORS 設定（`credentials: 'include'`）

## 3️⃣ 實際操作（程式碼範例）

### 🎯 任務：閱讀真實的 API 請求程式碼

以下是一個典型的前端 API 模組（類似後端的 Service Layer）：

```javascript
// api/users.js
import axios from 'axios';

// 建立 axios 實例（類似後端的 HTTP Client 配置）
const apiClient = axios.create({
  baseURL: '/api',
  timeout: 10000, // 10 秒超時
  headers: {
    'Content-Type': 'application/json'
  }
});

// 請求攔截器：自動加 Token
apiClient.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  error => Promise.reject(error)
);

// 響應攔截器：統一錯誤處理
apiClient.interceptors.response.use(
  response => response.data, // 只回傳 data 部分
  error => {
    if (error.response?.status === 401) {
      // Token 過期，跳轉到登入頁
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

// 使用範例
export const userService = {
  getAll: () => apiClient.get('/users'),
  
  getById: (id) => apiClient.get(`/users/${id}`),
  
  create: (userData) => apiClient.post('/users', userData),
  
  update: (id, userData) => apiClient.put(`/users/${id}`, userData),
  
  delete: (id) => apiClient.delete(`/users/${id}`)
};
```

### 📝 程式碼解讀

| 行號 | 程式碼 | 意思 |
|------|--------|------|
| 3-8 | `axios.create()` | 建立配置好的 HTTP Client（類似 requests.Session()） |
| 12 | `interceptors.request` | 請求發出去前「偷看」並加東西（加 Token） |
| 24 | `interceptors.response` | 響應回來後「偷看」處理（統一錯誤） |
| 28 | `401 處理` | Token 過期時自動跳登入頁 |
| 34-48 | `export const` | 類似後端 export 函數，供其他地方呼叫 |

### 🔄 前端如何呼叫上面的 API？

```javascript
// 在 React 元件中使用
import { userService } from '../api/users';
import { useState, useEffect } from 'react';

function UserList() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    loadUsers();
  }, []);

  async function loadUsers() {
    try {
      const data = await userService.getAll(); // 呼叫 API
      setUsers(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  }

  if (loading) return <div>載入中...</div>;
  if (error) return <div>載入失敗：{error}</div>;

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

### 📝 對比：後端 Controller vs 前端 Service

| 層面 | 後端（Spring Boot） | 前端（React） |
|------|---------------------|---------------|
| API 定義 | `@GetMapping("/users")` | `userService.getAll()` |
| 請求發送 | `restTemplate.getForObject()` | `axios.get()` |
| 統一配置 | `@Configuration` | `axios.create()` |
| 請求攔截 | `HandlerInterceptor` | `axios.interceptors.request` |
| 錯誤處理 | `@ControllerAdvice` | `axios.interceptors.response` |
| 回應格式 | `ResponseEntity` | 直接拿 `response.data` |

### 🎯 實際場景對話

#### 場景一：前端問「這個 API 回什麼格式？」

**後端應該給：**
```json
{
  "id": 1,
  "name": "小明",
  "email": "test@test.com",
  "createdAt": "2026-03-22T10:00:00Z"
}
```

**不要只說「回傳使用者資料」**

#### 場景二：前端問「分頁怎麼做？」

**後端應該給：**
```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 100,
    "totalPages": 5
  }
}
```

#### 場景三：前端問「錯誤格式是什麼？」

**後端應該給：**
```json
{
  "error": {
    "code": "USER_NOT_FOUND",
    "message": "找不到該使用者"
  }
}
```

**不要只回傳 `{"error": "找不到"}**

## 📚 課後延伸

### 常見問題 Q&A

**Q: 前端說「請給我 CORS」怎麼辦？**
> A: 在後端設定以下 Header：
> ```java
> response.setHeader("Access-Control-Allow-Origin", "http://frontend.com");
> response.setHeader("Access-Control-Allow-Methods", "GET,POST,PUT,DELETE");
> response.setHeader("Access-Control-Allow-Headers", "Content-Type,Authorization");
> ```
> 或使用 Spring 的 `@CrossOrigin` 註解

**Q: 前端說「請給我 OPTIONS」是什麼？**
> A: 這是 CORS 的預檢請求（Preflight）。瀏覽器在發送複雜請求前會先問：「我可以發這個請求嗎？」後端要正確回應 OPTIONS 請求。

**Q: 前端說「請給我冪等」是什麼？**
> A: 指「同一個請求發多次，結果要一樣」。GET 應該冪等，POST 可以多次新增。
