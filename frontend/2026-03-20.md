## 1️⃣ 基礎概念（用生活比喻）

### 🌐 HTML = 房子的骨架

**白話比喻：**
> 你要蓋一間房子，HTML 就是「鋼筋水泥」——決定房子長什麼樣子、有幾個房間、門在哪裡。它是網頁的「結構」。

**後端視角：**
> 你寫 API 回傳 JSON，前端用 HTML 把這些資料「組裝」成畫面。HTML 本身不處理資料運算，只負責「擺放」。

```html
<!-- 這就是一個最基本的網頁骨架 -->
<!DOCTYPE html>
<html>
  <head>
    <title>我的作品</title>
  </head>
  <body>
    <h1>歡迎來到我的網站</h1>
    <div id="app"></div>  <!-- 前端會把東西塞進這個容器 -->
  </body>
</html>
```

---

### 🎨 CSS = 房子的裝潢

**白話比喻：**
> CSS 就是「油漆、窗簾、燈具」——讓房子變得好看。沒有 CSS，網頁就像毛胚屋醜醜的。

**後端視角：**
> 你不需要會畫設計圖，但你要知道：CSS 是「裝飾」，不會影響功能。當前端說「樣式有問題」，通常是指 CSS 没寫好，不是你 API 的問題。

```css
/* 讓標題變大、變紅 */
h1 {
  font-size: 24px;
  color: red;
}

/* 讓區塊居中 */
.container {
  display: flex;
  justify-content: center;
}
```

---

### ⚡ JavaScript = 房子的電器設備

**白話比喻：**
> JS 是「水電」——讓房子動起來。燈會亮、門會開、暖氣會運作。沒有 JS，網頁就是靜態的展示牆。

**後端視角：**
> 這是你最需要關心的部分！因為 JS 負責：
> - 呼叫你的 API
> - 處理用戶操作（點擊、輸入）
> - 動態更新畫面

```javascript
// 呼叫後端 API
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => {
    console.log('拿到資料了:', data);
  });
```

## 2️⃣ 核心知識（專有名詞解釋）

### 📦 元件（Component）

**白話比喻：**
> 就像 LEGO 積木。一個「按鈕」是一個元件，一個「卡片」也是元件。把積木拼起來，就組成一隻機器人（網頁）。

**後端工程師怎麼理解：**
> 元件 = 函式/類別。一個 `UserCard` 元件，就像你的 `UserService`，可以重複使用。

```jsx
// React 元件範例
function UserCard({ name, email }) {
  return (
    <div className="card">
      <h3>{name}</h3>
      <p>{email}</p>
    </div>
  );
}
```

---

### 🔄 狀態（State）

**白話比喻：**
> 就像「開關」。燈泡有兩種狀態：開（亮）/ 關（暗）。網頁的「狀態」就是資料的當前值。

**後端工程師怎麼理解：**
> 狀態 = 變數。前端說「狀態改變」 = 變數被修改，畫面要重新渲染。

```javascript
// 這就是狀態：使用者名稱
const [username, setUsername] = useState('');
// 當 setUsername 被呼叫，畫面會自動更新
```

---

### 🔌 Props（屬性）

**白話比喻：**
> 就像「傳遞零件」。組裝 LEGO 時，把零件塞進去，這個零件就是 props。

**後端工程師怎麼理解：**
> Props = 函式參數。父元件把資料傳給子元件，就像你呼叫函式時傳參數。

```jsx
// 父元件把資料傳給子元件
<UserCard name="王小明" email="wang@example.com" />

// 子元件收到 props
function UserCard(props) {
  console.log(props.name); // "王小明"
}
```

---

### 🎯 虛擬 DOM（Virtual DOM）

**白話比喻：**
> 就像「施工藍圖」。真正的 DOM 是蓋好的房子，虛擬 DOM 是藍圖。先在藍圖上畫好，再一次性施工，省時省力。

**後端工程師怎麼理解：**
> 這是 React/Vue 的優化機制。你不用懂怎麼實作，但要知道：它讓畫面更新更快。

---

### 🏗️ 框架 vs 函式庫

| 名稱 | 比喻 | 代表 |
|------|------|------|
| 框架 (Framework) | 整套蓋房子方案 | React, Vue, Angular |
| 函式庫 (Library) | 工具箱 | jQuery, moment.js |

**後端視角：**
> 你可以選擇用「框架」自己蓋房子（React/Vue），或用「函式庫」只買幾把螺絲起子（jQuery）。現在主流是框架。

## 3️⃣ 實際操作（程式碼範例）

### 🎯 任務：理解前端如何呼叫你的 API

假設後端有一個 API：
```json
GET /api/users/1
Response: {
  "id": 1,
  "name": "張三",
  "email": "zhangsan@example.com"
}
```

前端用 React 這樣寫：

```jsx
import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    // 頁面載入時，呼叫後端 API
    fetch(`/api/users/${userId}`)
      .then(response => {
        if (!response.ok) {
          throw new Error('網路錯誤');
        }
        return response.json();
      })
      .then(data => {
        setUser(data);       // 把資料存進狀態
        setLoading(false);  // 關掉 loading
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, [userId]); // 當 userId 變時，重新執行

  if (loading) return <p>載入中...</p>;
  if (error) return <p>錯誤: {error}</p>;

  return (
    <div>
      <h2>{user.name}</h2>
      <p>{user.email}</p>
    </div>
  );
}
```

**🔍 讓後端工程師看懂的重點：**

| 前端程式 | 你要知道的 |
|----------|-----------|
| `useState` | 他在宣告變數存資料 |
| `useEffect` | 他在處理「副作用」（如 API 呼叫）|
| `fetch()` | 他在呼叫你的 API！|
| `.then()` | 他在處理非同步（你的 API 回覆了）|
| `loading` / `error` | 他在處理「載入中」和「錯誤」狀態 |

---

### 📡 API 對接常見情境

**情境 1：後端改了 API 格式**
> 前端會說：「你回傳的欄位跟之前不一樣，我的畫面壞了。」
> 
> 💡 原因：前端依賴特定欄位名稱（如 `user.name`），你改了欄位名稱他不知道。

**情境 2：後端改了 HTTP 方法**
> 前端會說：「你從 GET 改成 POST，我沒改程式會壞掉。」
> 
> 💡 原因：前端用 `fetch('/api/users', { method: 'GET' })`，你改成 POST 他沒改。

**情境 3：後端加了身份驗證**
> 前端會說：「我的請求被擋下來了，401 Unauthorized。」
> 
> 💡 原因：他沒傳 token，你後端擋掉了。

## 4️⃣ 常見前端術語（與後端對照）

| 前端術語 | 比喻 | 後端理解 |
|----------|------|----------|
| API 呼叫 (API Call) | 打電話給後端 | HTTP Request |
| 請求 (Request) | 寄信 | HTTP Request |
| 回應 (Response) | 回信 | HTTP Response |
| 渲染 (Render) | 畫出畫面 | 組裝 HTML |
| 重新渲染 (Re-render) | 重新畫畫面 | 重新組裝 HTML |
| 生命週期 (Lifecycle) | 網頁的一生 | - |
| 路由 (Router) | 網址導航 | URL routing |
| 打包 (Build) | 編譯打包 | 編譯 / 建置 |

| 節點 (Node) | 網頁元素 | DOM 元素 |
| 事件 (Event) | 用戶操作 | 用戶操作 |

## 5️⃣ 如何與前端順利溝通

### ✅ 建議做法

1. **文件先寫好**
   > 把 API 文件寫清楚：URL、方法、請求格式、回傳格式、錯誤碼。

2. **不要只說「壞了」**
   > 前端說「壞了」時，請他提供：錯誤訊息、請求的 URL、請求的 header。

3. **共同理解 HTTP 狀態碼**
   ```
   200 = 成功
   400 = 前端傳錯格式
   401 = 沒登入
   404 = 找不到資源
   500 = 後端炸了
   ```

4. **用 Postman 測試你的 API**
   > 後端也要會測自己的 API，這樣才能排除是前端問題還是後端問題。

5. **確認資料欄位命名**
   > 提前溝通好 snake_case / camelCase，别等到上線才發現對不上。

## 📝 今日小任務

### 🎯 動手練習：解讀前端程式碼

請閱讀以下 React 程式碼，回答問題：

```jsx
function ProductList() {
  const [products, setProducts] = useState([]);
  const [searchTerm, setSearchTerm] = useState('');

  useEffect(() => {
    fetch('/api/products')
      .then(res => res.json())
      .then(data => setProducts(data));
  }, []);

  const filteredProducts = products.filter(p => 
    p.name.toLowerCase().includes(searchTerm.toLowerCase())
  );

  return (
    <div>
      <input 
        type="text" 
        placeholder="搜尋商品" 
        onChange={e => setSearchTerm(e.target.value)}
      />
      <ul>
        {filteredProducts.map(product => (
          <li key={product.id}>{product.name}</li>
        ))}
      </ul>
    </div>
  );
}
```

**問題：**
1. `useState` 在這裡做了什麼？（提示：哪裡是「狀態」？）
2. `useEffect` 什麼時候會被執行？（提示：什麼時候呼叫 API？）
3. `filteredProducts` 是怎麼產生的？（提示：誰觸發這個動作？）

**答案：**
1. `products` 存商品列表，`searchTerm` 存搜尋關鍵字
2. 網頁載入時（第一次渲染）執行一次
3. 使用者輸入時（`onChange`），`searchTerm` 改變，篩選結果自動更新
