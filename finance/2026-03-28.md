## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 什麼是量化投資？

想象你在玩一款策略遊戲：
- **傳統投資（主動管理）**＝靠直覺和經驗玩遊戲的玩家，看基本面、技術面，憑「感覺」下判斷
- **量化投資**＝用外掛（電腦程式）玩的玩家，把所有「必勝策略」寫成程式，讓電腦自動執行

**核心差異**：量化投資把投資決策變成一連串的**數學公式和規則**，用程式自動執行。

> **比喻**：就像導航系統
> - **傳統投資**＝看地圖、問當地人、憑經驗找路
> - **量化投資**＝輸入座標，GPS 直接算出最佳路徑

---

### 什麼是因子（Factor）？

**因子**＝影響股票報酬的「特徵」或「屬性」

想象你在徵婚：
- 你會篩選對方的「身高、收入、年齡、學歷」等特徵
- 這些特徵＝**因子**，幫助你預測婚姻幸福程度（類似股票未來報酬）

投資中的常見因子：
| 因子 | 定義 | 篩選標準 |
|------|------|----------|
| **規模因子 (Size)** | 公司大小 | 小型股 > 大型股 |
| **價值因子 (Value)** | 股票是否便宜 | 低本益比、低股價淨值比 |
| **動量因子 (Momentum)** | 近期漲跌趨勢 | 過去漲幅大的股票 |
| **品質因子 (Quality)** | 公司體質 | 高ROE、低負債 |
| **波動率因子 (Volatility)** | 股價波動程度 | 低波動股票 |

> **比喻**：就像選大學
> - 規模因子＝選小型學校的潛力股
> - 價值因子＝選「分數低但有內涵」的學校
> - 動量因子＝選最近「進步最多」的學校

---

### 演算法交易是什麼？

**演算法交易**＝用電腦程式自動執行買賣，無需人為干預

想象自動販賣機：
- 你投入硬幣（訊號）→ 機器自動判斷庫存、找零、出貨（演算法）→ 拿到商品（成交）
- 過程完全自動化，沒有「人」在裡面決策

常見演算法類型：
1. **均值回歸**：股價偏離平均值太多，就會「回來」
2. **趨勢追蹤**：順勢而為，追漲殺跌
3. **套利**：抓住兩個市場的價格差
4. **冰山訂單**：分拆大單，慢慢買進

## 2️⃣ 核心知識（詳細教學，包含公式與計算）

### 2.1 Fama-French 三因子模型

這是量化投資的基石！由諾貝爾經濟學獎得主 Eugene Fama 和 Kenneth French 提出。

**核心公式**：

$$R_i - R_f = \alpha + \beta_1(Market) + \beta_2(SMB) + \beta_3(HML) + \epsilon$$

其中：
- $R_i$ = 股票 i 的報酬率
- $R_f$ = 無風險利率（國庫券利率）
- $\alpha$ = 超額報酬（無法被因子解釋的部分）
- $\beta_1$ = 市場風險係數
- $SMB$ = Small Minus Big（規模因子，小型股 - 大型股）
- $HML$ = High Minus Low（價值因子，高帳市值比 - 低帳市值比）
- $\epsilon$ = 殘差（未被解釋的部分）

**解讀**：
- 如果 $\beta_2$ 為正，表示這檔股票有「小型股偏好」
- 如果 $\beta_3$ 為正，表示這檔股票有「價值股偏好」

---

### 2.2 因子投資實務計算

**案例：篩選因子投資組合**

假設你想建立一個「價值 + 動量」因子投資組合：

**步驟1：計算因子分數**

| 股票 | 本益比 (PE) | 帳市值比 (PB) | 過去6個月報酬 |
|------|------------|--------------|--------------|
| A公司 | 8 | 0.9 | +15% |
| B公司 | 25 | 3.5 | -5% |
| C公司 | 12 | 1.2 | +8% |

**步驟2：標準化因子分數**

價值分數 = 1 / PE（越高越便宜）
動量分數 = 6個月報酬（越高越好）

| 股票 | 價值分數 (標準化) | 動量分數 (標準化) | 綜合分數 |
|------|------------------|------------------|---------|
| A公司 | 0.85 (高) | 0.90 (高) | 0.875 |
| B公司 | 0.10 (低) | 0.30 (低) | 0.200 |
| C公司 | 0.60 (中) | 0.60 (中) | 0.600 |

**步驟3：選股**

只選綜合分數 > 0.6 的股票 → 選 A公司 和 C公司

### 2.3 演算法交易策略設計

**均值回歸策略**：

$$Z = \frac{P_t - \mu}{\sigma}$$

- $P_t$ = 現在股價
- $\mu$ = 歷史平均價格
- $\sigma$ = 歷史標準差
- $Z$ = Z分數

**交易規則**：
- $Z > 2$：股價太高，卖出
- $Z < -2$：股價太低，買入
- $Z$ 回歸到 0 時：平倉

**實際計算案例**：

假設某檔股票：
- 過去 30 天平均價格 $\mu$ = 100 元
- 標準差 $\sigma$ = 10 元
- 今天收盤 $P_t$ = 122 元

$$Z = \frac{122 - 100}{10} = 2.2$$

$Z = 2.2 > 2$ → 觸發賣出訊號！

### 2.4 回測（Backtesting）

**什麼是回測**？用歷史資料測試策略是否有效

**關鍵指標**：
1. **總報酬率**：策略整體賺多少
2. **年化報酬率**：平均每年賺多少
3. **最大回撤**：最慘的時候跌多少
4. **夏普比率**：風險調整後的報酬
5. **勝率**：賺錢的次數比率

**計算案例**：

假設你用均值回歸策略交易 10 次：
- 賺錢 7 次，每次平均賺 5%
- 失敗 3 次，每次平均虧 3%

**總報酬** = $7 \times 5\% - 3 \times 3\% = 35\% - 9\% = 26\%$

**勝率** = 7/10 = 70%

**平均報酬** = 26% / 10 = 2.6% per trade

## 3️⃣ 實際應用（案例分析、計算範例）

### 案例：因子投資組合實務

**情境**：你想用 Python 實作一個簡單的因子投資策略

**Step 1：安裝所需套件**
```python
pip install pandas numpy yfinance
```

**Step 2：獲取資料並計算因子**
```python
import yfinance as yf
import pandas as pd
import numpy as np

# 下載股票資料
stocks = ['AAPL', 'MSFT', 'GOOGL', 'AMZN', 'META']
data = yf.download(stocks, start='2023-01-01')['Adj Close']

# 計算動量因子（過去6個月報酬）
momentum = data.pct_change(periods=126)  # 約126個交易日

# 計算價值因子（本益比需要財報，這裡用股價淨值比近似）
# 假設帳面值（簡化處理）
value_factor = data / data.mean()  # 簡化的價值指標

print("動量因子：")
print(momentum.tail())
```

**Step 3：選股邏輯**
```python
# 選取動量最強且價值被低估的股票
score = momentum * 0.5 + (1/value_factor) * 0.5  # 綜合分數
top_stocks = score.idxmax(axis=1)  # 選最高分的股票

print("推薦股票：", top_stocks.iloc[-1])
```

### 案例：簡單演算法交易回測

**情境**：如果你在 2023/1/1 開始用 100 萬投資台積電，用均值回歸策略：

**模擬結果**（簡化假設）：
- 初始資金：1,000,000 元
- 交易次數：20 次
- 平均報酬：+2.5% per trade
- 勝率：65%

**計算**：
- 總報酬 = 13 次賺錢 × 2.5% - 7 次失敗 × 1.5%
- = 32.5% - 10.5% = 22%
- 最終資金 = 1,000,000 × 1.22 = 1,220,000 元

**注意**：這只是模擬，實際交易會有：
- 手續費（0.1%~0.2%）
- 滑價（成交價與預期不符）
- 流動性風險

## 4️⃣ 常見錯誤提醒

### ❌ 過度擬合（Overfitting）

**錯誤**：用過去的歷史資料「硬湊」出一個完美策略，卻在未來失效

**比喻**：考試前把考古題全部背下來，但換了題目就不會

**解決方法**：
- 用不同的時間區間測試
- 簡化策略（不要用太多參數）
- 保留一部分資料做「樣本外測試」

---

### ❌ 忽視交易成本

**錯誤**：計算報酬時忘了扣手續費、滑價、稅金

**真實案例**：
- 頻繁交易策略看起來賺 30%
- 但扣掉 1% 手續費 + 0.5% 滑價 + 證交稅 = 實際可能倒虧

**建議**：保守估計交易成本 0.5%~1%

---

### ❌ 沒有風控機制

**錯誤**：沒有設定停損、沒有部位限制

**後果**：單筆交易可能虧掉 50% 或更多

**必備風控**：
- 單筆最大損失 ≤ 2%
- 每日最大損失 ≤ 5%
- 嚴格執行停損

---

### ❌ 混淆相關性與因果性

**錯誤**：以為某因子「有效」就代表它能「預測」未來

**事實**：歷史上有效的因子，未來可能完全失效（因子衰減）

**建議**：
- 因子輪動要定期檢視
- 不要把單一因子當作「聖杯」
