## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🌐 區塊鏈是什麼？

想像你在一個**沒有銀行的世界**裡，想轉帳給朋友。

傳統方式：
- 你把錢交給銀行 ➜ 銀行記帳 ➜ 銀行確認 ➜ 朋友收到錢
- 問題：**銀行倒閉怎麼辦？銀行亂記帳怎麼辦？銀行收手續費怎麼辦？**

區塊鏈的解法：
- **每個人都有一本帳本**，同時記錄所有交易
- 當你轉帳時，全世界所有人都檢查：「這筆交易合理嗎？」
- 如果大多數人說合理，**交易就成立**，寫入帳本
- **沒有人能偷偷改帳本**，因為改了自己那本沒用，必須同時改過半數人的帳本才有效

### 🏠 生活比喻：公寓公佈欄

想像你住在一棟公寓：

**傳統方式（銀行中心化）：**
- 有一位管理員負責記帳
- 所有房租、費用都經過管理員
- 管理員可以偷改帳目、可以倒閉跑路

**區塊鏈方式（去中心化）：**
- 一樓大廳有一塊**公佈欄**
- 每次有人轉帳，管理員在公佈欄上寫：「3樓住戶轉給5樓住戶 5000元」
- **所有住戶都拿手機拍照存證**
- 以後有人說「我沒收到錢」，直接翻公佈欄的歷史紀錄
- **無法偽造**，因為所有人的手機裡都有相同備份

### ⛓️ 「區塊」+「鏈」是什麼意思？

**區塊 (Block)：**
想像每一頁帳本就是一個區塊
- 這頁帳本記載了過去 10 分鐘內所有交易
- 例如：「張三轉給李四 1 萬」、「王五收到 5000」

**鏈 (Chain)：**
每一頁帳本都用**神奇的鎖**串在一起
- 每一頁都有前一頁的「指紋」在上面
- 如果你想改第 3 頁的內容，第 4、5、6... 所有頁面都會連鎖反應
- 就像把帳本釘死在一起，**撕掉一頁就等於全部作廢**

### 📜 區塊鏈的三大特性

| 特性 | 說明 | 生活比喻 |
|------|------|---------|
| **去中心化** | 沒有單一機構掌控 | 就像沒有人能掌控 Facebook，所有用戶共同維護 |
| **不可篡改** | 歷史資料無法修改 | 就像區塊鏈是時光機寫入的，過去的事實永遠存在 |
| **透明可驗證** | 任何人都能查詢 | 就像區塊鏈是公開的維基百科，但沒有管理員能刪改 |

## 2️⃣ 核心知識（詳細教學，包含公式與計算）

### 🔴 比特幣的誕生：2008 年金融危機

2008 年 9 月 15 日，**雷曼兄弟倒閉**，全球金融海嘯。

同年 11 月，一位神秘人物（或團隊）**中本聰 (Satoshi Nakamoto)** 發表白皮書：

> 標題：《Bitcoin: A Peer-to-Peer Electronic Cash System》
> 中文：《比特幣：一種點對點的電子現金系統》

**核心問題：** 如何在**沒有銀行**的情況下，實現安全的轉帳？

### 🚀 中本聰解決了三個歷史難題

| 問題 | 傳統解法 | 比特幣解法 |
|------|---------|-----------|
| **雙重支付 (Double Spending)** | 銀行統一記帳 | 全網公佈交易，礦工競爭記帳 |
| **拜占庭將軍問題** | 需要可信第三方 | PoW 共識機制 |
| **貨幣發行** | 政府決定 | **代碼決定總量**，2100 萬枚永不超發 |

### 🧮 比特幣關鍵數據

```
比特幣總量上限：21,000,000 BTC（永遠不會多）
區塊時間：平均 10 分鐘一個區塊
區塊獎勵：2009 年是 50 BTC，2024 年約 3.125 BTC（約每 4 年減半）
區塊大小：1 MB（後來用 SegWit 技術擴展）
預計挖完時間：2140 年
```

### 🎯 減半機制 (Halving) 是什麼？

比特幣設計了一個**自動減少獎勵**的機制：
- 每產生 **210,000 個區塊**（約 4 年），獎勵**減半**
- 2009 年：50 BTC
- 2012 年：25 BTC
- 2016 年：12.5 BTC
- 2020 年：6.25 BTC
- 2024 年：3.125 BTC
- 2140 年：**0 BTC**（全部挖完）

**比喻：** 像是一間公司設定獎金，每年員工數量不變，但獎金每年砍半，激勵員工在前期努力，後期則靠手續費維持網路安全。

### 🟣 以太坊的誕生：2013 年 Vitalik Buterin

比特幣只能轉帳，但無法執行**程式**。

2013 年，**Vitalik Buterin**（一個 19 歲的俄羅斯裔加拿大少年）提出：

> 「比特幣需要一個圖靈完整的平台，能執行智能合約」

2014 年：以太坊進行 ICO（首次代幣發行）
2015 年 7 月 30 日：以太坊主網正式上線

### 🌐 用 Python 查詢以太坊區塊鏈資料

```python
import requests

# 以太坊公開的 RPC 節點服務（由 Ankr 提供）
# RPC = Remote Procedure Call，翻譯：遠端程序呼叫
# 就像是你打電話給以太坊節點，問它：「現在區塊鏈怎麼樣？」

eth_rpc_url = "https://rpc.ankr.com/eth"

# 定義要發送的請求內容（JSON-RPC 格式，是以太坊的標準 API 格式）
payload = {
    "jsonrpc": "2.0",      # JSON-RPC 版本號，永遠是 "2.0"
    "method": "eth_blockNumber",  # 查詢最新區塊高度的方法
    "params": [],          # 這個方法不需要參數，所以是空陣列
    "id": 1                # 用戶端自定義的 ID，用來匹配請求與回應
}

# 發送 HTTP POST 請求（以太坊 RPC 用 POST 方法）
response = requests.post(eth_rpc_url, json=payload)

# 取得回應內容
result = response.json()

# 以太坊的區塊高度是十六進位（hex）格式，需要轉成十進位
# 例如："0x10d4f21" 代表某個十六進位數字
block_hex = result['result']

# 把十六進位字串轉成十進位整數
block_number = int(block_hex, 16)

print(f"📊 目前以太坊區塊高度：{block_number}")
print(f"   （十六進位格式：{block_hex}）")
```

```

   （十六進位格式：0x13f58a16）
```

## 4️⃣ 常見錯誤提醒

### ❌ 錯誤一：把區塊鏈當成萬能資料庫

**錯：** 「區塊鏈可以儲存所有資料，所以我要把 users 表格全部存上鏈」
**對：** 區塊鏈儲存成本極高，所有資料都上鏈會造成：
- Gas Fee 天價
- 節點同步時間變長
- 資料膨脹失控

**正確做法：** 只有需要「不可篡改性」的資料才上鏈，例如交易紀錄、智慧合約狀態。其他資料存普通資料庫即可。

### ❌ 錯誤二：認為區塊鏈完全匿名

**錯：** 「我用比特幣所以別人不知道我是誰，完全隱私」
**對：** 區塊鏈是**假名制 (Pseudonymous)**：
- 所有交易公開
- 如果你的地址與真實身份掛鉤（例如交易所 KYC），則完全可追溯
- 政府可用鏈上分析工具追蹤資金流向（已有多起案例）

### ❌ 錯誤三：把區塊鏈和加密貨幣混為一談

**錯：** 「區塊鏈就是比特幣 / 區塊鏈就是用來投機的」
**對：** 區塊鏈是一種**技術架構**，加密貨幣只是應用之一：
- 區塊鏈可用於：供應鏈管理、數位身份、投票系統、醫療紀錄、房地產登記...
- 比特幣只是區塊鏈技術的第一個應用

### ❌ 錯誤四：以為所有交易都即時完成

**錯：** 「我用比特幣轉帳，應該馬上到帳」
**對：** 比特幣平均 10 分鐘一個區塊，但：
- 交易所通常要求 6 個區塊確認（約 60 分鐘）才視為安全
- 以太坊 12 秒一個區塊，但仍需等待數個區塊確認
- 高峰期 Gas Fee 飆升時，可能需要更久
