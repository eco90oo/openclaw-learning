### 1️⃣ 基礎概念（用生活比喻，越詳細越好）

#### 🏠 區塊鏈是什麼？用村莊比喻讓你秒懂！

想像你住在一個小村莊裡，過去所有的交易記錄都寫在一本**公有帳本**上，由村長（中心化機構）保管。但問題來了：
- 村長可能喝醉酒記錯
- 村長可能被賄賂篡改記錄
- 村長退休或生病，帳本可能遺失

**區塊鏈的誕生就是為了解決這個問題！**

##### 🔗 什麼是區塊鏈？（Block + Chain）

| 傳統方式 | 區塊鏈方式 |
|---------|-----------|
| 村長記帳（中心化）| 每人一本帳本（去中心化）|
| 一本帳本 |  thousands 本帳本同步 |
| 村長說了算 | 多數人說了算 |
| 可以篡改 | 幾乎不可能篡改 |

**生活比喻**：想像你和 10 個朋友玩「記帳接力賽」：
1. 每人拿一本筆記本
2. 有人交易時，大家同時記下：「小明給了小華 100 元」
3. 每次記帳後，大家對一下彼此的筆記是否一樣
4. 確認一致後，這頁「帳」就「鎖死」不能改

這就是區塊鏈的核心精神：**大家一起記、大家一起對、記錄不可改**。

#### 💰 比特幣 Bitcoin - 第一個成功的區塊鏈

##### 比特幣的誕生背景（2008 年金融危機）

2008 年，全球金融海嘯讓人們對銀行和政府失去了信任。這時，一位神秘人物**中本聪**（至今不知是誰）發表了一份白皮書，標題是：

> **《比特幣：一種點對點的電子現金系統》**

##### 比特幣解決了什麼問題？

在數位世界裡，有一個經典的「雙重支付問題」：
- 你有一張數位照片，可以複製貼上給很多人
- 如果是錢，我能「複製」我的錢嗎？當然不行！

**比特幣用區塊鏈技術解決了雙重支付問題**，讓數位貨幣第一次成為可能。

##### 🪙 比特幣的核心特性

| 特性 | 說明 | 比喻 |
|-----|-----|-----|
| 總量固定 2100 萬枚 | 永不增發，對抗通膨 | 就像地球上的黃金，挖完就沒了 |
| 去中心化 | 沒有央行控制 | 像沒有人管理的公社 |
| 可分割 | 最小可到 0.00000001 BTC (1 satoshi) | 可以找零 |
| 匿名/假名 | 不需要實名 | 像戴面具交易 |
| 全球流通 | 跨境轉帳快速 | 就像數位黃金 |

#### 🌐 以太坊 Ethereum - 不只是錢，更是「世界電腦」

##### 以太坊的誕生（2014 年）

比特幣成功了，但它有一個限制：**只能用來轉帳**。

這時，一位 19 歲的天才少年 **Vitalik Buterin** 想到：
> 「如果區塊鏈不只是記帳，還能運行程式呢？」

於是，以太坊誕生了！

##### 以太坊 vs 比特幣

| 面向 | 比特幣 | 以太坊 |
|-----|-------|-------|
| 定位 | 數位黃金/貨幣 | 世界電腦/智慧合約平台 |
| 創辦人 | 中本聪（匿名） | Vitalik Buterin（公開） |
| 主要功能 | 轉帳 | 部署智能合約 |
| 區塊時間 | ~10 分鐘 | ~12-15 秒 |
| 最大供應量 | 2100 萬枚 | 無上限（目前流通中） |
| 程式語言 | Bitcoin Script | Solidity |

##### 🌟 什麼是智能合約 Smart Contract？

**生活比喻**：想象一臺自動販賣機！
- 你投入硬幣 → 機器自動判斷金額 → 彈出商品
- 這個過程不需要人類介入，機器「自動執行合約」

在區塊鏈上，智能合約就是：
```
如果 A 條件達成 → 自動執行 B 動作
```

例如：
- 如果我把 1 ETH 存入合約 → 合約自動給我相對應的代幣
- 如果我借貸逾期 → 合約自動清算我的抵押品

### 2️⃣ 核心知識（詳細教學，包含公式與計算）

#### 區塊鏈的核心技術組成

```
┌─────────────────────────────────────────────────────────┐
│                    區塊鏈架構圖                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐ │
│   │   區塊 #1   │───▶│   區塊 #2   │───▶│   區塊 #3   │ │
│   │  (Block)   │    │  (Block)    │    │  (Block)    │ │
│   └─────────────┘    └─────────────┘    └─────────────┘ │
│        │                  │                  │         │
│        ▼                  ▼                  ▼         │
│   ┌─────────────────────────────────────────────┐     │
│   │  每個區塊包含：                               │     │
│   │  • 前一個區塊的 Hash（指紋）                  │     │
│   │  • 交易數據                                   │     │
│   │  • 時間戳記                                   │     │
│   │  • 隨機數 (Nonce)                            │     │
│   └─────────────────────────────────────────────┘     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

##### 區塊結構詳解

```javascript
// 區塊的實際結構（概念圖）
Block {
    index: 1,                    // 區塊編號
    timestamp: "2026-03-26T23:04:00Z",  // 時間戳
    transactions: [              // 交易列表
        { from: "小明", to: "小華", amount: 100 },
        { from: "小美", to: "小強", amount: 50 }
    ],
    previousHash: "0000abc123...",  // 前一個區塊的指紋
    hash: "0000def456...",          // 這個區塊的指紋
    nonce: 12345                     // 隨機數（用於工作量證明）
}
```

##### 為什麼區塊鏈無法篡改？

**關鍵在於「連環扣」設計**：

假設你要篡改區塊 #2 的某一筆交易：
1. 你改了區塊 #2 的內容 → 區塊 #2 的 **Hash** 變了
2. 區塊 #3 存的是舊的 #2 Hash → **對不上**了！
3. 你必須連區塊 #3、#4、#5... 全部重新計算
4. 而且還要超過 51% 的節點同意（幾乎不可能）

**比喻**：就像把所有多米諾骨牌推倒重來，而且要在別人注意到之前完成！

### 3️⃣ 實際操作（程式碼範例，每行都要有中文註解）

#### 安裝 Web3.py 並連接以太坊測試網

```python
# 安裝 web3.py 庫（Python 操作區塊鏈的工具）
# pip install web3

from web3 import Web3  # 導入 Web3 類別

# =====================================================
# 連接以太坊測試網 (Sepolia Testnet)
# =====================================================
# 我們使用 Infura 的公共節點（非必須，自己運行節點也可以）
INFURA_PROJECT_ID = "your_infura_project_id"  # 請去 https://infura.io/ 申請免費金鑰

# 測試網 RPC URL
sepolia_rpc = f"https://sepolia.infura.io/v3/{INFURA_PROJECT_ID}"

# 建立 Web3 實例
web3 = Web3(Web3.HTTPProvider(sepolia_rpc))

# 檢查連接是否成功
if web3.is_connected():
    print("✅ 成功連接到以太坊 Sepolia 測試網！")
    print(f"   最新區塊號: {web3.eth.block_number}")
else:
    print("❌ 連接失敗，請檢查 RPC URL")

# =====================================================
# 取得區塊資訊（展示區塊鏈的結構）
# =====================================================
# 取得最新的區塊
latest_block = web3.eth.get_block('latest')

print("\n📦 最新區塊資訊：")
print(f"   區塊編號: {latest_block['number']}")
print(f"   時間戳: {latest_block['timestamp']}")
print(f"   區塊 Hash: {latest_block['hash'].hex()[:20]}...")
print(f"   父區塊 Hash: {latest_block['parentHash'].hex()[:20]}...")
print(f"   交易數量: {len(latest_block['transactions'])}")
print(f"   Gas 限制: {latest_block['gasLimit']}")

# =====================================================
# 體驗比特幣價格查詢（從比特幣節點）
# =====================================================
# 使用 public API 查詢比特幣價格（不需要連接比特幣節點）
import requests

# CoinGecko API（比特幣實時價格）
response = requests.get(
    "https://api.coingecko.com/api/v3/simple/price?ids=bitcoin&vs_currencies=usd"
)
btc_price = response.json()['bitcoin']['usd']

print(f"\n💰 比特幣當前價格: ${btc_price:,}")
print(f"   （數據來源: CoinGecko）")
```

### 4️⃣ 常見錯誤提醒

#### ❌ 誤區 1：把加密貨幣當成「股票」

**錯誤理解**：比特幣價格漲了 = 這是投資機會
**正確理解**：比特幣是「貨幣」也是「資產類別」，波動大且風險極高

#### ❌ 誤區 2：以為區塊鏈「完全匿名」

**錯誤理解**：區塊鏈交易是完全隱密的
**正確理解**：區塊鏈是「假名」（Pseudonym），地址雖然不綁定姓名，但透過分析工具（如 Chainalysis）可以追蹤到身份

#### ❌ 誤區 3：把「私鑰」當成「密碼」

**錯誤理解**：私鑰可以像密碼一樣修改
**正確理解**：私鑰就像「銀行卡密碼 + 银行卡 + 身份證」，一旦丟失或洩露，資產無法恢復！

#### ❌ 誤區 4：混淆「區塊鏈」和「代幣」

**錯誤理解**：所有區塊鏈都有自己的代幣
**正確理解**：區塊鏈是「技術平臺」，代幣是「平臺上的資產」。有些區塊鏈沒有原生代幣（如 Hyperledger）。
