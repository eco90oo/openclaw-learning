# 📚 Day 1：區塊鏈是什麼？比特幣與以太坊的誕生

## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🎯 區塊鏈是什麼？用「共享記事本」來比喻

想像你在一個辦公室裡，有一個**公開的記事本**：

- **每個人都有這個記事本的完整副本**
- **任何人都可以寫新的內容**，但寫完後大家都要確認
- **一旦寫上去，就幾乎不可能擦掉或修改**
- **每個人都可以驗證別人寫的內容是對的**

這就是區塊鏈的核心概念——一個**分散式、不可篡改、共同維護的帳本**。

### 🏠 比喻升級：村莊的「交易記帳本」

讓我們用更生動的比喻：

> 一個村莊裡，大家本來都把錢交給村長管理（傳統銀行），但村長可能會作弊、亂花錢、或破產。
>
> 後來，村民們決定：**不用村長了！我們每個人都拿一本相同的帳本**。
>
> 每次有人借錢或還錢，所有人就一起在帳本上記錄：
> - 「張三借給李四 100 元」
> - 「王五還了 50 元給陳六」
>
> 所有人都確認這筆交易後，才能寫入帳本。而且**一旦寫下去，就永遠留在那裡**，沒有人可以偷偷修改。

### ⛓️ 那「區塊」和「鏈」是什麼？

- **區塊（Block）**：就像帳本的一**頁**，裡面記錄了一段時間內的所有交易
- **鏈（Chain）**：每一頁都**用特殊的鎖環環相扣**，形成一條長鏈
- **如果有人想修改某一頁的內容**，他必須：
  1. 把那一頁撕下來
  2. 但這樣就會**扯斷鎖環**
  3. 然後還要重做所有後面的鎖環
  4. 而且還要說服全村超過一半的人（51% 攻擊）承認他的版本

這就是為什麼區塊鏈**幾乎不可能被篡改**。

### 💡 區塊鏈的三大特性

| 特性 | 比喻 | 說明 |
|------|------|------|
| **去中心化** | 沒有了村長，每個人都是村長 | 不再需要銀行、政府或任何中介機構 |
| **不可篡改** | 寫在石頭上的字 | 一旦確認就很難修改 |
| **透明可驗證** | 村裡每個人都有帳本副本 | 任何人都可以查詢、驗證交易 |

## 2️⃣ 核心知識（詳細教學，包含公式與計算）

### 📜 比特幣的誕生：2008 年的金融危機

**比特幣（Bitcoin, BTC）** 的誕生背景：

- **2008 年**：全球金融海嘯，銀行被認為是罪魁禍首
- **2008 年 10 月 31 日**：一個化名「中本聰（Satoshi Nakamoto）」的人發表了一篇論文
- **2009 年 1 月 3 日**：比特幣第一個區塊（創世區塊）被挖出

> 創世區塊裡，中本聰留下了一句話：「The Times 03/Jan/2009 Chancellor on brink of second bailout for banks」（財政大臣即將第二次銀行紓困）
> 這是對傳統金融系統的諷刺。

### 🎯 比特幣想要解決什麼問題？

傳統貨幣的問題：
1. **中心化**：貨幣由國家和銀行控制
2. **不透明**：你不知道銀行到底印了多少錢
3. **中介費用**：跨國轉帳要透過 Swift，手續費高、速度慢
4. **雙重支付問題**：如何確保同一筆錢没有被花兩次？

比特幣的解決方案：
1. **去中心化**：沒有發行機構，由網路共同維護
2. **固定總量**：永遠只有 2100 萬枚
3. **點對點轉帳**：不需要銀行
4. **共識機制**：透過密碼學和數學確保不會雙重支付

### 以太坊的誕生：比比特幣更「聰明」的區塊鏈

**以太坊（Ethereum）** 的誕生：

- **2013 年**：一個 19 歲的俄羅斯裔加拿大天才 Vitalik Buterin 提出了白皮書
- **2015 年 7 月 30 日**：以太坊正式上線
- **ETH**：以太坊的原生代幣

比特幣 vs 以太坊：

| 特性 | 比特幣 | 以太坊 |
|------|--------|--------|
| **定位** | 數位黃金、儲值貨幣 | 去中心化計算平台 |
| **主要功能** | 轉帳 | 運行「智能合約」 |
| **設計目的** | 取代傳統貨幣 | 取代傳統「合約」和「機構」 |
| **平均區塊時間** | 10 分鐘 | 12-15 秒 |
| **最大供應量** | 2100 萬枚（固定） | 無上限（目前） |

### 🔑 什麼是智能合約？

**智能合約（Smart Contract）** = **自動執行的合約**

比喻：
> 傳統合約就像自動販賣機：
> - 你投入硬幣（trigger）
> - 機器自動給你飲料（執行）
> - 不需要有人在中間操作

智能合約就像：
> - 你設定好條件（如果 A 完成 → 自動給 B 付款）
> - 條件觸發後，程式**自動執行**
> - 沒有人可以反悔或阻攔

## 3️⃣ 實際操作（程式碼範例，每行都要有中文註解）

讓我們用 Python 來建立一個**極度簡化的區塊鏈概念**，幫助理解區塊鏈的運作原理：

```python
import hashlib
import time
import json

# ========================
# 第一部分：區塊結構
# ========================

class Block:
    """
    區塊類別：代表區塊鏈中的「一頁帳本」
    """
    
    def __init__(self, index, timestamp, data, previous_hash):
        """
        初始化區塊
        
        參數:
            index: 區塊編號（第幾頁）
            timestamp: 創建時間
            data: 交易資料（例如：「張三轉給李四 100 元」）
            previous_hash: 前一個區塊的 Hash（就像鎖環）
        """
        self.index = index           # 區塊編號
        self.timestamp = timestamp   # 時間戳記
        self.data = data             # 交易資料（內容）
        self.previous_hash = previous_hash  # 前一個區塊的指紋
        self.hash = self.calculate_hash()  # 自己的 Hash（指紋）
    
    def calculate_hash(self):
        """
        計算這個區塊的 Hash（指紋）
        使用 SHA-256 演算法
        """
        # 將區塊的所有資訊組合成一個字串
        block_string = str(self.index) + str(self.timestamp) + \
                       str(self.data) + str(self.previous_hash)
        
        # 用 SHA-256 產生指紋（固定長度的字串）
        return hashlib.sha256(block_string.encode()).hexdigest()


# ========================
# 第二部分：區塊鏈結構
# ========================

class Blockchain:
    """
    區塊鏈類別：代表整本帳本
    """
    
    def __init__(self):
        """
        初始化區塊鏈：創建第一個區塊（創世區塊）
        """
        self.chain = []  # 用 list 儲存所有區塊
        self.create_genesis_block()  # 創建創世區塊
    
    def create_genesis_block(self):
        """
        創建創世區塊（区款鏈的第一個區塊）
        創世區塊的 previous_hash 為 0
        """
        genesis_block = Block(
            index=0,                    # 第一個區塊，編號是 0
            timestamp=time.time(),      # 現在的時間
            data="Genesis Block - 第一個區塊",  # 初始資料
            previous_hash="0"          # 沒有前一個，所以是 0
        )
        self.chain.append(genesis_block)  # 把創世區塊加入鏈中
    
    def get_latest_block(self):
        """
        取得最新的區塊（鏈的最後一個區塊）
        """
        return self.chain[-1]  # 取出 list 的最後一個元素
    
    def add_block(self, new_block):
        """
        添加新區塊到鏈上
        
        參數:
            new_block: 新的區塊
        """
        # 1. 先取得最新區塊
        previous_block = self.get_latest_block()
        
        # 2. 設定新區塊的前一個 Hash（鎖環相扣）
        new_block.previous_hash = previous_block.hash
        
        # 3. 重新計算新區塊的 Hash
        new_block.hash = new_block.calculate_hash()
        
        # 4. 把新區塊加入鏈中
        self.chain.append(new_block)
    
    def is_valid(self):
        """
        驗證整條區塊鏈是否有效
        檢查每個區塊的 Hash 是否正確、鏈是否斷裂
        """
        # 從第二個區塊開始檢查（跳過創世區塊）
        for i in range(1, len(self.chain)):
            current_block = self.chain[i]      # 目前的區塊
            previous_block = self.chain[i-1]  # 前一個區塊
            
            # 檢查 1：這個區塊的 Hash 是否正確
            if current_block.hash != current_block.calculate_hash():
                print(f"❌ 區塊 {i} 的 Hash 不正確！")
                return False
            
            # 檢查 2：這個區塊的 previous_hash 是否指向前一個區塊
            if current_block.previous_hash != previous_block.hash:
                print(f"❌ 區塊 {i} 的 previous_hash 與前一個區塊不匹配！")
                return False
        
        print("✅ 區塊鏈驗證通過！")
        return True


# ========================
# 第三部分：實際操作範例
# ========================

# 創建一條新的區塊鏈
my_blockchain = Blockchain()

# 添加第一個區塊（模擬第一筆交易）
block1 = Block(
    index=1,
    timestamp=time.time(),
    data="張三轉給李四 1 個比特幣",
    previous_hash="0"  # 暫時設定，稍後會被覆蓋
)
my_blockchain.add_block(block1)

# 添加第二個區塊（模擬第二筆交易）
block2 = Block(
    index=2,
    timestamp=time.time(),
    data="王五轉給趙六 2 個以太坊",
    previous_hash="0"
)
my_blockchain.add_block(block2)

# 添加第三個區塊（模擬第三筆交易）
block3 = Block(
    index=3,
    timestamp=time.time(),
    data="比特幣礦工獲得區塊獎勵 6.25 BTC",
    previous_hash="0"
)
my_blockchain.add_block(block3)

# ========================
# 第四部分：顯示區塊鏈內容
# ========================

print("=" * 60)
print("📜 區塊鏈內容展示")
print("=" * 60)

for block in my_blockchain.chain:
    print(f"""
    ╔══════════════════════════════════════════════════════════╗
    ║ 區塊 #{block.index}
    ╠══════════════════════════════════════════════════════════╣
    ║ 時間戳記: {block.timestamp}
    ║ 資料: {block.data}
    ║ 前一個 Hash: {block.previous_hash[:20]}...  （只顯示前 20 字）
    ║ 本區塊 Hash: {block.hash[:20]}...
    ╚══════════════════════════════════════════════════════════╝
    """)

# 驗證區塊鏈是否有效
my_blockchain.is_valid()

print("\n" + "=" * 60)
print("🎉 恭喜！你已經创建了一個簡單的區塊鏈原型！")
print("=" * 60)
```

## 4️⃣ 常見錯誤提醒

### ❌ 錯誤 1：把區塊鏈想成「資料庫」

很多人以為區塊鏈就像傳統資料庫，可以隨意查詢和修改。
**錯了！** 區塊鏈是**只能寫入（append-only）**的資料結構，
一旦寫入就不能修改或刪除。

### ❌ 錯誤 2：以為區塊鏈「完全匿名」

區塊鏈是**偽匿名（pseudo-anonymous）**，而不是完全匿名。
你的地址是公開的，只是不知道地址背後是誰。
但如果有人把你的地址和你的身份連接起來（例如你在交易所驗證身份），
就可以追查到所有交易。

### ❌ 錯誤 3：忽視「51% 攻擊」的風險

理論上，如果有人控制了超過 51% 的算力（或 stake），
就可以操控整條區塊鏈。比特幣目前是世界上最安全的網路，
但一些小幣種確實有被攻擊的風險。

### ❌ 錯誤 4：以為區塊鏈可以解決一切

區塊鏈不是萬能的！它適合：
- 需要去中心化、可驗證、不可篡改的場景

但不適合：
- 需要高速處理的場景（比特幣每秒 7 筆，以太坊 15-30 筆）
- 需要隱私的場景
