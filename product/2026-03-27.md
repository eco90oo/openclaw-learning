## 🎯 今日主題：產品願景與策略 - 如何制定長期方向

> **如果你不知道要去哪裡，你怎麼知道什麼時候到了？** 這是我在矽谷學到最重要的一課。很多新創公司和產品團隊失敗，不是因為执行力不夠，而是因為一開始就沒想清楚「我要做什麼」。今天我要把我的「願景制定框架」完整分享給你，這是我帶過 5 個從 0 到 1 產品的核心心法。

---

## 1️⃣ 基礎概念：用生活比喻讓你秒懂

### 🔥 什麼是「產品願景」？

**用生活比喻來解釋：**

> 想像你要去旅行：
> - **願景** = 你想要「體驗世界各國的文化」，這是你的終極目標
> - **策略** = 你決定「先亞洲、再歐洲、每年去 3 個國家」，這是你的路徑
> - **戰術** = 你買機票、訂住宿、安排行程，這是你的執行

**在產品開發裡的定義：**

| 術語 | 定義 | 例子 |
|------|------|------|
| **願景 (Vision)** | 產品最終要達成的「理想狀態」 | 「讓每個人都能輕鬆理財」 |
| **策略 (Strategy)** | 達成願景的「整體方向」 | 「從年輕人開始，提供簡單工具」 |
| **戰術 (Tactics)** | 具體的「執行方法」 | 「做一個理財 App，主打存錢功能」 |

### 📊 為什麼願景如此重要？

**資深 PM 的心法**：

> **願景是產品的北極星。** 當團隊方向不一致時，當資源不夠時，當誘惑太多時，願景幫助你做決定。

**沒有願景的團隊會怎樣？**

| 症狀 | 結果 |
|------|------|
| 做了很多功能，但不知道為什麼做 | 產品失去焦點 |
| 團隊成員各自為政 | 方向不一致 |
| 遇到誘惑就改方向 | 資源浪費 |
| 產品上線後不知道成不成功 | 沒有衡量的標準 |

## 2️⃣ 資深 PM 經驗分享：業界實際案例

### 💡 案例一：Spotify 的願景 - 「解鎖人類創造力的潛能」

**背景**：Spotify 從 2006 年創立到現在，成為全球最大的音樂串流平台。

**他們的願景演變**：

| 時間 | 願景 | 策略 |
|------|------|------|
| 2006 | 「讓每個人都能合法聽音樂」 | 從瑞典開始，打擊盜版 |
| 2012 | 「解鎖人類創造力的潛能」 | 擴展到全球，做 Podcast |
| 2020 | 「成為全球音樂的中心」 | 做音樂+Podcast+有聲書 |

**關鍵洞察**：

> Spotify 的願景不是「做一個音樂播放器」，而是**「解決人類的音樂需求」**。這個願景讓他們可以做音箱、做Podcast、做創作工具，因為這些都圍繞著「音樂」這個核心。

### 💡 案例二：Slack 的願景 - 「讓工作更有趣」

**背景**：Slack 從遊戲公司內部工具變成估值 270 億美元的企業通訊平台。

**他們的願景框架**：

```
願景：讓工作更有趣、更有效率
    │
    ├── 策略：從小型團隊切入 → 中型 → 大型企業
    │
    ├── 策略：先做訊息功能 → 加入檔案 → 加入整合
    │
    └── 策略：用免費版吸引 → 付費版變現
```

**獨特之處**：

> Slack 的願景不是「做一個企業通訊工具」，而是**「改變工作方式」**。這個願景讓他們專注在「使用者體驗」和「整合」，而不是和 Email 正面競爭。

### 💡 案例三：我的親身經歷 - 新創公司的願景迷失

**背景**：我曾經待過一家新創公司，做了 18 個月，燒了 500 萬美元，最後失敗。

**問題診斷**：

| 時期 | 問題 | 結果 |
|------|------|------|
| 一開始 | 願景是「做一個給年輕人的理財 App」 | 太模糊 |
| 第 3 個月 | 改成「做一個社交理財平台」 | 加了社群功能 |
| 第 6 個月 | 又改成「做一個區塊鏈理財 App」 | 追熱點 |
| 第 12 個月 | 再改成「做 AI 理財助手」 | 又追風口 |

**失敗原因**：

> 願景變來變去，每次改變都代表之前的開發白費。團隊不知道最終要去哪裡，每天只是「完成任務」。

> 💬 **我的筆記**：這個教訓讓我後來做任何產品，第一件事就是**寫下願景，並且確保團隊每個人都能複述**。

### 💡 案例四：Netflix 的願景演變

**背景**：Netflix 從 DVD 租借變成全球最大串流平台。

**願景演變時間線**：

| 年份 | 願景 | 策略轉變 |
|------|------|----------|
| 1997 | 「讓租 DVD 更方便」 | 郵寄 DVD，不用去門市 |
| 2007 | 「讓隨時看電影成為可能」 | 串流試探 |
| 2013 | 「成為全球娛樂內容創造者」 | 自製內容 |
| 2023 | 「成為全球娛樂的中心」 | 遊戲、直播、體育 |

**關鍵洞察**：

> Netflix 的願景不是「做DVD租借」或「做串流」，而是**「娛樂人類」**。這個願景讓他們可以從 DVD → 串流 → 自製劇 → 遊戲，每次轉型都是圍繞著核心願景。

## 3️⃣ 具體做法/步驟：制定願景與策略的框架

### 📐 Step 1：撰寫產品願景 (Vision Statement)

**我常用的公式**：

```
[目標用戶] 在 [使用情境] 時，能夠 [達成什麼結果]，
而不需要 [過去的痛點]。
```

**範例練習**：

```
範例 1 - Uber：
「 anyone 在 任何地方 需要移動 時，能夠 立刻叫到車，
而不需要 等待或溝通。」

範例 2 - Airbnb：
「 anyone 在 任何城市 想住宿 時，能夠 找到獨特的住處，
而不需要 付昂貴的飯店費用。」

範例 3 - Notion：
「 anyone 在 需要組織資訊 時，能夠 用單一工具管理一切，
而不需要 使用多個軟體。」
```

**撰寫技巧**：

| 技巧 | 說明 | 錯誤示範 |
|------|------|----------|
| **用「能夠」而不是「可以做」** | 結果導向 | ❌ 「提供理財功能」 |
| **具體但不過度細節** | 願景是方向，不是規格 | ❌ 「做一個有 Dark Mode 的 App」 |
| **以用戶為中心** | 從用戶視角出發 | ❌ 「增加營收 30%」 |
| **時間維度 3-5 年** | 願景是長期的 | ❌ 「這季要達到」 |

### 📐 Step 2：制定產品策略 (Product Strategy)

**策略是「如何」達成願景的規劃**：

```
                    ┌─────────────────┐
                    │   產品願景     │
                    │ (3-5 年目標)   │
                    └────────┬────────┘
                             │
            ┌────────────────┼────────────────┐
            │                │                │
     ┌──────▼──────┐  ┌──────▼──────┐  ┌──────▼──────┐
     │  策略一     │  │  策略二     │  │  策略三     │
     │  切入市場  │  │  差異化     │  │  變現模式   │
     └──────┬──────┘  └──────┬──────┘  └──────┬──────┘
            │                │                │
     ┌──────▼──────┐  ┌──────▼──────┐  ┌──────▼──────┐
     │   戰術 1    │  │   戰術 2    │  │   戰術 3    │
     │   功能 A    │  │   功能 B    │  │   功能 C    │
     └─────────────┘  └─────────────┘  └─────────────┘
```

**策略制定框架**：

| 面向 | 問題 | 答案範例 |
|------|------|----------|
| **目標市場** | 我們要服務誰？ | 20-35 歲的年輕專業人士 |
| **價值主張** | 我們提供什麼獨特價值？ | 一站式理財規劃 |
| **競爭定位** | 我們和別人有何不同？ | AI 驅動的個人化建議 |
| **商業模式** | 我們如何賺錢？ | 訂閱制 + 高端服務收費 |
| **核心指標** | 我們如何衡量成功？ | DAU、訂閱轉換率、客戶終身價值 |

### 📐 Step 3：設定階段性目標 (Milestones)

**願景太大，需要拆成可管理的里程碑**：

| 時間範圍 | 目標類型 | 範例 |
|----------|----------|------|
| **0-3 個月** | MVP 目標 | 發布最小可行產品，獲得 1000 用戶 |
| **3-12 個月** | 成長目標 | 月活躍用戶達 10,000，付費轉換 5% |
| **1-2 年** | 規模目標 | 市場佔有率 10%，營收達標 |
| **2-5 年** | 願景目標 | 成為市場領導者，拓展國際市場 |

**設定里程碑的 SMART 原則**：

| 原則 | 說明 | 例子 |
|------|------|------|
| **S**pecific | 具體 | 「獲得 1000 用戶」不是「獲得更多用戶」 |
| **M**easurable | 可衡量 | 有數字可以追蹤 |
| **A**chievable | 可達成 | 合理範圍內 |
| **R**elevant | 相關 | 和願景方向一致 |
| **T**ime-bound | 有時限 | 「3 個月內」 |

### 📐 Step 4：讓團隊對齊願景

**願景不是寫給自己看的，要讓每個人都理解**：

| 做法 | 說明 | 頻率 |
|------|------|------|
| **願景牆** | 團隊每天看的到的地方放願景 | 持續 |
| **入職培训** | 新人加入先講願景和策略 | 入職時 |
| **OKR 對齊** | 每季 OKR 連結到策略目標 | 每季 |
| **策略回顧** | 每月檢視是否偏離方向 | 每月 |

**我的經驗**：

> 每週團隊會議，我一定會問：「這週做的事情，和我們的願景有什麼關係？」如果回答不出來，那就是在做不該做的事。

## 4️⃣ 常見錯誤提醒

### ⚠️ 錯誤一：願景太模糊

**症狀**：願景是「做最好的產品」或「讓用戶滿意」。

**正確做法**：
> 願景要具體到可以被測量。例如「讓年輕人輕鬆存到第一桶金」比「做最好的理財App」更好。

### ⚠️ 錯誤二：願景和策略不一致

**症狀**：願景說「要服務年輕人」，但策略是「做企業市場」。

**正確做法**：
> 策略必須支撐願景。如果策略和願景衝突，要嘛改策略，要嘛改願景。

### ⚠️ 錯誤三：把功能規劃當成策略

**症狀**：策略是「做登入功能」、「做搜尋功能」。

**正確做法**：
> 策略是「方向」，不是「功能清單」。功能是戰術，策略是「為什麼要做這些功能」。

### ⚠️ 錯誤四：願景一成不變

**症狀**：5 年前的願景還在用，市場早就變了。

**正確做法**：
> 願景可以保持 3-5 年，但每年要檢視一次是否還適用。市場變化時，策略要靈活調整。

### ⚠️ 錯誤五：只有 PM 知道願景

**症狀**：只有 PM 寫了願景，其他人不曉得。

**正確做法**：
> 願景要讓團隊每個人都能複述。工程師要知道他們寫的 code 服務於什麼目標。

## 5️⃣ 今日小任務

### ✅ 任務 1：撰寫你產品的願景

用今天的公式：
```
[目標用戶] 在 [使用情境] 時，能夠 [達成什麼結果]，而不需要 [過去的痛點]。
```

寫出 3 個版本，然後選一個最好的。

### ✅ 任務 2：制定你的產品策略

回答這 5 個問題：
1. 目標市場是誰？
2. 我們提供什麼獨特價值？
3. 我們和競品有何不同？
4. 我們如何賺錢？
5. 我們用什麼指標衡量成功？

### ✅ 任務 3：設定 3 個里程碑

用 SMART 原則，設定：
- 3 個月的目標
- 12 個月的目標
- 3 年的目標

### ✅ 任務 4：向團隊或朋友分享你的願景

找一個人，用 1 分鐘解釋你的產品願景。看他能否複述。
