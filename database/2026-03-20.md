## 1️⃣ 基礎概念（用生活比喻）

### 📦 什麼是資料庫？

**比喻：圖書館 vs 隨便堆放**

想象你有兩種存放資料的方式：

| 隨便堆放（檔案） | 資料庫（圖書館） |
|----------------|----------------|
| 紙張隨便堆在桌上 | 書本整齊擺在書架上 |
| 找一本書要翻半天 | 瞬間找到目標書 |
| 容易弄丟、損壞 | 有管理員保護 |
| 一個人用 | 很多人同時用 |

**資料庫**就是一個**有組織的資料存放空間**，讓我們能夠：
- 高效地儲存資料
- 快速地找到資料
- 安全地保護資料
- 多人同時使用

### 🏠 生活中的資料庫

你其實天天都在接觸資料庫：

- **LINE 好友名單** → 通訊錄資料庫
- **Google 地圖** → 地點商家資料庫
- **蝦皮購物** → 商品、訂單資料庫
- **YouTube 影片** → 影片、留言、訂閱者資料庫

### 💾 資料庫 vs 檔案的差別

```
📄 檔案缺點：
- 搜尋慢（要一行一行讀）
- 多人同時寫會衝突
- 沒有權限管理
- 資料重複難管理

✅ 資料庫優點：
- 搜尋快速（索引幫忙）
- 并發控制（多人同時用）
- 權限管理（誰能看、誰能改）
- 資料不重複（正規化）
```

## 2️⃣ 資料庫的組成

### 🏗️ 資料庫架構（由小到大）

```
資料庫（Database）
    ↓
表格（Table）←→ 類似 Excel 的工作表
    ↓
列（Row / Record）←→ 每一筆資料（像是 Excel 的一行）
    ↓
欄位（Column / Field）←→ 資料的屬性（像是 Excel 的欄位標題）
```

### 📊 實際例子：學生資料庫

假設你要做一個「學生管理系統」：

```
📋 表格名稱：students（學生）

| id | name    | age | email              |
|----|---------|-----|-------------------|
| 1  | 小明    | 18  | ming@example.com  |
| 2  | 小華    | 19  | hua@example.com   |
| 3  | 小美    | 17  | mei@example.com   |

↑ 這叫「列」(Row)
        ↑ 這叫「欄位」(Column)
```

### 🔑 重要術語

| 術語 | 英文 | 意思 |
|------|------|------|
| 資料庫 | Database | 存放資料的倉庫 |
| 表格 | Table | 同一類資料的集合 |
| 列 | Row / Record | 一筆完整的資料 |
| 欄位 | Column / Field | 資料的某個屬性 |
| 主鍵 | Primary Key | 唯一識別每筆資料的欄位 |

## 3️⃣ 常見的資料庫類型

### 🗃️ 關聯式資料庫（Relational Database）

**代表：MySQL、PostgreSQL、SQL Server、Oracle**

- 用「表格」形式儲存資料
- 資料之間可以「關聯」
- 使用 **SQL** 語言操作
- 適合需要複雜查詢的交易系統

```
範例：MySQL
┌─────────────┐
│  students  │  ← 表格
│-------------│
│ id│ name   │
│───│────────│
│ 1 │ 小明   │
│ 2 │ 小華   │
└─────────────┘
```

### 📄 文件資料庫（Document Database）

**代表：MongoDB、CouchDB**

- 用「JSON 文件」形式儲存
- 不需要預先定義表格結構
- 適合快速開發、彈性大的應用

```
範例：MongoDB
{
  "name": "小明",
  "age": 18,
  "courses": ["數學", "英文"]
}
```

### ⚡ 鍵值資料庫（Key-Value Database）

**代表：Redis、Memcached**

- 最簡單的形式：Key → Value
- 速度極快
- 適合做快取、session 儲存

```
範例：Redis
key: "user:1" → value: "小明"
key: "user:2" → value: "小華"
```

### 📊 圖形資料庫（Graph Database）

**代表：Neo4j**

- 專門處理「關係」
- 適合社群網路、推薦系統

```
範例：Neo4j
(小明)-[朋友]→(小華)
(小華)-[朋友]→(小美)
```

## 4️⃣ 為什麼要學資料庫？

### 🎯 資料庫是現代應用的核心

```
網站 ←→ 資料庫 ←→ 使用者
App  ←→ 資料庫 ←→ 使用者
AI   ←→ 資料庫 ←→ 資料來源
```

無論你是：
- **前端工程師** → 從 API 拿資料
- **後端工程師** → 設計 API、寫 SQL
- **資料工程師** → 處理大量資料
- **AI/ML 工程師** → 存取訓練資料

**資料庫都是必備技能！**

### 💼 工作市場需求

```
常見面試問題：
- 資料庫設計題
- SQL 查詢題
- 索引與優化題
- 交易與鎖題
- 併發處理題
```

學好資料庫 = 增加競爭力 💪

## 5️⃣ 今日小任務

### ✅ 實作任務

1. **安裝 MySQL（或使用線上環境）**
   - Windows: 下載 MySQL Installer
   - Mac: `brew install mysql`
   - 或使用線上練習：https://www.db4free.org/

2. **建立第一個資料庫**
   ```sql
   CREATE DATABASE my_first_db;
   USE my_first_db;
   ```

3. **建立第一個表格**
   ```sql
   CREATE TABLE students (
       id INT PRIMARY KEY,
       name VARCHAR(50),
       age INT,
       email VARCHAR(100)
   );
   ```

4. **插入一筆資料**
   ```sql
   INSERT INTO students (id, name, age, email)
   VALUES (1, '小明', 18, 'ming@example.com');
   ```

5. **查詢資料**
   ```sql
   SELECT * FROM students;
   ```
