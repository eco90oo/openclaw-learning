## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🔍 什麼是資料庫？

想象一下你是個小餐廳的老闆：

**沒有資料庫的時代（紙本時期）：**
- 你把所有顧客資料寫在紙本表格
- 每次要找某位顧客的電話，得一頁一頁翻
- 如果兩個人同時修改同一頁，就會打架（字跡糊掉）
- 客人一多，管理超級混亂

**有資料庫的時代（數位化）：**
- 所有顧客資料存在電腦裡
- 輸入名字就能立刻搜到電話
- 系統幫你排隊、處理同時來的多筆訂單
- 資料永遠乾淨整齊

> **資料庫 = 電子化的檔案櫃 + 超級聰明的管理員**

### 🗂️ 資料庫的核心組成

```
資料庫 (Database)
    │
    ├── 表格 (Table)     ← 就像是 Excel 的工作表
    │      │
    │      ├── 列 (Row/Record)  ← 每一筆資料（一位顧客）
    │      │
    │      └── 欄 (Column)      ← 資料的屬性（姓名、電話、地址）
    │
    └── 索引 (Index)     ← 像是書本的目錄，加速搜尋
```

### 💡 為什麼要學資料庫？

| 現實問題 | 資料庫解決方案 |
|---------|--------------|
| 資料散落各地很難找 | 統一儲存，快速搜尋 |
| 多人同時修改會衝突 | 交易機制確保資料正確 |
| 資料安全感善風險 | 權限管理 + 備份還原 |
| 網站爆多人會當機 | 連線池 + 效能優化 |

## 2️⃣ SQL 語法/語句解說

### 什麼是 SQL？

**SQL (Structured Query Language)** = 資料庫的「命令語言」
就像我們用中文和電腦溝通，SQL 是和資料庫溝通的語言。

### 常見 SQL 指令分類

| 指令類型 | 功能 | 範例 |
|---------|------|------|
| **SELECT** | 查詢資料 | 找出所有顧客 |
| **INSERT** | 插入資料 | 新增一位顧客 |
| **UPDATE** | 更新資料 | 修改顧客電話 |
| **DELETE** | 刪除資料 | 刪除某位顧客 |
| **CREATE** | 建立表格 | 創建顧客表 |

### 基礎語法示範（今天先看概念，之後會實作）

```sql
-- 查詢所有顧客資料（像是問：「給我所有顧客名單」）
SELECT * FROM customers;

-- 新增一位顧客（像是說：「幫我記錄這位新顧客」）
INSERT INTO customers (name, phone) VALUES ('王小明', '0912-345678');

-- 修改顧客資料（像是說：「把這位顧客的電話改掉」）
UPDATE customers SET phone = '0988-888888' WHERE name = '王小明';

-- 刪除顧客（像是說：「把這位顧客除名」）
DELETE FROM customers WHERE name = '王小明';
```

> **小提醒**：SQL 指令通常用大寫，但小寫也能執行。這是給人類看的「 convention 」。

## 3️⃣ 實際操作（一步一步帶著做）

### 🛠️ 環境準備：我們使用 SQLite（最簡單的資料庫）

SQLite 是一個**不用安裝伺服器**的資料庫，檔案就是一個 `.db` 檔案，非常適合學習！

#### Step 1：開啟終端機，進入 SQLite

```bash
# macOS / Linux
sqlite3 test.db

# Windows (如果沒安裝，可以先跳過，下一篇會教怎麼用 Python 操作)
```

#### Step 2：創建第一張表格

```sql
-- 建立顧客表格
-- CREATE TABLE = 創建表格
-- customers = 表格名稱
CREATE TABLE customers (
    -- id 是編號，用 INTEGER（整數），PRIMARY KEY 表示這是主鍵（唯一的）
    id INTEGER PRIMARY KEY,
    -- name 是姓名，用 TEXT（文字）
    name TEXT NOT NULL,          -- NOT NULL = 不能空白
    -- phone 是電話
    phone TEXT,
    -- created_at 是建立時間
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP  -- 自動產生時間
);
```

> **比喻**：這就像在 Excel 建立一個表格，定義好欄位名稱（id, name, phone, created_at）。

#### Step 3：插入資料

```sql
-- INSERT INTO = 插入資料
-- customers = 表格名稱
-- (name, phone) = 要填哪些欄位
-- VALUES = 填入的值
INSERT INTO customers (name, phone) VALUES ('王小明', '0912-345678');

INSERT INTO customers (name, phone) VALUES ('李美華', '0988-999999');

INSERT INTO customers (name, phone) VALUES ('張志強', '0933-111222');
```

#### Step 4：查詢資料

```sql
-- SELECT * = 查詢所有欄位
-- FROM customers = 從 customers 表格
SELECT * FROM customers;

-- 只查姓名和電話
SELECT name, phone FROM customers;

-- 只查特定條件（id = 1 的顧客）
SELECT * FROM customers WHERE id = 1;
```

#### Step 5：更新資料

```sql
-- UPDATE = 更新
-- SET = 設定新的值
-- WHERE = 條件（只有 id=1 的會被改）
UPDATE customers SET phone = '0900-000000' WHERE id = 1;
```

#### Step 6：刪除資料

```sql
-- DELETE FROM = 刪除
-- WHERE = 條件（只有 id=2 的會被刪）
DELETE FROM customers WHERE id = 2;
```

#### Step 7：查看結果

```sql
-- 最終結果
SELECT * FROM customers;
```

#### Step 8：離開 SQLite

```sql
-- 離開資料庫
.exit
```

## 4️⃣ 常見錯誤/雷點提醒

| 錯誤情境 | 問題 | 解決方式 |
|---------|------|----------|
| 輸入 `SELECT * FROM customers` 沒有結果 | 表格是空的（正常） | 先插入資料 |
| 執行 SQL 時語法錯誤 | 大小寫、括號、逗點錯 | 仔細對照語法 |
| 中文字變成問號 | 編碼問題 | 確認資料庫編碼是 UTF-8 |
| 刪除資料後悔 | 沒有備份 | 先用 SELECT 確認再刪除 |
| WHERE 條件寫錯 | 改到不該改的資料 | 千萬記得加 WHERE！ |

> ⚠️ **重要觀念**：SQL 刪除/更新資料**是永久性的**！不像 Word 可以 Undo，請務必小心！
