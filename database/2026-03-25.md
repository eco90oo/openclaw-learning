## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🎯 什麼是慢查詢？

**比喻：高速公路 vs 小巷子**

想象你要從台北開車到高雄：

| 小巷子（慢查詢） | 高速公路（快查詢） |
|-----------------|-------------------|
| 沿途都是紅綠燈 | 沒有紅綠燈直通 |
| 路小車多會塞車 | 車道多不塞車 |
| 100公里要開3小時 | 100公里只要1小時 |
| 浪費時間又費油 | 快速又省油 |

**慢查詢就是那條「小巷子」—— 資料庫在繞遠路！**

### 🔍 慢查詢怎麼產生的？

**常見原因：**

1. **沒有索引** → 像在圖書館找書沒有分類卡
2. **查詢寫爛** → 像用鏟子挖井不用鑽頭
3. **資料太多** → 像在幾十萬本書裡找一本
4. **關聯太複雜** → 像同時查幾百個表格

### ⏱️ 什麼是「慢」？

- 一般來說：超過 1秒就算慢
- 嚴格標準：超過 100毫秒就要注意
- 支付系統：超過 50毫秒都不行

### 🛠️ EXPLAIN 是什麼？

**比喻：EXPLAIN 就像是「GPS導航」**

它會告訴你：
- 資料庫準備怎麼找資料
- 會用到哪些索引
- 預計要讀多少資料
- 哪裡可能會「塞車」

## 2️⃣ SQL 語法/語句解說

### 語法 1：EXPLAIN - 查看查詢計劃

```sql
EXPLAIN [ANALYZE] [VERBOSE] 你的SQL語句;
```

**參數說明：**
- `EXPLAIN`：只分析，不實際執行（安全）
- `EXPLAIN ANALYZE`：實際執行並測量時間（更準確）
- `EXPLAIN VERBOSE`：顯示更詳細的資訊

### 語法 2：看慢查詢日誌

```sql
-- MySQL 查看慢查詢設定
SHOW VARIABLES LIKE 'slow_query_log%';

-- 查看慢查詢閾值（超過這個秒數算慢）
SHOW VARIABLES LIKE 'long_query_time%';
```

### 語法 3：看正在執行的查詢

```sql
-- MySQL 查看執行中的查詢
SHOW PROCESSLIST;

-- 或者查詢 information_schema
SELECT * FROM information_schema.PROCESSLIST 
WHERE TIME > 5;
```

## 3️⃣ 實際操作（一步一步帶著做）

### 步驟 1：創建測試資料

```python
import sqlite3

# 創建資料庫
conn = sqlite3.connect('shop.db')
cursor = conn.cursor()

# 創建产品表格（沒有索引！）
cursor.execute('''
    CREATE TABLE IF NOT EXISTS products (
        id INTEGER PRIMARY KEY,
        name TEXT,
        category TEXT,
        price DECIMAL(10,2),
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
''')

# 創建訂單表格
cursor.execute('''
    CREATE TABLE IF NOT EXISTS orders (
        id INTEGER PRIMARY KEY,
        product_id INTEGER,
        quantity INTEGER,
        total_price DECIMAL(10,2),
        order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
''')

# 插入測試資料（製造大量數據）
import random
from datetime import datetime, timedelta

categories = ['電子產品', '服裝', '食品', '書籍', '運動']
products_data = []

for i in range(10000):  # 插入 10000 筆產品資料
    products_data.append((
        f'產品{i+1}',
        random.choice(categories),
        round(random.uniform(10, 10000), 2),
        datetime.now() - timedelta(days=random.randint(0, 365))
    ))

cursor.executemany(
    'INSERT INTO products (name, category, price, created_at) VALUES (?, ?, ?, ?)',
    products_data
)

# 插入訂單資料
orders_data = []
for i in range(50000):  # 插入 50000 筆訂單
    orders_data.append((
        random.randint(1, 10000),  # 隨機產品ID
        random.randint(1, 10),
        round(random.uniform(100, 50000), 2),
        datetime.now() - timedelta(days=random.randint(0, 180))
    ))

cursor.executemany(
    'INSERT INTO orders (product_id, quantity, total_price, order_date) VALUES (?, ?, ?, ?)',
    orders_data
)

conn.commit()
print("✅ 測試資料創建完成！")
print(f"產品數量: {cursor.execute('SELECT COUNT(*) FROM products').fetchone()[0]}")
print(f"訂單數量: {cursor.execute('SELECT COUNT(*) FROM orders').fetchone()[0]}")
conn.close()
```

### 步驟 2：分析慢查詢 - 沒有索引的狀況

```python
import sqlite3
import time

conn = sqlite3.connect('shop.db')
cursor = conn.cursor()

# ====== 測試 1：沒有索引的全表掃描 ======
print("\n" + "="*50)
print("測試 1：查詢特定類別的產品（無索引）")
print("="*50)

# 先看看查詢計劃
print("\n📊 EXPLAIN 分析：")
cursor.execute("EXPLAIN QUERY PLAN SELECT * FROM products WHERE category = '電子產品'")
for row in cursor.fetchall():
    print(row)

# 實際執行並測量時間
start = time.time()
cursor.execute("SELECT * FROM products WHERE category = '電子產品'")
results = cursor.fetchall()
end = time.time()

print(f"\n⏱️ 查詢時間: {(end-start)*1000:.2f} 毫秒")
print(f"📦 回傳筆數: {len(results)}")

# ====== 測試 2：使用 LIKE 模糊查詢 ======
print("\n" + "="*50)
print("測試 2：使用 LIKE 模糊查詢（無索引）")
print("="*50)

print("\n📊 EXPLAIN 分析：")
cursor.execute("EXPLAIN QUERY PLAN SELECT * FROM products WHERE name LIKE '%產品1%'")
for row in cursor.fetchall():
    print(row)

start = time.time()
cursor.execute("SELECT * FROM products WHERE name LIKE '%產品1%'")
results = cursor.fetchall()
end = time.time()

print(f"\n⏱️ 查詢時間: {(end-start)*1000:.2f} 毫秒")
print(f"📦 回傳筆數: {len(results)}")

# ====== 測試 3：JOIN 查詢 ======
print("\n" + "="*50)
print("測試 3：訂單關聯查詢（無索引）")
print("="*50)

print("\n📊 EXPLAIN 分析：")
cursor.execute("""
    EXPLAIN QUERY PLAN 
    SELECT p.name, o.quantity, o.total_price 
    FROM products p 
    JOIN orders o ON p.id = o.product_id 
    WHERE p.category = '電子產品'
""")
for row in cursor.fetchall():
    print(row)

start = time.time()
cursor.execute("""
    SELECT p.name, o.quantity, o.total_price 
    FROM products p 
    JOIN orders o ON p.id = o.product_id 
    WHERE p.category = '電子產品'
    LIMIT 100
""")
results = cursor.fetchall()
end = time.time()

print(f"\n⏱️ 查詢時間: {(end-start)*1000:.2f} 毫秒")
print(f"📦 回傳筆數: {len(results)}")

conn.close()
```

### 步驟 4：解讀 EXPLAIN 結果

```
EXPLAIN 輸出重點解讀：

┌─────────────────────────────────────────────────────────────┐
│  type（查詢類型）- 越前面越好                                │
├─────────────────────────────────────────────────────────────┤
│  const  >  eq_ref  >  ref  >  range  >  index  >  ALL      │
│   ⭐最好        次好        普通      尚可      差         │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  key（使用的索引）- 看這裡用什麼索引                        │
├─────────────────────────────────────────────────────────────┤
│  NULL = 沒有使用索引（全表掃描！）                          │
│  idx_xxx = 使用了這個索引                                  │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  rows（預計掃描的行數）- 越少越好                           │
├─────────────────────────────────────────────────────────────┤
│  rows = 10000 表示要掃描 1 萬行                             │
│  rows = 10 表示只掃描 10 行 ⭐                              │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  Extra（額外資訊）- 注意這些警示                            │
├─────────────────────────────────────────────────────────────┤
│  Using filesort = 需要額外排序（慢！）                     │
│  Using temporary = 需要暫時表格（慢！）                     │
│  Using index = 使用覆蓋索引（快！）                         │
│  Using where = 需要過濾資料                                 │
└─────────────────────────────────────────────────────────────┘
```

### 步驟 5：優化 SQL 語句

```python
import sqlite3

conn = sqlite3.connect('shop.db')
cursor = conn.cursor()

# ====== 壞寫法 vs 好寫法 ======
print("\n" + "="*50)
print("SQL 優化實戰")
print("="*50)

# ❌ 壞寫法 1：SELECT *
print("\n❌ 壞寫法 1：SELECT *")
cursor.execute("EXPLAIN QUERY PLAN SELECT * FROM products")
for row in cursor.fetchall():
    print(row)
print("→ 問題：取所有欄位，傳輸資料量大")

# ✅ 好寫法：只要需要的欄位
print("\n✅ 好寫法：只要需要的欄位")
cursor.execute("EXPLAIN QUERY PLAN SELECT id, name, price FROM products")
for row in cursor.fetchall():
    print(row)
print("→ 減少傳輸量，使用覆蓋索引可能更快")

# ❌ 壞寫法 2：LIKE %開頭
print("\n❌ 壞寫法 2：LIKE '%xxx%'（前後都有%）")
cursor.execute("EXPLAIN QUERY PLAN SELECT * FROM products WHERE name LIKE '%產品1%'")
for row in cursor.fetchall():
    print(row)
print("→ 問題：無法使用索引，全表掃描")

# ✅ 好寫法：LIKE 開頭
print("\n✅ 好寫法：LIKE 'xxx%'（只有前綴%）")
cursor.execute("EXPLAIN QUERY PLAN SELECT * FROM products WHERE name LIKE '產品1%'")
for row in cursor.fetchall():
    print(row)
print("→ 可以使用索引（前綴匹配）")

# ❌ 壞寫法 3：OR 條件
print("\n❌ 壞寫法 3：使用 OR")
cursor.execute("EXPLAIN QUERY PLAN SELECT * FROM products WHERE category = '電子產品' OR category = '服裝'")
for row in cursor.fetchall():
    print(row)
print("→ 問題：可能變成全表掃描")

# ✅ 好寫法：IN 條件
print("\n✅ 好寫法：使用 IN")
cursor.execute("EXPLAIN QUERY PLAN SELECT * FROM products WHERE category IN ('電子產品', '服裝')")
for row in cursor.fetchall():
    print(row)
print("→ 更好，可以利用索引")

# ❌ 壞寫法 4：函數運算
print("\n❌ 壞寫法 4：對欄位使用函數")
cursor.execute("EXPLAIN QUERY PLAN SELECT * FROM products WHERE UPPER(category) = '電子產品'")
for row in cursor.fetchall():
    print(row)
print("→ 問題：無法使用索引")

# ✅ 好寫法：直接比對
print("\n✅ 好寫法：直接比對")
cursor.execute("EXPLAIN QUERY PLAN SELECT * FROM products WHERE category = '電子產品'")
for row in cursor.fetchall():
    print(row)
print("→ 可以使用索引")

conn.close()
```

## 4️⃣ 常見錯誤/雷點提醒

| 錯誤 | 解決方法 |
|------|----------|
| EXPLAIN 顯示 ALL（全表掃描） | 檢查是否缺少索引 |
| rows 太大（上百萬） | 考慮加索引或分頁 |
| Using filesort 出現 | 避免 ORDER BY 非索引欄位 |
| Using temporary 出現 | 優化 GROUP BY/ORDER BY |
| LIKE %開頭無法用索引 | 考慮全文索引或 Elasticsearch |
| OR 條件變全表掃描 | 改用 IN 或 UNION |

### 🚨 性能殺手排行榜

1. **SELECT *** → 傳輸過多不需要的資料
2. **沒有索引 + WHERE** → 全表掃描
3. **JOIN 沒有索引** → 災難級慢
4. **子查詢濫用** → 重複執行
5. **分頁沒用 LIMIT** → 取幾十萬筆
