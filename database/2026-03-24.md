# 📚 【今日學習：Day 1 - 資料庫基礎（什麼是資料庫、為什麼要學）】

## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🎯 什麼是資料庫？

**比喻：圖書館 vs 雜物間**

想象你有兩種存放資料的方式：

| 雜物間（沒結構） | 圖書館（有結構） |
|-----------------|------------------|
| 書本隨便堆在地上 | 書本整齊排列在書架上 |
| 找書要翻遍整個房間 | 根據分類標籤很快找到 |
| 書很容易損壞、遺失 | 書本被妥善保護管理 |
| 一個人找書都困難 | 多人同時借書有系統 |

**資料庫（Database）就是那個「圖書館系統」！**

它幫你：
- 📦 **儲存資料** - 像圖書館藏書
- 🔍 **快速查找** - 像圖書館的索引卡片
- ✏️ **修改更新** - 像圖書館更新書籍資訊
- 🗑️ **刪除資料** - 像圖書館移除舊書
- 👥 **多人共用** - 像圖書館服務多位讀者

### 💡 為什麼要學資料庫？

**如果你不學資料庫...**

想象你要管理一家網店的訂單：
```
❌ 沒有資料庫：
- 用 Excel 表格存訂單（幾千筆就卡死）
- 客戶電話改了要找半天
- 兩個人同時修改會亂掉
- 系統突然壞了資料全不見
```

```
✅ 有資料庫：
- 百萬筆訂單瞬間查詢
- 客戶資料一次更新全部同步
- 多人同時操作不出錯
- 自動備份不怕資料遺失
```

**現實中：**
- 淘寶、京東背後都是資料庫
- 你的銀行存款、醫院病歷、學校成績單...
- 所有「需要長期儲存+多人共用+快速查詢」的系統都用資料庫

### 🔧 常見的資料庫有哪些？

| 資料庫類型 | 代表產品 | 比喻 |
|-----------|---------|------|
| 關聯式資料庫 | MySQL, PostgreSQL, SQL Server | 就像有標準格式的表格一本本 |
| 文件資料庫 | MongoDB | 就像雜誌攤，什麼格式都可能 |
| 鍵值資料庫 | Redis | 就像字典，查單字對應意思 |
| 圖形資料庫 | Neo4j | 就像人際關係圖 |

### 🏗️ 資料庫的組成結構

```
┌─────────────────────────────────────────┐
│           資料庫（Database）             │
│  ┌─────────────────────────────────┐    │
│  │     表格（Table）- 像Excel表    │    │
│  │  ┌─────────────────────────┐   │    │
│  │  │ 欄位（Column）: ID       │   │    │
│  │  │ 欄位（Column）: 名稱     │   │    │
│  │  │ 欄位（Column）: 價格     │   │    │
│  │  │         ↓                │   │    │
│  │  │   列（Row/Record）       │   │    │
│  │  │   列（Row/Record）       │   │    │
│  │  │   列（Row/Record）       │   │    │
│  │  └─────────────────────────┘   │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

### 🔑 核心關鍵字小抄

- **DBMS**（Database Management System）= 資料庫管理系統＝圖書館管理員
- **SQL**（Structured Query Language）= 資料庫的語言＝跟圖書館溝通的話
- **Table** = 表格 = 圖書館的一個分類書架
- **Row** = 資料列 = 一本書
- **Column** = 欄位 = 書的屬性（作者、書名、出版年）

## 3️⃣ 實際操作（一步一步帶著做）

### 環境準備：我們用 SQLite（最簡單的資料庫）

SQLite 不用安裝伺服器，是一個檔案就能運作的資料庫，非常適合初學者！

#### Windows 安裝 SQLite：

1. 去官方網站下載：`https://www.sqlite.org/download.html`
2. 下載 `sqlite-tools-win-x64-xxxx.zip`
3. 解壓縮後，把資料夾路徑加入系統 PATH
4. 打開 CMD，輸入 `sqlite3` 測試

#### macOS（已內建）：

打開 Terminal，直接輸入：
```bash
sqlite3
```

#### Linux（Ubuntu/Debian）：
```bash
sudo apt install sqlite3
```

### 🚀 第一次操作：創建你的第一個資料庫！

```bash
# 啟動 SQLite（進入資料庫世界）
sqlite3 my_first_db.sqlite

# 創建一個表格（產品資料表）
CREATE TABLE products (
    id INTEGER PRIMARY KEY,      -- 產品編號（主鍵，像身份證號）
    name TEXT,                    -- 產品名稱
    price DECIMAL(10,2),          -- 產品價格
    stock INTEGER                 -- 庫存數量
);

# 插入一筆資料（新增一本書）
INSERT INTO products (name, price, stock) 
VALUES ('iPhone 15', 29900, 100);

# 插入更多資料
INSERT INTO products (name, price, stock) 
VALUES ('MacBook Pro', 59900, 50);

INSERT INTO products (name, price, stock) 
VALUES ('AirPods Pro', 7500, 200);

# 查詢所有資料（看看圖書館裡有什麼書）
SELECT * FROM products;

# 查詢特定資料（找特定書）
SELECT * FROM products WHERE price > 10000;

# 更新資料（修改書的內容）
UPDATE products SET stock = 99 WHERE id = 1;

# 刪除資料（移除書）
DELETE FROM products WHERE id = 3;

# 離開資料庫
.quit
```

### 💻 在 Python 中使用 SQLite（更實用！）

```python
import sqlite3

# 連接資料庫（如果不存在會自動創建）
conn = sqlite3.connect('shop.db')
cursor = conn.cursor()

# 創建表格
cursor.execute('''
    CREATE TABLE IF NOT EXISTS products (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        price REAL NOT NULL,
        stock INTEGER DEFAULT 0
    )
''')

# 插入資料
cursor.execute(
    "INSERT INTO products (name, price, stock) VALUES (?, ?, ?)",
    ('iPhone 15', 29900, 100)
)

# 提交変更
conn.commit()

# 查詢資料
cursor.execute("SELECT * FROM products")
rows = cursor.fetchall()
for row in rows:
    print(f"ID: {row[0]}, 名稱: {row[1]}, 價格: {row[2]}, 庫存: {row[3]}")

# 關閉連線
conn.close()

print("✅ 資料庫操作完成！")
```

## 4️⃣ 常見錯誤/雷點提醒

| 錯誤 | 解決方法 |
|------|----------|
| 資料庫檔案找不到 | 檢查路徑是否正確 |
| 表格已經存在 | 使用 `CREATE TABLE IF NOT EXISTS` |
| 插入資料失敗 | 檢查欄位類型是否正確 |
| 中文變成亂碼 | 確認使用 UTF-8 編碼 |
| 忘記 commit | 每次修改後要記得 `.commit()` |
