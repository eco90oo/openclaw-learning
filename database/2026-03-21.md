# 📚 Day 1：資料庫基礎（什麼是資料庫、為什麼要學）

---

## 1️⃣ 基礎概念（用生活比喻）

### 什麼是資料庫？

想像一下：
- ** Excel 是什麼？** 就是一張大紙，上面寫滿了各種資料
- ** 資料庫是什麼？** 就是一個**超級智慧的文件櫃**，會幫你管理、搜尋、保护你的資料

### 生活中的比喻

| 現實生活 | 資料庫世界 |
|---------|-----------|
| 文件櫃 | 資料庫（Database） |
| 抽屜 | 表格（Table） |
| 紙上的表格欄位 | 欄位（Column） |
| 每張紙的一行資料 | 資料列（Row/Record） |
| 查找文件 | 查詢（Query） |
| 文件管理員 | DBMS（資料庫管理系統） |

### 為什麼不用 Excel 要用資料庫？

**Excel 的問題：**
- 同時很多人開，檔案會壞掉（併發問題）
- 資料幾萬筆就變慢了
- 搜尋條件複雜一點就卡住
- 沒有辦法保護資料安全（誰都能改）
- 沒有辦法做複雜的關聯（比如：找出「買過 A 商品的顧客，他們還買過什麼」）

**資料庫的好處：**
- ✅ 可以很多人同時使用
- ✅ 資料幾百萬筆也很快
- ✅ 搜尋、篩選功能強大
- ✅ 可以設定誰能看、誰能改
- ✅ 可以做複雜的資料分析
- ✅ 資料不會輕易不見（有備份機制）

### 常見的資料庫種類

```
📦 關聯式資料庫（Relational Database）
   ├── MySQL（最流行，免費）
   ├── PostgreSQL（功能最強大）
   ├── Oracle（企業級，貴）
   └── SQLite（輕量，內嵌式）

📦 NoSQL 資料庫（不是傳統表格）
   ├── MongoDB（文件資料庫）
   ├── Redis（鍵值資料庫）
   └── Cassandra（寬欄位資料庫）
```

**初學者建議：** 從 MySQL 或 SQLite 開始！

## 2️⃣ 術語大補帖

在繼續之前，先認識一下這些常用詞：

| 術語 | 意思是... |
|------|----------|
| DB | 資料庫（Database） |
| DBMS | 資料庫管理系統（幫你管資料庫的軟體） |
| Table | 表格（像 Excel 的工作表） |
| Row | 資料列（表格的一行） |
| Column | 欄位（表格的一欄） |
| Query | 查詢指令（要資料庫做事的命令） |
| SQL | 查詢語言（和資料庫溝通的語言） |

## 3️⃣ 實際操作（一步一步帶著做）

### 安裝 SQLite（最簡單的資料庫）

SQLite 超級輕量，不用安裝伺服器，直接就是一個檔案。最適合初學者！

**在電腦上安裝：**

```bash
# macOS
brew install sqlite

# Ubuntu/Debian
sudo apt-get install sqlite3

# Windows
# 去官網下載：https://www.sqlite.org/download.html
```

**確認安裝成功：**
```bash
sqlite3 --version
```

### 創建你的第一個資料庫

```bash
# 創建資料庫（就像創建一個新文件夾）
sqlite3 my_first_database.db
```

你會看到這樣的畫面：
```
SQLite version 3.xx.x
Enter ".help" for instructions.
Enter SQL statements terminated with a ";"
sqlite>
```

### 創建第一張表格

我們來創建一個「學生資料表」：

```sql
CREATE TABLE students (
    id INTEGER PRIMARY KEY,           -- 學生ID（主鍵）
    name TEXT NOT NULL,               -- 姓名
    age INTEGER,                      -- 年齡
    email TEXT                        -- 電子郵件
);
```

**解說：**
- `CREATE TABLE` = 創建表格
- `students` = 表格名字
- `id INTEGER PRIMARY KEY` = ID 是數字，而且是主鍵（唯一不能重複）
- `TEXT NOT NULL` = 文字類型，不能是空的

### 插入資料

```sql
INSERT INTO students (name, age, email) VALUES ('王小明', 20, 'wang@example.com');
INSERT INTO students (name, age, email) VALUES ('李美華', 22, 'lee@example.com');
INSERT INTO students (name, age, email) VALUES ('張志強', 19, 'zhang@example.com');
```

### 查詢資料

```sql
-- 查詢所有學生
SELECT * FROM students;

-- 查詢特定欄位
SELECT name, email FROM students;

-- 條件查詢
SELECT * FROM students WHERE age > 20;
```

### 常用 SQLite 命令

```sql
-- 看到所有表格
.tables

-- 看到表格的結構
.schema students

-- 離開 SQLite
.exit
```

## 4️⃣ 常見錯誤/雷點提醒

### ⚠️ 新手常犯的錯

1. **忘記加分號**
   ```sql
   -- 錯的 ❌
   SELECT * FROM students

   -- 對的 ✅
   SELECT * FROM students;
   ```

2. **表格名稱用中文（初期不建議）**
   ```sql
   -- 可以但麻煩 ❌
   CREATE TABLE 學生資料表 (...)

   -- 建議用英文 ✅
   CREATE TABLE students (...);
   ```

3. **SQL 關鍵字大小寫**
   - SQL 不分大小寫，但**慣例**是大寫
   ```sql
   -- 都可以執行，但建議這樣寫 ✅
   SELECT * FROM students WHERE age > 20;
   ```

4. **中文字編碼問題**
   - 如果顯示乱码，記得加：
   ```sql
   .charset utf8
   ```

5. **以為 SELECT 會改變資料**
   - SELECT 只是「讀取」，不會刪除或修改資料
   - 要改變資料需要用 INSERT、UPDATE、DELETE

## 💡 小知識

**SQL 怎麼念？**
- 官方發音：「S-Q-L」（三個字母分開讀）
- 有些人讀「sequel」
- 兩種都可以，聽得懂最重要！

**資料庫 vs 資料表：**
- 資料庫 = 整個文件櫃
- 資料表 = 櫃子裡的一個抽屜
- 一個資料庫可以有很多張表格！
