## 1️⃣ 基礎概念

### 📝 什麼是 SQL？

**SQL（Structured Query Language，結構化查詢語言）** 是用來操作資料庫的標準語言。

```
比喻：SQL 就像是資料庫的「命令提示字元」

你對資料庫說話 → 資料庫聽懂 → 執行任務 → 回覆結果
```

### 🔄 CRUD - 資料庫的四大天王

```
C reate（新增）→ INSERT 新增資料
R ead（讀取）→ SELECT 查詢資料
U pdate（修改）→ UPDATE 修改資料
D elete（刪除）→ DELETE 刪除資料
```

**口號：沒有 CRUD，就沒有資料庫！** 📣

## 2️⃣ 實際操作

### 📋 事前準備：建立練習環境

如果還沒建立資料庫，先執行以下指令：

```sql

CREATE TABLE students (
    id INT PRIMARY KEY AUTO_INCREMENT,

```

### 📖 R - Read（查詢資料）

#### 基本查詢：SELECT

```sql
-- 查詢所有資料（* 代表所有欄位）
SELECT * FROM students;

-- 查詢特定欄位
SELECT name, email FROM students;

-- 給欄位取暱稱（as）
SELECT name AS '姓名', email AS '信箱' FROM students;
```

#### 篩選資料：WHERE

```sql
-- 找出年齡大於 18 歲的學生
SELECT * FROM students WHERE age > 18;

-- 找出住在台北的學生
SELECT * FROM students WHERE city = '台北';

-- 多重條件： AND（並且）
SELECT * FROM students WHERE city = '台北' AND age > 17;

-- 多重條件： OR（或者）
SELECT * FROM students WHERE city = '高雄' OR city = '台中';

-- 找出年齡在 18-20 歲之間
SELECT * FROM students WHERE age BETWEEN 18 AND 20;
```

#### 模糊搜尋：LIKE

```sql
-- 名字以「小」開頭
SELECT * FROM students WHERE name LIKE '小%';

-- 名字包含「強」
SELECT * FROM students WHERE name LIKE '%強%';

-- 電子郵件以 @school.com 結尾
SELECT * FROM students WHERE email LIKE '%@school.com';
```

> **💡 提示**：`%` 代表「任意字元」，`_` 代表「單一字元」

#### 排序資料：ORDER BY

```sql
-- 按年齡從小到大排序（預設 ASC）
SELECT * FROM students ORDER BY age;

-- 按年齡從大到小排序（DESC）
SELECT * FROM students ORDER BY age DESC;

-- 先按城市排序，再按年齡排序
SELECT * FROM students ORDER BY city, age DESC;
```

#### 限制筆數：LIMIT

```sql
-- 只查詢前 3 筆資料
SELECT * FROM students LIMIT 3;

-- 跳過前 2 筆，查詢接下來的 3 筆（分頁用）
SELECT * FROM students LIMIT 2, 3;

-- 查詢最年長的學生
SELECT * FROM students ORDER BY age DESC LIMIT 1;
```

#### 聚合函數：COUNT, SUM, AVG, MAX, MIN

```sql
-- 計算總人數
SELECT COUNT(*) FROM students;

-- 計算平均年齡
SELECT AVG(age) FROM students;

-- 找出最大年齡
SELECT MAX(age) FROM students;

-- 找出最小年齡
SELECT MIN(age) FROM students;

-- 按城市分組統計人數
SELECT city, COUNT(*) as '人數' FROM students GROUP BY city;
```

#### DISTINCT - 去除重複

```sql
-- 找出所有不重複的城市
SELECT DISTINCT city FROM students;
```

### ➕ C - Create（新增資料）

#### INSERT - 插入單筆資料

```sql
-- 指定所有欄位
INSERT INTO students (name, age, email, city)
VALUES ('小龍', 21, 'lung@school.com', '台北');

-- 只指定必要欄位（其他會用預設值）
INSERT INTO students (name, email)
VALUES ('小虎', 'hu@school.com');
```

#### INSERT - 一次插入多筆

```sql
INSERT INTO students (name, age, email, city) VALUES
('小兔子', 19, 'rabbit@school.com', '高雄'),
('小狗狗', 18, 'dog@school.com', '台北'),
('小貓咪', 20, 'cat@school.com', '台中');
```

---

### ✏️ U - Update（修改資料）

#### UPDATE - 更新資料

```sql
-- 修改單筆資料（一定要加 WHERE！）
UPDATE students
SET age = 19
WHERE id = 1;

-- 修改多個欄位
UPDATE students
SET age = 20, city = '新北市'
WHERE name = '小明';

-- 符合條件的都修改（小心！沒有 WHERE 會改全部）
UPDATE students
SET city = '台北市';
```

> ⚠️ **重要守則**：UPDATE 和 DELETE 一定要加 WHERE 條件，否則會改壞或刪掉所有資料！

---

### 🗑️ D - Delete（刪除資料）

#### DELETE - 刪除資料

```sql
-- 刪除特定資料
DELETE FROM students WHERE id = 5;

-- 刪除符合條件的資料
DELETE FROM students WHERE city = '台中';

-- 刪除所有資料（小心！）
DELETE FROM students;
```

> 💡 **建議**：正式環境刪除資料前，先用 SELECT 確認範圍對不對！

## ❓ 常見問題

**Q: SQL 語法大小寫有差嗎？**
A: 資料庫通常不區分大小寫，但建議關鍵字用大寫（SELECT、FROM），欄位名用小寫，這是業界慣例。

**Q: 刪除的資料能復原嗎？**
A: 大部分情況下，DELETE 後就沒了！所以正式環境請先用 BEGIN TRANSACTION 或確認影響筆數。

**Q: WHERE 條件那麼多，要怎麼記？**
A: 不用死記！記住邏輯：篩選（WHERE）→ 排序（ORDER BY）→ 限制（LIMIT），多練習就會了。
