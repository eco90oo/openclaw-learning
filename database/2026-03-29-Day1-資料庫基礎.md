# 📚 【今日學習：Day 1 - 資料庫基礎（什麼是資料庫、為什麼要學）】

## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🎯 什麼是資料庫？

想象一下，你有一個超級大的玩具箱，這個玩具箱有以下特點：

| 玩具箱特性 | 資料庫對應概念 | 生活比喻 |
|-----------|--------------|---------|
| 儲存空間 | 資料庫（Database） | 一個有組織的大倉庫 |
| 玩具格子 | 資料表（Table） | 倉庫裡的分類架子 |
| 每個玩具 | 資料列（Row/Record） | 架子上的具体玩具 |
| 玩具屬性 | 資料欄位（Column） | 玩具的名稱、顏色、大小等 |

### 🏠 更具體的比喻：圖書館

想象你要去圖書館借書：

- **圖書館** = 資料庫（整個系統）
- **書籍分類區** = 資料表（例如：電腦書區、小說區）
- **每本書** = 一筆資料（Row）
- **書的資訊** = 欄位（書名、作者、出版日期、ISBN）

```
📚 圖書館（資料庫）
├── 📖 電腦書區（資料表：books）
│   ├── 第1本書：{書名:"Python入門", 作者:"張三", 頁數:300}
│   ├── 第2本書：{書名:"資料庫設計", 作者:"李四", 頁數:450}
│   └── ...
├── 📕 小說區（資料表：novels）
│   └── ...
└── 📗 雜誌區（資料表：magazines）
    └── ...
```

### 💡 為什麼我們需要資料庫？

在沒有資料庫之前，人類是怎麼儲存資料的？

#### 時代1️⃣：紙本時代
```
📝 紙本紀錄
缺點：
- 難以搜尋（要找一筆資料可能要翻半天）
- 資料重複（很多人都有相同的筆記）
- 不易修改（涂改容易出錯）
- 不易分享（一人一份，版本不同步）
```

####時代2️⃣：Excel/試算表
```
📊 Excel
優點：
- 電腦儲存，搜尋方便
- 可以排序、篩選

缺點：
- 資料量大時很慢（超過100萬筆）
- 多人同時編輯會衝突
- 沒有權限控制（誰都能修改）
- 沒有保護機制（誤刪救不回來）
```

#### 時代3️⃣：資料庫（現代）
```
🗄️ 資料庫
優點：
- 處理海量資料（上百萬筆沒問題）
- 多人同時使用（併發控制）
- 資料安全（權限管理、備份還原）
- 資料一致性（不會出現矛盾）
- 高效能查詢（索引加速）
```

### 🎓 資料庫管理器（DBMS）

資料庫本身只是「資料的集合」，你需要一個「軟體」來管理它，這個軟體叫做 **DBMS（Database Management System）**。

常見的 DBMS 有：
- **MySQL** - 免費开源，最多人使用 🌟
- **PostgreSQL** - 功能強大，高級特性多
- **Oracle** - 企業級，付費，大型系統用
- **SQL Server** - 微軟出品，企業用
- **SQLite** - 輕量級，嵌入式（手機App常用）

```
┌─────────────────────────────────────┐
│           應用程式                   │
│    （你的網站、App、Python程式）       │
└─────────────────┬───────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│         DBMS（資料庫管理系統）         │
│    （MySQL、PostgreSQL、Oracle）      │
└─────────────────┬───────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│          資料庫（實際的檔案）          │
│    （data.mdb、data.ibd 等）         │
└─────────────────────────────────────┘
```

### 🔑 重點複習

| 術語 | 意思 | 比喻 |
|-----|-----|-----|
| DBMS | 資料庫管理軟體 | 圖書館管理員 |
| Database | 資料庫（整個系統） | 整個圖書館 |
| Table | 資料表（表格） | 圖書分類區 |
| Row | 資料列（一筆資料） | 一本書 |
| Column | 資料欄位（欄） | 書的屬性（書名、作者） |
| SQL | 查詢語言 | 跟管理員說話的語言 |

## 2️⃣ SQL 語法/語句解說

SQL（Structured Query Language，結構化查詢語言）就是我們跟資料庫「溝通」的語言。

### 第一個 SQL：建立資料庫

```sql
-- 建立一個叫做 shop_database 的資料庫
-- CREATE DATABASE 是創建資料庫的指令
CREATE DATABASE shop_database;
```

```sql
-- 切換到要使用的資料庫
-- USE 就像打開圖書館的某個分區
USE shop_database;
```

### 查看資料庫

```sql
-- 顯示所有資料庫
-- 就像環顧整個圖書館有哪些區域
SHOW DATABASES;
```

## 3️⃣ 實際操作（一步一步帶著做）

### 步驟1️⃣：安裝 MySQL（可選，本地練習用）

如果你在 Linux 伺服器上，可以安裝 MySQL：

```bash
# Ubuntu/Debian 安裝 MySQL
sudo apt update
sudo apt install mysql-server

# 啟動 MySQL 服務
sudo systemctl start mysql
sudo systemctl enable mysql
```

### 步驟2️⃣：連線到 MySQL

```bash
# 使用 root 帳號登入（本地測試用）
mysql -u root -p
```

### 步驟3️⃣：執行我們的第一個 SQL

登入後，輸入以下指令：

```sql
-- 建立資料庫
CREATE DATABASE demo_db;

-- 使用這個資料庫
USE demo_db;

-- 顯示所有資料庫，確認建立成功
SHOW DATABASES;
```

你應該會看到：
```
+--------------------+
| Database           |
+--------------------+
| information_schema |
| demo_db            |
| mysql              |
| performance_schema |
| sys                |
+--------------------+
```

恭喜！你已經建立了第一個資料庫！🎉

## 4️⃣ 常見錯誤/雷點提醒

### ❌ 錯誤1：語法順序錯誤

```sql
-- 錯誤示範：先USE再CREATE
USE demo_db;  -- 錯誤！資料庫還沒創建
CREATE DATABASE demo_db;
```

✅ **正確順序**：先 CREATE，再 USE

### ❌ 錯誤2：名稱使用特殊字元

```sql
-- 錯誤！資料庫名稱不能有空格或特殊符號
CREATE DATABASE my database;  -- ❌

-- 正確：使用底線
CREATE DATABASE my_database;  -- ✅
```

### ❌ 錯誤3：分號忘了加

```sql
-- SQL 語句要以分號結尾
SHOW DATABASES  -- ❌ 沒分號

SHOW DATABASES; -- ✅ 有分號
```

### ❌ 錯誤4：大小寫問題

```sql
-- MySQL 在 Linux 上區分大小寫
-- 建立 test_db 和 Test_DB 是不同的！

CREATE DATABASE test_db;
CREATE DATABASE Test_DB;  -- 這會建立另一個資料庫，不是錯誤！
```

> 💡 **建議**：雖然 MySQL 對關鍵字不區分大小寫，但資料庫/表格名稱在不同作業系統有不同行為。建議養成**統一小寫**的習慣。

## 5️⃣ 今日小任務

### 🎯 任務1：建立你的第一個資料庫

```sql
-- 建立一個名為 online_shop 的資料庫（代表網路商店）
CREATE DATABASE online_shop;

-- 使用它
USE online_shop;
```
