# 📚 【今日學習：Day 8 - 交易基礎（Transaction）是什麼？】

## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🎯 什麼是交易（Transaction）？

**比喻：銀行轉帳**

想象你要轉帳 1000 元給朋友：

```
你帳戶：原本有 5000 元
朋友帳戶：原本有 3000 元

你要轉帳 1000 元，過程是：
1. 從你的帳戶扣掉 1000 元 → 你剩下 4000 元
2. 加上 1000 元到朋友的帳戶 → 朋友剩下 4000 元
```

**問題來了！** 如果步骤 1 成功，但步骤 2 失敗了會怎樣？
- 你的帳戶少了 1000 元
- 朋友的帳戶沒變
- **你的 1000 元就這樣消失了！**

這就是為什麼需要「交易」機制——**要嘛全部成功，要嘛全部失敗！**

### 🔄 交易的四大特性（ACID）

交易必須保證四件事（我們明天會詳細講，今天先簡單介紹）：

| 特性 | 比喻 | 說明 |
|------|------|------|
| **Atomic（原子性）** | 「全有或全無」 | 要嘛全部成功，要嘛全部失敗 |
| **Consistency（一致性）** | 「收支平衡」 | 轉帳前後，總金額不變 |
| **Isolation（隔離性）** | 「排隊辦理」 | 同時很多人轉帳也不會亂 |
| **Durability（持久性）** | 「寫入存摺」 | 交易完成後永不消失 |

### 📊 生活中的交易例子

| 範例 | 為什麼需要交易 |
|------|----------------|
| 銀行轉帳 | 扣款和入帳必須同時成功 |
| 網路購物 | 庫存扣款 + 訂單成立 + 發貨通知 |
| 股票交易 | 買方扣款 + 賣方入帳 + 股份轉讓 |
| 註冊新用戶 | 建立帳號 + 發送驗證信 + 建立預設設定 |

## 2️⃣ SQL 語法/語句解說

### 語法 1：開始交易（BEGIN TRANSACTION）

```sql
BEGIN TRANSACTION;
-- 或者簡寫：
BEGIN;
-- 或者：
START TRANSACTION;
```

**作用**：告訴資料庫「我要開始一筆交易」，之後的 SQL 指令都屬於這個交易區塊。

### 語法 2：提交交易（COMMIT）

```sql
COMMIT;
```

**作用**：告訴資料庫「交易成功」，所有修改永久寫入資料庫。

### 語法 3：回滾交易（ROLLBACK）

```sql
ROLLBACK;
```

**作用**：告訴資料庫「交易失敗」，所有修改全部撤消，恢復到交易開始前的狀態。

### 語法 4：設定保存點（SAVEPOINT）

```sql
SAVEPOINT 保存點名稱;
-- 例如：
SAVEPOINT point1;
```

**作用**：在交易中設立「檢查點」，可以在只回滾到某個特定點，而不是全部回滾。

### 語法 5：回滾到保存點

```sql
ROLLBACK TO SAVEPOINT 保存點名稱;
-- 例如：
ROLLBACK TO SAVEPOINT point1;
```

## 3️⃣ 實際操作（一步一步帶著做）

### 步驟 1：創建餘額測試表格

```python
import sqlite3

# 連接到資料庫（如果不存在會自動建立）
conn = sqlite3.connect('bank.db')
cursor = conn.cursor()

# 創建帳戶表格
cursor.execute('''
    CREATE TABLE IF NOT EXISTS accounts (
        id INTEGER PRIMARY KEY,
        name TEXT NOT NULL,
        balance DECIMAL(10, 2) NOT NULL DEFAULT 0
    )
''')

# 插入測試資料 - 兩個帳戶
cursor.execute("DELETE FROM accounts")  # 清空表格
cursor.execute("INSERT INTO accounts (name, balance) VALUES ('小明', 5000)")
cursor.execute("INSERT INTO accounts (name, balance) VALUES ('小華', 3000)")

conn.commit()

# 查看餘額
cursor.execute("SELECT * FROM accounts")
print("轉帳前的餘額：")
for row in cursor.fetchall():
    print(f"  {row[1]}: {row[2]} 元")

conn.close()
```

### 步驟 2：模擬沒有交易的轉帳（會出問題！）

```python
import sqlite3

conn = sqlite3.connect('bank.db')
cursor = conn.cursor()

print("\n" + "="*50)
print("❌ 錯誤示範：沒有使用交易")
print("="*50)

# 嘗試轉帳 1000 元（小明 -> 小華）
# 步驟 1：小明的帳戶扣款
cursor.execute("UPDATE accounts SET balance = balance - 1000 WHERE name = '小明'")
print("✅ 小明帳戶已扣款")

# 這裡故意讓第二步失敗（例如斷電、當機）
# 我們手動模擬失敗場景
# raise Exception("系統錯誤！第二步失敗！")

# 步驟 2：小華的帳戶加款（如果上面失敗，這裡不會執行）
cursor.execute("UPDATE accounts SET balance = balance + 1000 WHERE name = '小華'")
print("✅ 小華帳戶已加款")

conn.commit()

# 檢查結果
cursor.execute("SELECT * FROM accounts")
print("\n📊 轉帳後的餘額（沒有交易保護）：")
for row in cursor.fetchall():
    print(f"  {row[1]}: {row[2]} 元")

print("\n⚠️ 問題：小明的 1000 元不見了！")

conn.close()
```

### 步驟 3：使用交易的正確轉帳

```python
import sqlite3

conn = sqlite3.connect('bank.db')
cursor = conn.cursor()

print("\n" + "="*50)
print("✅ 正確做法：使用交易（Transaction）")
print("="*50)

try:
    # ====== 開始交易 ======
    cursor.execute("BEGIN TRANSACTION")
    
    # 步驟 1：小明的帳戶扣款
    cursor.execute("UPDATE accounts SET balance = balance - 1000 WHERE name = '小明'")
    print("✅ 小明帳戶已扣款")
    
    # 故意製造錯誤（模擬第二步失敗）
    # raise Exception("系統錯誤！第二步失敗！")
    
    # 步驟 2：小華的帳戶加款
    cursor.execute("UPDATE accounts SET balance = balance + 1000 WHERE name = '小華'")
    print("✅ 小華帳戶已加款")
    
    # ====== 交易成功，提交 ======
    cursor.execute("COMMIT")
    print("✅ 交易已提交")

except Exception as e:
    # ====== 交易失敗，回滾 ======
    cursor.execute("ROLLBACK")
    print(f"❌ 交易已回滾：{e}")

# 檢查結果
cursor.execute("SELECT * FROM accounts")
print("\n📊 轉帳後的餘額（有交易保護）：")
for row in cursor.fetchall():
    print(f"  {row[1]}: {row[2]} 元")

conn.close()
```

### 步驟 4：使用保存點（Savepoint）

```python
import sqlite3

conn = sqlite3.connect('bank.db')
cursor = conn.cursor()

print("\n" + "="*50)
print("🔧 進階示範：使用保存點（Savepoint）")
print("="*50)

# 重置餘額
cursor.execute("UPDATE accounts SET balance = 5000 WHERE name = '小明'")
cursor.execute("UPDATE accounts SET balance = 3000 WHERE name = '小華'")
conn.commit()

try:
    # 開始交易
    cursor.execute("BEGIN TRANSACTION")
    
    # 第一步：扣款
    cursor.execute("UPDATE accounts SET balance = balance - 500 WHERE name = '小明'")
    print("✅ 扣款 500 元")
    
    # 設立保存點 point1
    cursor.execute("SAVEPOINT point1")
    
    # 第二步：嘗試加款 1000 元
    cursor.execute("UPDATE accounts SET balance = balance + 1000 WHERE name = '小華'")
    print("✅ 加款 1000 元")
    
    # 第三步：再加款 500 元（故意讓這一步失敗）
    # 這裡我們選擇回滾到保存點，而不是全部回滾
    # raise Exception("第三步失敗")
    
    # 如果第三步失敗，回滾到保存點
    cursor.execute("ROLLBACK TO SAVEPOINT point1")
    print("⚠️ 第三步失敗，已回滾到保存點")
    
    # 第四步：換成加款 300 元
    cursor.execute("UPDATE accounts SET balance = balance + 300 WHERE name = '小華'")
    print("✅ 加款 300 元（替代方案）")
    
    # 交易提交
    cursor.execute("COMMIT")
    print("✅ 交易完成")

except Exception as e:
    cursor.execute("ROLLBACK")
    print(f"❌ 交易失敗：{e}")

# 檢查結果
cursor.execute("SELECT * FROM accounts")
print("\n📊 最終餘額：")
for row in cursor.fetchall():
    print(f"  {row[1]}: {row[2]} 元")

# 小明：5000 - 500 = 4500 元
# 小華：3000 + 300 = 3300 元

conn.close()
```

### 步驟 5：自動交易模式（Python 在提交時自動 BEGIN）

```python
import sqlite3

# Python 的 connection 对象可以开启自动提交模式
# connection.autocommit = True  # 但这样就失去了交易保护！

# 正确的做法：使用 context manager
print("\n" + "="*50)
print("🔧 現代做法：使用 with 语句")
print("="*50)

conn = sqlite3.connect('bank.db')

# 重置餘額
conn.execute("UPDATE accounts SET balance = 5000 WHERE name = '小明'")
conn.execute("UPDATE accounts SET balance = 3000 WHERE name = '小華'")
conn.commit()

# 使用 with 语句（SQLite 会自动管理交易）
try:
    with conn:
        # 在 with 块中，所有修改会在块结束时自动 COMMIT
        # 如果发生异常，会自动 ROLLBACK
        cursor = conn.cursor()
        cursor.execute("UPDATE accounts SET balance = balance - 1000 WHERE name = '小明'")
        cursor.execute("UPDATE accounts SET balance = balance + 1000 WHERE name = '小華'")
        print("✅ 轉帳完成（使用 with 語句）")
except Exception as e:
    print(f"❌ 錯誤：{e}")

# 檢查結果
cursor = conn.cursor()
cursor.execute("SELECT * FROM accounts")
print("\n📊 最終餘額：")
for row in cursor.fetchall():
    print(f"  {row[1]}: {row[2]} 元")

conn.close()
```

## 4️⃣ 常見錯誤/雷點提醒

| 錯誤 | 解決方法 |
|------|----------|
| 忘記 COMMIT，資料沒寫入 | 交易完成後一定要 COMMIT |
| 發生錯誤沒 ROLLBACK | 用 try-except 包裹交易程式碼 |
| 長時間持有交易 | 儘快完成交易，不要在交易中做昂貴操作 |
| 交易範圍太大 | 只把需要原子性的操作放進交易 |
| SQLite 預設自動 COMMIT | 每條 SQL 自動提交，無法回滾 |

### 🚨 交易地雷排行榜

1. **在交易中查詢大量資料** → 鎖住資料庫太久
2. **交易中呼叫外部 API** → API 失敗無法回滾
3. **巢狀交易** → SQLite 不支援（會報錯）
4. **SELECT 也會鎖資料** → 注意隔離層級
5. **长时间不 COMMIT** → 造成鎖競爭
