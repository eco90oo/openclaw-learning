## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🔍 什麼是資料庫？

想像一下，你有一間超級市場。

- **沒有資料庫的時代**：你可以把所有商品資訊寫在紙上，但當你要找「所有庫存少於 10 件的商品」或「上個月銷售額最高的前 10 名商品」時，你可能需要花幾小時甚至幾天，一張一張翻開紙本來統計。

- **有了資料庫之後**：就像是有了一個超級聰明的電子化管理系統。你只要下達一個指令，系統就會在幾秒鐘內幫你找到所有符合條件的資訊，而且**不會出錯**。

> **比喻總結**：資料庫 = 超級市場的電子化存貨管理系統。它幫你**儲存**、**整理**、**查找**資料，而且效率極高。

### 🗂️ 資料庫的核心組成

| 組成部分 | 比喻 | 實際意義 |
|---------|------|----------|
| **資料表 (Table)** | 就像 Excel 的工作表 | 儲存某一類型的資料（如：顧客資料表、訂單資料表）|
| **列 (Row/Record)** | 就像工作表中的一row | 一筆完整的資料（例如：一位顧客的所有資訊）|
| **欄位 (Column/Field)** | 就像工作表中的欄標題 | 資料的屬性（例如：姓名、年齡、地址）|
| **主鍵 (Primary Key)** | 就像的身份證字號 | 唯一識別每一筆記錄的編號，不能重複 |

### 📦 常見的資料庫種類

1. **關聯式資料庫 (Relational Database)** - 例如：MySQL、PostgreSQL
   - 最適合需要精確資料和複雜查詢的應用（電商、金融系統）
   - 使用 SQL 語言操作
   
2. **文件資料庫 (Document Database)** - 例如：MongoDB
   - 適合資料結構不固定、JSON 格式的資料
   - 靈活性高，但犧牲了一些嚴謹性

3. **鍵值資料庫 (Key-Value Database)** - 例如：Redis
   - 適合高速存取的場景（快取、session 儲存）
   - 速度極快，但功能簡單

### 🤔 為什麼要學資料庫？

| 理由 | 說明 |
|------|------|
| **幾乎所有網站都需要** | 臉書、Google、Netflix 背後都有資料庫支撐 |
| **薪水高** | 後端工程師、資料庫工程師都是高薪職位 |
| **解決实际问题** | 學會設計資料庫，就能開發真正的應用（購物網站部落格系統等）|
| **面試必考** | 無論是應徵前端、後端、DevOps，都會被問到資料庫觀念 |

## 2️⃣ SQL 語法/語句解說（每行都要有中文註解）

> SQL 是「Structured Query Language」的縮寫，中文叫做「結構化查詢語言」。這是操作資料庫的標準語言，就像英語是國際通用語言一樣，幾乎所有資料庫都支援 SQL。

### 第一個 SQL 指令：建立資料庫

```sql
-- 建立一個名為 shop 的資料庫
-- CREATE DATABASE 是創建資料庫的指令
CREATE DATABASE shop;
```

### 建立資料表

```sql
-- 使用 shop 資料庫
-- USE 是切換當前操作資料庫的指令
USE shop;

-- 建立顧客資料表 customers
-- CREATE TABLE 是創建資料表的指令
CREATE TABLE customers (
    id INT NOT NULL AUTO_INCREMENT,  -- id：整數型態，不能為空，自動遞增（流水號）
    name VARCHAR(100) NOT NULL,      -- name：最多100個字元的字串，不能為空
    email VARCHAR(100) NOT NULL,     -- email：電子郵件欄位
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  -- 建立的時間，預設為現在時間
    PRIMARY KEY (id)                 -- 設定 id 為主鍵（唯一識別碼）
);
```

### 插入資料（INSERT）

```sql
-- 向 customers 資料表插入一筆記錄
-- INSERT INTO 是插入資料的指令
-- VALUES 是要插入的具體數值
INSERT INTO customers (name, email) VALUES ('王小明', 'wang@example.com');
```

### 查詢資料（SELECT）

```sql
-- 查詢 customers 資料表中的所有資料
-- SELECT 是查詢指令，* 代表所有欄位
-- FROM 指定要查詢哪個資料表
SELECT * FROM customers;
```

### 更新資料（UPDATE）

```sql
-- 更新 id 為 1 的顧客 Email
-- UPDATE 是更新資料的指令
-- SET 後面是要修改的欄位和值
-- WHERE 是篩選條件
UPDATE customers SET email = 'newemail@example.com' WHERE id = 1;
```

### 刪除資料（DELETE）

```sql
-- 刪除 id 為 1 的顧客
-- DELETE FROM 是刪除資料的指令
-- WHERE 是條件（必加，否則會刪除所有資料！）
DELETE FROM customers WHERE id = 1;
```

## 3️⃣ 實際操作（一步一步帶著做）

### 環境準備：使用 SQLite（最簡單的資料庫）

> SQLite 是一個**內嵌式**資料庫，不需要安裝伺服器，直接用一個檔案就可以存資料。最適合初學者練習！

### 步驟一：建立資料庫檔案

```bash
# 在終端機輸入以下指令，會建立一個 shop.db 檔案
sqlite3 shop.db
```

### 步驟二：建立資料表（使用 SQL）

```sql
-- 在 sqlite3 指令列中輸入：
CREATE TABLE customers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT NOT NULL,
    phone TEXT
);
```

### 步驟三：插入資料

```sql
INSERT INTO customers (name, email, phone) VALUES ('王小明', 'wang@example.com', '0912345678');
INSERT INTO customers (name, email, phone) VALUES ('李美華', 'li@example.com', '0988765432');
```

### 步驟四：查詢所有資料

```sql
SELECT * FROM customers;
```

預期輸出：
```
1|王小明|wang@example.com|0912345678
2|李美華|li@example.com|0988765432
```

### 步驟五：查詢特定條件

```sql
-- 找出 id 大於 1 的顧客
SELECT * FROM customers WHERE id > 1;
```

### 步驟六：離開 SQLite

```sql
-- 輸入 .quit 離開
.quit
```

## 4️⃣ 常見錯誤/雷點提醒

| 錯誤情況 | 說明 | 解決方法 |
|---------|------|----------|
| **忘記加 WHERE 直接 UPDATE/DELETE** | 這會更新或刪除**所有資料**！非常危險！ | 永遠先使用 SELECT 確認要操作的資料範圍 |
| **SQL 語法沒有分號** | 大多數 SQL 指令需要用分號結尾 | 檢查語法是否完整 |
| **字串忘記加引號** | SQL 中的字串要用單引號 ' 包起來 | 使用 '字串' 而非 字串 |
| **資料類型不匹配** | 例如：把文字放進數字欄位 | 確認欄位型態再插入資料 |
| **沒有設定 PRIMARY KEY** | 資料可能會重複，無法唯一識別 | 永遠為每個表設計主鍵 |
