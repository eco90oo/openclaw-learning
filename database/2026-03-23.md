## 1️⃣ 基礎概念

### 為什麼要學資料類型？

昨天我們學會了 CRUD，但有一個問題沒解決：

**「欄位要存什麼類型的資料？」**

比如：
- 年齡：存數字？還是存文字？
- 日期：存文字不行嗎？
- 電話：為什麼不能用數字？

這就是今天要解決的問題！

### 常見資料類型家族

SQL 有很多資料類型，但我們把它簡化成 5 個家族：

| 家族 | 用途 | 常見類型 | 範例 |
|------|------|----------|------|
| 數字家族 | 存數字 | INTEGER、REAL、FLOAT | 18、3.14、99.9 |
| 文字家族 | 存文字 | TEXT、VARCHAR | "王小明"、"0912345678" |
| 日期家族 | 存時間 | DATE、DATETIME、TIMESTAMP | "2026-03-23"、"2026-03-23 10:30:00" |
| 是非家族 | 存真假 | BOOLEAN | TRUE、FALSE |
| 二進位 | 存檔案 | BLOB、BINARY | 圖片、影片 |

### SQLite 專用的簡化版本

SQLite 超級貼心，它把資料類型簡化只剩 5 種：

| SQLite 類型 | 可以存什麼 | 範例 |
|-------------|-----------|------|
| INTEGER | 整數（沒有小數點的數） | 1、100、-5 |
| REAL | 小數（有小數點的數） | 3.14、99.9 |
| TEXT | 文字、字串 | "Hello"、"2026-03-23" |
| BLOB | 原本的樣子（不轉換） | 圖片、影片、檔案 |
| NULL | 什麼都沒有（空值） | 無 |

**超級重要：SQLite 的 TEXT 可以存日期！** 這就是為什麼很多教學都用 SQLite，因为它超級簡單！

### 什麼時候用什麼類型？

#### 年齡 → INTEGER
```
年齡是數字，而且不會有「18.5歲」這種說法（對，我們只用整數）
→ 用 INTEGER
```

#### 姓名 → TEXT
```
姓名是文字，中文、英文都可能
→ 用 TEXT
```

#### 薪水 → REAL
```
薪水可能有小數，比如 35000.50
→ 用 REAL（或 INTEGER 存「分」）
```

#### 日期 → TEXT（SQLite）
```
日期看起來像文字，存起來方便
→ SQLite 用 TEXT
→ 其他資料庫用 DATE
```

#### 電話 → TEXT
```
電話看起來是數字，但「0912」開頭的 0 很 important！
如果用 INTEGER，001 會變成 1，開頭的 0 就不見了！
→ 用 TEXT
```

#### 身份證字號 → TEXT
```
有英文字母，不可能用數字
→ 用 TEXT
```

### 表格設計基本原則

設計表格時要想三件事：

**1. 這個欄位要存什麼？**
   - 數字？文字？日期？

**2. 這個欄位可以空白嗎？**
   - 可以 → 不用特別設定
   - 不可以 → 要設 NOT NULL

**3. 這個欄位需要唯一嗎？**
   - 需要（比如 email）→ 設 UNIQUE
   - 不需要 → 不用特別設定

### 主鍵（Primary Key）是什麼？

**主鍵 = 這張表的身份證號**

特性：
- 唯一：每筆資料的主鍵都不會重複
- 不能空白：NOT NULL
- 一張表只能有一個主鍵

常見設計：
- 流水號（1, 2, 3...）← 最常用
- UUID（像密碼的一大串文字）
- 業務邏輯（學號、員工編號）

## 2️⃣ 實際操作

### 情境：我們要設計一張「學生資料表」

需求：
- 學生編號（要唯一）
- 姓名（必填）
- 年齡
- 性別
- Email（要唯一）
- 生日
- 入學日期

### 建立表格

```sql
CREATE TABLE students (
    id INTEGER PRIMARY KEY,           -- 流水號，主鍵
    student_no TEXT UNIQUE NOT NULL,  -- 學號，唯一且必填
    name TEXT NOT NULL,               -- 姓名，必填
    age INTEGER,                      -- 年齡，可空白
    gender TEXT,                      -- 性別，可空白
    email TEXT UNIQUE,                -- Email，唯一（但不強制必填）
    birthday TEXT,                    -- 生日，可空白
    enroll_date TEXT NOT NULL         -- 入學日期，必填
);
```

**解讀每一行：**
- `INTEGER PRIMARY KEY` → 整數主鍵，自動遞增
- `TEXT UNIQUE NOT NULL` → 文字、唯一、不能空白
- `TEXT NOT NULL` → 文字、不能空白
- `TEXT` → 文字，可空白

### 實際操作看看

```bash
# 打開資料庫
sqlite3 school.db
```

```sql
-- 創建表格
CREATE TABLE students (
    id INTEGER PRIMARY KEY,
    student_no TEXT UNIQUE NOT NULL,
    name TEXT NOT NULL,
    age INTEGER,
    email TEXT UNIQUE,
    birthday TEXT,
    enroll_date TEXT NOT NULL
);
```

```sql
-- 查看表格結構（確認一下）
.schema students
```

```sql
-- 試著插入一筆資料
INSERT INTO students (student_no, name, age, email, birthday, enroll_date)
VALUES ('S001', '王小明', 20, 'wang@example.com', '2005-05-15', '2024-09-01');
```

```sql
-- 再插入一筆
INSERT INTO students (student_no, name, age, email, enroll_date)
VALUES ('S002', '李美華', 22, 'lee@example.com', '2024-09-01');
-- 注意：age、email、birthday 都沒填，會自動變成 NULL（空白）
```

```sql
-- 查詢所有資料
SELECT * FROM students;
```

### 嘗試犯錯（學習用）

```sql
-- 錯誤示範 1：學號重複（會失敗）
INSERT INTO students (student_no, name, enroll_date)
VALUES ('S001', '張三', '2024-09-01');
-- 結果：Error: UNIQUE constraint failed: students.student_no

-- 錯誤示範 2：必填欄位空白（會失敗）
INSERT INTO students (student_no, enroll_date)
VALUES ('S003', '2024-09-01');
-- 結果：Error: NOT NULL constraint failed: students.name
```

---

### 常見錯誤修正語法

如果表格已經建好，想要修改結構：

```sql
-- 新增欄位
ALTER TABLE students ADD COLUMN phone TEXT;

-- 刪除欄位（SQLite 不支援，改用重建表格）
```

```sql
-- 重新命名表格
ALTER TABLE students RENAME TO students_old;
```

## 💡 小知識

**為什麼身份證字號不能存 INTEGER？**

因為身份證有英文字母！

另外，電話號碼也是：
- INTEGER 存 0912345678 → 變成 912345678（少一個 0！）
- TEXT 存 "0912345678" → 完整保留

**記住法則：**
「**有英文字**或**開頭可能是 0**的→用 TEXT」
