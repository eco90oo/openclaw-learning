## 1️⃣ 基礎概念（用生活比喻）

### 什麼是 DevOps？——像餐廳開店一樣！

想像你要開一家餐廳：

**傳統做法（沒有 DevOps）：**
- 廚師做好菜 👨‍🍳 → 端給客人 🍽️
- 如果客人說「太鹹」，廚師要重新做 → 又要等半天
- 每次改菜單都要整家店關門重來

**DevOps 做法：**
- 廚房（開發团队）快速做好菜
- 傳菜帶（自動化流程）一直運轉
- 客人反映問題 → 馬上改 → 立刻上菜
- **重點：速度快、誤差小、大家都有飯吃**

### DevOps = Development + Operations

| 傳統角色 | DevOps 角色 |
|---------|------------|
| 開發者只管寫程式 | 開發者也要懂部署 |
| 運維只管維護伺服器 | 運維也要懂程式 |
| 兩個部門互相怪罪 | 大家同一個團隊 |

## 2️⃣ 核心知識（專有名詞解釋）

### 🔴 Docker —— 「帶著走的廚房」

**比喻：**  
想像 Docker 就像一個**真空包裝的料理包**。不管你在哪家餐廳、用什麼爐子，只要按照包裝上的說明加熱，就能做出一樣的料理。

**正式解釋：**
- Docker 是一個**容器化平台**
- 容器 =  把應用程式和它的環境「打包」在一起
- 這樣無論在哪裡執行，結果都一樣

### 🔴 Kubernetes (K8s) —— 「餐廳管理系統」

**比喻：**
一家大型餐廳有很多廚師、很多桌子、很多存貨。誰來管理？當然是**餐廳管理系統**！

- 某個廚師太累 → 系統自動調配人力
- 某桌客人太多 → 系統自動加開桌子
- 某個食材快用完 → 系統自動補貨

**Kubernetes 就是這樣的管理系統**，幫你管理很多 Docker 容器。

**簡稱：**
- K8s = Kubernetes（K + 8個字母 + s）

### 🔴 CI/CD —— 「自動化的傳菜帶」

| 縮寫 | 全名 | 比喻 |
|-----|------|------|
| **CI** | Continuous Integration（持續整合） | 廚師做好一道菜，馬上有人試吃 |
| **CD** | Continuous Deployment（持續部署） | 試吃通過，自動端上餐桌 |

**CI/CD 流程：**
```
寫程式 → 自動測試 → 自動打包 → 自動部署 → 自動監控
```

### 🔴 雲端部署 —— 「租別人的廚房」

| 選項 | 比喻 |
|-----|------|
| IaaS | 租一間空廚房（你自己買爐具） |
| PaaS | 租一間有設備的廚房（只要帶食材） |
| SaaS | 別人做好端給你吃（直接叫外賣） |

常見雲端：AWS、Azure、GCP（Google Cloud）

### 🔴 監控與日誌 —— 「餐廳的監視器」

- **監控：** 看今天賣了多少菜、翻桌率多少
- **日誌：** 記錄、誰在幾點點了什麼菜、發生什麼問題
- **警報：** 冰箱溫度太高 → 自動打電話通知你

## 3️⃣ 實際操作（步驟教學）

### ️⃣️ 實戰一：安裝 Docker

**Step 1：更新系統**
```bash
sudo apt update
sudo apt upgrade -y
```

**Step 2：安裝 Docker**
```bash
# 安裝必要工具
sudo apt install -y curl wget

# 下載並執行 Docker 安裝腳本
curl -fsSL https://get.docker.com | sh

# 啟動 Docker
sudo systemctl start docker
sudo systemctl enable docker
```

**Step 3：確認安裝成功**
```bash
docker --version
# 應該看到類似：Docker version 24.x.x
```

**Step 4：讓當前用戶可以使用 Docker（免得每次都要sudo）**
```bash
sudo usermod -aG docker $USER
# 需要重新登入才能生效
```

### ️⃣️ 實戰二：第一個 Docker 容器

**比喻：製作第一道料理**

**Step 1：找一個「料理包」（映像檔）**
```bash
# 從 Docker Hub 拉取一個簡單的映像檔
docker pull hello-world
```

**Step 2：啟動容器（把料理微波）**
```bash
docker run hello-world
```

**Step 3：看看我們的「冰箱裡有什麼」（列出所有容器）**
```bash
docker ps -a
```

**Step 4：清理「吃剩的料理」（刪除容器）**
```bash
docker rm <容器ID>
```

### ️⃣️ 實戰三：部署一個簡單的網站

**比喻：開一家小餐廳**

**Step 1：創建一個簡單的 HTML 檔案**
```html
<!-- 創建檔案 -->
mkdir my-website
cd my-website
echo '<h1>你好！這是我的第一個網站 🐳</h1>' > index.html
```

**Step 2：創建 Dockerfile（料理包說明書）**
```dockerfile
# 選擇基礎映像檔（作業系統）
FROM nginx:alpine

# 複製我們的網站到容器裡
COPY index.html /usr/share/nginx/html/index.html

# 對外开放的端口
EXPOSE 80

# 啟動時執行的命令
CMD ["nginx", "-g", "daemon off;"]
```

**Step 3：構建映像檔（做成真空料理包）**
```bash
docker build -t my-website:1.0 .
```

**Step 4：運行容器（把料理端上桌）**
```bash
docker run -d -p 8080:80 --name my-website my-website:1.0
```
- `-d` = 在背景运行
- `-p 8080:80` = 把容器的 80 端口映射到電腦的 8080 端口

**Step 5：打開瀏覽器看結果**
```
http://localhost:8080
```

### ️⃣️ 實戰四：Docker Compose —— 同時做好幾道菜

當你需要多個容器（資料庫 + 網站 + 快取）時，一個一個啟動太麻煩。

**比喻：一次準備套餐**

**創建 docker-compose.yml：**
```yaml
version: '3.8'

services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html
  
  database:
    image: redis:alpine
    ports:
      - "6379:6379"
```

**啟動所有服務：**
```bash
docker-compose up -d
```

**停止所有服務：**
```bash
docker-compose down
```
