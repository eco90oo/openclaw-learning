### 1️⃣ 基礎概念（用生活比喻，越詳細越好）

#### 回顧：昨天的概念

昨天我們學了 Docker 三大核心：
- **Image（映像檔）** = 食譜（唯讀模板）
- **Container（容器）** = 用食譜做出來的菜（運行中的實例）
- **Registry（倉庫）** = 存放食譜的圖書館（Docker Hub）

#### 今天學什麼？

今天我們要學會「如何操作這些食譜和菜」。就像餐廳廚房，你需要知道：
- 怎麼拿食譜（pull）
- 怎麼做菜（run）
- 怎麼看正在做什麼（ps）
- 怎麼進入廚房檢查（exec）

#### 🟢 `docker run` - 啟動一個容器（做菜）

**最基本語法**：
```bash
# docker run [選項] 映像檔名稱 [命令]
docker run nginx
```

**生活比喻**：就像從冰箱拿出一個食譜，按照步驟開始做菜。

**常見選項詳解**：

```bash
# -d：Detached mode（背景運行）
# 想像：廚師把菜放進烤箱後去忙其他事，不用一直盯著
docker run -d nginx

# --name：給容器取個名字
# 想像：給每道菜貼上標籤，方便識別
docker run --name 我的第一個容器 nginx

# -p：Port mapping（連接埠對應）
# 格式：主機連接埠:容器連接埠
# 想像：餐廳外牆有門牌號（主機），廚房內有工作區編號（容器）
# 外面的人找「8080」，廚房內實際在「80」工作
docker run -d -p 8080:80 nginx

# -e：環境變數（設定參數）
# 想像：調整食譜的辣度、鹹度
docker run -d -e MY_VAR=hello nginx

# --rm：容器停止後自動刪除
# 想像：吃完飯後，服務生立刻收拾碗盤
docker run --rm nginx echo "完成後就消失"

# -it：互動模式（進入容器內部操作）
# -i：interactive（保持標準輸入）
# -t：pseudo-TTY（虛擬終端機）
# 想像：親自走進廚房，拿起鍋鏟自己操作
docker run -it ubuntu /bin/bash

# -v：Volume 掛載（分享資料夾）
# 格式：主機路徑:容器路徑
# 想像：廚房裡有個共用冰箱，外面可以放食材進去
docker run -v /home/user/data:/container/data nginx

# --network：指定網路
# 想像：把這個廚房接到哪個管線系統
docker run --network my-net nginx
```

**完整範例**：
```bash
# 啟動一個 Nginx，背景運行，取名 web-server，連接埠 8080
# 這就像：派一個廚師啟動一台烤箱（nginx），命名為主烤箱，
# 把餐廳的 8080 號門對應到烤箱的 80 號出風口
docker run -d \
  --name web-server \     # 命名為 web-server
  -p 8080:80 \           # 對外開放 8080
  nginx                  # 使用 nginx 映像檔
```

#### 🟢 `docker ps` - 查看運行中的容器（看廚房狀態）

```bash
# 列出運行中的容器
# ps：process status
docker ps

# 列出所有容器（包含已停止的）
# -a：all
docker ps -a

# 列出所有容器，只顯示 ID
# -q：quiet（只顯示 ID）
docker ps -aq

# 格式化輸出（更好閱讀）
# --format：指定輸出格式
docker ps --format "table {{.ID}}\t{{.Image}}\t{{.Status}}"
```

**輸出欄位說明**：

```
CONTAINER ID   IMAGE   COMMAND   CREATED   STATUS   PORTS   NAMES
容器ID         映像檔  啟動命令  建立時間  狀態     連接埠  名稱
```

| 欄位 | 比喻 | 說明 |
|-----|------|------|
| CONTAINER ID | 廚房編號 | 容器唯一識別碼（縮寫顯示12位） |
| IMAGE | 食譜名稱 | 使用哪個映像檔啟動的 |
| STATUS | 工作狀態 | Up 2 hours（運行2小時）/ Exited（已停止） |
| PORTS | 對外窗口 | 哪些連接埠對外開放 |
| NAMES | 暱稱 | 容器的名字（可自訂） |

**實際輸出範例**：
```
CONTAINER ID   IMAGE   COMMAND                  CREATED        STATUS        PORTS                  NAMES
a1b2c3d4e5f6   nginx   "/docker-entrypoint…"   2 hours ago    Up 2 hours    0.0.0.0:8080->80/tcp   web-server
```

#### 🟢 `docker exec` - 進入容器執行命令（走進廚房）

**基本語法**：
```bash
# docker exec [選項] 容器名稱/ID 命令
docker exec 容器名稱 命令
```

**生活比喻**：廚房正在運作，你想親自進去看看裡面什麼情況。

```bash
# 在容器內執行一個命令（不走進去）
# 想像：透過對講機問廚房「現在幾點了？」
docker exec web-server date

# 以互動模式進入容器（-it）
# 想像：親自穿上廚師服，走進廚房
docker exec -it web-server /bin/bash

# 以 root 身份進入
# -u：指定用戶，root 是最高權限
docker exec -u root -it web-server /bin/bash

# 在容器內執行多個命令
# -c：用一行指令執行
docker exec web-server sh -c "ls -la /app && echo '完成'"
```

**常見使用場景**：

```bash
# 場景1：檢查容器內的檔案結構
docker exec -it web-server ls /var/log

# 場景2：檢查 Nginx 設定檔
docker exec -it web-server cat /etc/nginx/nginx.conf

# 場景3：確認網路設定
docker exec -it web-server ip addr

# 場景4：安裝工具（除錯用）
docker exec -it web-server apt-get update && apt-get install -y curl

# 場景5：檢查執行中的程序
docker exec -it web-server ps aux
```

#### 🟢 `docker build` - 從 Dockerfile 建構映像檔（根據食譜做罐頭）

**基本語法**：
```bash
# docker build [選項] 上下文路徑
docker build .
```

**生活比喻**：有了詳細的食譜文件（Dockerfile），廚房按照步驟做出可以保存的罐頭（Image）。

```bash
# 在當前目錄建構映像檔
# .：代表「當前目錄」（Dockerfile 所在位置）
docker build .

# 給映像檔取名字和標籤
# -t：tag，命名格式是「名稱:標籤」
docker build -t 我的網站:1.0 .

# 完整命名（含 Registry）
# 格式：registry.com/repository:tag
docker build -t myregistry.com/myapp:v1.0 .

# 指定 Dockerfile 檔名（預設是 Dockerfile）
# -f：file，指定使用哪個檔案
docker build -f Dockerfile.dev .

# 不使用快取（強制的重新建構）
# --no-cache：每次都從頭開始，確保最新版本
docker build --no-cache -t myapp:latest .

# 建構時傳入變數（ARG）
docker build --build-arg VERSION=1.0 -t myapp:1.0 .
```

**建構過程解讀**：

```
Step 1/5 : FROM python:3.9
 -> 從冰箱拿出 python:3.9 這個食材

Step 2/5 : WORKDIR /app
 -> 設定工作檯面為 /app

Step 3/5 : COPY requirements.txt .
 -> 把需求清單從外面複印到工作檯

Step 4/5 : RUN pip install -r requirements.txt
 -> 執行 pip 安裝（就像照著清單採購）

Step 5/5 : COPY . .
 -> 把所有東西搬進來

Successfully built abc123def456
Successfully tagged myapp:1.0
```

#### 🟢 其他常用指令

```bash
# 停止容器
# stop：優雅停止（等它處理完手上的事）
docker stop 容器名稱

# 強制終止容器
# kill：直接切斷電源
docker kill 容器名稱

# 啟動已停止的容器
# start：重新啟動
docker start 容器名稱

# 重啟容器
# restart：等於 stop + start
docker restart 容器名稱

# 刪除已停止的容器
# rm：remove
docker rm 容器名稱

# 強制刪除運行中的容器
docker rm -f 容器名稱

# 一次刪除所有已停止的容器
docker container prune

# 檢視容器日誌
# logs：查看輸出/錯誤日誌
docker logs 容器名稱

# 即時追蹤日誌（像 tail -f）
docker logs -f 容器名稱

# 查看最近 100 行日誌
docker logs --tail 100 容器名稱

# 檢視資源使用（CPU、記憶體）
# stats：statistics
docker stats

# 只看特定容器
docker stats 容器名稱

# 檢視容器詳細資訊
# inspect：深入查看設定
docker inspect 容器名稱

# 檢視容器行程樹
# top：就像 Linux 的 top 指令
docker top 容器名稱

# 複製檔案到/從容器
# cp：copy
docker cp 主的檔案.txt 容器名稱:/container/路徑/
docker cp 容器名稱:/container/檔案.txt 主的路徑/

# 暫停/恢復容器
# pause/unpause
docker pause 容器名稱
docker unpause 容器名稱
```

#### 🟢 映像檔相關指令

```bash
# 列出本地所有映像檔
docker images

# 刪除映像檔
# rmi：remove image
docker rmi 映像檔名稱

# 拉取映像檔
# pull：從 Registry 下載
docker pull nginx:latest

# 推送映像檔
# push：上傳到 Registry
docker push myregistry.com/myapp:v1.0

# 清理未使用的資源（映像檔、容器、網路）
docker system prune

# 清理所有未使用的資源（更徹底）
docker system prune -a

# 查看磁碟使用
docker system df

# 標記映像檔（常見於準備推送）
# tag：新增標籤/別名
docker tag myapp:latest myregistry.com/myapp:v1.0
```

### 3️⃣ 實際操作（一步一步帶著做）

#### Step 1：啟動第一個 Nginx 容器

```bash
# 背景運行 Nginx，取名 my-nginx，对外 8080 端口，内部 80
# -d：背景運行
# --name：命名
# -p 8080:80：主機 8080 → 容器 80
docker run -d \
  --name my-nginx \
  -p 8080:80 \
  nginx

# 立刻檢查是否啟動成功
# -n 1：只顯示最近 1 條
docker logs -n 1 my-nginx

# 用瀏覽器或 curl 測試
# 如果在本地，開瀏覽器訪問 http://localhost:8080
# 或用 curl：
curl http://localhost:8080
```

**預期結果**：
```
<!DOCTYPE html>
<html>
<head>
<title>Welcome to nginx!</title>
...
```

#### Step 2：操作容器（ps, exec, logs）

```bash
# 查看運行中的容器
docker ps

# 查看所有容器（包含已停止的）
docker ps -a

# 進入容器內部（互動模式）
# 這時你會看到 root@xxxx:/# 提示符，代表你在容器裡了
docker exec -it my-nginx /bin/bash

# 在容器內執行一些命令（試完後 Ctrl+D 或 exit 離開）
# 檢查 nginx 版本
nginx -v

# 查看已安裝的套件
ls /usr/share/nginx/html/

# 查看 /etc 目錄
ls /etc/

# 離開容器（回到主機）
exit

# 檢查 Nginx 日誌（在外面操作）
docker logs my-nginx

# 即時追蹤日誌
docker logs -f my-nginx
```

#### Step 3：建立自己的 Dockerfile 並建構

```bash
# 建立一個資料夾
mkdir ~/my-docker-app && cd ~/my-docker-app

# 建立一個簡單的 Python Flask 應用
cat > app.py << 'EOF'
from flask import Flask
app = Flask(__name__)

@app.route('/')
def hello():
    return '<h1>Hello Docker!</h1><p>我在容器裡跑喔！</p>'

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
EOF

# 建立 Dockerfile
# 注意：這是最基本的，Day 3 會詳細講解
cat > Dockerfile << 'EOF'
# 從 Python 3.9 映像檔開始（像拿一個乾淨的鍋子）
FROM python:3.9-slim

# 設定工作目錄（廚房的工作檯）
WORKDIR /app

# 將當前目錄的檔案複製到容器的 /app
COPY . .

# 安裝依賴（按照需求清單採購）
RUN pip install flask

# 對外開放 5000 端口
EXPOSE 5000

# 啟動命令
CMD ["python", "app.py"]
EOF

# 建構映像檔
# -t：命名為 my-flask-app:1.0
# .：使用當前目錄的 Dockerfile
docker build -t my-flask-app:1.0 .

# 等待建構完成，查看結果
docker images | grep my-flask-app

# 啟動容器
docker run -d \
  --name my-flask \
  -p 5000:5000 \
  my-flask-app:1.0

# 測試
curl http://localhost:5000

# 查看日誌
docker logs my-flask

# 進入容器內部（如果需要除錯）
docker exec -it my-flask /bin/bash

# 在容器內可以執行（自己試試看）：
#   pip list
#   cat app.py
#   env | grep PYTHON

exit
```

#### Step 4：清理環境

```bash
# 停止所有容器
docker stop $(docker ps -aq)

# 刪除所有容器
docker rm $(docker ps -aq)

# 刪除測試用的映像檔
docker rmi my-flask-app:1.0

# 或者一次清理所有未使用的資源
docker system prune -a
```

### 4️⃣ 今日小任務

#### ✅ 必做任務

1. **啟動一個 Ubuntu 容器**：
```bash
docker run -d --name my-ubuntu ubuntu tail -f /dev/null
```

```bash
docker exec -it my-ubuntu /bin/bash

# 離開後執行：docker stop my-ubuntu && docker rm my-ubuntu
```

3. **啟動 Nginx + 自訂 HTML**：
```bash
# 建立一個資料夾，放一個 index.html
mkdir ~/web-test && cd ~/web-test
echo "<h1>我的第一個 Docker 網站</h1>" > index.html

# 啟動 Nginx，掛載這個資料夾
docker run -d \
  --name my-web \
  -p 8080:80 \
  -v $(pwd):/usr/share/nginx/html \
  nginx

# 測試：http://localhost:8080
# 清理：docker stop my-web && docker rm my-web
```
