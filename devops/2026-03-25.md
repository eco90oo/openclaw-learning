## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 什麼是容器化？

想像一下：

**傳統方式（虛擬機）** 🏠
> 就像你要開一家餐廳，你需要蓋一整棟大樓，每層樓獨立的廚房、餐廳、電力系統。即使只想賣一碗泡麵，也要蓋整棟樓！

**容器化（Docker）** 📦
> 就像是用標準貨櫃來包裝餐點。貨櫃有統一的大小、標準的接口，不管哪家餐廳都能放到同一輛卡車上运输。貨櫃裡面有餐具、調味包、熱水，**你只管打開就能吃**，不用管餐廳怎麼做的。

### 容器 vs 虛擬機

| 特性 | 虛擬機 (VM) | 容器 (Container) |
|------|-------------|------------------|
| **大小** | 幾 GB 到幾十 GB | 幾 MB 到幾百 MB |
| **啟動時間** | 幾分鐘 | 幾秒鐘 |
| **資源占用** | 獨占 OS，資源浪費 | 共享 OS，高效利用 |
| **比喻** | 獨立別墅 | 統一貨櫃 |

### Docker 是什麼？

**Docker** 就像是一個**標準化的貨櫃系統**：
- **Container（容器）**：貨櫃本身
- **Image（映像檔）**：貨櫃的設計圖/模板
- **Registry（倉庫）**：存放貨櫃設計圖的圖書館
- **Dockerfile**：寫在紙上的建造說明書

### 為什麼要學 Docker？

1. **環境一致性**：開發、測試、生產環境都一樣，不會有「在我電腦上可以啊」的問題
2. **快速部署**：幾秒鐘就能啟動一個完整系統
3. **資源隔離**：不同服務之間不會互相影響
4. **易於擴展**：需要更多服務？再開一個容器就好

## 2️⃣ 核心知識（指令詳解）

### Docker 基本架構

```
┌─────────────────────────────────────────────────────────────┐
│                        Docker Host                          │
│  ┌───────────────────────────────────────────────────────┐  │
│  │                    Docker Daemon                      │  │
│  │   (這是老大，管理所有容器的大小事)                       │  │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐              │  │
│  │  │Container│  │Container│  │Container│              │  │
│  │  │   A     │  │   B     │  │   C     │              │  │
│  │  └─────────┘  └─────────┘  └─────────┘              │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────┐
│ Docker Hub      │  ← 公開的映像檔倉庫
│ (Registry)      │  ← 可以放私人倉庫
└─────────────────┘
```

### 常用指令速查表

```bash
# 查看 Docker 版本（確認安裝成功）
docker --version

# 查看 Docker 詳細資訊（包含伺服器資訊）
docker info

# 顯示所有容器（包括已停止的）
docker ps -a

# 顯示正在運行的容器
docker ps

# 顯示所有映像檔
docker images

# 拉取（下載）映像檔
docker pull nginx:latest

# 執行（啟動）一個容器
docker run -d -p 80:80 nginx

# 停止容器
docker stop <容器ID>

# 刪除容器
docker rm <容器ID>

# 刪除映像檔
docker rmi <映像檔ID>
```

## 3️⃣ 實際操作（一步一步帶著做）

### 步驟 1：確認 Docker 安裝

```bash
# 檢查 Docker 版本
docker --version

# 預期輸出：
# Docker version 24.0.0, build 1234567
```

### 步驟 2：啟動第一個容器（Hello World）

```bash
# 執行 hello-world 映像檔
# docker run = 啟動一個容器
# hello-world = 要執行的映像檔名稱
docker run hello-world

# 輸出會類似：
# Hello from Docker!
# This message shows that your installation appears to be working correctly.
```

### 步驟 3：實際操作 Nginx 網頁伺服器

```bash
# 啟動一個 Nginx 容器（網頁伺服器）
# -d = detach mode（背景執行，不會卡住你的 terminal）
# -p 80:80 = 端口映射（把容器的 80 端口映射到主機的 80 端口）
# nginx = 映像檔名稱
docker run -d -p 80:80 nginx

# 查看正在運行的容器
docker ps

# 結果會類似：
# CONTAINER ID   IMAGE   COMMAND                  CREATED         STATUS        PORTS                NAMES
# a1b2c3d4e5f6   nginx   "/docker-entrypoint.…"   5 seconds ago  Up 5 seconds  0.0.0.0:80->80/tcp   peaceful_mayer
```

### 步驟 4：停止和刪除容器

```bash
# 停止容器（用容器 ID 或名稱）
docker stop a1b2c3d4e5f6

# 查看容器是否已停止
docker ps

# 刪除容器
docker rm a1b2c3d4e5f6

# 查看所有容器（包括已停止的）
docker ps -a

# 強制刪除運行中的容器
docker rm -f <容器ID>
```

### 步驟 5：探索映像檔

```bash
# 列出所有映像檔
docker images

# 結果類似：
# REPOSITORY   TAG       IMAGE ID       CREATED        SIZE
# nginx        latest    a6bd71f48f68   2 days ago     187MB
# hello-world  latest    feb5d9fea3a5   5 months ago   13.3kB

# 刪除映像檔
docker rmi nginx

# 強制刪除映像檔（即使有容器在使用）
docker rmi -f nginx
```

### 步驟 6：互動式容器（進入容器內部）

```bash
# 啟動一個互動式的 Ubuntu 容器
# -i = interactive（保持 STDIN 開放）
# -t = pseudo-TTY（分配一個虛擬終端機）
# /bin/bash = 執行 Bash  shell
docker run -it ubuntu /bin/bash

# 在容器內，你可以執行：
# cat /etc/os-release  # 查看系統版本
# ls                   # 列出檔案
# exit                 # 離開容器

# 離開容器（但容器會停止）
exit

# 如果想保持容器運行並離開，按 Ctrl+P+Q
```

### 步驟 7：查看容器日誌

```bash
# 查看容器日誌（-f 像是 tail -f 即時查看）
docker logs -f <容器ID>

# 只看最後 100 行
docker logs --tail 100 <容器ID>
```

## 4️⃣ 今日小任務

### 🎯 任務目標：完成 Docker 基礎操作

1. **確認環境**：執行 `docker --version` 確認 Docker 已安裝
2. **啟動 Hello World**：執行 `docker run hello-world`
3. **啟動 Nginx**：在背景啟動 Nginx 容器並映射端口
4. **驗證服務**：使用瀏覽器或 curl 訪問 localhost:80
5. **清理環境**：停止並刪除所有容器

### 預期產出

完成後，請在終端機執行以下命令確認：

```bash
# 顯示所有容器（應該是空的或只有運行中的）
docker ps -a

# 顯示所有映像檔
docker images
```
