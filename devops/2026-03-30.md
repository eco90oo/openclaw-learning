## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🏠 什麼是容器？用「公寓」來比喻！

想象你是一個房東，要出租房子給不同的房客。

#### 傳統方式（虛擬機）= 每間公寓都是獨立別墅
- 一棟大樓（伺服器）
- 每層樓是完全隔離的別墅（VM）
- 每個別墅有自己的水電系統、瓦斯管線
- 隔音超好，但：
  - 建設時間很長（啟動慢）
  - 空間浪費嚴重（資源浪費）
  - 費用昂貴（硬體成本高）

#### Docker 方式（容器）= 每間公寓都是酒店房間
- 一樣是一棟大樓（伺服器）
- 每個房間（容器）共享整棟大樓的水電
- 建設快速（秒級啟動）
- 空間利用率高（可同時運行數十個容器）
- 設備標準化，随处可部署

### 📦 容器 vs 傳統虛擬機

| 特性 | 傳統 VM | Docker 容器 |
|------|---------|-------------|
| 隔離級別 | 硬體隔離（完整 OS） | 行程隔離（共享 Kernel） |
| 啟動時間 | 分鐘級 | 秒級 |
| 資源消耗 | 高（完整 OS + Hypervisor） | 低（只運行必要程序） |
| 大小 | GB 級（完整 OS） | MB 級（只含應用+依賴） |
| 可移植性 | 一般 | 極高（Build Once, Run Anywhere） |

### 🐳 Docker 的核心理念

```
Build Once, Run Anywhere
一次建構，到處運行
```

這句話的意思是：
- 開發人員在本地電腦 build 一個容器
- 這個容器可以在任何有 Docker 的環境運行
- 不會再有「在我電腦上明明可以」的問題

## 2️⃣ 核心知識

### 🔑 Docker 三個核心概念

#### 1. Image（映像檔）= 建築藍圖/模板
- 只讀的模板
- 定義了你要運行的環境
- 例如：一個 Node.js 映像檔包含了 Node.js runtime、相關函式庫
- 可以想像成：房屋的設計圖

#### 2. Container（容器）= 根據藍圖蓋的房子
- Image 的實例（Instance）
- 可運行、可停止、可刪除
- 例如：根據 Node.js 映像檔 запущен 10 個容器
- 可以想像成：根據設計圖蓋好的實際房子

#### 3. Registry（倉庫）= 存放藍圖的圖書館
- 存放 Image 的地方
- 最常用的是 Docker Hub (https://hub.docker.com)
- 可以想像成：房屋設計圖書館

## 3️⃣ 實際操作（一步一步帶著做）

### 📋 步驟 1：確認 Docker 是否已安裝

打開終端機，輸入以下指令：

```bash
# 檢查 Docker 版本（-v 是 --version 的簡寫）
docker -v

# 顯示結果類似：
# Docker version 24.0.7, build 311b9fc
```

> 💡 註解：`-v` 或 `--version` 用來顯示 Docker 客戶端的版本號碼

### 📋 步驟 2：理解 Docker 的客戶端-伺服器架構

```bash
# 檢查 Docker 服務是否正在運行（相當於檢查引擎是否啟動）
docker info

# 這會顯示：
#  - Docker 客戶端版本
#  - Docker 服務端（Daemon）版本
#  - 運行的容器數量
#  - 儲存驅動資訊
```

> 💡 註解：`docker info` 是診斷 Docker 環境的第一步，如果這裡出錯，通常是 Docker 服務沒有啟動

### 📋 步驟 3：理解 Image 和 Container 的關係

讓我們用一個生活中的比喻來解釋：

```bash
# 先想像一下：
# Image（映像檔）就像 DNA/藍圖 - 靜態的、只讀的
# Container（容器）就像根據 DNA 生成的活生生的生物 - 動態的、可運行的
```

### 📋 步驟 4：人生中的第一個 Docker 操作

讓我們拉取一個最基本的 Image：

```bash
# docker pull：從 Registry 下載 Image
# hello-world：這是 Docker 官方提供的一個測試 Image
docker pull hello-world
```

> 💡 註解：
> - `pull` 是「拉取」的意思，把遠端的 Image 拉到本地
> - `hello-world` 是一個極簡單的測試程式，用來驗證 Docker 是否正常運作

### 📋 步驟 5：運行第一個容器

```bash
# docker run：根據 Image 創建並運行一個容器
docker run hello-world
```

> 💡 註解：
> - `run` 會先檢查本地是否有這個 Image
> - 如果沒有，會自動執行 `docker pull`
> - 然後根據 Image 創建一個新的容器並運行

### 📋 步驟 6：查看正在運行的容器

```bash
# docker ps：列出正在運行的容器
# ps 是 "Process Status" 的縮寫
docker ps

# 加上 -a 參數：列出所有容器（包括已停止的）
docker ps -a
```

> 💡 註解：
> - `ps` 就像 Linux 的 `ps` 指令，用來查看程序狀態
> - `-a` 是 `--all` 的簡寫，顯示所有狀態的容器

### 📋 步驟 7：查看本地現有的 Images

```bash
# docker images：列出本地快取的所有 Image
docker images

# 顯示結果類似：
# REPOSITORY    TAG       IMAGE ID       CREATED        SIZE
# hello-world   latest    feb5d9fea6a5   5 months ago   13.3kB
```

> 💡 註解：
> - `REPOSITORY`：Image 的名稱
> - `TAG`：版本標籤（預設是 `latest`）
> - `IMAGE ID`：Image 的唯一識別碼
> - `SIZE`：Image 的大小

## 4️⃣ 今日小任務

### 🎯 練習題

1. **操作題**：在你的電腦上執行上述所有指令，確認都能正常運行
2. **思考題**：為什麼 Container 啟動比 VM 快？嘗試用生活中的比喻解釋
3. **延伸題**：嘗試拉取並運行 `nginx` 這個 Image（這是一個網頁伺服器）

### 🆘 常見問題解答

| 問題 | 解答 |
|------|------|
| Docker 服務没启动？ | 執行 `sudo systemctl start docker`（Linux）或開啟 Docker Desktop |
| 權限不足？ | 執行 `sudo usermod -aG docker $USER` 將自己加入 docker 群組 |
| pull 很慢？ | 建議配置 Docker 鏡像加速器（阿裡雲、DaoCloud 等） |
