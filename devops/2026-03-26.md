## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 複習一下：Docker 的核心角色

想像你在**管理一家物流公司**：

| Docker 概念 | 生活比喻 | 說明 |
|-------------|---------|------|
| **Image（映像檔）** | 貨櫃的**設計圖** | 定義裡面要放什麼（軟體、系統、程式） |
| **Container（容器）** | 實際的**貨櫃** | 根據設計圖做出來的實體 |
| **Dockerfile** | 組裝說明書 | 告訴 Docker 怎麼建造貨櫃 |
| **Registry（倉庫）** | 設計圖圖書館 | 存放和分享設計圖的地方 |

### 今日重點：四大核心指令

今天要學的指令就像**物流公司的四大工具**：

1. **`docker run`** - 就像「打開并啟動一個貨櫃」
2. **`docker build`** - 就像「根據設計圖建造貨櫃」
3. **`docker ps`** - 就像「清點貨櫃清單」
4. **`docker exec`** - 就像「進入貨櫃裡面檢查」

## 2️⃣ 核心知識（每個指令都要有中文註解）

### 📦 docker run - 啟動容器

```bash
# 基本語法
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

# 常用範例：
```

| 選項 | 說明 | 比喻 |
|------|------|------|
| `-d` | detach mode，背景執行 | 貨櫃在倉庫裡運行，不擋住你的路 |
| `-p` | 端口映射（主機:容器） | 貨櫃的「送貨地址」 |
| `-it` | 互動模式（-i + -t） | 進入貨櫃裡面操作 |
| `--name` | 給容器取名字 | 給貨櫃貼標籤 |
| `-v` | 掛載 volume | 給貨櫃加一個「儲物櫃」 |
| `--rm` | 停止後自動刪除 | 用完就丟的紙盒 |
| `-e` | 設定環境變數 | 設定貨櫃內的溫度、濕度 |
| `--network` | 指定網路 | 選擇要用哪條公路 |

**範例コード：**

```bash
# 範例 1：最基本的運行
docker run nginx

# 範例 2：背景執行 + 端口映射
# -d: 背景執行，不會卡住 terminal
# -p 8080:80: 把主機的 8080 端口對應到容器的 80 端口
docker run -d -p 8080:80 nginx

# 範例 3：自定義名字 + 環境變數
# --name: 給容器取名「my-web-server」
# -e: 設定環境變數 TZ=Asia/Taipei（時區）
docker run -d --name my-web-server -e TZ=Asia/Taipei -p 80:80 nginx

# 範例 4：互動式容器（進入裡面操作）
# -it: 打開互動模式
# ubuntu: 使用 Ubuntu 映像檔
# /bin/bash: 執行 Bash  shell
docker run -it ubuntu /bin/bash

# 範例 5：掛載本地目錄
# -v /home/node/mywebsite:/usr/share/nginx/html
# 主機的 /home/node/mywebsite 對應到容器內的 /usr/share/nginx/html
docker run -v /home/node/mywebsite:/usr/share/nginx/html -p 80:80 nginx
```

### 🔨 docker build - 從 Dockerfile 建造映像檔

```bash
# 基本語法
docker build [OPTIONS] PATH | URL | -

# 常用選項：
# -t, --tag: 給映像檔取名（格式：名稱:標籤）
# -f, --file: 指定 Dockerfile 路徑
# --no-cache: 不使用快取，重新建構
# --progress=plain: 顯示詳細建構過程
```

**範例コード：**

```bash
# 範例 1：在目前目錄建構映像檔
# . 代表目前目錄（會找 Dockerfile）
# -t myapp:latest 給映像檔取名為 myapp，標籤為 latest
docker build -t myapp:latest .

# 範例 2：指定 Dockerfile 路徑
# -f ./Dockerfile.dev 指定使用哪個 Dockerfile
docker build -f ./Dockerfile.dev -t myapp:dev .

# 範例 3：從 Git repository 建構
# 從 GitHub 建構，需要指定 Git URL 和分支
docker build -t myapp:git https://github.com/user/myapp.git#main

# 範例 4：建構但不使用快取（確保每次重新下載）
docker build --no-cache -t myapp:latest .
```

### 📋 docker ps - 查看容器清單

```bash
# 基本語法
docker ps [OPTIONS]

# 常用選項：
# -a, --all: 顯示所有容器（包括已停止的）
# -q, --quiet: 只顯示容器 ID
# -l, --latest: 顯示最近創建的容器
# --format: 自訂輸出格式
# -n <數字>: 顯示最近 N 個容器
```

**範例コード：**

```bash
# 範例 1：只看正在運行的容器
docker ps

# 結果輸出：
# CONTAINER ID   IMAGE   COMMAND                  CREATED         STATUS        PORTS                NAMES
# a1b2c3d4e5f6   nginx   "/docker-entrypoint…"   5 seconds ago  Up 5 seconds  0.0.0.0:80->80/tcp   peaceful_mayer

# 範例 2：顯示所有容器（包括已停止的）
docker ps -a

# 範例 3：只顯示容器 ID（適合腳本用）
docker ps -q

# 範例 4：顯示最近 5 個容器
docker ps -n 5

# 範例 5：自訂格式輸出（只顯示名字和狀態）
docker ps --format "table {{.Names}}\t{{.Status}}"

# 範例 6：顯示容器 ID + 詳細資訊
docker ps -a --format "{{.ID}}: {{.Names}} - {{.Status}}"
```

### 🔧 docker exec - 進入容器內部操作

```bash
# 基本語法
docker exec [OPTIONS] CONTAINER COMMAND [ARG...]

# 常用選項：
# -i, --interactive: 保持 STDIN 開放（互動模式）
# -t, --tty: 分配虛擬終端機
# -d, --detach: 在背景執行
# -w, --workdir: 指定工作目錄
# -e, --env: 設定環境變數（可多個）
```

**範例コード：**

```bash
# 範例 1：進入容器執行命令（互動模式）
# a1b2c3d4e5f6 是容器 ID
# /bin/bash 執行 Bash
docker exec -it a1b2c3d4e5f6 /bin/bash

# 範例 2：執行單一命令（不用進入容器）
# 列出容器內的檔案
docker exec a1b2c3d4e5f6 ls -la

# 範例 3：在特定目錄執行命令
# -w /app 進入 /app 目錄後執行
docker exec -it -w /app a1b2c3d4e5f6 /bin/bash

# 範例 4：背景執行命令（不阻塞）
# -d 在背景執行
docker exec -d mycontainer touch /tmp/test.txt

# 範例 5：執行多個命令
docker exec -it a1b2c3d4e5f6 sh -c "ls -la && pwd"
```

### 🧹 docker rm / docker rmi - 清理

```bash
# 刪除容器
docker rm [OPTIONS] CONTAINER [CONTAINER...]

# 選項：
# -f, --force: 強制刪除（正在運行的容器也能刪）
# -v, --volumes: 同時刪除關聯的 volume

# 刪除映像檔
docker rmi [OPTIONS] IMAGE [IMAGE...]

# 選項：
# -f, --force: 強制刪除

# 範例：
docker rm mycontainer          # 刪除單一容器
docker rm -f mycontainer       # 強制刪除（正在運行也能刪）
docker rm $(docker ps -aq)     # 刪除所有容器
docker rmi myimage             # 刪除映像檔
docker rmi -f myimage          # 強制刪除映像檔
docker system prune            # 清理所有停止的容器、網路、build 快取
```

## 3️⃣ 實際操作（一步一步帶著做）

### 步驟 1：建構自己的第一個映像檔

首先，建立一個簡單的專案：

```bash
# 建立一個資料夾
mkdir -p /home/node/docker-demo
cd /home/node/docker-demo

# 建立一個簡單的網頁
echo "<h1>Hello Docker!</h1>" > index.html
```

建立 Dockerfile：

```bash
# 建立 Dockerfile（注意大小寫）
cat > Dockerfile << 'EOF'
# 使用輕量級的 Nginx 映像檔
FROM nginx:alpine

# 複製網頁檔案到容器內
COPY index.html /usr/share/nginx/html/

# 設定預設命令（這行可省略，因為 nginx 預設就會跑）
CMD ["nginx", "-g", "daemon off;"]
EOF
```

解說：
- `FROM nginx:alpine`：基底映像檔，用 Alpine 版本的 Nginx（更小）
- `COPY index.html`：把本機的 index.html 複製進容器
- `CMD`：容器啟動時執行的命令

### 步驟 2：建構映像檔

```bash
# 在當前目錄建構映像檔
# -t my-web:latest 給映像檔取名
# . 代表 Dockerfile 在當前目錄
docker build -t my-web:latest .

# 查看建構好的映像檔
docker images

# 輸出類似：
# REPOSITORY   TAG       IMAGE ID       CREATED        SIZE
# my-web       latest    abc123def456   10 seconds ago  23MB
# nginx        alpine    def789ghi012   3 days ago     23MB
```

### 步驟 3：運行並測試容器

```bash
# 背景執行容器
# -d: 背景執行
# -p 8080:80: 主機 8080 端口對應容器 80 端口
# --name web-test: 給容器取名
docker run -d -p 8080:80 --name web-test my-web:latest

# 查看運行的容器
docker ps

# 測試網頁是否能訪問
curl http://localhost:8080

# 輸出應該是：<h1>Hello Docker!</h1>
```

### 步驟 4：進入容器內部看看

```bash
# 進入容器內部的 Bash
docker exec -it web-test /bin/sh

# 在容器內可以執行：
# ls -la                    # 列出檔案
# cat /usr/share/nginx/html/index.html  # 查看網頁內容
# ps aux                    # 查看行程
# exit                      # 離開容器

# 示範：直接執行命令（不用進入）
docker exec web-test cat /usr/share/nginx/html/index.html
```

### 步驟 5：修改內容並重新建構

```bash
# 修改網頁內容
echo "<h1>Hello Docker - Updated!</h1>" > index.html

# 重新建構（會使用快取，只有變更的層會重新建構）
docker build -t my-web:latest .

# 停止舊容器
docker stop web-test

# 刪除舊容器
docker rm web-test

# 用新映像檔啟動新容器
docker run -d -p 8080:80 --name web-test my-web:latest

# 測試更新後的內容
curl http://localhost:8080

# 輸出應該是：<h1>Hello Docker - Updated!</h1>
```

### 步驟 6：清理環境

```bash
# 停止容器
docker stop web-test

# 刪除容器
docker rm web-test

# 刪除映像檔
docker rmi my-web:latest

# 或者使用一鍵清理（小心！）
docker system prune -a
```
