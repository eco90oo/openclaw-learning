## 1️⃣ 基礎概念：用生活比喻搞懂容器化

### 🐳 什麼是 Docker？讓我們用郵輪貨櫃來比喻！

大家一定有看過港口的**貨櫃輪**吧？為什麼全球貿易都使用標準大小的貨櫃？

**答案**：因為**標準化**！無論裡面放的是玩具、食品還是電子產品，全部用同一種規格的貨櫃，這樣起重機、卡車、船舶都能統一處理，效率超高！

**Docker 就是資訊世界的「貨櫃系統」**：

| 現實世界 | Docker 世界 |
|---------|------------|
| 貨櫃 (Container) | 容器 - 標準化的軟體封裝 |
| 貨櫃輪船 | Docker Engine - 運輸工具 |
| 起重機 | Docker CLI - 操作介面 |
| 貨櫃規格 | Docker Image - 模板/藍圖 |
| 港口 | Docker Hub - 存放點 |

### 🏠 容器 vs 虛擬機：用公寓大樓來比喻

#### 傳統虛擬機（VM）- 就像獨棟別墅🏠
- 每個別墅都有獨立的土地、獨立的供電系統、獨立的水管
- 隔離性極佳，但**資源浪費嚴重**
- 啟動慢（像蓋房子），佔用空間大

#### Docker 容器 - 就像酒店房間🏨
- 共享大樓的電力、水管、保全系統
- 每個房間有自己的獨立空間和隱私
- 隔離性足夠，**資源利用率高**
- 啟動超快（像開門入住），佔用空間小

### 💡 為什麼需要容器化？

**開發人員的痛點**：
```
😫 "在我的機器上可以執行啊..."
😫 "這個版本跟正式環境不一樣..."
😫 "安裝環境要花 3 天..."
```

**容器化的好處**：
1. **一致性を保證**（保持一致性）- 開發、測試、生產環境都一樣
2. **快速部署** - 幾秒鐘就能啟動一個完整的服務
3. **資源隔離** - 每個容器獨立運行，互不干擾
4. **弹性擴展** - 需要多少容器就複製多少

## 2️⃣ 核心知識：Docker 核心組成

### 🧩 Docker 三大核心元件

```
┌────────────────────────────────────────────────────────┐
│                     Docker 生態系                       │
├────────────────────────────────────────────────────────┤
│                                                        │
│   ┌─────────────┐     ┌─────────────┐                  │
│   │    Image    │────▶│  Container  │                  │
│   │   (模板)     │     │   (實例)     │                  │
│   └─────────────┘     └─────────────┘                  │
│         ▲                    │                         │
│         │                    │                         │
│   ┌─────┴─────┐        ┌─────┴─────┐                  │
│   │  Registry │        │  Docker   │                  │
│   │ (存放處)  │        │  Engine   │                  │
│   └───────────┘        └───────────┘                  │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 1. Docker Image（映像檔）- 就像是食譜🥣
- **定義**：一個唯讀的模板，包含運行應用所需的所有東西
- **包含**：程式碼、函式庫、環境變數、配置文件
- **特性**：可以版本控制、可以重複使用、像樂高一樣堆疊

#### 2. Docker Container（容器）- 就像是按照食譜做出來的菜🍳
- **定義**：Image 的運行實例
- **特性**：可讀寫層、獨立進程、隨時創建/刪除
- **生命週期**：創建 → 運行 → 暫停 → 停止 → 刪除

#### 3. Docker Registry（倉庫）- 就像是餐廳的食材儲藏室🏪
- **存放**： готовых Docker Image
- **種類**：
  - Docker Hub（公開/官方）
  - Amazon ECR
  - Google Container Registry
  - 私有 Registry

### 🔄 Docker 工作流程

```
1. 開發者寫 Dockerfile（食譜）
        │
        ▼
2. docker build（按照食譜做菜）
        │
        ▼
3. docker push（把菜放到餐廳儲藏室）
        │
        ▼
4. docker pull（從儲藏室拿菜）
        │
        ▼
5. docker run（把菜端上桌/啟動容器）
```

## 3️⃣ 實際操作：安裝 Docker

### 📦 在 Linux 安裝 Docker（以 Ubuntu 為例）

```bash
# Step 1: 更新系統套件列表（確保取得最新版本資訊）
sudo apt-get update

# Step 2: 安裝依賴套件（允許 apt 透過 HTTPS 下載套件）
sudo apt-get install -y \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

# Step 3: 新增 Docker 的官方 GPG 金鑰（驗證套件來源是否正確）
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Step 4: 設定 Docker 官方 APT 倉庫（讓系統知道要去哪裡下載 Docker）
echo \
    "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
    https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Step 5: 再次更新套件列表（現在會包含 Docker 的套件）
sudo apt-get update

# Step 6: 安裝 Docker Engine（本體）+ containerd（容器運行時）+ docker-compose（編排工具）
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Step 7: 驗證 Docker 是否安裝成功（顯示版本號）
docker --version
```

### ✅ 確認 Docker 安裝成功

```bash
# 顯示 Docker 版本資訊（確認安裝正確）
docker --version

# 顯示更詳細的資訊（包含 Build, API 版本等）
docker info
```

如果看到類似以下的輸出，代表安裝成功：
```
Docker version 24.0.7, build 311b9ff
Docker Compose version v2.23.0
```

### 🐳 讓非 root 使用者也能使用 Docker（可選）

```bash
# 建立 docker 使用者群組（避免每次都要用 sudo）
sudo usermod -aG docker $USER

# 重新登入後生效，或者用以下命令立即生效
newgrp docker
```

## 4️⃣ 今日小任務：Hello World 容器初體驗

### 🎯 任務目標
完成 Docker 安裝後，運行第一個容器！

### 📝 執行步驟

```bash
# 1. 拉取官方 hello-world 映像檔（從 Docker Hub 下載到本地）
docker pull hello-world

# 2. 列出本地所有映像檔（確認 hello-world 已下載）
docker images

# 3. 運行 hello-world 容器（會在螢幕上顯示一段說明訊息）
docker run hello-world

# 4. 列出正在運行的容器（此時 hello-world 已結束）
docker ps

# 5. 列出所有容器（包括已停止的，-a 代表 all）
docker ps -a
```

### 🎓 預期輸出
執行 `docker run hello-world` 時，你會看到：
```
Hello from Docker!
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
 1. The Docker client contacted the Docker daemon.
  2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
    (amd64)
  3. The Docker daemon created a new container from that image which runs the
    executable that produces the output you are currently reading.
  4. The Docker daemon streamed that output to the Docker client, which sent it
    to your terminal.
```
