## 1️⃣ 基礎概念（用生活比喻）

### 🐳 Docker 是什麼？

**比喻：餐廳的外帶便當盒**

想象你要開一家餐廳，客人可能點：
- 義大利麵
- 咖哩飯
- 壽司

以前廚師要為每道菜準備獨立的爐具、鍋具、調料，麻煩死了！

**Docker 的概念就像「標準化便當盒」：**
- 每一個「便當盒」裡，已經包含了那份菜需要的所有東西（程式、資料庫、系統設定）
- 這個便當盒可以在任何廚房（電腦、伺服器、雲端）打開就開始煮
- 不會出現「在家做很好吃，餐廳做不好吃」的問題

> **Docker = 把應用程式和它的環境包裝在一起的技術**

### ☸️ Kubernetes (K8s) 是什麼？

**比喻：大型主題樂園的經營管理**

假設你開了一個主題樂園，裡面有很多設施：
- 雲霄飛車（人多的時候要多幾節、人少可以減少）
- 旋轉木馬（隨時待命）
- 紀念品店（根據天氣調整庫存）

**以前的方式：** 人工調度，累死主管
**Kubernetes 的方式：** 自動化管理！

- **自動擴充**：人多的時候，自動多開幾台雲霄飛車；人少就減少
- **故障復原**：如果一台雲霄飛車壞了，系統自動啟動備用的
- **負載平衡**：把客人平均分配到各個設施，不要讓某個地方排隊太久

> **Kubernetes = 自動化管理眾多 Docker 容器（便當盒）的系統**

### 🔄 CI/CD 是什麼？

**比喻：工廠的生產線**

想象一間玩具工廠：
1. **設計師畫好圖** → 2. **師傅做出樣品** → 3. **品管檢查** → 4. **包裝出貨**

**DevOps 的 CI/CD 就是軟體的自動化生產線：**

| 階段 | 全名 | 比喻 |
|------|------|------|
| CI | Continuous Integration（持續整合） | 設計師每次畫完圖，馬上有人檢查對不對 |
| CD | Continuous Delivery（持續交付） | 檢查通過後，玩具自動包裝好準備出貨 |

**簡單說：** 開發者寫好程式碼 → 自動測試 → 自動部署到伺服器

## 2️⃣ 核心知識（專有名詞解釋）

### Docker 相關

| 專有名詞 | 白話解釋 |
|----------|----------|
| **Image（映像檔）** | 就像「食譜」，告訴電腦這個便當盒需要什麼材料 |
| **Container（容器）** | 實際做出來的「便當盒」，可以打開來用 |
| **Dockerfile** | 寫著「如何做這個便當」的食譜書 |
| **Volume（ Volume）** | 讓容器和外面共享資料的「傳送門」 |

### Kubernetes 相關

| 專有名詞 | 白話解釋 |
|----------|----------|
| **Pod** | K8s 管理的最小單位，通常是「一個或幾個相關的便當盒」 |
| **Deployment** | 告訴 K8s「你要管理哪些便當盒」的說明書 |
| **Service** | 讓外界可以找到並訪問這些便當盒的「地址牌」 |
| **Namespace** | 幫便當盒分門別類的「置物櫃」 |

### CI/CD 相關

| 專有名詞 | 白話解釋 |
|----------|----------|
| **Pipeline（管線）** | 自動化生產線，一關過了一關 |
| **Build（建置）** | 把程式碼變成可執行的程式 |
| **Test（測試）** | 檢查程式有沒有壞掉 |
| **Deploy（部署）** | 把程式放到伺服器上 |

## 3️⃣ 實際操作（步驟教學）

### 🐳 步驟一：安裝 Docker（Windows/Mac）

1. **下載 Docker Desktop**
   - 前往 https://www.docker.com/products/docker-desktop
   - 點擊「Download for Windows」或「Download for Mac」

2. **安裝程式**
   - 執行下載的安裝檔
   - 按照指示一直點「下一步」

3. **啟動 Docker**
   - 安裝完成後，會看到 Docker Desktop 視窗
   - 看到綠色的「Docker is running」就成功了！

### 🐳 步驟二：跑第一個 Docker 容器

**目標：讓 Docker 幫你跑一個簡單的網頁伺服器**

打開終端機（Windows 用 PowerShell，Mac 用 Terminal）：

```bash
# 這句話的意思是：請 Docker 幫我找一個叫 nginx 的映像檔並且跑起來
docker run -d -p 8080:80 nginx
```

**解讀這行命令：**
- `docker run` = 幫我跑一個容器
- `-d` = 在背景執行（不要擋住終端機）
- `-p 8080:80` = 把電腦的 8080 端口映射到容器的 80 端口
- `nginx` = 要跑的程式名稱（這是一個網頁伺服器）

**驗證結果：**
1. 打開瀏覽器
2. 輸入 `http://localhost:8080`
3. 你應該會看到 nginx 的歡迎頁面！

**常用 Docker 命令：**

```bash
# 查看正在運行的容器
docker ps

# 停止容器
docker stop [容器ID]

# 查看所有容器（包括停止的）
docker ps -a

# 刪除容器
docker rm [容器ID]
```

### ☸️ 步驟三：安裝 Kubernetes（學習用）

**要在本地學習 K8s，最簡單的方式是使用 Minikube！**

1. **安裝 Minikube（Windows）：**
```powershell
# 用管理員權限開啟 PowerShell
winget install minikube
```

2. **安裝 kubectl（控制 K8s 的工具）：**
```powershell
winget install kubernetes-kubectl
```

3. **啟動 Minikube：**
```bash
minikube start
```

### ☸️ 步驟四：在 K8s 部署一個簡單應用

**目標：把剛才的 nginx 部署到 K8s 裡**

```bash
# 創建一個 Deployment（部署設定）
kubectl create deployment my-nginx --image=nginx

# 讓外界可以訪問
kubectl expose deployment my-nginx --type=NodePort --port=80

# 查看部署狀態
kubectl get pods
kubectl get services
```

**獲得訪問網址：**
```bash
minikube service my-nginx
```

### 🔄 步驟五：動手玩 CI/CD（GitHub Actions）

**目標：讓 GitHub 自動幫你測試和部署**

1. **建立一個 GitHub 倉庫**
   - 前往 github.com
   - 點擊「New repository」
   - 取名為 `my-first-cicd`

2. **建立一個簡單的程式**
   - 在倉庫建立 `index.html`：
   ```html
   <!DOCTYPE html>
   <html>
   <head><title>我的第一個網站</title></head>
   <body>
       <h1>🎉 Hello DevOps!</h1>
   </body>
   </html>
   ```

3. **建立 CI/CD 流程檔**
   - 在倉庫建立 `.github/workflows/deploy.yml`：
   ```yaml
   name: Deploy to GitHub Pages
   
   on:
     push:
       branches: [main]
   
   jobs:
     deploy:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v3
         
         - name: Deploy
           uses: peaceiris/actions-gh-pages@v3
           with:
             github_token: ${{ secrets.GITHUB_TOKEN }}
             publish_dir: ./
   ```

4. **啟用 GitHub Pages**
   - 進入倉庫 Settings → Pages
   - Source 選擇「Deploy from a branch」
   - Branch 選擇「gh-pages」

5. **推送到 GitHub！**
   ```bash
   git add .
   git commit -m "Add CI/CD"
   git push origin main
   ```

6. **觀察結果**
   - 進入倉庫的「Actions」頁面
   - 你會看到自動化流程正在跑！
   - 跑完後，會得到一個網址可以訪問你的網頁
