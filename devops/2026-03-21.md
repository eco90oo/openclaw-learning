## 1️⃣ 基礎概念（用生活比喻）

### 什麼是 DevOps？想像你在開餐廳！

想象你要開一家餐廳，傳統的做法是：
- 廚師（開發者）只負責做菜
- 服務生（運維）只負責端菜
- 兩邊互不溝通，結果：菜涼了、客人抱怨、廚師不知道客人愛吃什麼

**DevOps** 就是讓開發和運維變成一個團隊，大家一起為了「讓客人吃到熱騰騰的菜」而努力！

> 🏷️ **核心口號**：開發快、部署快、出錯少！

---

### Docker 是什麼？就像美食的「保鮮盒」！

**問題**：廚師在家做了一道菜，端到餐廳卻變味了？
- 因為家裡的爐火不一樣、調味料不一樣、廚房環境不一樣

**Docker** 的解決方案：
- 就像把所有食材、調味料、烹飪步驟都打包進一個「保鮮盒」
- 不管在哪裡，只要打開盒子，都能做出同一道菜
- 這個「保鮮盒」叫做 **Container（容器）**

---

### Kubernetes (K8s) 是什麼？就像「餐廳管理系統」！

假設你開了 10 家分店：
- 每家店要幾個廚師？幾個服務生？
- 哪家店客人多？要不要調派人手？
- 某家店廚師生病了，怎麼快速調人頂替？

**Kubernetes** 就是幫你管理這一切的系統：
- 自動調配資源（人力）
- 健康檢查（有人不舒服就替換）
- 擴充/縮減（生意好就多請人，生意差就少請）

> 📝 K8s 是 Kubernetes 的簡稱（因為 K 和 s 中間有 8 個字母）

---

### CI/CD 是什麼？就像「流水線工廠」！

**傳統做法**（手工組裝）：
1. 開發者寫完程式 → 手工測試 → 手工打包 → 手工上傳
2. 慢！容易錯！沒人敢改程式！

**CI/CD 流水線**（自動化）：
1. 開發者寫完程式 → **自動測試** → **自動打包** → **自動部署**
2. 像工廠流水線一樣，快速又穩定！

> 📝 CI = Continuous Integration（持續整合）
> 📝 CD = Continuous Deployment（持續部署）

## 2️⃣ 核心知識（專有名詞解釋）

### Docker 核心概念

| 名詞 | 白話解釋 | 比喻 |
|------|---------|------|
| **Image（映像檔）** | 模板/食譜 | 食譜書（寫好怎麼做菜的配方） |
| **Container（容器）** | 實際做出來的菜 | 端上桌的料理 |
| **Dockerfile** | 制作料理的食譜 | 烹飪步驟說明書 |
| **Volume** | 存放數據的硬碟 | 冰箱（存放食材） |
| **Network** | 容器之間的通訊 | 餐廳內部對講機 |

### Kubernetes 核心概念

| 名詞 | 白話解釋 | 比喻 |
|------|---------|------|
| **Pod** | 最小的部署單元 | 一個員工 |
| **Node** | 工作伺服器 | 一家分店 |
| **Service** | 對外服務的門面 | 餐廳大門 |
| **Deployment** | 管理多個 Pod | 人事部門 |
| **Ingress** | 入口（外部訪問） | 餐廳大門口接待員 |
| **ConfigMap** | 設定檔 | 菜單 |
| **Secret** | 敏感資料（密碼鑰匙） | 保險箱密碼 |

### CI/CD 核心概念

| 名詞 | 白話解釋 |
|------|---------|
| **Pipeline** | 一條龍的自動化流程 |
| **Build** | 編譯程式 |
| **Test** | 自動化測試 |
| **Deploy** | 部署到伺服器 |
| **Artifact** | 最終產出的檔案（像 APK、ZIP） |
| **Stage** | 流程中的階段 |

## 3️⃣ 實際操作（步驟教學）

### 🐳 Part 1：Docker 實戰

#### 步驟 1：安裝 Docker（Ubunu/CentOS）

```bash
# Ubuntu
sudo apt update
sudo apt install docker.io

# 啟動 Docker
sudo systemctl start docker
sudo systemctl enable docker

# 確認安裝成功
docker --version
```

#### 步驟 2：幫你的程式穿衣服（建立 Dockerfile）

假設你有一個簡單的 Node.js 程式 `app.js`：

```javascript
// app.js
const http = require('http');
const server = http.createServer((req, res) => {
  res.send('Hello DevOps! 🎉');
});
server.listen(3000, () => console.log('Server running!'));
```

建立 `Dockerfile`（就像寫食譜）：

```dockerfile
# 第一步：選基礎食材（要用什麼環境）
FROM node:18

# 第二步：設置工作地點
WORKDIR /app

# 第三步：把程式碼放進來
COPY package.json ./
COPY app.js ./

# 第四步：安裝依賴（就像買調味料）
RUN npm install

# 第五步：啟動程式
CMD ["node", "app.js"]
```

#### 步驟 3：把程式包成容器

```bash
# 进入项目目录
cd /home/node/.openclaw/workspace

# 建立 Docker 映像檔（就像把食譜做成料理包）
docker build -t my-first-app .

# 看看我們做了什麼
docker images

# 啟動容器（把料理包打開端上桌）
docker run -p 3000:3000 my-first-app

# 測試一下
curl http://localhost:3000
```

> 🎉 **成功了！** 你的程式已經被 Docker 包起來了，無論在哪裡都能運行！

#### 步驟 4：常用 Docker 命令

```bash
# 列出正在運行的容器
docker ps

# 列出所有容器（包括停止的）
docker ps -a

# 停止容器
docker stop <容器ID>

# 刪除容器
docker rm <容器ID>

# 查看容器日誌
docker logs <容器ID>

# 進入容器內部（像打開冰箱看裡面）
docker exec -it <容器ID> bash
```

### ☸️ Part 2：Kubernetes 實戰

#### 步驟 1：安裝 Minikube（在本地玩 K8s）

```bash
# 安裝 minikube（學習用，本地跑 K8s）
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# 啟動 minikube
minikube start

# 確認狀態
minikube status
```

#### 步驟 2：把你的 Docker 映像放到 K8s

先把映像放到 Docker Hub（或其他 registry）：

```bash
# 登入 Docker Hub
docker login

# 標記映像
docker tag my-first-app your-username/my-first-app:latest

# 上傳
docker push your-username/my-first-app:latest
```

#### 步驟 3：寫 K8s 部署文件（Deployment）

建立 `deployment.yaml`：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app-deployment
spec:
  replicas: 3  # 需要 3 個員工
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: your-username/my-first-app:latest
        ports:
        - containerPort: 3000
        # 健康檢查
        livenessProbe:
          httpGet:
            path: /
            port: 3000
          initialDelaySeconds: 5
          periodSeconds: 10
```

#### 步驟 4：部署到 K8s

```bash
# 安裝 kubectl（K8s 的命令工具）
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install kubectl /usr/local/bin/

# 部署！
kubectl apply -f deployment.yaml

# 查看部署狀態
kubectl get pods
kubectl get deployment

# 查看詳細
kubectl describe deployment my-app-deployment

# 擴充到 5 個副本
kubectl scale deployment my-app-deployment --replicas=5

# 刪除部署
kubectl delete -f deployment.yaml
```

> 🚀 **太棒了！** 你已經把應用程式部署到 K8s 了！

### 🔄 Part 3：CI/CD 實戰（用 GitHub Actions）

#### 步驟 1：建立 GitHub Actions 設定

在專案根目錄建立 `.github/workflows/deploy.yml`：

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    
    steps:
    # 1. 取出程式碼
    - uses: actions/checkout@v3
    
    # 2. 設定 Node.js 環境
    - uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
    
    # 3. 安裝依賴
    - run: npm install
    
    # 4. 執行測試
    - run: npm test
    
    # 5. 建立 Docker 映像
    - name: Build Docker image
      run: docker build -t my-app:${{ github.sha }} .
    
    # 6. 登入 Docker Hub
    - name: Login to Docker Hub
      uses: docker/login-action@v2
      with:
        username: ${{ secrets.DOCKER_USERNAME }}
        password: ${{ secrets.DOCKER_PASSWORD }}
    
    # 7. 推送 Docker 映像
    - name: Push Docker image
      run: |
        docker tag my-app:${{ github.sha }} ${{ secrets.DOCKER_USERNAME }}/my-app:latest
        docker push ${{ secrets.DOCKER_USERNAME }}/my-app:latest
    
    # 8. 部署到 K8s（如果有 K8s 叢集）
    - name: Deploy to Kubernetes
      run: |
        kubectl set image deployment/my-app my-app=${{ secrets.DOCKER_USERNAME }}/my-app:latest
      env:
        KUBECONFIG: ${{ secrets.KUBECONFIG }}
```

> 🔐 **記得設定密碼！**
> 在 GitHub 專案設定中，加入這些 secrets：
> - DOCKER_USERNAME
> - DOCKER_PASSWORD
> - KUBECONFIG

### 📊 Part 5：監控與日誌

#### 安裝 Prometheus + Grafana（監控系統）

```yaml
# prometheus-config.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: prometheus-config
data:
  prometheus.yml: |
    global:
      scrape_interval: 15s
    scrape_configs:
    - job_name: 'my-app'
      static_configs:
      - targets: ['my-app-service:3000']
```

```bash
# 安裝 Prometheus
kubectl apply -f https://raw.githubusercontent.com/prometheus-operator/prometheus-operator/main/bundle.yaml

# 部署 Prometheus
kubectl apply -f prometheus-config.yaml
```

#### 常見日誌命令

```bash
# Docker 看日誌
docker logs -f <容器ID>

# K8s 看日誌
kubectl logs -f deployment/my-app-deployment

# 查詢特定時間的日誌
kubectl logs --since=1h deployment/my-app
```
