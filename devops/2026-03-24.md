## 1️⃣ 基礎概念（用生活比喻）

### 🎯 今天的比喻： Docker 是「物流公司」

想像 Docker 是一間**物流公司**：

| 物流概念 | Docker 對應 | 比喻說明 |
|---------|-----------|---------|
| 貨櫃 | **Container 容器** | 標準化的箱子，不管裝什麼都能統一運輸 |
| 貨櫃碼頭 | **Docker Host 主機** | 存放和管理所有貨櫃的地方 |
| 貨櫃之間的通道 | **Network 網路** | 讓不同貨櫃能互相溝通 |
| 貨櫃裡的儲物箱 | **Volume 卷** | 存放重要物品，搬動貨櫃時不會遺失 |

### 🔌 什麼是 Docker 網路？

**比喻：**  
想象你有兩家餐廳：
- A 餐廳（廚房）
- B 餐廳（外場）

如果沒有網路，A 餐廳做的菜無法送到 B 餐廳，客人就沒東西吃。

Docker 網路就是**連接不同容器的道路**：

```
┌─────────────┐      網路（道路）      ┌─────────────┐
│  Web 容器   │ ◀─────────────────▶ │  DB 容器    │
│  (前端餐廳) │                      │  (廚房)     │
└─────────────┘                      └─────────────┘
```

### 📦 什麼是 Docker Volume？

**比喻：**  
想像你在船上生活，船（容器）會移動，但有些東西你不希望跟著船走：

- 船上的傢俱（容器內的檔案）→ 會隨船丟棄
- 陸地上的倉庫（Volume）→ 永遠存在，隨時存取

**Volume 就是「陸地上的倉庫」**，讓資料獨立於容器之外。

## 2️⃣ 核心知識（專有名詞詳解）

### 🌐 Docker 網路類型

Docker 提供多種網路驅動，就像不同的道路類型：

| 網路類型 | 用途 | 比喻 |
|---------|------|------|
| **bridge** | 默認網路，單主機內容器互聯 | 社區內的道路 |
| **host** | 移除容器隔離，直接用主機網路 | 沒有大門的建築 |
| **overlay** | 跨主機容器互聯（如 Swarm） | 高速公路連接不同城市 |
| **none** | 完全隔離，無網路 | 與世隔絕的孤島 |

### 📦 Docker Volume 類型

| Volume 類型 | 用途 | 比喻 |
|------------|------|------|
| **named volume** | 持久化資料，支援多容器共享 | 公共倉庫 |
| **bind mount** | 映射主機特定目錄到容器 | 專屬貨架 |
| **tmpfs mount** | 暫存資料，只存在於記憶體 | 隨手筆記（斷電消失） |
| **anonymous volume** | 臨時儲存，容器刪除後消失 | 餐巾紙（用完即丟） |

## 3️⃣ 實際操作（一步一步帶著做）

### 🎯 實戰一：認識預設網路

**Step 1：查看目前網路**
```bash
# 列出所有 Docker 網路
# docker network ls = list networks（列出網路）
docker network ls

# 輸出會像：
# NETWORK ID     NAME        DRIVER    SCOPE
# abc123...      bridge      bridge    local
# def456...      host        host      local
# ghi789...      none        null      local
```

**Step 2：查看 bridge 網路詳細資訊**
```bash
# docker network inspect = 檢視網路詳細資訊
# bridge 是預設的網路驅動程式
docker network inspect bridge
```

### 🎯 實戰二：建立自訂網路

**比喻：建立一條社區內的道路**

**Step 1：建立一個 bridge 網路**
```bash
# docker network create = 建立新網路
# --driver bridge = 使用橋接網路驅動（預設）
# my-network = 網路名稱（你自己取的名字）
docker network create --driver bridge my-network
```

**Step 2：啟動容器並加入這個網路**
```bash
# docker run = 運行容器
# -d = 在背景運行（detached mode）
# --network my-network = 把容器加入自訂網路
# --name web = 幫容器取個名字
# nginx:alpine = 使用輕量版 Nginx 映像檔
docker run -d --network my-network --name web nginx:alpine
```

**Step 3：再啟動另一個容器加入同一網路**
```bash
# 啟動一個 Debian 系統的容器
docker run -d --network my-network --name app debian:latest sleep infinity
```

**Step 4：測試容器間的通訊**
```bash
# docker exec = 在執行中的容器裡執行命令
# -it = 互動模式（interactive terminal）
# app = 容器名稱
# ping -c 3 = 發送 3 次 ping 測試
docker exec -it app ping -c 3 web

# 如果成功，會看到：
# PING web (172.18.0.2) 56(84) bytes of data.
# 64 bytes from 172.18.0.2: icmp_seq=1 ttl=64 time=0.XXX ms
```

**📝 指令解說：**
- 兩個容器在**同一個網路**中，可以透過**容器名稱**直接通訊
- 這就像在同一個社區裡，你可以叫鄰居的名字找到他

### 🎯 實戰三：建立網路並連接資料庫

**比喻：讓網站連接到資料庫（廚房要能連接到倉庫）**

**Step 1：建立一個網路**
```bash
docker network create app-network
```

**Step 2：啟動資料庫容器（PostgreSQL）**
```bash
# docker run = 運行容器
# -d = 在背景運行
# --network app-network = 加入網路
# --name postgres = 容器名稱
# -e POSTGRES_PASSWORD=secret = 設定環境變數（資料庫密碼）
# -e POSTGRES_DB=mydb = 設定環境變數（資料庫名稱）
# postgres:15 = PostgreSQL 15 版本映像檔
docker run -d --network app-network --name postgres \
  -e POSTGRES_PASSWORD=secret \
  -e POSTGRES_DB=mydb \
  postgres:15
```

**Step 3：啟動網站容器並連接到資料庫**
```bash
# 啟動一個簡單的 Node.js 應用
docker run -d --network app-network --name webapp \
  -e DATABASE_URL=postgres://postgres:secret@postgres:5432/mydb \
  node:18-alpine \
  sh -c "echo '已連接到資料庫：$DATABASE_URL'"
```

**Step 4：驗證網路連接**
```bash
# 在 webapp 容器中測試連接到 postgres
docker exec -it webapp sh -c "apk add --no-cache postgresql-client && psql -h postgres -U postgres -d mydb -c 'SELECT 1;'"

# 如果成功，會看到：
# SELECT 1
# --------
#      1
# (1 row)
```

### 🎯 實戰四：使用 Named Volume 持久化資料

**比喻：建立一個永久倉庫，資料不會隨容器消失**

**Step 1：建立一個 named volume**
```bash
# docker volume create = 建立 Volume（卷）
# mydata = Volume 名稱
docker volume create mydata
```

**Step 2：查看 Volume 列表**
```bash
# docker volume ls = 列出所有 Volume
docker volume ls

# 輸出：
# DRIVER    VOLUME NAME
# local     mydata
```

**Step 3：使用 Volume 啟動容器**
```bash
# docker run = 運行容器
# -v mydata:/data = 將名為 mydata 的 Volume 掛載到容器的 /data 目錄
# -d = 在背景運行
# --name data-container = 容器名稱
# debian:latest = 使用 Debian 映像檔
# 這裡的 -v 是 volume（卷）的簡寫
docker run -v mydata:/data -d --name data-container debian:latest
```

**Step 4：在 Volume 中寫入資料**
```bash
# 在容器的 /data 目錄建立一個檔案
# echo "Hello Volume" = 寫入文字
# > /data/hello.txt = 輸出到 hello.txt
docker exec data-container sh -c "echo '這是我的重要資料' > /data/hello.txt"
```

**Step 5：刪除容器，驗證資料還在**
```bash
# docker rm = 刪除容器（-f 強制刪除）
docker rm -f data-container

# 建立新容器，掛載同一個 Volume
docker run -v mydata:/data -d --name new-container debian:latest

# 檢查資料是否還在
docker exec new-container cat /data/hello.txt

# 輸出：這是我的重要資料
```

### 🎯 實戰五：使用 Bind Mount 映射主機目錄

**比喻：像在餐廳裡設定一個「專屬貨架」，直接放東西**

**Step 1：建立主機上的目錄**
```bash
# mkdir = make directory（建立目錄）
# -p = 建立多層目錄
mkdir -p ~/docker-test/app
```

**Step 2：在主機目錄建立測試檔案**
```bash
# echo = 輸出文字
# > index.html = 寫入檔案
echo '<h1>這是從主機掛載的網頁</h1>' > ~/docker-test/app/index.html
```

**Step 3：使用 bind mount 啟動 Nginx**
```bash
# docker run = 運行容器
# -v ~/docker-test/app:/usr/share/nginx/html = 
#   主機的 ~/docker-test/app 目錄
#   映射到容器的 /usr/share/nginx/html 目錄（ Nginx 網頁根目錄）
# -p 8080:80 = 映射端口（主機8080 → 容器80）
# -d = 在背景運行
# --name nginx-mounted = 容器名稱
# nginx:alpine = Nginx 輕量版映像檔
docker run -v ~/docker-test/app:/usr/share/nginx/html -p 8080:80 -d --name nginx-mounted nginx:alpine
```

**Step 4：驗證結果**
```bash
# 打開瀏覽器訪問 http://localhost:8080
# 或者使用 curl 測試
curl http://localhost:8080

# 應該看到：<h1>這是從主機掛載的網頁</h1>
```

**Step 5：修改主機檔案，即時更新**
```bash
# 修改主機上的檔案
echo '<h1>即時更新的內容！</h1>' > ~/docker-test/app/index.html

# 再次訪問，應該看到新內容
curl http://localhost:8080

# 輸出：<h1>即時更新的內容！</h1>
```

### 🎯 實戰六：使用 tmpfs 進行高效暫存

**比喻：就像隨手寫在黑板上的備忘錄，電腦關機就消失**

**Step 1：使用 tmpfs 啟動容器**
```bash
# docker run = 運行容器
# --tmpfs /tmp = 將容器的 /tmp 目錄掛載到記憶體
# -d = 在背景運行
# --name tmpfs-demo = 容器名稱
# debian:latest = 映像檔
docker run --tmpfs /tmp -d --name tmpfs-demo debian:latest
```

**Step 2：驗證 tmpfs**
```bash
# docker exec = 在容器中執行命令
# df -h = 查看磁碟使用情況
docker exec tmpfs-demo df -h

# 應該看到 /tmp 是 tmpfs（記憶體檔案系統）
# tmpfs           64M     0   64M      0% /tmp
```

**📝 補充說明：**
- tmpfs 的優點：**速度超快**（因為在記憶體中）
- tmpfs 的缺點：**重開機資料不見**
- 適合場景：暫存檔、session 資料、緩存

### 🎯 實戰七：多容器網路應用範例

**比喻：完整的餐廳系統（網站 + 資料庫 + 快取）**

**Step 1：建立 docker-compose.yml**
```yaml
# docker-compose 版本（相容性）
version: '3.8'

# services（服務）= 要啟動的容器
services:
  # 網站服務
  web:
    image: nginx:alpine
    # ports（端口映射）：主機:容器
    ports:
      - "8080:80"
    # networks（網路）：加入哪個網路
    networks:
      - app-network
    # volumes（卷）：掛載哪些磁碟
    volumes:
      - ./html:/usr/share/nginx/html
    # depends_on（依賴）：等待哪個服務先啟動
    depends_on:
      - cache
      - database

  # 快取服務
  cache:
    image: redis:alpine
    networks:
      - app-network

  # 資料庫服務
  database:
    image: postgres:15
    # 環境變數（敏感資訊）
    environment:
      POSTGRES_PASSWORD: secret
      POSTGRES_DB: myapp
    networks:
      - app-network
    # volumes（持久化）
    volumes:
      - db-data:/var/lib/postgresql/data

# networks（網路定義）
networks:
  app-network:
    driver: bridge

# volumes（卷定義）
volumes:
  db-data:
```

**Step 2：建立 HTML 目錄和檔案**
```bash
mkdir -p html
echo '<h1>Docker Compose 多容器應用</h1>' > html/index.html
```

**Step 3：啟動所有服務**
```bash
# docker-compose up = 啟動所有服務
# -d = 在背景運行
docker-compose up -d
```

**Step 4：查看運行的容器**
```bash
# docker-compose ps = 查看服務狀態
docker-compose ps

# 輸出：
# NAME                IMAGE       COMMAND                  SERVICE    CREATED   STATUS
# myapp-cache-1      redis:alpine   "docker-entrypoint.sh"   cache     ...       Up
# myapp-database-1   postgres:15    "postgres"              database  ...       Up
# myapp-web-1        nginx:alpine    "/docker-entrypoint.sh"  web       ...       Up
```

**Step 5：停止所有服務**
```bash
# docker-compose down = 停止並刪除所有容器
docker-compose down
```
