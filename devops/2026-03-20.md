## 💡 為什麼重要

你有沒有遇過這種情況？

```
開發者：「在我電腦上明明可以跑啊...」
工程師：「奇怪，為什麼到你那邊就壞了？」
```

**這就是著名的「在我機器上可以跑」（It Works on My Machine）問題！**

### Docker 的解決方案

Docker 就像 **「標準化的貨櫃」**：

- 貨櫃有統一尺寸 → 任何船都能運
- 貨櫃裡裝什麼很清楚 → 開箱就知道
- 貨櫃可以堆疊存放 → 資源利用率高

**Docker 容器**：把應用程式和它的「環境」打包在一起，無論在哪台電腦都能完美運行。

## 🧠 核心概念

### 🐳 Docker 是什麼？

**Docker** 是一個「容器化平台」—— 讓你可以把應用程式和它的所有依賴（資料庫、函式庫、設定檔等）打包成一個可攜帶的單元。

生活比喻：
- 就像把所有食材、鍋具、食譜包成一個「料理包」
- 任何人拿到料理包，按照說明操作，都能做出相同的菜

### 📦 映像檔（Image）vs 容器（Container）

| 概念 | 比喻 | 說明 |
|------|------|------|
| **映像檔（Image）** | 食譜/模具 | 靜態的模板，定義要放什麼 |
| **容器（Container）** | 做出來的料理 | 映像檔的「執行個體」，正在運行的東西 |

### 🆚 容器 vs 虛擬機

```
傳統虛擬機：
┌─────┐ ┌─────┐ ┌─────┐
│ App │ │ App │ │ App │
├─────┤ ├─────┤ ├─────┤
│ OS  │ │ OS  │ │ OS  │
├─────┴─┴─────┴─────┤
│    Hypervisor    │
└──────────────────┘

Docker 容器：
┌─────┐ ┌─────┐ ┌─────┐
│ App │ │ App │ │ App │
├─────┴─┴─────┴─────┤
│   Docker Engine   │
└──────────────────┘
```

**差別**：
- 虛擬機每個都要裝作業系統 → 慢、佔空間
- 容器共享主機 OS → 快、輕量

## 🛠️ 實際應用

1. **本地開發環境** —— 新人加入團隊，不用折騰環境設定，拉下 Docker 就搞定
2. **持續部署（CI/CD）** —— 自動化測試和部署
3. **微服務架構** —— 每個服務獨立包裝，互不干擾
4. **雲端部署** —— AWS、GCP、Azure 都原生支援 Docker

## 📝 範例教學

### 第一步：安裝 Docker

```bash
# macOS
brew install --cask docker

# Ubuntu/Debian
sudo apt-get update
sudo apt-get install docker.io

# 啟動服務
sudo systemctl start docker
```

### 第二步：跑一個現成的容器

```bash
# 跑一個簡單的網頁伺服器
docker run -d -p 8080:80 nginx

# 參數說明：
# -d: 在背景執行（detached）
# -p 8080:80: 把容器的 80 端口映射到本機的 8080
```

打開瀏覽器訪問 `http://localhost:8080`，你會看到 Nginx 預設頁面！

### 第三步：寫第一個 Dockerfile

建立一個簡單的 Node.js 應用：

```dockerfile
# 1. 指定基礎映像檔
FROM node:18-alpine

# 2. 設定工作目錄
WORKDIR /app

# 3. 複製檔案
COPY package*.json ./

# 4. 安裝依賴
RUN npm install

# 5. 複製應用程式碼
COPY . .

# 6. 暴露端口
EXPOSE 3000

# 7. 啟動命令
CMD ["node", "index.js"]
```

### 第四步：建構與運行

```bash
# 根據 Dockerfile 建立映像檔
docker build -t my-app .

# 運行容器
docker run -p 3000:3000 my-app
```

### 常用指令速查表

| 指令 | 用途 |
|------|------|
| `docker ps` | 列出運行中的容器 |
| `docker ps -a` | 列出所有容器 |
| `docker images` | 列出映像檔 |
| `docker stop <id>` | 停止容器 |
| `docker rm <id>` | 刪除容器 |
| `docker rmi <name>` | 刪除映像檔 |
| `docker logs <id>` | 查看容器日誌 |
| `docker exec -it <id> sh` | 進入容器內部 |
