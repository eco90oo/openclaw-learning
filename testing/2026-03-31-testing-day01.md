# Day 1：單元測試基礎（unittest / pytest）

## 📚 今日學習：自動化測試

### 1️⃣ 基礎概念（用生活比喻，越詳細越好）

#### 什麼是單元測試？

想像一下你要組裝一台樂高機器人：
- **沒有單元測試**：組裝完後才發現手臂不會動、齒輪卡住，這時你要全部拆掉重來，超級崩潰！
- **有單元測試**：每組裝好一隻手臂，就馬上測試它能不能動；裝好齒輪，馬上檢查轉不轉。這樣發現問題立刻修正，不用等到全部組完才來debug。

**單元測試（Unit Test）** 就是這個概念：
- 測試程式的最小單位（通常是一個函式、一個類別的方法）
- 隔離外部依賴（資料庫、網路、檔案系統）
- 自動化執行，快速驗證程式碼是否正確

#### 為什麼要做自動化測試？

| 手動測試 | 自動化測試 |
|---------|-----------|
| 每次改程式都要人工重測 | 幾百個測試按一個鍵就跑完 |
| 容易漏測、忘記測 | 每次都會跑，不會漏掉 |
| 耗時費力 | 速度快，幾秒鐘跑完 |
| 依賴測試人員記憶力 | 機器不會累，永遠記得 |

#### 兩大框架：unittest vs pytest

- **unittest**：Python 內建的測試框架，標準庫的一部份。語法參考 Java 的 JUnit，適合喜歡傳統 OOP 風格的人。
- **pytest**：目前最流行的第三方測試框架。語法簡潔、擴充性強、插件豐富。**强烈建議新手從 pytest 開始！**

### 2️⃣ 核心知識（每行程式碼都要有中文註解！）

#### 第一個 pytest 測試

```python
# test_example.py

# 這是我們要測試的函式 - 假設這是一個簡單的計算機功能
def add(a, b):
    """
    簡單的加法函式
    a: 第一個數字
    b: 第二個數字
    回傳：a + b 的結果
    """
    return a + b

# 測試函式的命名規則：test_開頭 + 描述性名稱
# 這讓 pytest 知道這是一個測試，而不是一般函式
def test_add_two_numbers():
    """
    測試 add() 函式 - 正常情況
    驗證兩個正整數相加是否正確
    """
    # 呼叫要測試的函式
    result = add(2, 3)
    
    # 斷言（Assert）：假設結果應該是 5
    # 如果不是 5，測試就會失敗並顯示錯誤訊息
    assert result == 5

def test_add_negative_numbers():
    """
    測試 add() 函式 - 負數相加
    驗證負數也能正確相加
    """
    result = add(-1, -2)
    assert result == -3

def test_add_zeros():
    """
    測試 add() 函式 - 零的處理
    確保加零不會出問題
    """
    result = add(0, 5)
    assert result == 5
```

#### 使用 unittest 框架（了解另一種風格）

```python
# test_unittest_example.py
import unittest  # 匯入 unittest 模組

# 定義測試類別，繼承自 unittest.TestCase
# 這樣 pytest 就知道這是一個測試類別
class TestCalculator(unittest.TestCase):
    """
    計算機測試類別
    繼承 TestCase 讓我們可以使用各種斷言方法
    """
    
    def setUp(self):
        """
        setUp() - 每個測試執行前會自動呼叫
        用來初始化測試所需的資源
        就像每次考試前發空白卷給學生
        """
        self.calculator = Calculator()
    
    def test_add_positive_numbers(self):
        """
        測試加法 - 正數
        使用 assertEqual 來斷言結果
        """
        # assertEqual(預期值, 實際值, 錯誤訊息)
        result = self.calculator.add(3, 4)
        self.assertEqual(result, 7, "3 + 4 應該等於 7")
    
    def test_add_negative_numbers(self):
        """
        測試加法 - 負數
        """
        result = self.calculator.add(-2, -5)
        self.assertEqual(result, -7, "負數相加應該正確")

# 這行很重要！讓我們可以直接執行這個檔案來跑測試
if __name__ == '__main__':
    # unittest.main() 會自動執行這個檔案中的所有測試
    unittest.main()
```

#### 實際的Calculator類別（被測試的對象）

```python
# calculator.py

class Calculator:
    """
    簡單的計算機類別
    用來展示如何測試一個類別
    """
    
    def add(self, a, b):
        """加法"""
        return a + b
    
    def subtract(self, a, b):
        """減法"""
        return a - b
    
    def multiply(self, a, b):
        """乘法"""
        return a * b
    
    def divide(self, a, b):
        """
        除法
        注意：除以零會拋出例外
        """
        if b == 0:
            raise ValueError("不能除以零！")
        return a / b
```

#### pytest 的多 assert 寫法（更簡潔）

```python
# test_calculator.py
from calculator import Calculator

# 建立一個共用的 Calculator 實例
calc = Calculator()

def test_add_multiple_cases():
    """
    一個測試函式可以包含多個斷言
    這種寫法更精簡
    """
    assert calc.add(2, 3) == 5
    assert calc.add(0, 0) == 0
    assert calc.add(-1, 1) == 0

def test_subtract():
    """測試減法"""
    assert calc.subtract(10, 3) == 7
    assert calc.subtract(5, 5) == 0

def test_multiply():
    """測試乘法"""
    assert calc.multiply(3, 4) == 12
    assert calc.multiply(0, 100) == 0

def test_divide():
    """測試除法"""
    assert calc.divide(10, 2) == 5
    assert calc.divide(9, 3) == 3

def test_divide_by_zero():
    """
    測試除以零的例外情況
    使用 pytest.raises() 來檢查是否拋出正確的例外
    """
    import pytest
    
    # pytest.raises() 區塊內的程式碼應該要拋出 ValueError
    with pytest.raises(ValueError):
        calc.divide(10, 0)
```

class Calculator:

    def add(self, a, b):
        """加法：回傳 a + b"""
        return a + b
    
    def subtract(self, a, b):

#### 步驟 3：建立測試檔案

```python
# tests/test_calculator.py
"""計算機的單元測試"""
from my_calculator.calculator import Calculator

# 建立 Calculator 實例（每個測試都會用到）
calc = Calculator()

def test_add_two_positive_numbers():
    """
    測試 1：兩個正數相加
    驗證：2 + 3 = 5
    """
    result = calc.add(2, 3)
    assert result == 5, f"預期 5 but got {result}"

def test_add_two_negative_numbers():
    """
    測試 2：兩個負數相加
    驗證：-2 + (-3) = -5
    """
    result = calc.add(-2, -3)
    assert result == -5

def test_add_positive_and_negative():
    """
    測試 3：正數加負數
    驗證：5 + (-3) = 2
    """
    result = calc.add(5, -3)
    assert result == 2

def test_add_zeros():
    """
    測試 4：零加零
    驗證：0 + 0 = 0
    """
    result = calc.add(0, 0)
    assert result == 0

def test_subtract():
    """
    測試 5：減法
    驗證：10 - 4 = 6
    """
    result = calc.subtract(10, 4)
    assert result == 6

def test_multiply():
    """
    測試 6：乘法
    驗證：6 * 7 = 42
    """
    result = calc.multiply(6, 7)
    assert result == 42

def test_divide():
    """
    測試 7：除法
    驗證：20 / 4 = 5
    """
    result = calc.divide(20, 4)
    assert result == 5

def test_divide_with_decimal():
    """
    測試 8：除法（小數結果）
    驗證：7 / 2 = 3.5
    """
    result = calc.divide(7, 2)
    assert result == 3.5

def test_divide_by_zero_raises_error():
    """
    測試 9：除以零應該拋出 ValueError
    這是一個「例外測試」
    """
    import pytest
    
    # 使用 with pytest.raises() 來驗證例外
    with pytest.raises(ValueError) as exc_info:
        calc.divide(10, 0)
    
    # 驗證錯誤訊息是否正確
    assert "不能除以零" in str(exc_info.value)
```

def add(self, a, b):

```

```

這樣你就知道什麼叫「測試失敗」了！

### 📝 今日重點回顧

| 概念 | 說明 |
|------|------|
| **單元測試** | 測試程式的最小單位（函式/方法），隔離依賴，快速驗證正確性 |
| **pytest** | 最流行的 Python 測試框架，語法簡潔直觀 |
| **斷言 (assert)** | 驗證「預期值」與「實際值」是否相等 |
| **測試命名** | 檔案用 `test_` 開頭，函式用 `test_` 開頭 |
| **例外測試** | 使用 `pytest.raises()` 驗證程式碼是否正確拋出例外 |
