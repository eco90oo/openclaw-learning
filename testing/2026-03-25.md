## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是單元測試？

**比喻**：就像買手機時會測試每個按鍵、螢幕、相機功能是否正常。如果只按下電源鍵發現開不了機，我們不會說「這支手機壞了」，而是知道「電源鍵這個單元有問題」。

單元測試就是這樣的概念：
- **單元（Unit）**：程式中最小的功能單位，通常是一個函數或一個方法
- **測試（Test）**：驗證這個單元是否按照預期工作

### 🤔 為什麼要自動化測試？

**比喻**：想像你是餐廳廚師，每天要試吃每一道菜。

- **手動測試（人工測試）**：廚師每次做好菜後自己親口吃吃看
  - 缺點：費時費力、重複做很無聊、可能會忘記某道菜沒試
  
- **自動化測試**：像是有個試吃機器人，每次出菜自動檢查
  - 優點：快速、準確、可重複、節省人力

### 📊 單元測試 vs 其他測試

| 測試類型 | 範圍 | 速度 | 比喻 |
|---------|------|------|------|
| 單元測試 | 一個函數/方法 | ⚡ 超快 | 試吃一颗調味料 |
| 整合測試 | 多個功能一起 | 🔄 中等 | 試吃一道完整的菜 |
| E2E 測試 | 整個系統 | 🐢 較慢 | 客人完整吃一餐 |

## 2️⃣ 核心知識（詳細語法教學）

### 🔧 Python 單元測試框架介紹

Python 有兩大單元測試框架：

1. **unittest**：Python 內建框架，就像手機原廠配件
2. **pytest**：第三方強大框架，就像功能更多的第三方 APP

兩者都能完成測試任務，我們會用 **pytest** 為主（因為更強大且更流行）。

### 📝 第一個測試案例

讓我們從最基本的開始：

```python
# 假設我們有一個簡單的計算機功能
def add(a, b):
    """這是一個加法函數"""
    return a + b

def subtract(a, b):
    """這是一個減法函數"""
    return a - b
```

**使用 unittest 框架撰寫測試：**

```python
# 引入 unittest 模組（Python 內建的測試框架）
import unittest

# 引入我們要測試的函數
from my_calculator import add, subtract

# 創建一個測試類別，繼承自 unittest.TestCase
# 類別名稱通常以 Test 結尾，這是慣例
class TestCalculator(unittest.TestCase):
    
    # 每一個測試方法都要以 test_ 開頭！
    # unittest 會自動找到所有 test_ 開頭的方法並執行
    
    def test_add_two_positive_numbers(self):
        """測試：兩個正數相加"""
        
        # 呼叫 add 函數，傳入 2 和 3
        result = add(2, 3)
        
        # 斷言（Assertion）：斷言 result 應該等於 5
        # 如果不相等，測試會失敗並顯示錯誤訊息
        self.assertEqual(result, 5)
    
    def test_add_negative_numbers(self):
        """測試：兩個負數相加"""
        
        # 測試 -1 + (-2) = -3
        result = add(-1, -2)
        
        # 斷言結果應該等於 -3
        self.assertEqual(result, -3)
    
    def test_subtract(self):
        """測試：減法功能"""
        
        # 測試 10 - 5 = 5
        result = subtract(10, 5)
        
        # 斷言結果應該等於 5
        self.assertEqual(result, 5)

# 這一行很重要！當直接執行這個檔案時，才會執行測試
# 如果這個檔案被其他檔案 import，不會自動執行
if __name__ == '__main__':
    # 啟動 unittest 的測試執行器
    unittest.main()
```

### 📝 使用 pytest 框架（更強大！）

pytest 的語法更簡潔，不需要繼承特定類別：

```python
# 引入 pytest（需要先 pip install pytest）
import pytest

# 引入要測試的函數
from my_calculator import add, subtract

# 函數名稱以 test_ 開頭，這樣 pytest 才會執行
def test_add_positive():
    """測試：兩個正數相加"""
    
    # 呼叫函數
    result = add(2, 3)
    
    # 使用 assert 語句（Python 原生語法）
    # 這比 unittest 的 self.assertEqual 更直觀
    assert result == 5

def test_add_negative():
    """測試：兩個負數相加"""
    result = add(-1, -2)
    assert result == -3

def test_subtract():
    """測試：減法"""
    result = subtract(10, 5)
    assert result == 5

# 不用寫 if __name__ == '__main__'
# 執行命令：pytest test_calculator.py
```

### 🧐 深入理解：斷言方法

pytest 提供多種斷言方式：

```python
import pytest

def test_assertions():
    """各種斷言範例"""
    
    # === 基本斷言 ===
    x = 10
    assert x == 10              # x 等於 10
    assert x != 5               # x 不等於 5
    
    # === 布林斷言 ===
    is_valid = True
    assert is_valid             # 為 True
    assert not False            # 為 False
    
    # === 包含關係 ===
    name = "Hello World"
    assert "World" in name      # name 包含 "World"
    assert "Python" not in name # name 不包含 "Python"
    
    # === 型別斷言 ===
    value = "hello"
    assert isinstance(value, str)       # value 是字串
    assert isinstance(value, int) is False # value 不是整數
    
    # === 空值斷言 ===
    empty_list = []
    assert len(empty_list) == 0         # 空列表長度為 0
    assert not empty_list               # 列表為空（falsy）
    
    # === 數值比較（浮點數）===
    price = 19.99
    assert abs(price - 20) < 0.01       # 接近 20（處理浮點數誤差）

# === 測試異常情況 ===
def test_division_by_zero():
    """測試除以零會拋出異常"""
    
    # 使用 pytest.raises() 來斷言會拋出異常
    with pytest.raises(ZeroDivisionError):
        result = 10 / 0  # 這會拋出 ZeroDivisionError
```

### 📊 運行測試的方式

```bash
# 方法 1：運行所有測試
pytest

# 方法 2：運行特定檔案
pytest test_calculator.py

# 方法 3：運行特定測試函數
pytest test_calculator.py::test_add_positive

# 方法 4：顯示詳細輸出（-v = verbose）
pytest -v

# 方法 5：顯示更多細節（-vv 可以看到具體斷言失敗原因）
pytest -vv

# 方法 6：停在第一個失敗（-x）
pytest -x

# 方法 7：顯示測試覆蓋範圍（需要安裝 pytest-cov）
pytest --cov=my_calculator
```

### 🛠️ 步驟 2：撰寫被測試的程式

打開 `calculator.py`，寫入：

```python
"""
計算機模組 - 提供基本的數學運算功能
"""

def add(a, b):
    """
    加法函數
    
    參數:
        a: 第一個數字
        b: 第二個數字
    
    返回:
        兩個數字的和
    """
    return a + b

def subtract(a, b):
    """
    減法函數
    
    參數:
        a: 被減數
        b: 減數
    
    返回:
        a - b 的結果
    """
    return a - b

def multiply(a, b):
    """
    乘法函數
    
    參數:
        a: 第一個因數
        b: 第二個因數
    
    返回:
        兩個數字的積
    """
    return a * b

def divide(a, b):
    """
    除法函數
    
    參數:
        a: 被除數
        b: 除數（不能為 0）
    
    返回:
        a / b 的結果
    
    異常:
        ZeroDivisionError: 當 b 為 0 時拋出
    """
    if b == 0:
        raise ValueError("除數不能為 0")
    return a / b
```

### 🛠️ 步驟 3：撰寫測試檔案

打開 `test_calculator.py`，寫入：

```python
"""
計算機測試檔案
使用 pytest 框架測試 calculator.py 的功能
"""

# 引入 pytest 框架
import pytest

# 引入要被測試的模組
from calculator import add, subtract, multiply, divide


# ===== 加法測試 =====
def test_add_two_positive():
    """測試：兩個正數相加"""
    result = add(3, 5)
    assert result == 8, "3 + 5 應該等於 8"

def test_add_two_negative():
    """測試：兩個負數相加"""
    result = add(-2, -4)
    assert result == -6, "-2 + (-4) 應該等於 -6"

def test_add_positive_and_negative():
    """測試：正數加負數"""
    result = add(10, -3)
    assert result == 7, "10 + (-3) 應該等於 7"

def test_add_zero():
    """測試：0 加任何數"""
    result = add(0, 5)
    assert result == 5, "0 + 5 應該等於 5"


# ===== 減法測試 =====
def test_subtract_positive():
    """測試：兩個正數相減"""
    result = subtract(10, 3)
    assert result == 7, "10 - 3 應該等於 7"

def test_subtract_negative_result():
    """測試：結果為負數"""
    result = subtract(3, 10)
    assert result == -7, "3 - 10 應該等於 -7"


# ===== 乘法測試 =====
def test_multiply_two_positive():
    """測試：兩個正數相乘"""
    result = multiply(4, 5)
    assert result == 20, "4 * 5 應該等於 20"

def test_multiply_by_zero():
    """測試：乘以 0"""
    result = multiply(100, 0)
    assert result == 0, "100 * 0 應該等於 0"


# ===== 除法測試 =====
def test_divide_positive():
    """測試：兩個正數相除"""
    result = divide(20, 4)
    assert result == 5, "20 / 4 應該等於 5"

def test_divide_with_remainder():
    """測試：除法有餘數"""
    result = divide(7, 2)
    assert result == 3.5, "7 / 2 應該等於 3.5"

def test_divide_by_zero_raises_error():
    """測試：除以 0 應該拋出異常"""
    # 使用 pytest.raises 來斷言會拋出 ValueError
    with pytest.raises(ValueError) as exc_info:
        divide(10, 0)
    
    # 進一步檢查錯誤訊息
    assert "除數不能為 0" in str(exc_info.value)
```

### 🛠️ 步驟 5：故意讓測試失敗看看效果

修改 `calculator.py` 中的 `add` 函數，故意讓它回傳錯誤的結果：

```python
def add(a, b):
    # 故意寫錯！應該是 a + b
    return a * b  # 錯誤：做成乘法了
```

再次運行測試：

```bash
pytest test_calculator.py -v
```

你會看到測試失敗的輸出：

```

...
AssertionError: assert 15 == 8
```

這就是單元測試的強大之處：**它能快速發現程式碼中的錯誤！**
