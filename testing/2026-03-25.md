## 1️⃣ 基礎概念（用生活比喻）

**什麼是單元測試？**

想象你是餐廳的廚師，要端出一盤美味的料理。在端上桌之前，你會先「試吃」一下，確保味道對了、鹹淡適中、食材煮熟了。這就是「單元測試」的概念——**在把程式碼交給使用者之前，先自己驗證它是否正確運作**。

單元測試就像「把大問題拆成一小塊一小塊」來測試：
- 想像組裝一台樂高機器人，每個零件（函數/類別）都要先確認沒問題，最後組裝起來的機器人才會動
- 單元（Unit）指的是程式的最小單位，通常是一個函數（function）或一個方法（method）

**為什麼需要自動化？**
- 手動測試就像每次吃飯前都要試吃，浪費時間
- 自動化測試就像請了一個機器人幫你試吃，一次設定好，永遠自動幫你檢查

## 2️⃣ 核心知識

### 2.1 Python 內建的 unittest（像新手村）

```python
# 首先，匯入 unittest 模組（內建不用安裝）
import unittest

# 假設我們有一個計算機模組，先定義要測試的函數
def add(a, b):
    """這是一個簡單的加法函數"""
    return a + b

def divide(a, b):
    """這是一個除法函數，注意不能除以 0"""
    if b == 0:
        raise ValueError("不能除以零！")
    return a / b

# 建立測試類別，繼承 unittest.TestCase
# 類別名稱通常以 Test 開頭，這是慣例
class TestMathFunctions(unittest.TestCase):
    """測試數學函數的類別"""
    
    # 測試方法必須以 test_ 開頭，這樣 unittest 才會自動執行
    def test_add_positive_numbers(self):
        """測試：兩個正數相加"""
        # 使用 assertEqual 斷言（斷言 = 斷定什麼應該是對的）
        # 語法：assertEqual(實際結果, 預期結果, 錯誤訊息)
        self.assertEqual(add(2, 3), 5, "2 + 3 應該等於 5")
    
    def test_add_negative_numbers(self):
        """測試：兩個負數相加"""
        self.assertEqual(add(-2, -3), -5, "-2 + (-3) 應該等於 -5")
    
    def test_add_with_zero(self):
        """測試：加上零"""
        self.assertEqual(add(5, 0), 5, "5 + 0 應該等於 5")
    
    def test_divide_normal(self):
        """測試：正常除法"""
        self.assertEqual(divide(10, 2), 5, "10 / 2 應該等於 5")
    
    def test_divide_by_zero(self):
        """測試：除以零應該拋出例外"""
        # assertRaises 用來驗證程式碼會拋出指定的錯誤
        with self.assertRaises(ValueError):
            divide(10, 0)  # 這行應該會拋出 ValueError

# 這行很重要！只有在直接執行這個檔案時才會執行測試
# 如果是被匯入（import），則不會執行
if __name__ == '__main__':
    # verbosity=2 會顯示更詳細的輸出（顯示每個測試方法的名稱）
    unittest.main(verbosity=2)
```

**執行結果會像這樣：**
```
test_add_negative_numbers ... ok
test_add_positive_numbers ... ok
test_add_with_zero ... ok
test_divide_normal ... ok
test_divide_by_zero ... ok
----------------------------------------------------------------------
Ran 5 tests in 0.001s

OK
```

### 2.2 更流行的 pytest（專業工具）

pytest 是目前最流行的測試框架，語法更簡潔、功能更強大。

**先安裝 pytest：**
```bash
pip install pytest
```

**pytest 版本的測試：**

```python
# 定義要測試的函數
def add(a, b):
    """加法函數"""
    return a + b

def divide(a, b):
    """除法函數"""
    if b == 0:
        raise ValueError("不能除以零！")
    return a / b

# pytest 不需要繼承任何類別
# 直接定義以 test_ 開頭的函數即可

def test_add_positive():
    """測試正數相加 - pytest 語法更簡潔"""
    assert add(2, 3) == 5  # 直接用 assert

def test_add_negative():
    """測試負數相加"""
    assert add(-2, -3) == -5

def test_add_zero():
    """測試加零"""
    assert add(100, 0) == 100

def test_divide_normal():
    """測試正常除法"""
    assert divide(20, 4) == 5

def test_divide_by_zero():
    """測試除以零 - pytest 用 raises"""
    import pytest
    with pytest.raises(ValueError):
        divide(10, 0)
```

**執行 pytest：**
```bash
pytest test_example.py -v
```

## 3️⃣ 實際操作（一步一步帶著做）

### Step 1: 建立測試檔案

打開你的程式碼編輯器（或文字編輯器），建立一個新檔案 `test_math.py`：

```python
# test_math.py

# ===== 第一部分：被測試的程式 =====
def multiply(a, b):
    """乘法函數"""
    return a * b

def is_even(number):
    """判斷是否為偶數"""
    return number % 2 == 0

# ===== 第二部分：測試程式 =====

# 使用 pytest 語法
def test_multiply_positive():
    """測試：正數相乘"""
    assert multiply(3, 4) == 12

def test_multiply_negative():
    """測試：負數相乘"""
    assert multiply(-3, 4) == -12

def test_multiply_zero():
    """測試：乘以零"""
    assert multiply(100, 0) == 0

def test_is_even_true():
    """測試：是偶數"""
    assert is_even(4) == True

def test_is_even_false():
    """測試：不是偶數"""
    assert is_even(5) == False

def test_is_even_zero():
    """測試：零是偶數"""
    assert is_even(0) == True
```

### Step 2: 執行測試

在終端機輸入：
```bash
pytest test_math.py -v
```

你應該會看到：
```
test_math.py::test_multiply_positive PASSED
test_math.py::test_multiply_negative PASSED
test_math.py::test_multiply_zero PASSED
test_math.py::test_is_even_true PASSED
test_math.py::test_is_even_false PASSED
test_math.py::test_is_even_zero PASSED

======================== 6 passed in 0.05s ========================
```

### Step 3: 故意測試失敗看看效果

修改 `test_math.py`，故意寫錯一個測試：

```python
def test_multiply_positive():
    """測試：正數相乘"""
    assert multiply(3, 4) == 999  # 故意寫錯！
```

執行後會看到：
```
FAILED test_math.py::test_multiply_positive
AssertionError: assert 12 == 999
```

這就是測試失敗的訊息，幫助你快速找到問題！
