## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是自動化測試？

想象一下：

> 你是一家餐廳的廚師，每天要端出 100 道菜。客人吃之前，你會先試吃一口，確認味道對了再端出去。

**傳統手動測試** = 每道菜你都用筷子夾一點來吃吃看
**自動化測試** = 請一個機器人幫你試吃，而且它可以 24 小時不斷試吃

---

### 🍕 用「做披薩」來比喻三種測試

| 測試類型 | 比喻 | 檢查什麼？ |
|---------|------|-----------|
| **單元測試** | 檢查麵糰、番茄醬、起司各自有沒有壞掉 | 程式最小單元（函數、類別） |
| **整合測試** | 把麵糰、醬料、起司放在一起烤，看看能不能組合成披薩 | 多個元件一起運作 |
| **E2E 測試** | 把披薩端給客人，看客人吃完是否滿意 | 整個系統模擬真實使用情境 |

---

### 🔍 為什麼需要自動化測試？

**手動測試的問題：**
- ❌ 每次改程式碼都要重新測試 → 浪費時間
- ❌ 無聊、重複的事情做多了會恍神 → 容易漏掉錯誤
- ❌ 專案變大後測試項目爆增 → 根本測不完

**自動化測試的優點：**
- ✅ 寫一次，永久使用
- ✅ 幾秒鐘跑完數百個測試
- ✅ 每次改程式碼都能快速確認「我有沒有把之前的功能弄壞」

## 2️⃣ 核心知識（專有名詞解釋）

### 📖 必學詞彙

| 詞彙 | 白話解釋 | 生活中的例子 |
|-----|---------|-------------|
| **Unit Test (單元測試)** | 測試程式的最小單位（像測試一顆螺絲釘） | 檢查螺絲釘有沒有生鏽 |
| **Integration Test (整合測試)** | 測試多個零件組在一起能不能運作（像組裝家具） | 把桌子腿和桌面組在一起，看能不能站穩 |
| **E2E Test (端對端測試)** | 模擬真實使用者操作完整流程 | 客人從進餐廳、點餐、吃飯、結帳全程 |
| **TDD (測試驅動開發)** | 先寫測試，再寫程式 | 先寫食譜再做菜 |
| **Test Framework (測試框架)** | 幫助你寫測試的工具（像廚具組） | 鍋子、鏟子、爐子 |
| **Assertion (斷言)** | 斷言是「斷言」測試結果是對的 | 就像大聲說「這道菜應該是鹹的！」 |
| **Mock (模擬物件)** | 假的代替品，專門用來測試 | 用假人模特兒練習 CPR |
| **CI/CD** | 每次提交程式碼，自動跑測試 | 餐廳每做好一道菜，機器人自動試吃 |

---

### 🔄 TDD 是什麼？測試驅動開發

**傳統開發流程：**
```
寫程式 → 手動測試 → 發現問題 → 修改 → 再手動測試
```

**TDD 流程：**
```
1. 先寫一個會失敗的測試（紅燈）
2. 寫最少的程式讓它通過（綠燈）
3. 重構程式碼，讓它更乾淨（藍燈）
```

> 💡 **比喻**：就像學做菜時，先想好「什麼樣的菜算成功」（比如：鹹淡適中、軟嫩適口），然後動手做，做完對照標準，不滿意就調整。

## 3️⃣ 實際操作（程式碼範例）

### 🍳 用 JavaScript + Jest 來練習

#### 步驟 1：安裝 Jest

```bash
# 建立一個資料夾
mkdir my-first-test
cd my-first-test

# 初始化專案
npm init -y

# 安裝 Jest
npm install --save-dev jest
```

#### 步驟 2：寫一個簡單的函數

**檔案：calculator.js**
```javascript
// 加法函數
function add(a, b) {
  return a + b;
}

// 減法函數
function subtract(a, b) {
  return a - b;
}

// 匯出函數讓測試可以使用
module.exports = { add, subtract };
```

#### 步驟 3：寫單元測試

**檔案：calculator.test.js** (Jest 會自動找到這個檔案)
```javascript
// 引入要測試的函數
const { add, subtract } = require('./calculator');

describe('計算機功能', () => {
  // 第一個測試：測試加法
  test('add(2, 3) 應該等於 5', () => {
    expect(add(2, 3)).toBe(5);
  });

  // 第二個測試：測試負數
  test('add(-1, 1) 應該等於 0', () => {
    expect(add(-1, 1)).toBe(0);
  });

  // 第三個測試：測試減法
  test('subtract(10, 3) 應該等於 7', () => {
    expect(subtract(10, 3)).toBe(7);
  });
});
```

#### 步驟 4：修改 package.json

```json
{
  "scripts": {
    "test": "jest"
  }
}
```

#### 步驟 5：執行測試

```bash
npm test
```

**預期輸出：**
```
PASS  calculator.test.js
  計算機功能
    ✓ add(2, 3) 應該等於 5
    ✓ add(-1, 1) 應該等於 0
    ✓ subtract(10, 3) 應該等於 7

Test Suites: 1 passed, 3 tests passed
```

### 🎯 實作 E2E 測試（用簡單的概念）

假設我們要測試「使用者登入」功能：

**概念流程：**
1. 打開瀏覽器
2. 輸入帳號密碼
3. 點擊登入按鈕
4. 檢查是否看到「歡迎回來」

**程式碼概念（使用 Playwright）：**
```javascript
const { test, expect } = require('@playwright/test');

test('使用者登入成功', async ({ page }) => {
  // 1. 前往登入頁面
  await page.goto('https://example.com/login');
  
  // 2. 輸入帳號密碼
  await page.fill('#username', 'testuser');
  await page.fill('#password', 'password123');
  
  // 3. 點擊登入
  await page.click('#login-button');
  
  // 4. 檢查結果
  await expect(page.locator('.welcome-message')).toContainText('歡迎回來');
});
```

## 4️⃣ 今日小任務

### 🎓 任務：為你的第一個函數寫測試

**任務描述：**
1. 在之前的專案資料夾中，建立一個 `stringUtils.js` 檔案
2. 寫一個反轉字串的函數 `reverseString(str)`
3. 寫一個測試檔案 `stringUtils.test.js` 來測試它

**參考答案：**

**stringUtils.js**
```javascript
function reverseString(str) {
  return str.split('').reverse().join('');
}

module.exports = { reverseString };
```

**stringUtils.test.js**
```javascript
const { reverseString } = require('./stringUtils');

describe('字串工具', () => {
  test('reverseString("hello") 應該返回 "olleh"', () => {
    expect(reverseString("hello")).toBe("olleh");
  });

  test('reverseString("ABC") 應該返回 "CBA"', () => {
    expect(reverseString("ABC")).toBe("CBA");
  });

  test('reverseString("") 應該返回 ""', () => {
    expect(reverseString("")).toBe("");
  });
});
```

**執行：**
```bash
npm test
```
