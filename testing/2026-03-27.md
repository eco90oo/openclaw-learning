## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 什麼是測試？為什麼要自動化？

想象一下：
- **手動測試**就像聘請一位「人工質檢員」，每當生產線（即你的程式碼）生產出一件產品（功能），這位質檢員就要親手去檢查一遍。有時候檢查得很快，但有時候會很累、會漏檢、會因為太晚而犯困。
- **自動化測試**就像在生產線上安裝了一個「自動質檢機器」。每當產品出來，機器就會自動跑一遍標準檢查流程，速度快、不會累、永遠一樣嚴格。

### 什麼是單元測試（Unit Test）？

想象你在組裝一輛汽車：
- **單元測試**就像測試「每一顆螺絲釘」是否能正常鎖緊、是否符合規格。我們不測試整輛車能不能開（那是整合測試），我們只測試最小的零件是否正常。

在程式碼中：
- 一個 **單元（Unit）** 通常是一個 **function（函數）** 或 **class（類別）**
- 單元測試就是「驗證這個函數，給定輸入 A，是否會輸出預期結果 B」

### 為什麼要用 pytest 或 unittest？

- **unittest**：Python 內建的測試框架，就像「標配的家具」
- **pytest**：第三方強大框架，就像「升級版的家具」，更簡潔、更强大、擴充性更好

### 第一個 unittest 範例

```python
# 首先，引入 unittest 模組（Python 內建的測試框架）
import unittest

# 定義一個簡單的加法函數（這是我們要測試的目標）
def add(a, b):
    """
    簡單的加法函數
    輸入：a（數字）, b（數字）
    輸出：a + b（兩數相加的結果）
    """
    return a + b

# 建立一個測試類別，繼承 unittest.TestCase
# 命名慣例：被測試的類別/函數名稱 + Test
class TestAddFunction(unittest.TestCase):
    """
    測試 add 函數的測試類別
    我們會在這個類別中定義多個測試方法
    """
    
    # 每一個測試方法都要以 "test_" 開頭
    # unittest 會自動找到這些方法並執行
    def test_add_positive_numbers(self):
        """
        測試案例 1：測試正數相加
        預期：2 + 3 = 5
        """
        # 使用 assertEqual 來斷言（斷言 = 斷定某件事為真）
        # 語法：assertEqual(實際結果, 預期結果, 錯誤訊息)
        self.assertEqual(add(2, 3), 5, "正數相加應該正確")
    
    def test_add_negative_numbers(self):
        """
        測試案例 2：測試負數相加
        預期：-1 + -2 = -3
        """
        self.assertEqual(add(-1, -2), -3, "負數相加應該正確")
    
    def test_add_zero(self):
        """
        測試案例 3：測試零
        預期：5 + 0 = 5
        """
        self.assertEqual(add(5, 0), 5, "任何數加零應該是自己")

# 這一行是標準的測試入口點
# 當直接執行這個檔案時，會啟動 unittest 的測試執行器
if __name__ == '__main__':
    unittest.main()
```

### 執行結果會像這樣：

```
...
----------------------------------------------------------------------
Ran 3 tests in 0.001s

OK
```

### 認識更多的 unittest 斷言方法

```python
import unittest

class TestAssertionsDemo(unittest.TestCase):
    """
    展示各種常用的斷言方法
    """
    
    # assertEqual：斷言兩個值相等
    def test_equal(self):
        self.assertEqual(1 + 1, 2)  # 預期 1+1 等於 2
    
    # assertNotEqual：斷言兩個值不相等
    def test_not_equal(self):
        self.assertNotEqual(1 + 1, 3)  # 預期 1+1 不等於 3
    
    # assertTrue：斷言條件為 True
    def test_true(self):
        self.assertTrue(10 > 5)  # 預期 10 > 5 為真
    
    # assertFalse：斷言條件為 False
    def test_false(self):
        self.assertFalse(10 < 5)  # 預期 10 < 5 為假
    
    # assertIsNone：斷言值為 None（空）
    def test_none(self):
        result = None
        self.assertIsNone(result)  # 預期 result 是 None
    
    # assertIsNotNone：斷言值不為 None
    def test_not_none(self):
        result = "Hello"
        self.assertIsNotNone(result)  # 預期 result 不是 None
    
    # assertIn：斷言某元素在列表/字串中
    def test_in(self):
        self.assertIn("la", "hello")  # 預期 "la" 在 "hello" 中
        # 這個會失敗！因為 "hello" 中沒有 "la"
    
    # assertNotIn：斷言某元素不在列表/字串中
    def test_not_in(self):
        self.assertNotIn("z", "abc")  # 預期 "z" 不在 "abc" 中
    
    # assertRaises：斷言會拋出特定例外
    def test_raises(self):
        # 預期除以零會拋出 ZeroDivisionError
        with self.assertRaises(ZeroDivisionError):
            x = 10 / 0  # 這會觸發例外

if __name__ == '__main__':
    unittest.main()
```

### pytest 的寫法（更簡潔！）

```python
# 這是 pytest 版本的測試
# 注意：不需要 import 任何東西（除了被測試的模組）

# 直接定義測試函數，不用類別也可以！（這是 pytest 的特色）
def add(a, b):
    """簡單的加法函數"""
    return a + b

# 測試函數也是以 test_ 開頭
def test_add_positive():
    """
    測試正數相加
    pytest 的斷言直接使用 Python 的 assert 語句
    """
    assert add(2, 3) == 5, "正數相加應該正確"

def test_add_negative():
    """測試負數相加"""
    assert add(-1, -2) == -3, "負數相加應該正確"

def test_add_zero():
    """測試零"""
    assert add(5, 0) == 5, "任何數加零應該是自己"

# 執行方式：pytest 檔名.py
# 或者：python -m pytest 檔名.py
```

### pytest 的多種斷言語法

```python
# pytest 支援所有 Python 原生 assert 語法
# 而且當 assert 失敗時，pytest 會自動顯示詳細的錯誤訊息

def test_pytest_assertions():
    """展示 pytest 的各種斷言方式"""
    
    # 簡單的 assert（最常用）
    assert 1 + 1 == 2
    
    # 斷言不相等
    assert 1 + 1 != 3
    
    # 斷言大於/小於
    assert 10 > 5
    assert 5 < 10
    
    # 斷言列表包含某元素
    fruits = ["apple", "banana", "orange"]
    assert "apple" in fruits
    
    # 斷言字典有某個 key
    user = {"name": "Tom", "age": 25}
    assert "name" in user
    
    # 斷言為 None
    result = None
    assert result is None
    
    # 使用 assert 配合條件表達式（可自訂錯誤訊息）
    x = 100
    assert x > 0, f"x 應該是正數，但實際是 {x}"

# 當測試失敗時，pytest 會顯示：
# AssertionError: x 應該是正數，但實際是 100
```

### 步驟 3：建立被測試的程式

```python
# 建立一個名為 calculator.py 的檔案
# 這就是我們要測試的「待測系統」（System Under Test, SUT）

def add(a, b):
    """加法：返回 a + b"""
    return a + b

def subtract(a, b):
    """減法：返回 a - b"""
    return a - b

def multiply(a, b):
    """乘法：返回 a * b"""
    return a * b

def divide(a, b):
    """除法：返回 a / b"""
    # 如果除數為零，拋出 ValueError（這是一種例外的處理方式）
    if b == 0:
        raise ValueError("除數不能為零")
    return a / b
```

### 步驟 4：建立測試檔案

```python
# 建立一個名為 test_calculator.py 的檔案
# 檔名一定要以 test_ 開頭，pytest 才會自動發現它

# 1. 引入待測的模組
from calculator import add, subtract, multiply, divide

# 2. 開始寫測試（使用 pytest 風格）

def test_add_two_positive_numbers():
    """測試兩個正數相加"""
    # 輸入：2 和 3，預期輸出：5
    assert add(2, 3) == 5, "2 + 3 應該等於 5"

def test_add_two_negative_numbers():
    """測試兩個負數相加"""
    assert add(-5, -3) == -8, "-5 + -3 應該等於 -8"

def test_add_positive_and_negative():
    """測試正數和負數相加"""
    assert add(10, -4) == 6, "10 + (-4) 應該等於 6"

def test_add_zero():
    """測試加零"""
    assert add(0, 0) == 0, "0 + 0 應該等於 0"
    assert add(100, 0) == 100, "100 + 0 應該等於 100"

def test_subtract():
    """測試減法"""
    assert subtract(10, 5) == 5, "10 - 5 應該等於 5"
    assert subtract(5, 10) == -5, "5 - 10 應該等於 -5"

def test_multiply():
    """測試乘法"""
    assert multiply(3, 4) == 12, "3 * 4 應該等於 12"
    assert multiply(0, 100) == 0, "0 * 任何數應該等於 0"
    assert multiply(-2, 3) == -6, "-2 * 3 應該等於 -6"

def test_divide():
    """測試除法"""
    assert divide(10, 2) == 5, "10 / 2 應該等於 5"
    assert divide(9, 3) == 3, "9 / 3 應該等於 3"

def test_divide_by_zero():
    """測試除以零應該拋出例外"""
    # 使用 pytest 的 raises 來檢查是否拋出正確的例外
    with pytest.raises(ValueError):
        divide(10, 0)  # 嘗試除以零，應該觸發 ValueError
```

### 步驟 5：執行測試

```bash
# 在終端機執行：
pytest test_calculator.py -v

# 參數說明：
# -v = verbose，顯示詳細的測試結果
# 預期輸出會像：

# test_calculator.py::test_add_two_positive_numbers PASSED
# test_calculator.py::test_add_two_negative_numbers PASSED
# test_calculator.py::test_add_positive_and_negative PASSED
# ...
# ===================== 8 passed in 0.02s ====================
```

### 步驟 6：嘗試讓測試失敗（看看失敗的樣子）

```python
# 修改 calculator.py 中的 add 函數，故意寫錯：
def add(a, b):
    return a * b  # 錯誤：應該是加法，卻寫成乘法

# 再次執行測試：
pytest test_calculator.py -v

# 你會看到測試失敗，類似：
# FAILED test_calculator.py::test_add_two_positive_numbers
# AssertionError: assert 6 == 5
```
