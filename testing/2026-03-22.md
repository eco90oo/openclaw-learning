# 📚 【今日學習：自動化測試】

> 自動化測試是現代軟體開發的核心技能，讓我們用生活比喻來徹底搞懂它！

---

## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是自動化測試？

**比喻：保全系統 vs 人工巡邏**

想象你家裡的保全系統：
- **手動測試** = 請一個保全人員每天敲完所有房門，確認沒有人闖入
- **自動化測試** = 安裝一套智慧保全系統，它會**自動**檢查門窗、感應異常

> 保全人員會累、會漏、會请假；自動化系統 24 小時全年无休，每次都能同樣認真檢查！

### 🔍 為什麼需要自動化測試？

| 情境 | 沒有自動化測試 | 有自動化測試 |
|------|---------------|--------------|
| 改一行程式碼 | 要人工檢查所有功能（2小時） | 點一下，3分鐘完成 |
| 加新功能 | 擔心破壞舊功能 | 自動驗證舊功能是否正常 |
| 釋出版本 | 熬夜測試超緊張 | 睡覺前讓程式跑，早上喝咖啡看結果 |

### 📊 測試金字塔

```
        ╱╲
       ╱  ╲    ← E2E 測試（少量，5-10%）
      ╱────╲
     ╱      ╲  ← 整合測試（中等，20-30%）
    ╱────────╲
   ╱          ╲← 單元測試（大量，60-70%）
  ╱────────────╲
  
 數量：少 → 多
 速度：慢 → 快
 成本：高 → 低
```

**簡單理解：**
- 底部（單元測試）像「體檢」，每天做，快速發現小問題
- 中間（整合測試）像「聯絡簿」，檢查零件組合後能否正常運作
- 頂部（E2E 測試）像「驗收」，模擬真實使用者操作，確保整個系統正常

## 2️⃣ 核心知識（專有名詞解釋）

### 📝 單元測試（Unit Test）

**定義**：針對程式最小的單位（通常是一個函數或方法）進行的測試

**生活比喻**：
- 就像測試一粒米會不會熟（單元）
- 而不是測試整鍋飯（整合）

**特點**：
- 執行速度最快（毫秒級）
- 不需要網路、資料庫
- 每個測試獨立運作

### 🔗 整合測試（Integration Test）

**定義**：測試多個元件組合在一起是否能正常運作

**生活比喻**：
- 就像把電話和路由器接在一起，測試能否上網
- 單獨測電話 ✅ 單獨測路由器 ✅ 放在一起能否上網？→ 整合測試

### 🌐 E2E 測試（End-to-End Test）

**定義**：從頭到尾模擬真實使用者的操作流程

**生活比喻**：
- 就像請一天客人來家裡，從開門、換鞋、坐下、喝茶、聊天、離開，全部體驗一遍
- 這是**最接近真實使用情境**的測試

### 🔄 TDD（Test-Driven Development，測試驅動開發）

**定義**：先寫測試，再寫程式碼

**傳統開發流程**：
```
寫程式碼 → 手動測試 → 發現問題 → 修改 → 再次手動測試
```

**TDD 開發流程**：
```
寫測試（失敗） → 寫最少的程式碼讓測試通過 → 重構（優化） → 下一個測試
         ↑                                        │
         ╰────────────────────────────────────────╯
```

**TDD 口號**：Red（紅）→ Green（綠）→ Refactor（重構）

**比喻**：就像先寫食譜（測試），再照著食譜做菜（寫程式），確保做出來的味道符合預期！

### 🛠️ 常見測試框架

| 語言 | 單元測試框架 | E2E 測試框架 |
|------|-------------|-------------|
| JavaScript/TypeScript | Jest, Mocha, Jasmine | Playwright, Cypress, Selenium |
| Python | pytest, unittest | Playwright, Selenium |
| Java | JUnit, TestNG | Selenium, Cypress |
| Go | testing (內建), testify | Playwright |

## 3️⃣ 實際操作（程式碼範例）

### 🧪 用 JavaScript + Jest 寫單元測試

**安裝 Jest：**
```bash
npm install --save-dev jest
```

**第一個單元測試：**

```javascript
// math.js - 我們要測試的程式
function add(a, b) {
  return a + b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = { add, multiply };
```

```javascript
// math.test.js - 測試檔案
const { add, multiply } = require('./math');

describe('數學運算測試', () => {
  
  test('加法：2 + 3 應該等於 5', () => {
    expect(add(2, 3)).toBe(5);
  });
  
  test('加法：負數相加', () => {
    expect(add(-1, -1)).toBe(-2);
  });
  
  test('乘法：3 * 4 應該等於 12', () => {
    expect(multiply(3, 4)).toBe(12);
  });
  
});
```

**執行測試：**
```bash
npx jest
```

**輸出結果：**
```
 PASS  math.test.js
  ✓ 加法：2 + 3 應該等於 5
  ✓ 加法：負數相加
  ✓ 乘法：3 * 4 應該等於 12
```

### 🎯 用 TDD 開發一個計算機功能

**步驟 1：先寫測試（還沒實作功能！）**

```javascript
// calculator.test.js
const { calculate } = require('./calculator');

describe('計算機功能', () => {
  
  test('減法：10 減 3 應該等於 7', () => {
    expect(calculate(10, '-', 3)).toBe(7);
  });
  
});
```

**執行 → 會失敗（Red）：**
```
FAIL calculator.test.js
Cannot find module './calculator'
```

**步驟 2：寫最少的程式碼讓測試通過**

```javascript
// calculator.js
function calculate(a, operator, b) {
  if (operator === '-') {
    return a - b;
  }
  return null; // 先隨便返回
}
```

**執行 → 通過（Green）：**
```
PASS calculator.test.js
```

**步驟 3：重構（優化程式碼）**

```javascript
// calculator.js - 完善的版本
function calculate(a, operator, b) {
  switch (operator) {
    case '+': return a + b;
    case '-': return a - b;
    case '*': return a * b;
    case '/': return b !== 0 ? a / b : null;
    default: return null;
  }
}

module.exports = { calculate };
```

### 🌐 用 Cypress 寫 E2E 測試

**安裝 Cypress：**
```bash
npm install --save-dev cypress
```

**E2E 測試範例：**

```javascript
// cypress/integration/login.spec.js
describe('登入功能測試', () => {
  
  beforeEach(() => {
    // 每次測試前訪問登入頁面
    cy.visit('http://localhost:3000/login');
  });
  
  it('應該能夠成功登入', () => {
    // 輸入帳號密碼
    cy.get('[data-testid="username"]').type('testuser');
    cy.get('[data-testid="password"]').type('password123');
    
    // 點擊登入按鈕
    cy.get('[data-testid="login-button"]').click();
    
    // 驗證結果：應該跳轉到首頁
    cy.url().should('include', '/dashboard');
    
    // 驗證：應該看到歡迎訊息
    cy.contains('Welcome, testuser').should('be.visible');
  });
  
  it('應該顯示錯誤訊息 when 密碼錯誤', () => {
    cy.get('[data-testid="username"]').type('testuser');
    cy.get('[data-testid="password"]').type('wrongpassword');
    cy.get('[data-testid="login-button"]').click();
    
    // 驗證：應該看到錯誤訊息
    cy.contains('Invalid credentials').should('be.visible');
  });
  
});
```

## 4️⃣ 今日小任務

### 🌱 初級挑戰（30 分鐘）

1. **安裝 Jest**：
   ```bash
   mkdir my-first-test
   cd my-first-test
   npm init -y
   npm install --save-dev jest
   ```

2. **建立一個簡單的函數**（例如：反轉字串）：
   ```javascript
   // stringHelper.js
   function reverseString(str) {
     return str.split('').reverse().join('');
   }
   module.exports = { reverseString };
   ```

3. **寫測試**：
   ```javascript
   // stringHelper.test.js
   const { reverseString } = require('./stringHelper');
   
   test('反轉 "hello" 應該得到 "olleh"', () => {
     expect(reverseString('hello')).toBe('olleh');
   });
   ```

4. **執行**：`npx jest`

### 🚀 中級挑戰（1 小時）

嘗試用 TDD 開發一個「檢查密碼強度」的功能：
- 密碼長度 < 8 → 回傳 "弱"
- 密碼 >= 8 且包含數字 → 回傳 "中"
- 密碼 >= 8 且包含數字和英文 → 回傳 "強"

先寫測試，再寫程式碼！
