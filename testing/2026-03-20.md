## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是自動化測試？

想像你在組裝一台樂高機器人。你會先看說明書，然後一步步組裝。組完後，你要檢查：

- ✅ 手臂可以轉動嗎？
- ✅ 輪子會轉嗎？
- ✅ 開關可以按嗎？

**手動測試**就像你用手逐一檢查每一個零件，檢查完還要寫下結果，確認沒問題後才能交給客人。

**自動化測試**就像請一個機器人幫你檢查！你事先設定好檢查清單（就像寫劇本），然後讓機器人自動跑一遍，自動告訴你哪裡有問題。

### 🔄 為什麼需要自動化測試？

| 手動測試 | 自動化測試 |
|---------|-----------|
| 每次都要人來測 | 設定一次，無限次使用 |
| 容易漏掉項目 | 不會漏 |
| 測到一半累了會出錯 | 不會累，永遠一樣認真 |
| 測 100 次需要 5 小時 | 測 100 次只需要 5 分鐘 |

**簡單說**：自動化測試就像聘請一個全年無休、超級細心的 QA 小幫手！

## 2️⃣ 核心知識（專有名詞解釋）

### 📌 單元測試（Unit Testing）

**比喻**：就像測試汽車的每一個零件
- 測試引擎：發動了嗎？
- 測試輪胎：會轉嗎？
- 測試煞車：會停嗎？

**定義**：單元測試是測試程式中最小的單位（通常是一個函數或一個方法），確保每個小零件單獨運作正常。

**特點**：
- 測試範圍小
- 執行速度快
- 容易找出問題在哪裡

### 📌 整合測試（Integration Testing）

**比喻**：就像把零件組裝成引擎後，測試引擎是否能正常運作

**定義**：測試多個單元組合在一起時，是否能正確配合運作。

**特點**：
- 測試多個元件的合作
- 檢查資料流是否正確
- 模擬真實情況

### 📌 E2E 測試（End-to-End Testing）

**比喻**：就像整台車組裝好後，實際開上路測試

**定義**：從頭到尾模擬真實使用者的操作，測試整個系統是否符合使用者需求。

**特點**：
- 測試完整的使用者流程
- 最接近真實使用情境
- 執行時間較長

### 📌 TDD（Test-Driven Development，測試驅動開發）

**比喻**：就像先寫食譜，再做菜

傳統開發順序：
1. 寫程式 → 2. 寫測試 → 3. 發現問題修改

TDD 開發順序：
1. 先寫測試（像先設標準）→ 2. 寫讓測試通過的程式 → 3. 重構改進

**TDD 核心口號**：紅燈 → 綠燈 → 重構！

### 📌 常見測試框架

| 語言 | 單元測試框架 | E2E 測試框架 |
|-----|-------------|-------------|
| JavaScript/TypeScript | Jest, Mocha, Vitest | Cypress, Playwright, Selenium |
| Python | pytest, unittest | Playwright, Selenium |
| Java | JUnit, TestNG | Selenium |
| C# | NUnit, xUnit | Playwright |

## 3️⃣ 實際操作（程式碼範例）

### 🟢 第一步：安裝 Jest（JavaScript 單元測試框架）

```bash
# 建立專案資料夾
mkdir my-first-test
cd my-first-test

# 初始化專案（回答問題直接按 Enter）
npm init -y

# 安裝 Jest
npm install --save-dev jest
```

### 🟢 第二步：寫一個簡單的函數

建立檔案 `math.js`：

```javascript
// 這是我們要測試的程式
function 加法(a, b) {
  return a + b;
}

function 判斷是否成年(年齡) {
  if (年齡 >= 18) {
    return true;
  }
  return false;
}

module.exports = { 加法, 判斷是否成年 };
```

### 🟢 第三步：寫測試！

建立檔案 `math.test.js`（Jest 會自動找到這個檔案）：

```javascript
// 引入要測試的程式
const { 加法, 判斷是否成年 } = require('./math.js');

describe('加法函數測試', () => {
  test('1 + 1 應該等於 2', () => {
    expect(加法(1, 1)).toBe(2);
  });

  test('5 + 3 應該等於 8', () => {
    expect(加法(5, 3)).toBe(8);
  });

  test('負數相加也應該正確', () => {
    expect(加法(-1, -1)).toBe(-2);
  });
});

describe('判斷是否成年測試', () => {
  test('18 歲應該成年', () => {
    expect(判斷是否成年(18)).toBe(true);
  });

  test('17 歲不應該成年', () => {
    expect(判斷是否成年(17)).toBe(false);
  });

  test('20 歲應該成年', () => {
    expect(判斷是否成年(20)).toBe(true);
  });
});
```

### 🟢 第四步：執行測試！

在 `package.json` 的 `"scripts"` 加入：

```json
"test": "jest"
```

然後在終端機執行：

```bash
npm test
```

**你應該會看到類似這樣的結果：**

```
✓ 判斷是否成年測試
  ✓ 18 歲應該成年
  ✓ 17 歲不應該成年
  ✓ 20 歲應該成年

✓ 加法函數測試
  ✓ 1 + 1 應該等於 2
  ✓ 5 + 3 應該等於 8
  ✓ 負數相加也應該正確

Test Suites: 1 passed, 6 tests passed
```

### 🎯 測試程式碼解讀

```javascript
test('測試描述', () => {
  // expect(實際值).toBe(預期值)
  expect(加法(1, 1)).toBe(2);
});
```

- `test()` = 定義一個測試案例
- `expect()` = 放你要測試的東西
- `toBe()` = 預期應該得到的結果

其他常見斷言：
- `toBe()` - 完全相等
- `toEqual()` - 物件/陣列內容相等
- `toBeTruthy()` - 應該是真值
- `toBeFalsy()` - 應該是假值
- `toContain()` - 應該包含這個內容
- `toBeGreaterThan()` - 應該大於
