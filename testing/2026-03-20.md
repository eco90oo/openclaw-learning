## 💡 為什麼重要

### ❌ 沒有測試的悲劇故事

想象一下：
- 你寫了一個購物網站功能
- 人工測試了「加入購物車」
- 卻忘了測試「結帳時優惠券過期了怎麼辦」
- 結果上線後，客戶用過期優惠券結帳，公司損失 100 萬

**自動化測試就是為了避免這種悲劇！**

### ✅ 有測試的好處

1. **省時間** — 跑一次測試 = 人工檢查 100 次
2. **更放心** — 修改程式碼後，測試幫你確認沒破壞舊功能
3. **更快樂** — 不用一直做無聊的重複測試
4. **敢重構** — 有測試保護，改程式碼不怕改壞

## 🧠 核心概念

### 📝 自動化測試是什麼？

**自動化測試** = 寫一段程式，自動檢查另一段程式是否正確運作

就像：
- 飛機起飛前會做「自動檢查系統」
- 工廠生產時會用「自動品質檢測機」

### 🧪 三種主要測試類型

#### 1. 單元測試（Unit Test）

**定義**：測試程式的最小單位（通常是一個函數或一個類別）

生活比喻：
- 像測試一輛汽車的「引擎單元」是否能正常啟動
- 只測試一個零件，不測試整輛車

```
// 我們要測試的函數：加法
function add(a, b) {
  return a + b;
}

// 單元測試
function testAdd() {
  // 測試 1 + 1 = 2
  if (add(1, 1) !== 2) {
    throw new Error('1 + 1 應該等於 2');
  }
  // 測試 0 + 5 = 5
  if (add(0, 5) !== 5) {
    throw new Error('0 + 5 應該等於 5');
  }
  console.log('✅ 全部通過！');
}

testAdd();
```

#### 2. 整合測試（Integration Test）

**定義**：測試多個元件一起運作是否正常

生活比喻：
- 像測試「引擎 + 變速箱 + 輪子」是否能讓車子前進
- 把多個零件組裝在一起測試

```
// 假設我們有這些元件
function getUser(id) { /* 從資料庫取得用戶 */ }
function sendEmail(email) { /* 發送 email */ }

// 整合測試：確認「新增用戶並發送歡迎信」這個流程正常
function testNewUserFlow() {
  const user = getUser(123);
  if (!user) throw new Error('找不到用戶');
  
  const result = sendEmail(user.email);
  if (!result.success) throw new Error('郵件發送失敗');
  
  console.log('✅ 新用戶流程測試通過！');
}
```

#### 3. E2E 測試（End-to-End Test）

**定義**：模擬真實用戶行為，從頭到尾測試整個系統

生活比喻：
- 像請一個人真的去買車：開車去展示中心 → 試駕 → 簽約 → 開回家
- 完全模擬真實使用情境

```
// 用 Selenium 或 Playwright 模擬用戶操作
// 情境：用戶註冊 → 登入 → 購買商品 → 結帳

// 偽程式碼
function testCompletePurchase() {
  // 1. 打開網站
  browser.open('https://shop.com');
  
  // 2. 點擊註冊
  browser.click('#register-btn');
  browser.fill('#email', 'test@test.com');
  browser.click('#submit');
  
  // 3. 登入
  browser.fill('#email', 'test@test.com');
  browser.fill('#password', '123456');
  browser.click('#login');
  
  // 4. 購買商品
  browser.click('#product-1');
  browser.click('#add-to-cart');
  browser.click('#checkout');
  
  // 5. 確認購買成功
  const message = browser.getText('#success-message');
  if (!message.includes('購買成功')) {
    throw new Error('購買流程失敗');
  }
  
  console.log('✅ 完整購買流程測試通過！');
}
```

### 📊 測試金字塔

```
        /\
       /E2E\        ← 最少（慢、昂貴）
      /------\
     /整合測試\     ← 中等
    /----------\
   / 單元測試 \    ← 最多（快、便宜）
  /------------\
```

**核心原則**：
- **單元測試最多**（70%）：最快、最便宜，多寫
- **整合測試中等**（20%）：測試元件之間的配合
- **E2E 測試最少**（10%）：最慢、最貴，只測核心功能

## 🛠️ 實際操作

### 用 JavaScript + Jest 寫單元測試

Jest 是最多人用的 JavaScript 測試框架。

#### 步驟 1：安裝 Jest

```bash
# 建立專案資料夾
mkdir my-test-project
cd my-test-project

# 初始化 npm
npm init -y

# 安裝 Jest
npm install --save-dev jest
```

#### 步驟 2：修改 package.json

```json
{
  "scripts": {
    "test": "jest"
  }
}
```

#### 步驟 3：建立要測試的檔案

**math.js**（要測試的程式）
```javascript
// 一個簡單的加法函數
function add(a, b) {
  return a + b;
}

// 一個簡單的乘法函數
function multiply(a, b) {
  return a * b;
}

// 匯出函數讓測試可以使用
module.exports = { add, multiply };
```

#### 步驟 4：建立測試檔案

**math.test.js**（測試程式）
```javascript
const { add, multiply } = require('./math');

describe('加法測試', () => {
  test('1 + 1 應該等於 2', () => {
    expect(add(1, 1)).toBe(2);
  });

  test('-1 + 1 應該等於 0', () => {
    expect(add(-1, 1)).toBe(0);
  });

  test('0.1 + 0.2 應該等於 0.3', () => {
    expect(add(0.1, 0.2)).toBeCloseTo(0.3);
  });
});

describe('乘法測試', () => {
  test('2 * 3 應該等於 6', () => {
    expect(multiply(2, 3)).toBe(6);
  });

  test('0 * 100 應該等於 0', () => {
    expect(multiply(0, 100)).toBe(0);
  });
});
```

#### 步驟 5：執行測試

```bash
npm test
```

你應該會看到：
```
 PASS  math.test.js
  加法測試
    ✓ 1 + 1 應該等於 2
    ✓ -1 + 1 應該等於 0
    ✓ 0.1 + 0.2 應該等於 0.3
  乘法測試
    ✓ 2 * 3 應該等於 6
    ✓ 0 * 100 應該等於 0

Test Suites: 1 passed, 3 tests passed
```

### ⚠️ 常見錯誤提醒

1. **忘記匯出函數** → 測試無法引入要測試的程式
2. **測試自己寫的程式** → 不要測試第三方程式（如 npm 套件）
3. **測試太多東西** → 每個測試只測一件事
4. **沒有定期跑測試** → 要把測試加入開發流程
