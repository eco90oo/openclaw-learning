## 1️⃣ 基礎概念（用生活比喻，越詳細越好）

### 🏠 什麼是單元測試？

想象你要組裝一台樂高機器人。在組裝之前，你會先檢查每個零件是否完好：
- 齒輪能不能轉動？
- 輪子是否平衡？
- 關節連接處緊不緊？

**單元測試（Unit Test）** 就是類似這樣的概念——在軟體中，我們要「測試最小、最基本的零件」，確認每個「零件」都能正常工作。

> **比喻**：單元測試就像「品管部門」在產品出廠前，對每個零件進行的獨立檢驗。它不檢查零件之間的配合（那是整合測試的事），只檢查「這個零件本身有沒有問題」。

### 🔍 為什麼需要單元測試？

1. **及時發現 Bug**：就像在組裝前發現零件壞了，總比組裝好後發現問題容易處理
2. **重構的保障**：當你想「優化代碼」時，有測試保護就不怕改壞東西
3. **文件的效果**：好的測試就像「範例代碼」，讓後人知道功能應該怎麼運作

### 🛠️ unittest vs pytest

Python 提供了兩種主流的測試框架：

| 特性 | unittest（標準庫） | pytest（第三方） |
|------|-------------------|-----------------|
| **出生** | Python 內建，自帶 | 需要安裝 `pip install pytest` |
| **語法** | 比較傳統，類別風格 | 更簡潔，更 Pythonic |
| **社區** | 穩定但較老派 | 活躍，插件多 |

> **建議**：初學者先學 unittest 理解概念，再學 pytest 體驗現代語法。今天我們會同時介紹兩者！

### 📖 unittest 基礎語法

```python
# ============================================================
# unittest_test_sample.py
# 這是一個 unittest 的基本範例
# ============================================================

# 匯入 unittest 模組（Python 標準庫，內建不用安裝）
import unittest

# ============================================================
# 我們要測試的「待測試程式」
# 假設這是一個簡單的計算機模組
# ============================================================

def add(a, b):
    """加法函數：回傳兩個數字的和"""
    return a + b

def subtract(a, b):
    """減法函數：回傳兩個數字的差"""
    return a - b

def multiply(a, b):
    """乘法函數：回傳兩個數字的乘積"""
    return a * b

# ============================================================
# 開始撰寫測試類別
# 命名規則：被測試的類/模組 + TestCase
# 例如：CalculatorTest, StringHelperTestCase
# ============================================================

class TestCalculator(unittest.TestCase):
    """
    測試計算機功能的類別
    繼承自 unittest.TestCase 才能使用測試功能
    """
    
    # ============================================================
    # setUp() 方法：在每個測試方法執行前自動運行
    # 適合放「準備工作」，例如建立測試資料、初始化物件
    # 類似：考試前發試卷
    # ============================================================
    def setUp(self):
        """每個測試執行前都會先執行這裡"""
        print("▶ setUp 執行中...")
        # 這裡可以做一些初始化
    
    # ============================================================
    # tearDown() 方法：在每個測試方法執行後自動運行
    # 適合放「清理工作」，例如關閉檔案、清除測試資料
    # 類似：考試後收試卷
    # ============================================================
    def tearDown(self):
        """每個測試執行後都會先執行這裡"""
        print("▶ tearDown 執行中...")
        # 這裡可以做清理
    
    # ============================================================
    # 測試方法命名規範：test_開頭 + 描述性名稱
    # 為什麼要 test_ 開頭？因為 unittest 會自動找到所有 test_ 開頭的方法
    # ============================================================
    
    def test_add_two_positive_numbers(self):
        """測試：兩個正數相加"""
        # act（執行）- 呼叫我們要測試的函數
        result = add(3, 5)
        
        # assert（斷言）- 檢查結果是否符合預期
        # assertEqual(預期值, 實際值, 失敗時的訊息)
        self.assertEqual(result, 8, "正數相加應該得到正確結果")
    
    def test_add_two_negative_numbers(self):
        """測試：兩個負數相加"""
        result = add(-3, -5)
        # assertEqual 斷言結果應該等於 -8
        self.assertEqual(result, -8)
    
    def test_add_positive_and_negative(self):
        """測試：正數 + 負數"""
        result = add(10, -4)
        self.assertEqual(result, 6)
    
    def test_subtract(self):
        """測試：減法功能"""
        result = subtract(10, 3)
        self.assertEqual(result, 7)
    
    def test_multiply(self):
        """測試：乘法功能"""
        result = multiply(4, 5)
        self.assertEqual(result, 20)

# ============================================================
# 運行測試的入口點
# 當直接執行這個檔案時，會執行 unittest.main()
# 如果是被 import 引入，則不會執行（因為 __name__ != '__main__'）
# ============================================================

if __name__ == '__main__':
    # verbosity=2 表示顯示詳細輸出（會顯示每個測試的名稱和結果）
    unittest.main(verbosity=2)
```

### 📖 pytest 基礎語法（更簡潔！）

```python
# ============================================================
# pytest_test_sample.py
# pytest 的版本 - 語法更簡潔、更 Pythonic
# ============================================================

# 先定義要測試的函數
def add(a, b):
    """加法函數"""
    return a + b

def divide(a, b):
    """除法函數"""
    # 注意：這裡還沒處理除以零的情況！
    return a / b

# ============================================================
# pytest 測試函數
# 函數名稱必須以 test_ 開頭
# 不需要繼承任何類別，直接寫函數！
# ============================================================

def test_add_positive():
    """測試：正數相加"""
    result = add(3, 5)
    assert result == 8, "正數相加應該正確"

def test_add_negative():
    """測試：負數相加"""
    result = add(-3, -5)
    assert result == -8

def test_add_mixed():
    """測試：正負數混合"""
    result = add(10, -3)
    assert result == 7

# ============================================================
# 測試除法功能
# 這個測試會失敗，因為 divide 函數沒有處理除以零的情況
# 這正好說明：測試可以幫助我們發現 bug！
# ============================================================

def test_divide_normal():
    """測試：正常除法"""
    result = divide(10, 2)
    assert result == 5

def test_divide_by_zero():
    """測試：除以零會拋出例外"""
    # pytest 用 with pytest.raises() 來檢查是否拋出例外
    # 這是一個「預期行為」的測試
    with pytest.raises(ZeroDivisionError):
        divide(10, 0)

# ============================================================
# 必須匯入 pytest 才能使用 pytest.raises
# ============================================================
import pytest
```

### 📖 常見斷言方法一覽

```python
# ============================================================
# assertions_demo.py
# 展示各種常用的斷言方法
# ============================================================

import unittest

class TestAssertions(unittest.TestCase):
    """演示各種斷言的使用方式"""
    
    # ============================================================
    # assertEqual(a, b) - 斷言 a 等於 b
    # 最常用！檢查兩個值是否相等
    # ============================================================
    def test_assert_equal(self):
        self.assertEqual(10 + 5, 15)
        self.assertEqual("hello", "hello")
    
    # ============================================================
    # assertNotEqual(a, b) - 斷言 a 不等於 b
    # ============================================================
    def test_assert_not_equal(self):
        self.assertNotEqual(10, 20)
    
    # ============================================================
    # assertTrue(x) - 斷言 x 為 True
    # ============================================================
    def test_assert_true(self):
        self.assertTrue(10 > 5)
    
    # ============================================================
    # assertFalse(x) - 斷言 x 為 False
    # ============================================================
    def test_assert_false(self):
        self.assertFalse(10 < 5)
    
    # ============================================================
    # assertIsNone(x) - 斷言 x 為 None
    # 常用於檢查函數是否回傳 None
    # ============================================================
    def test_assert_is_none(self):
        result = print("test")  # print 回傳 None
        self.assertIsNone(result)
    
    # ============================================================
    # assertIsNotNone(x) - 斷言 x 不為 None
    # ============================================================
    def test_assert_is_not_none(self):
        result = "hello"
        self.assertIsNotNone(result)
    
    # ============================================================
    # assertIn(item, container) - 斷言 item 在 container 內
    # 常用於檢查字串、列表、字典中是否包含某元素
    # ============================================================
    def test_assert_in(self):
        self.assertIn("apple", "apple banana cherry")
        self.assertIn(3, [1, 2, 3, 4, 5])
    
    # ============================================================
    # assertNotIn(item, container) - 斷言 item 不在 container 內
    # ============================================================
    def test_assert_not_in(self):
        self.assertNotIn("dog", "cat bird fish")
    
    # ============================================================
    # assertIsInstance(obj, class) - 斷言 obj 是 class 的實例
    # ============================================================
    def test_assert_is_instance(self):
        self.assertIsInstance("hello", str)
        self.assertIsInstance([1, 2], list)
    
    # ============================================================
    # assertRaises(exception) - 斷言會拋出指定的例外
    # 測試「錯誤處理」必用！
    # ============================================================
    def test_assert_raises(self):
        with self.assertRaises(ZeroDivisionError):
            x = 10 / 0
    
    # ============================================================
    # assertAlmostEqual(a, b, places=7) - 斷言近似相等
    # 解決浮點數精度問題（例如 0.1 + 0.2 != 0.3）
    # ============================================================
    def test_assert_almost_equal(self):
        self.assertAlmostEqual(0.1 + 0.2, 0.3, places=7)
```

```python
# calculator.py
# ============================================================
# 計算機模組 - 這是我們要測試的「產品」
# ============================================================

class Calculator:
    """簡單的計算機類別"""
    
    def add(self, a, b):
        """加法：回傳 a + b"""
        return a + b
    
    def subtract(self, a, b):
        """減法：回傳 a - b"""
        return a - b
    
    def multiply(self, a, b):
        """乘法：回傳 a * b"""
        return a * b
    
    def divide(self, a, b):
        """除法：回傳 a / b"""
        if b == 0:
            # 故意不處理除以零，讓測試發現這個問題！
            pass
        return a / b
```

### 🧪 第四步：撰寫測試程式

```python
# test_calculator.py
# ============================================================
# 計算機測試 - 守護我們的程式碼
# ============================================================

import unittest
# 匯入待測試的模組
from calculator import Calculator

class TestCalculator(unittest.TestCase):
    """
    計算機測試類別
    這裡的每個 test_ 方法都是一個「測試案例」
    """
    
    # ============================================================
    # setUp：在每個測試前建立一個新的計算機物件
    # 為什麼每次都要新的？確保測試之間互不影響！
    # ============================================================
    def setUp(self):
        self.calc = Calculator()
    
    # ============================================================
    # 測試加法
    # ============================================================
    def test_add_two_positive_numbers(self):
        """測試：兩個正數相加應該得到正確結果"""
        result = self.calc.add(5, 3)
        self.assertEqual(result, 8)
    
    def test_add_two_negative_numbers(self):
        """測試：兩個負數相加"""
        result = self.calc.add(-5, -3)
        self.assertEqual(result, -8)
    
    def test_add_zero(self):
        """測試：任何數加 0 應該不變"""
        result = self.calc.add(100, 0)
        self.assertEqual(result, 100)
    
    # ============================================================
    # 測試減法
    # ============================================================
    def test_subtract(self):
        """測試：基本減法"""
        result = self.calc.subtract(10, 3)
        self.assertEqual(result, 7)
    
    def test_subtract_negative_result(self):
        """測試：減到變負數"""
        result = self.calc.subtract(3, 10)
        self.assertEqual(result, -7)
    
    # ============================================================
    # 測試乘法
    # ============================================================
    def test_multiply(self):
        """測試：基本乘法"""
        result = self.calc.multiply(4, 5)
        self.assertEqual(result, 20)
    
    def test_multiply_by_zero(self):
        """測試：乘以 0 應該是 0"""
        result = self.calc.multiply(999, 0)
        self.assertEqual(result, 0)
    
    # ============================================================
    # 測試除法
    # ============================================================
    def test_divide(self):
        """測試：基本除法"""
        result = self.calc.divide(10, 2)
        self.assertEqual(result, 5)
    
    def test_divide_by_zero_should_raise_error(self):
        """測試：除以零應該拋出錯誤"""
        # 這個測試會失敗，因為我們的程式碼還沒處理除以零！
        with self.assertRaises(ZeroDivisionError):
            self.calc.divide(10, 0)

if __name__ == '__main__':
    # 執行測試
    unittest.main(verbosity=2)
```

### 📊 執行結果解讀

當你執行 `pytest test_calculator.py -v` 時，應該會看到類似這樣的輸出：

```
============================= test session starts ==============================
platform win32 -- Python 3.x.x, pytest-7.x.x
collected 8 items

test_calculator.py::TestCalculator::test_add_two_positive_numbers PASSED
test_calculator.py::TestCalculator::test_add_two_negative_numbers PASSED
test_calculator.py::TestCalculator::test_add_zero PASSED
test_calculator.py::TestCalculator::test_subtract PASSED
test_calculator.py::TestCalculator::test_subtract_negative_result PASSED
test_calculator.py::TestCalculator::test_multiply PASSED
test_calculator.py::TestCalculator::test_multiply_by_zero PASSED
test_calculator.py::TestCalculator::test_divide_by_zero_should_raise_error FAILED
========================= 7 passed, 1 failed in 0.05s =========================
```

**解讀**：
- ✅ PASSED = 測試通過
- ❌ FAILED = 測試失敗（這是預期的，因為除法還沒處理除以零！）
- 這就是 TDD（Test-Driven Development）的第一步：先寫測試，再寫功能
