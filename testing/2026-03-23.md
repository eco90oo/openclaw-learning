## 1️⃣ 基礎概念

### 為什麼需要 Mock？

**生活中的比喻**：

想象你要測試「自動販賣機」：
- **完整測試**：真的放硬幣、真的出飲料 → 麻煩又慢
- **Mock 測試**：用假的硬幣偵測器、假的飲料出口 → 快又可控

**在測試中**：
- Mock = 假的物件/函數
- 讓測試不依賴外部資源（資料庫、API、檔案）

## 2️⃣ Python Mock 語法

### 基本使用

```python
from unittest.mock import Mock, patch, MagicMock

# 建立假的物件
mock_api = Mock()
mock_api.get_user.return_value = {"id": 1, "name": "小明"}

# 使用假的物件
result = mock_api.get_user(1)
print(result)  # {'id': 1, 'name': '小明'}
```

### Mock 函數

```python
# 假的 random 函數
with patch('random.randint', return_value=5):
    result = my_function()  # random.randint 會回傳 5
```

## 3️⃣ 常見 Mock 場景

### 場景 1：Mock API 呼叫

```python
import requests
from unittest.mock import patch

@patch('requests.get')
def test_get_user(mock_get):
    # 假裝 API 回傳的資料
    mock_get.return_value.json.return_value = {
        "id": 1,
        "name": "小明",
        "email": "test@example.com"
    }
    
    # 測試我們的函數
    user = get_user(1)
    
    assert user["name"] == "小明"
    mock_get.assert_called_once_with("https://api.example.com/users/1")
```

### 場景 2：Mock 檔案操作

```python
from unittest.mock import patch, mock_open

@patch("builtins.open", new_callable=mock_open)
def test_read_file(mock_file):
    mock_file.read.return_value = "Hello, World!"
    
    result = read_file("test.txt")
    
    assert result == "Hello, World!"
```

### 場景 3：Mock 時間

```python
from datetime import datetime
from unittest.mock import patch

@patch('datetime.datetime')
def test_is_business_hours(mock_datetime):
    # 假裝現在是早上 10 點
    mock_datetime.now.return_value = datetime(2024, 1, 15, 10, 0, 0)
    
    result = is_business_hours()
    
    assert result == True
```

## 4️⃣ Spy - 監控行為

### 什麼是 Spy？

**比喻**：
- **Mock**：請一個「假廚師」來做菜
- **Spy**：請一個「監視器」看真實廚師怎麼做

**Spy 用於**：驗證函數是否被呼叫、呼叫幾次、傳什麼參數

```python
from unittest.mock import Mock, call

# 建立 Spy
notifier = Mock()

# 執行會呼叫 notifier 的函數
send_notifications(["user1", "user2"], notifier)

# 驗證行為
notifier.assert_called()  # 確認有被呼叫
notifier.call_count == 2  # 確認被呼叫 2 次
notifier.assert_has_calls([
    call("user1"),
    call("user2")
])
```

## 5️⃣ Fixture 與 Mock 結合

```python
import pytest
from unittest.mock import Mock

@pytest.fixture
def mock_api():
    api = Mock()
    api.get_user.return_value = {"id": 1, "name": "小明"}
    return api

def test_get_user(mock_api):
    result = get_user(1, api=mock_api)
    assert result["name"] == "小明"
    mock_api.get_user.assert_called_once_with(1)
```

## 6️⃣ 什麼時候用 Mock？

| 情況 | 建議 |
|------|------|
| 測試需要呼叫外部 API | Mock 掉 API |
| 測試需要讀寫資料庫 | Mock 掉 DB |
| 測試需要讀檔案 | Mock 掉 file system |
| 測試需要時間/日期 | Mock 掉 datetime |
| 只想驗證函數有沒有被呼叫 | 用 Spy |
