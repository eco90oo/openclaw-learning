## 1️⃣ 基礎概念（生活比喻）

### 🎯 什麼是 Mock？

**場景**：假設你要測試一台「咖啡機」的功能，但咖啡機需要「供水系統」才能運作。

- **現實做法**：你可以真的安裝水管、打開水龍頭，測試咖啡機
- **問題**：麻煩！萬一水管漏了、停水了，測試就會失敗
- **Mock 的概念**：做一個「假的」水龍頭出來，它看起來像真的，但實際上只會輸出「假水」（模擬數據），讓咖啡機以為真的有水供應

### 🎯 為什麼需要 Mock？

| 情況 | 沒有 Mock | 有 Mock |
|------|-----------|---------|
| 測試需要呼叫外部 API | 可能超時、失敗、費用高 | 瞬間完成，假資料 |
| 測試需要寫入資料庫 | 需要清理資料、耗時 | 用記憶體模擬，假資料 |
| 測試需要發送 Email | 會真的寄信出去 | 模擬發送，驗證呼叫次數 |
| 測試需要讀取檔案 | 可能檔案不存在 | 假檔案，想給什麼給什麼 |

**核心精神**：測試 A 功能時，只關心 A，不要被 B、C、D 影響。

## 2️⃣ 核心知識（程式碼範例）

### 2.1 unittest.mock 模組

Python 內建了 `unittest.mock`，我們稱之為 **Mock**（模擬物件）。

```python
# 引入 unittest 模組的 mock 子模組
from unittest.mock import Mock
```

### 2.2 建立第一個 Mock 物件

```python
# 建立一個「假的」購物車物件
mock_cart = Mock()

# 假設我們的購物車有 .total() 方法可以計算總金額
# Mock 物件會自動擁有這個方法（即使它本來沒有！）
mock_cart.total.return_value = 999  # 設定它回傳 999 元

# 呼叫這個假方法
result = mock_cart.total()

print(result)  # 輸出：999
```

> 💡 **解說**：Mock 物件就像一個「橡皮圖章」，它會自動長出任何你需要的方法。

### 2.3 用 Mock 隔離外部依賴

**真實情境**：你的程式碼會呼叫 Payment API（付款 API），但你不希望在測試時真的扣款。

```python
# ==========================================
# 範例程式：要測試的「結帳系統」
# ==========================================

class CheckoutSystem:
    """結帳系統，需要一個付款處理器"""
    
    def __init__(self, payment_processor):
        # 接收一個「付款處理器」物件
        self.payment_processor = payment_processor
    
    def checkout(self, cart, user_info):
        """
        執行結帳流程
        :param cart: 購物車物件
        :param user_info: 使用者資訊（字典）
        :return: 結帳結果
        """
        # 計算總金額
        total = cart.total()
        
        # 呼叫付款 API（這裡會真的扣款！）
        payment_result = self.payment_processor.charge(
            amount=total,
            card=user_info['card_number']
        )
        
        if payment_result['success']:
            return {'status': 'success', 'total': total}
        else:
            return {'status': 'failed', 'message': payment_result['error']}
```

**問題**：測試 `CheckoutSystem` 時，會真的呼叫付款 API！

**解決方案**：用 Mock 取代真的 `payment_processor`。

```python
# ==========================================
# 測試程式：使用 Mock 隔離付款 API
# ==========================================

import unittest
from unittest.mock import Mock  # 引入 Mock 類別

class TestCheckoutSystem(unittest.TestCase):
    """測試結帳系統"""
    
    def test_checkout_success(self):
        """
        測試案例：結帳成功
        """
        # ===== Step 1: 建立 Mock 物件 =====
        # 建立一個「假的」付款處理器
        mock_payment_processor = Mock()
        
        # ===== Step 2: 設定 Mock 的回傳值 =====
        # 當 charge() 被呼叫時，回傳「成功」的模擬結果
        mock_payment_processor.charge.return_value = {
            'success': True,
            'transaction_id': 'TXN-12345'
        }
        
        # ===== Step 3: 建立其他假物件 =====
        # 建立一個「假的」購物車，總金額是 500 元
        mock_cart = Mock()
        mock_cart.total.return_value = 500
        
        # ===== Step 4: 執行測試 =====
        # 將假物件傳入真的系統
        checkout = CheckoutSystem(payment_processor=mock_payment_processor)
        result = checkout.checkout(
            cart=mock_cart,
            user_info={'card_number': '4111-1111-1111-1111'}
        )
        
        # ===== Step 5: 斷言驗證結果 =====
        self.assertEqual(result['status'], 'success')
        self.assertEqual(result['total'], 500)
        
        # 額外驗證：確認 mock_payment_processor.charge() 確實被呼叫了
        mock_payment_processor.charge.assert_called_once()  # 至少呼叫一次
```

### 2.4 Mock 的三種主要用法

#### 用法 1：直接建立 Mock 物件

```python
# 直接建立一個 Mock 物件
mock_obj = Mock()
mock_obj.some_method.return_value = 'fake result'
result = mock_obj.some_method()  # 回傳 'fake result'
```

#### 用法 2：用 patch 裝飾器替換

```python
# 假設有個模組在網路上抓匯率
# from external import get_exchange_rate

# 用 patch 暫時把 get_exchange_rate 換成假的
@patch('external.get_exchange_rate')  # 替換這個函數
def test_currency_conversion(mock_get_rate):
    # 設定假函數回傳 1 USD = 30 TWD
    mock_get_rate.return_value = 30
    
    # 執行真的轉換邏輯（裡面會呼叫 get_exchange_rate）
    result = convert_usd_to_twd(100)  # 100 USD
    
    assert result == 3000  # 100 * 30 = 3000 TWD
```

#### 用法 3：用 patch 上下文管理器

```python
# 有時候不想用裝飾器，可以用 with 語句
def test_with_patch():
    with patch('external.get_exchange_rate') as mock_rate:
        mock_rate.return_value = 31
        result = convert_usd_to_twd(100)
        assert result == 3100
```

## 3️⃣ 實際操作（一步一步帶著做）

### 📋 練習題：測試「天氣通知系統」

**情境**：系統會每天早上 8 點檢查天氣，如果下雨就發送推播通知。

**目標**：測試「發送推播」功能，但不真的發送！（用 Mock）

#### Step 1：建立練習檔案

```python
# weather_notifier.py
# 天氣通知系統（要測試的程式碼）

class WeatherService:
    """天氣服務：用來取得天氣資料"""
    
    def get_weather(self):
        """
        取得天氣資訊
        回傳：天氣字典 {'condition': 'sunny/rainy', 'temp': 25}
        """
        # 這裡可能會呼叫外部天氣 API（真實情況）
        # 我們先假設它回傳「下雨」
        return {'condition': 'rainy', 'temp': 18}

class NotificationService:
    """推播服務：用來發送通知"""
    
    def send_push(self, message):
        """
        發送推播訊息
        :param message: 訊息內容
        """
        # 這裡可能會呼叫 Firebase / APNs（真實發送）
        print(f"[REAL PUSH] 發送通知：{message}")

class WeatherNotifier:
    """天氣通知器：整合天氣服務與推播服務"""
    
    def __init__(self, weather_service, notification_service):
        # 依賴注入：接收外部服務
        self.weather_service = weather_service
        self.notification_service = notification_service
    
    def notify_if_rain(self):
        """
        檢查天氣，如果下雨就發送推播
        """
        # 取得天氣資訊
        weather = self.weather_service.get_weather()
        
        # 如果下雨，發送通知
        if weather['condition'] == 'rainy':
            self.notification_service.send_push(
                message=f"提醒：今天天氣下雨，溫度 {weather['temp']} 度"
            )
            return True
        else:
            return False
```

#### Step 2：撰寫測試（使用 Mock）

```python
# test_weather_notifier.py
# 使用 Mock 隔離外部依賴的測試

import unittest
from unittest.mock import Mock, call  # 引入 Mock 和 call（檢查多次呼叫）

class TestWeatherNotifier(unittest.TestCase):
    """測試天氣通知器"""
    
    def test_notify_if_rain_should_send_push_when_rainy(self):
        """
        測試：當天氣是下雨時，應該發送推播
        """
        # ===== Step 1: 建立 Mock 天氣服務 =====
        mock_weather_service = Mock()
        # 設定 get_weather() 回傳「下雨」的天氣資料
        mock_weather_service.get_weather.return_value = {
            'condition': 'rainy',
            'temp': 18
        }
        
        # ===== Step 2: 建立 Mock 推播服務 =====
        mock_notification_service = Mock()
        # 不需要設定回傳值，因為 send_push() 回傳 None
        
        # ===== Step 3: 建立測試對象 =====
        notifier = WeatherNotifier(
            weather_service=mock_weather_service,
            notification_service=mock_notification_service
        )
        
        # ===== Step 4: 執行測試 =====
        result = notifier.notify_if_rain()
        
        # ===== Step 5: 斷言驗證 =====
        # 驗證：應該回傳 True（因為有下雨）
        self.assertTrue(result)
        
        # 驗證：推播服務的 send_push() 應該被呼叫 1 次
        mock_notification_service.send_push.assert_called_once()
        
        # 驗證：呼叫時的參數內容
        mock_notification_service.send_push.assert_called_with(
            message="提醒：今天天氣下雨，溫度 18 度"
        )
    
    def test_notify_if_rain_should_not_send_push_when_sunny(self):
        """
        測試：當天氣是晴天時，不應該發送推播
        """
        # ===== Step 1: 建立 Mock 天氣服務 =====
        mock_weather_service = Mock()
        # 設定 get_weather() 回傳「晴天」的天氣資料
        mock_weather_service.get_weather.return_value = {
            'condition': 'sunny',
            'temp': 28
        }
        
        # ===== Step 2: 建立 Mock 推播服務 =====
        mock_notification_service = Mock()
        
        # ===== Step 3: 建立測試對象 =====
        notifier = WeatherNotifier(
            weather_service=mock_weather_service,
            notification_service=mock_notification_service
        )
        
        # ===== Step 4: 執行測試 =====
        result = notifier.notify_if_rain()
        
        # ===== Step 5: 斷言驗證 =====
        # 驗證：應該回傳 False（因為晴天）
        self.assertFalse(result)
        
        # 驗證：推播服務的 send_push() 不應該被呼叫
        mock_notification_service.send_push.assert_not_called()


# 執行測試
if __name__ == '__main__':
    unittest.main()
```

#### Step 3：執行測試

```bash
# 在終端機執行
python -m unittest test_weather_notifier.py -v
```

**預期輸出**：

```
test_notify_if_rain_should_send_push_when_rainy ... ok
test_notify_if_rain_should_not_send_push_when_sunny ... ok
----------------------------------------------------------------------
Ran 2 tests in 0.001s

OK
```

> ✅ **成功**：即使沒有真的天氣 API、沒有真的發推播，測試依然通過！

## 📚 總結重點

| 概念 | 說明 |
|------|------|
| **Mock 物件** | 假的物件，用來取代真實的外部依賴 |
| **隔離測試** | 只測試目標程式，不被其他系統影響 |
| **return_value** | 設定 Mock 方法的回傳值 |
| **assert_called_once()** | 驗證 Mock 方法至少被呼叫一次 |
| **assert_called_with()** | 驗證 Mock 方法被呼叫時的參數 |
