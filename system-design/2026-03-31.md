## 2️⃣ 思考框架

### Step 1：比特幣解決了什麼問題？

**比喻**：想象一個**「共享Excel表格」**，裡面記錄了所有人的餘額。

- 傳統銀行：只有銀行能修改這個表格（中心化）
- 比特幣：每個人都有一份拷貝，大家一起維護（去中心化）

**核心問題**：如果每個人都能隨意修改自己的餘額，怎麼辦？

### Step 3：核心機制拆解

| 概念 | 比喻 | 實際運作 |
|------|------|----------|
| **區塊 (Block)** | 每一頁Excel | 包含交易列表 + 時間戳 + 前一區塊Hash |
| **鏈 (Chain)** | 訂書針 | 每個區塊包含前一區塊的指紋，形成不可篡改的鏈 |
| **Hash** | 指紋 | SHA-256 演算法，輸入任何東西產生固定64字元指紋 |
| **Nonce** | 彩票號碼 | 調整這個數字讓Hash符合目標條件 |
| **工作量證明 (PoW)** | 抽獎 | 誰先算出符合條件的Nonce，誰獲得記帳權 |

## 3️⃣ 詳細解答

### 3.1 區塊鏈如何確保「沒人作弊」？

**比喻**：像全班同學互相檢查作業

1. **每筆交易都需要「簽名」**
   - 只有持有私鑰的人才能發起轉帳
   - 就像只有本人才能開啟自己的銀行帳戶

2. **共識機制：工作量證明 (Proof of Work)**
   ```
   步驟：
   1. 礦工收集待處理的交易
   2. 將交易打包成區塊
   3. 不斷調整 Nonce，計算 Hash
   4. 當 Hash 開頭符合「0000...」（難度目標）時，公布區塊
   5. 其他節點驗證後，接受區塊並繼續
   ```

3. **最長鏈原則**
   - 如果有人想篡改歷史，需要重新計算該區塊之後的所有區塊
   - 但他同時要和全球所有礦工競爭
   - 實際上不可能（51%攻擊的代價）

### 3.3 比特幣的「貨幣學」

| 參數 | 數值 | 意義 |
|------|------|------|
| 區塊時間 | 10分鐘 | 平均產生一個區塊的時間 |
| 區塊大小 | 1MB | 每個區塊能容納的交易數量上限 |
| 比特幣總量 | 2100萬枚 | 上限，永遠不會通貨膨脹 |
| 減半週期 | 4年 | 每21萬區塊，挖礦獎勵減半 |
| 難度調整 | 2週 | 每2016區塊調整一次難度 |

**比喻**：比特幣就像一個「數學版的黃金」
- 黃金開採越來越難，比特幣也是
- 黃金有總量上限，比特幣也是
- 黃金不需要任何人批准，比特幣也是

### 3.4 如果你要設計一個新區塊鏈？

```python
# 簡化的區塊類別（Python 概念示範）
import hashlib
import time

class Block:
    def __init__(self, index, transactions, previous_hash):
        self.index = index
        self.timestamp = time.time()
        self.transactions = transactions  # 交易列表
        self.previous_hash = previous_hash   # 前一區塊的Hash
        self.nonce = 0                        # 隨機數（挖礦用）
        self.hash = self.calculate_hash()
    
    def calculate_hash(self):
        # SHA-256 Hash
        data = (
            str(self.index) +
            str(self.timestamp) +
            str(self.transactions) +
            str(self.previous_hash) +
            str(self.nonce)
        )
        return hashlib.sha256(data.encode()).hexdigest()
    
    def mine_block(self, difficulty):
        # 工作量證明：找出符合難度的Nonce
        target = "0" * difficulty  # 例如 "0000" 表示難度4
        while self.hash[:difficulty] != target:
            self.nonce += 1
            self.hash = self.calculate_hash()
        print(f"區塊 #{self.index} 挖礦成功！ Nonce: {self.nonce}")

# 區塊鏈類別
class Blockchain:
    def __init__(self):
        self.chain = [self.create_genesis_block()]
        self.difficulty = 4
    
    def create_genesis_block(self):
        # 創世區塊
        return Block(0, "Genesis Block", "0")
    
    def add_block(self, transactions):
        new_block = Block(
            index=len(self.chain),
            transactions=transactions,
            previous_hash=self.chain[-1].hash
        )
        new_block.mine_block(self.difficulty)
        self.chain.append(new_block)
    
    def is_valid(self):
        # 驗證區塊鏈完整性
        for i in range(1, len(self.chain)):
            current = self.chain[i]
            previous = self.chain[i-1]
            
            # 檢查Hash是否正確
            if current.hash != current.calculate_hash():
                return False
            
            # 檢查鏈接是否正確
            if current.previous_hash != previous.hash:
                return False
        return True
```

## 4️⃣ 延伸問題

面試官可能會追問：

### Q1：區塊鏈有什麼缺點？
- **效率低**：比特幣每秒只能處理約7筆交易，遠低於Visa的數千筆
- **能耗高**：全球比特幣挖礦耗電超過某些國家
- **擴展性差**：區塊大小有限制

### Q2：有哪些區塊鏈共識機制？
| 機制 | 優點 | 缺點 | 代表幣種 |
|------|------|------|----------|
| PoW (工作量證明) | 安全、去中心化 | 耗電、慢 | BTC |
| PoS (權益證明) | 省電、快 | 富者恆富 | ETH (現在) |
| DPoS (委託權益證明) | 超快 | 中心化風險 | EOS, TRX |
| PBFT (實用拜占庭) | 快、確定性 | 節點數有限 | Hyperledger |

### Q3：什麼場景適合用區塊鏈？
- ✅ 需要「不可篡改」的記錄
- ✅ 需要「去中心化」、無信任第三方
- ✅ 需要「跨境支付」或「資產代幣化」

### Q4：什麼場景不適合用區塊鏈？
- ❌ 需要高速交易處理
- ❌ 數據需要頻繁修改
- ❌ 已有可信的中央機構
