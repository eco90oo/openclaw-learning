# 📚 【今日學習：設計 URL 短網址服務（TinyURL）】

## 1️⃣ 題目說明

### 面試官會怎麼問？

「請設計一個像 TinyURL 或 Bitly 的 URL 短網址服務。用戶可以輸入一個長網址，我們的系統會產生一個短網址；當用戶訪問這個短網址時，會被重新導向到原始的長網址。」

### 需要澄清什麼問題？

在動手畫圖之前，先問清楚這些：

1. **流量規模**：每天產生多少短網址？每天有多少點擊訪問？
   - 是每天 100 個？100 萬個？還是 10 億個？
   
2. **功能範圍**：
   - 只需要基本的「短網址產生 → 重新導向」功能？
   - 還是需要「自訂短網址」？「網址過期時間」？「統計點擊次數」？

3. **服務品質**：
   - 需要多少個 9 的可用性？（99% vs 99.99%）
   - 延遲要求是多少？（毫秒級？秒級？）

4. **對象**：
   - 公開服務？還是僅供內部使用？

## 2️⃣ 思考框架

### Step 1: 需求釐清

讓我們假設一個中等規模的設計：
- **每日新增短網址**：100 萬個
- **每日點擊訪問**：1 億次
- **功能**：基本短網址產生 + 重新導向 + 點擊統計
- **可用性**：99.9%（三個 9）

### Step 2: High-Level 設計

想像你在開一家「網址當鋪」：
- 你收下長網址（一個複雜的抵押品）
- 給他一個「號碼牌」（短網址）
- 客戶拿號碼牌來換回長網址

| 元件 | 比喻 | 職責 |

| **Load Balancer** | 餐廳接待員 | 分散流量到多個伺服器 |
| **API Server** | 櫃台服務生 | 處理請求邏輯 |
| **Key-Value Store** | 倉庫管理員 | 儲存短網址 ↔ 長網址 的對應關係 |

### Step 3: 資料庫設計

核心就是一張表，儲存「短碼」和「長網址」的對應關係：

```sql
-- 用 SQL 表格來理解（實際可能用 NoSQL）
CREATE TABLE urls (
    short_code    CHAR(6) PRIMARY KEY,  -- 例如 "abc123"
    original_url  TEXT NOT NULL,        -- 原始長網址
    created_at    TIMESTAMP DEFAULT NOW(),
    expires_at    TIMESTAMP NULL,       -- 過期時間（可選）
    click_count   BIGINT DEFAULT 0      -- 點擊統計
);
```

**為什麼用 CHAR(6)？**

- 6 個字元 = 26 (小寫) + 26 (大寫) + 10 (數字) = 62⁶ = 568 億個組合
- 足夠容納數百億個短網址！

### Step 4: API 設計

兩個核心 API：

```
1. 產生短網址
   POST /api/shorten
   Request:  { "url": "https://very-long-url.com/..." }
   Response: { "short_url": "https://tinyurl.com/abc123" }

2. 訪問短網址（重新導向）
   GET /api/redirect/{short_code}
   Response: HTTP 302 → Location: https://original-long-url.com
```

## 3️⃣ 詳細解答

### 核心演算法：如何生成短碼？

**方法一：隨機生成（像抽獎）**

```python
import random
import string

def generate_short_code(length=6):
    """從 62 個字元中隨機選 6 個"""
    chars = string.ascii_letters + string.digits  # a-z, A-Z, 0-9
    return ''.join(random.choice(chars) for _ in range(length))

# 範例輸出：abc123, XyZ987, 4K9mNp
```

**優點**：簡單實作
**缺點**：可能碰撞（已存在的短碼），需要檢查重複

**方法二：遞增 ID（像排隊號碼牌）**

```
資料庫 ID: 1      → 短碼: 000001 (Base62)
資料庫 ID: 2      → 短碼: 000002
資料庫 ID: 999999 → 短碼: 15Fs7K
```

```python
def id_to_short_code(id_num):
    """將數字轉換為 Base62 編碼"""
    chars = "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ"
    result = ""
    while id_num > 0:
        id_num, remainder = divmod(id_num, 62)
        result = chars[remainder] + result
    return result.zfill(6)  # 補零到 6 位

# 測試
print(id_to_short_code(1))    # "000001"
print(id_to_short_code(62))   # "000010"
print(id_to_short_code(999)) # "00087f"
```

**優點**：不會碰撞，效能好
**缺點**：需要一個遞增的 ID 產生器

### 完整程式碼範例

```python
# 簡化版實作（Python + Flask）
from flask import Flask, request, redirect, jsonify
import redis
import random
import string

app = Flask(__name__)
redis_client = redis.Redis(host='localhost', port=6379, db=0)

CHARS = string.ascii_letters + string.digits

@app.route('/api/shorten', methods=['POST'])
def shorten_url():
    """產生短網址"""
    original_url = request.json.get('url')
    if not original_url:
        return jsonify({"error": "URL required"}), 400
    
    # 生成短碼
    short_code = ''.join(random.choice(CHARS) for _ in range(6))
    
    # 儲存到 Redis（key: 短碼, value: 長網址）
    redis_client.setex(f"url:{short_code}", 86400*30, original_url)
    
    return jsonify({
        "short_url": f"https://tinyurl.com/{short_code}"
    })

@app.route('/<short_code>', methods=['GET'])
def redirect_to_url(short_code):
    """重新導向到原始網址"""
    original_url = redis_client.get(f"url:{short_code}")
    
    if not original_url:
        return "URL not found", 404
    
    return redirect(original_url.decode())

if __name__ == '__main__':
    app.run(port=8080)
```

## 4️⃣ 延伸問題

面試官可能會追問：

### Q1: 如何處理高並發訪問？

> **提示**：快取！將熱門的短網址放在記憶體中（Redis），減少資料庫查詢。

### Q2: 如果某個短網址被大量訪問（例如熱門貼文）？

> **提示**：Redis Cluster + 讀寫分離，讓熱門資料庫變成「多個複製」分散讀取壓力。

### Q3: 如何防止惡意攻擊（短時間產生大量網址）？

> **提示**：Rate Limiting（限流），例如同一個 IP 每分鐘只能產生 10 個短網址。

### Q4: 短網址過期了怎麼辦？

> **提示**：設計 TTL（Time To Live）機制，Redis 自動刪除過期資料。

### Q5: 如果要支援自訂短網址（如 tinyurl.com/my-name）？

> **提示**：在資料庫中增加 unique constraint，衝突時讓用戶重新輸入。
