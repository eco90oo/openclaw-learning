# 📚 【今日學習：設計 YouTube/Netflix 影片串流系統】

## 1️⃣ 題目說明

### 面試官會怎麼問？

「請設計一個像 YouTube 或 Netflix 的影片串流服務。用戶可以上傳影片、觀看影片、搜尋影片，並根據用戶興趣推薦影片。」

### 需要澄清什麼問題？

1. **影片規模**：
   - 多少影片？總容量多大？
   - 影片長度？（短影片 1 分鐘 vs 電影 2 小時）
   
2. **觀看體驗**：
   - 需要支援哪些畫質？（720p, 1080p, 4K）
   - 需要離線觀看嗎？
   - 延遲要求？（按下播放多久要開始）

3. **功能範圍**：
   - 只是觀看？還是需要評論、、按讚、分享？
   - 需要推薦系統嗎？

## 4️⃣ 核心元件設計

### 4.1 影片上傳與處理

```python
class VideoUploadService:
    """影片上傳服務"""
    
    def __init__(self, storage: StorageService):
        self.storage = storage
    
    async def upload_video(self, user_id: str, video_data: bytes):
        # 1. 建立上傳任務
        task_id = await self.create_upload_task(user_id)
        
        # 2. 儲存原始影片
        video_id = await self.storage.save_raw(video_data)
        
        # 3. 加入轉碼佇列
        await self.transcode_queue.add({
            "video_id": video_id,
            "qualities": ["720p", "1080p", "4K"]
        })
        
        return task_id

class TranscodeService:
    """影片轉碼服務"""
    
    async def transcode(self, video_id: str, qualities: list):
        """將影片轉碼成多種畫質"""
        
        # 使用 FFmpeg 轉碼
        for quality in qualities:
            output_path = f"/storage/{video_id}_{quality}.mp4"
            
            # 轉碼命令
            cmd = f"ffmpeg -i input.mp4 -s {quality} {output_path}"
            await self.run_transcode(cmd)
            
            # 產生縮圖
            await self.generate_thumbnail(video_id)
            
            # 產生 HLS 切片
            await self.generate_hls(video_id, quality)
```

### 4.2 影片播放（HTTP Range Request）

```python
class StreamingService:
    """影片串流服務"""
    
    async def stream_video(self, video_id: str, quality: str, range_header: str):
        """HTTP Range Request 實現串流"""
        
        # 解析 Range header
        start, end = self.parse_range(range_header)
        
        # 取得影片檔案
        video_path = f"/storage/{video_id}_{quality}.mp4"
        
        # 返回特定範圍的影片資料
        return StreamingResponse(
            media=open(video_path, "rb"),
            media_type="video/mp4",
            headers={
                "Content-Range": f"bytes {start}-{end}/*",
                "Accept-Ranges": "bytes"
            }
        )
```

### 4.3 HLS 切片串流

```python
class HLSSegmenter:
    """HLS 切片服務"""
    
    async def segment_video(self, video_path: str, output_dir: str):
        """將影片切成多個小片段"""
        
        # 使用 FFmpeg 產生 m3u8 播放清單
        cmd = f"""
        ffmpeg -i {video_path} \
          -profile:v baseline -level 3.0 \
          -start_number 0 \
          -hls_time 10 \
          -hls_list_size 0 \
          -f hls {output_dir}/output.m3u8
        """
        await self.run_command(cmd)
        
        # 產生的檔案：
        # output.m3u8 (播放清單)
        # output0.ts (第1個片段)
        # output1.ts (第2個片段)
        # ...
```

```python
class RecommendationService:
    """影片推薦服務"""
    
    def __init__(self, ml_model: MLModel):
        self.model = ml_model
    
    async def get_recommendations(self, user_id: str, limit: int = 10):
        # 取得用戶行為數據
        user_data = await self.get_user_behavior(user_id)
        
        # 取得觀看歷史
        watch_history = await self.get_watch_history(user_id)
        
        # ML 模型推薦

            user_features=user_data,
            watch_history=watch_history,
            candidates=await self.get_all_videos(),
            k=limit
        )
        
        return recommendations
```

```sql
CREATE TABLE users (

);

CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_status ON users(status);
```

### 5.2 影片表

```sql
CREATE TABLE videos (
    video_id BIGINT PRIMARY KEY,
    user_id BIGINT REFERENCES users(user_id),
    title VARCHAR(200),
    description TEXT,
    
    -- 影片資訊
    duration_seconds INT,  -- 秒數
    view_count BIGINT DEFAULT 0,
    like_count BIGINT DEFAULT 0,
    
    -- 儲存位置
    storage_path VARCHAR(500),
    thumbnail_path VARCHAR(500),
    
    -- 狀態
    status VARCHAR(20) DEFAULT 'processing',  -- processing, ready, failed
    published_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_videos_user ON videos(user_id);
CREATE INDEX idx_videos_published ON videos(published_at DESC);
CREATE INDEX idx_videos_view_count ON videos(view_count DESC);
```

### 5.3 觀看歷史表

```sql
CREATE TABLE watch_history (
    id BIGINT PRIMARY KEY,
    user_id BIGINT REFERENCES users(user_id),
    video_id BIGINT REFERENCES videos(video_id),

CREATE INDEX idx_watch_user_time ON watch_history(user_id, watch_at DESC);
```

## 7️⃣ 估算與效能

### 頻寬估算

```
假設：
- 10 億觀看次數/天
- 平均觀看時間：10 分鐘
- 平均畫質：720p (2 Mbps)

頻寬計算：
= 10億 × 10分鐘 × 60秒 × 2Mbps
= 10億 × 600秒 × 2Mbps
= 1.2 兆 bps
≈ 1.2 Tbps

需要：
- 多個 CDN 節點
- 影片壓縮優化
- 彈性擴展能力
```

### 儲存估算

```
假設：
- 1 億部影片
- 平均大小：500 MB
- 3 種畫質

儲存空間：
= 1億 × 500MB × 3
= 150 PB

加上備份：
= 150 PB × 3 (副本)
= 450 PB

需要：
- S3 / HDFS 分散式儲存
- 冷熱資料分層
```
