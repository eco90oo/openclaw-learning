## 1️⃣ 基礎概念（用生活比喻）

### 什麼是 MVP？

**傳統思維**：我要打造一台完美的咖啡機，包含磨豆、萃取、奶泡、清潔系統，開發 2 年後才上架。

**MVP 思維**：先做一個「紙杯 + 熱水龍頭」，看用戶是否願意為「咖啡味熱水」付費。如果願意，再一步步加磨豆功能。

**白話文**：MVP 就是「最陽春但能用的版本」，目的是「快速知道方向對不對」，不是「一次做到完美」。

### 為什麼 AI 時代要 7 天驗證？

以前做 MVP 需要工程師寫程式碼，1 個功能可能就要 1 週。現在有了 AI：
- **Copilot**：幫你寫程式碼
- **Claude/GPT**：幫你設計架構、寫文案、想策略
- **No-code 工具**：不用寫程式也能做產品原型

相當於從「手動組裝一台電腦」變成「用積木玩具快速拼出一個展示品」。

### 驗證 vs 開發：你在找金礦 vs 你在挖金礦

| 階段 | 目標 | 心態 |
|------|------|------|
| 驗證 (Validation) | 確認這個山頭有金子 | 偵察兵：用最少力氣確認方向 |
| 開發 (Development) | 把金子挖出來規模化 | 礦工：用全力穩定產出 |

**多數人的錯誤**：還沒確認有沒有金礦，就砸重金買挖礦設備。

## 2️⃣ 核心知識

### MVP 的 5 種經典形式

#### 1. **Landing Page MVP**（登陸頁 MVP）
- **適用時機**：驗證「有沒有人對這個概念感興趣」
- **做法**：做一個漂亮的行銷頁，收集 Email 訂閱
- **驗收標準**：轉換率 > 5%（每 100 個訪客至少 5 人留 email）

#### 2. **Wizard of Oz MVP**（綠野仙蹤 MVP）
- **適用時機**：還不確定技術能否實現，但想先看市場反應
- **做法**：用戶看到的是「自動化系統」，實際上後面有人手動處理
- **經典案例**：Food Robotics 早期沒有廚房機器人，用的是外包廚房，但用戶以為是 AI 在做

#### 3. **Concierge MVP**（專屬管家 MVP）
- **適用時機**：服務複雜、每個用戶需求不同
- **做法**：一對一服務前 10 個用戶，親自了解需求
- **適合**：顧問型、教練型、客製化服務

#### 4. **Prototype MVP**（原型 MVP）
- **適用時機**：產品複雜、需要展示 UX 流程
- **做法**：用 Figma 做出可互動的原型，給用戶測試
- **驗收標準**：用戶能完成核心任務嗎？

#### 5. **Single Feature MVP**（單一功能 MVP）
- **適用時機**：你有很多想法，想先測試哪個最殺手
- **做法**：只做 1 個功能，其他全部不做
- **原則**：「如果用戶只能用 1 個功能，你希望它是什麼？」

#### Day 1：定義假設

**問題錨定**：我發現很多人每天早上浪費 15-30 分鐘想「今天穿什麼」。

**假設陳述（用 HMW 格式）**：

**驗證維度**：
- 用戶真的有這個痛點嗎？
- 他們現在怎麼解決？
- 願意付多少錢？

#### Day 2-3：用戶訪談腳本（AI 協助生成）

```
訪談開場（30秒）：
「你好，我正在研究一個解決穿衣選擇困難的產品想法，很想聽聽你的真實經驗。」

核心問題：

Q1：「你每天早上通常花多久決定穿什麼？」
   → 挖掘：<15分（無痛點）vs >20分（有痛點）

Q2：「你曾經因為穿搭問題遲到或心情不好嗎？」
   → 挖掘：影響程度

Q3：「你現在用什麼方法解決這個問題？」
   → 挖掘：現有解法（對照組）

   → 直接驗證意願

Q5：「你願意為這個服務付多少錢？」
   → 付費意願（不要問「你覺得多少合理」，要問「你上次為類似的東西付過多少」）

# 使用 python-telegram-bot 建立基礎 Bot
# 這個 Bot 在 MVP 階段會由你手動回覆，穿著「AI」的外衣

from telegram import Update
from telegram.ext import Application, CommandHandler, MessageHandler, filters

# 簡單的天氣回覆邏輯（實驗室版本）
async def start(update: Update, context):
    await update.message.reply_text(
        "👋 嗨！我是 StyleBot，你的AI穿搭顧問！\n\n"
        "請告訴我：\n"
        "1️⃣ 今天的天氣（晴/雨/陰/熱/冷）\n"
        "2️⃣ 今天的主要場合（上班/約會/運動/休閒）\n\n"
        "我會給你今日最佳穿搭建議！✨"
    )

async def outfit_suggestion(update: Update, context):
    user_input = update.message.text.lower()
    
    # 簡單規則引擎（實驗室：手動專家模式）
    suggestions = {
        "晴_上班": "🌤️ 晴天上班：淺藍衬衫 + 卡其褲 + 白鞋",
        "雨_上班": "🌧️ 雨天上班：深色風衣 + 牛仔褲 + 防滑靴",
        "熱_休閒": "🔥 高溫休閒：白色背心 + 短褲 + 涼鞋",
    }
    
    # 這裡是 MVP 的魔法：用 AI 包裝成「智慧推薦」
    response = f"""🎽 根據你說的「{user_input}」，
我的 AI 分析建議：

✨ 今日精選：簡約白 T + 高腰牛仔褲 + 白色運動鞋
💡 理由：天氣舒適，這套既涼爽又時尚，
適合一般休閒場合！"""

    await update.message.reply_text(response)

# 啟動 Bot
app = Application.builder().token("YOUR_BOT_TOKEN").build()
app.add_handler(CommandHandler("start", start))
app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, outfit_suggestion))
app.run_polling()
```
