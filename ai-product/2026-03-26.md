## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是 Vibe Coding？

想象一下：

- **傳統開發** = 你去餐廳吃飯，需要先決定要什麼食材、怎麼烹飪、步驟是什麼，然後自己動手做
- **Vibe Coding** = 你走進一家高級餐廳，告訴廚師「我想吃那種吃了會開心的料理」，師就端出美味的菜

**Vibe Coding** 就是用「感覺」來開發軟體。你不需要知道怎麼寫程式碼，只要清楚「想要什麼結果」，AI 幫你把想法變成產品。

### 🔄 傳統開發 vs Vibe Coding

| 面向 | 傳統開發 | Vibe Coding |
|------|----------|--------------|
| **開發者角色** | 技術工匠（自己動手做） | 產品指揮家（告訴 AI 要什麼） |
| **程式碼產出** | 從零手寫 | AI 生成 + 人類審核 |
| **學習曲線** | 陡峭（要學會語法、框架、工具） | 平緩（學會溝通prompt） |
| **開發速度** | 幾週～幾個月 | 幾小時～幾天 |
| **錯誤處理** | 自己debug | AI + 人類一起debug |

## 2️⃣ 核心知識

### 🧠 Vibe Coding 的核心思維

1. **Prompt 即產品**
   - 你對 AI 描述問題的方式，決定了產品的品質
   - 好的 prompt = 清楚的上下文 + 具體的期望 + 明確的邊界

2. **迭代式開發**
   - 先讓 AI 生成一個「大概能用的版本」
   - 再逐步 refine（優化）
   - 傳統開發：一次寫完整個系統；Vibe Coding：不斷問、不断改

3. **人類把關**
   - AI 生成的程式碼可能有漏洞、安全問題
   - 人類需要審核、測試、優化
   - **AI 是 copilot，不是 autopilot（副駕駛，不是自動駕駛）**

4. **工具生態系**
   - Cursor、Windsurf、GitHub Copilot：IDE 整合
   - v0、 bolt.new：網頁/UI 快速生成
   - Replit Agent：全端開發輔助

### 📊 Vibe Coding 適合的場景

| 適合 ✅ | 不適合 ❌ |
|--------|-----------|
| MVP、原型、快速驗證想法 | 需要高度安全性的系統 |
| 簡單的 CRUD 應用 | 複雜的系統架構設計 |
| 學習新技術/框架 | 對效能要求極高的系統 |
| 小型工具/腳本 | 需要完全掌控的底層系統 |

## 3️⃣ 實際操作/程式碼範例

### 🚀 實戰：用 Vibe Coding 開發一個天氣查詢工具

#### 步驟 1：描述你的需求（Prompt）

```text
我要一個簡單的天氣查詢網站：
- 用戶輸入城市名稱
- 顯示現在天氣、温度、濕度
- 介面要乾淨、響應式
- 用 Tailwind CSS 設計
- 不要後端，用 Open-Meteo API（免費）
```

#### 步驟 2：讓 AI 生成（以 v0/bolt.new 為例）

把上面的描述貼进去，AI 會生成完整的程式碼。

#### 步驟 3：驗證與優化

生成的程式碼可能長這樣：

```html
<!DOCTYPE html>
<html lang="zh-TW">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>天氣查詢</title>
    <script src="https://cdn.tailwindcss.com"></script>
</head>
<body class="bg-gradient-to-br from-blue-400 to-purple-500 min-h-screen flex items-center justify-center">
    <div class="bg-white rounded-2xl shadow-2xl p-8 w-full max-w-md">
        <h1 class="text-3xl font-bold text-center mb-6 text-gray-800">🌤️ 天氣查詢</h1>
        
        <div class="flex gap-2 mb-6">
            <input 
                type="text" 
                id="cityInput" 
                placeholder="輸入城市名稱" 
                class="flex-1 px-4 py-3 rounded-lg border-2 border-gray-200 focus:border-blue-500 focus:outline-none transition"
            >
            <button 
                onclick="getWeather()" 
                class="bg-blue-500 text-white px-6 py-3 rounded-lg hover:bg-blue-600 transition font-medium"
            >
                查詢
            </button>
        </div>

        <div id="result" class="hidden text-center">
            <div class="text-6xl mb-4" id="weatherIcon">☀️</div>
            <div class="text-4xl font-bold text-gray-800 mb-2" id="temperature">--°C</div>
            <div class="text-gray-600" id="humidity">濕度: --%</div>
            <div class="text-gray-500 mt-2" id="cityName">--</div>
        </div>

        <div id="error" class="hidden text-red-500 text-center mt-4"></div>
    </div>

    <script>
        // 使用 Open-Meteo API（免費、無需 API Key）
        async function getWeather() {
            const city = document.getElementById('cityInput').value;
            if (!city) return;

            try {
                // 先用 geocoding API 取得座標
                const geoRes = await fetch(
                    `https://geocoding-api.open-meteo.com/v1/search?name=${encodeURIComponent(city)}&count=1`
                );
                const geoData = await geoRes.json();

                if (!geoData.results || geoData.results.length === 0) {
                    throw new Error('找不到這個城市');
                }

                const { latitude, longitude, name } = geoData.results[0];

                // 再取得天氣資料
                const weatherRes = await fetch(
                    `https://api.open-meteo.com/v1/forecast?latitude=${latitude}&longitude=${longitude}&current=temperature_2m,relative_humidity_2m,weather_code`
                );
                const weatherData = await weatherRes.json();

                // 更新 UI
                const temp = weatherData.current.temperature_2m;
                const humidity = weatherData.current.relative_humidity_2m;
                const code = weatherData.current.weather_code;

                document.getElementById('temperature').textContent = `${temp}°C`;
                document.getElementById('humidity').textContent = `濕度: ${humidity}%`;
                document.getElementById('cityName').textContent = name;
                document.getElementById('weatherIcon').textContent = getWeatherEmoji(code);
                
                document.getElementById('result').classList.remove('hidden');
                document.getElementById('error').classList.add('hidden');
            } catch (err) {
                document.getElementById('error').textContent = err.message;
                document.getElementById('error').classList.remove('hidden');
                document.getElementById('result').classList.add('hidden');
            }
        }

        function getWeatherEmoji(code) {
            const map = {
                0: '☀️', 1: '🌤️', 2: '⛅', 3: '☁️',
                45: '🌫️', 48: '🌫️',
                51: '🌧️', 53: '🌧️', 55: '🌧️',
                61: '🌧️', 63: '🌧️', 65: '🌧️',
                71: '🌨️', 73: '🌨️', 75: '🌨️',
                80: '🌧️', 81: '🌧️', 82: '🌧️'
            };
            return map[code] || '❓';
        }
    </script>
</body>
</html>
```

#### 步驟 4：優化 Prompt（Refine）

如果結果不满意，改進 prompt：

```text
天氣查詢網站需要：
- 城市搜尋要有自動完成建議
- 顯示 5 天預報
- 夜間模式切換按鈕
- 錯誤處理要更友善
```

## 4️⃣ 今日小任務

### 🎯 挑戰：用 Vibe Coding 開發一個簡單的「理財記錄工具」

**需求描述（直接複製給 AI）：**

```
建立一個個人理財記錄應用：
- 功能：收入/支出記錄、每月統計圖表
- UI：使用淺綠色調，卡片式設計
- 技術：單頁 HTML，用 Chart.js 畫圖表
- 資料存 localStorage（不需要後端）
- 響應式設計（手機也能用）
```

**驗收標準：**
1. ✅ 能新增收入/支出
2. ✅ 能看到本月總結
3. ✅ 圖表能顯示支出分類比例
4. ✅ 重新整理資料不會消失

**預期產出：** 一個完整的 HTML 檔案，複製到瀏覽器就能運行。
