# 📚 Day 1：Vibe Coding 開發革命 - AI 時代開發方式與傳統開發的差異

> *"以前我們寫程式，現在我們「說」程式。"*

---

## 1️⃣ 基礎概念（用生活比喻）

### 🏗️ 傳統開發：像蓋一棟房子

想像你要蓋一間房子：
1. 先畫詳細的建築藍圖（系統設計）
2. 請土木師傅一磚一瓦蓋（寫程式碼）
3. 每個細節都要親自監工（Debug）
4. 蓋到一半想改設計？對不起，要敲掉重來

**傳統開發的痛點**：工程師是「翻譯機」——把產品經理的想法翻譯成電腦語言。過程冗長、溝通耗費心力、一個小改動可能折騰半天。

### 🚀 AI 時代開發：像用樂高拼積木

現在你有 AI 幫手，就像有了一個超強的樂高助手：
1. 你說「我要一棟有花園的城堡」
2. AI 立刻幫你拼出來
3. 你說「改成歐式風格」
4. AI 刷刷刷改好

這就是 **Vibe Coding（氛圍編程）**——你用自然語言描述需求，AI 幫你生成程式碼。

### 💡 到底什麼是 Vibe Coding？

> **Vibe Coding = 用「感覺」寫程式**
> 
> 不需要懂語法細節，只要清楚表達「我要什麼」，AI 幫你搞定「怎麼做」。

---

## 2️⃣ 核心知識

### 傳統開發 vs Vibe Coding

| 面向 | 傳統開發 | Vibe Coding |
|------|----------|--------------|
| **輸入方式** | 寫程式碼 (Python, JavaScript...) | 說人話 / 寫提示詞 |
| **核心技能** | 語法、框架、API 文件 | 描述需求、批判性思考 |
| **生產力** | 1 人/月 | 1 人/天 |
| **學習曲線** | 陡峭（學好幾年）| 平緩（幾天上手）|
| **適合場景** | 複雜系統、底層優化 | MVP、原型、快速驗證 |

### ⚠️ Vibe Coding 的陷阱

> *AI 很強，但不代表你可以當甩手掌櫃！*

1. **Prompt 寫不好 = 做出來的東西爛**
   - 就像請設計師做海報，你只說「幫我做一個酷的」
   - 結果：設計師做出來的可能跟你想的天差地遠

2. **AI 會一本正經地胡說八道**
   - 程式碼看起來很專業，但可能隱藏 Bug
   - **一定要自己測試！**

3. **複雜系統仍需要架構設計**
   - AI 能幫你寫一個按鈕，但無法幫你設計整個系統
   - 就像 AI 可以幫你寫一段劇情，但整部電影的導演還是你

### 📝 第一個 Vibe Coding 練習

**任務**：讓 AI 幫你寫一個「今日天氣小幫手」網頁

**Step 1：打開你的 AI 開發工具**

打開 Cursor 或 VS Code + Claude，創建新檔案 `weather.html`

**Step 2：寫下你的「氛圍」Prompt**

```
幫我寫一個天氣查詢網頁，要有：
- 輸入城市名稱
- 顯示現在天氣（溫度、濕度、天氣狀況）
- 用簡單漂亮的 UI（漸層背景、卡片式設計）
- 用 Open-Meteo API（免費不用金鑰）
```

**Step 3：AI 會生成什麼？**

AI 會幫你產出類似這樣的程式碼：

```html
<!DOCTYPE html>
<html lang="zh-TW">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>天氣小幫手</title>
    <style>
        * { margin: 0; padding: 0; box-sizing: border-box; }
        body {
            min-height: 100vh;
            background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
            display: flex;
            justify-content: center;
            align-items: center;
            font-family: 'Noto Sans TC', sans-serif;
        }
        .card {
            background: white;
            padding: 2rem;
            border-radius: 20px;
            box-shadow: 0 20px 60px rgba(0,0,0,0.2);
            width: 90%;
            max-width: 400px;
        }
        h1 { color: #333; margin-bottom: 1.5rem; text-align: center; }
        input {
            width: 100%;
            padding: 1rem;
            border: 2px solid #ddd;
            border-radius: 10px;
            font-size: 1rem;
            margin-bottom: 1rem;
        }
        button {
            width: 100%;
            padding: 1rem;
            background: linear-gradient(135deg, #667eea, #764ba2);
            color: white;
            border: none;
            border-radius: 10px;
            font-size: 1rem;
            cursor: pointer;
            transition: transform 0.2s;
        }
        button:hover { transform: scale(1.02); }
        .result {
            margin-top: 1.5rem;
            padding: 1rem;
            background: #f8f9fa;
            border-radius: 10px;
            display: none;
        }
    </style>
</head>
<body>
    <div class="card">
        <h1>🌤️ 天氣小幫手</h1>
        <input type="text" id="city" placeholder="請輸入城市名稱（如：台北）">
        <button onclick="getWeather()">查詢天氣</button>
        <div class="result" id="result"></div>
    </div>

    <script>
        async function getWeather() {
            const city = document.getElementById('city').value;
            if (!city) return alert('請輸入城市名稱');

            // 先取得座標
            const geoUrl = `https://geocoding-api.open-meteo.com/v1/search?name=${city}&count=1&language=zh&format=json`;
            const geoRes = await fetch(geoUrl);
            const geoData = await geoRes.json();

            if (!geoData.results) {
                return alert('找不到這個城市');
            }

            const { latitude, longitude, name } = geoData.results[0];

            // 再取得天氣
            const weatherUrl = `https://api.open-meteo.com/v1/forecast?latitude=${latitude}&longitude=${longitude}&current=temperature_2m,relative_humidity_2m,weather_code`;
            const weatherRes = await fetch(weatherUrl);
            const weatherData = await weatherRes.json();

            const temp = weatherData.current.temperature_2m;
            const humidity = weatherData.current.relative_humidity_2m;
            const code = weatherData.current.weather_code;

            const weatherIcons = {
                0: '☀️ 晴朗', 1: '🌤️ 晴時多雲', 2: '⛅ 多雲', 3: '☁️ 陰天',
                45: '🌫️ 霧', 51: '🌧️ 小雨', 61: '🌧️ 中雨', 80: '🌧️大雨'
            };
            const weather = weatherIcons[code] || '🌤️';

            document.getElementById('result').style.display = 'block';
            document.getElementById('result').innerHTML = `
                <h3>${name}</h3>
                <p style="font-size: 2rem; margin: 0.5rem 0;">${weather}</p>
                <p>🌡️ 溫度：${temp}°C</p>
                <p>💧 濕度：${humidity}%</p>
            `;
        }
    </script>
</body>
</html>
```

**Step 4：測試你的作品**

```bash
# 用瀏覽器打開
open weather.html
```

恭喜！你用「說」的就完成了一個功能完整的天氣 App！
