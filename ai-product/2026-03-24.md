## 1️⃣ 傳統競品分析的問題

### 傳統競品分析的做法

1. **Google 搜尋**：找類似產品
2. **下載試用**：註冊、玩一遍
3. **功能對照表**：畫一個表格，這個有那個沒有
4. **結論**：「我們要做更的功能」

### 問題在哪裡？

| 傳統問題 | AI 時代更嚴重的問題 |
|---------|-------------------|
| 只能看見有形的功能 | 忽略「AI 能力」這個維度 |
| 分析靜態產品 | 忽略產品的迭代速度 |
| 看得見現在 | 看不見未來方向 |
| 10 個競爭對手 | 任何人都能用 AI 快速做出類似產品 |

## 2️⃣ AI 時代競品分析的新框架

### 新框架：五維度分析

```
┌─────────────────────────────────────────┐
│           AI 時代競品分析                │
├──────────┬──────────┬──────────────────┤
│ 維度 1   │ 維度 2   │     維度 3       │
│ 產品功能 │ AI 能力  │   用戶體驗       │
├──────────┼──────────┼──────────────────┤
│ 維度 4   │ 維度 5   │                  │
│ 迭代速度 │ 商業模式 │   獨特優勢       │
└──────────┴──────────┴──────────────────┘
```

### 維度 1：產品功能

**傳統分析內容**：
- 有哪些功能？
- 功能完整度如何？
- UI/UX 好不好？

**AI 時代新問題**：
- 這個功能如果用 AI 重新做，會怎麼樣？
- 對手有沒有在用 AI 增強功能？

### 維度 2：AI 能力（全新維度！）

```markdown
## 評估對手的 AI 能力

1. **有沒有 AI 功能？**
   - 用的是什麼模型？
   - 自建還是外包？

2. **AI 功能好用嗎？**
   - 回應品質如何？
   - 延遲多久？

3. **AI 是不是核心功能？**
   - 還是只是裝飾用的？

4. **有沒有 Prompt/提示詞優化？**
   - 對手在這塊有沒有投入？
```

### 維度 3：迭代速度

**AI 時代的關鍵問題**：

| 問題 | 為什麼重要 |
|------|-----------|
| 對手多久發布一次新功能？| AI 時代，幾天就可能翻盤 |
| 對手用什麼方式收集用戶回饋？| 迭代速度決定生存 |
| 對手的社群活躍度？| 反映產品生命力 |

### 維度 4：商業模式

**傳統問題**：
- 怎麼賺錢？
- 定價策略？

**AI 時代新問題**：
- 用 AI 後成本結構有什麼變化？
- AI 有沒有改變定價模式？
- 訂閱制 vs 用量制哪個更合理？

### 維度 5：獨特優勢（護城河）

**在 AI 時代，什麼是真正的護城河？**

| 護城河類型 | 強度 |
|-----------|------|
| 獨家數據 | ⭐⭐⭐⭐⭐ |
| 網路效應 | ⭐⭐⭐⭐⭐ |
| 品牌 | ⭐⭐⭐⭐ |
| 技術專利 | ⭐⭐⭐ |
| 先發優勢 | ⭐⭐（AI 時代變弱了）|

## 3️⃣ 實際演練：如何做 AI 時代的競品分析？

### Case Study：AI 寫作工具市場

假設你要做一個「AI 寫作助手」，該怎麼分析？

#### Step 1：列出所有對手

```
已知對手：
- Grammarly（文法檢查）
- Jasper AI（行銷文案）
- Copy.ai（廣告文案）
- Notion AI（筆記 + AI）
- 秘塔寫作（中文）
```

#### Step 2：用五維度分析

```markdown
## 對手分析：Jasper AI

### 維度 1：產品功能
- 長文寫作 ✅
- 模板豐富 ✅
- SEO 優化 ✅
- 缺少中文優化 ❌

### 維度 2：AI 能力
- 使用 GPT 模型 ✅
- 自有 fine-tuned 模型 ❌
- API 延遲偏高 ❌

### 維度 3：迭代速度
- 每週更新 ✅
- 用戶回饋管道順暢 ✅

### 維度 4：商業模式
- 訂閱制 $49/月起 ✅
- 企業版有收入 ✅

### 維度 5：護城河
- 品牌知名度 ✅
- 社群規模 ✅
- 數據累積 ✅
- 先發優勢 ✅
```

#### Step 3：找出切入點

```
分析結論：
- 大多數對手都是英文為主
- 中文市場相對空白
- 沒有針對「學術論文」的專業 AI 寫作工具
- 可以做「中文 + 學術論文」的垂直切入
```

## 4️⃣ 市場挖掘的新方法

### 傳統市場挖掘

| 方法 | 問題 |
|------|------|
| 問卷調查 | 問不到真正需求 |
| 焦點團體 | 樣本太小 |
| 竞品分析 | 只能抄襲 |

### AI 時代的新方法

#### 方法 1：觀察 AI 產品評論

```
去哪裡看？
- Product Hunt 評論區
- Reddit r/ArtificialIntelligence
- Twitter/X 推文
- 小紅書評價
- Google Play / App Store 評論

看什麼？
- 用戶在抱怨什麼？
- 用戶想要什麼功能？
- 哪些問題一直沒解決？
```

#### 方法 2：分析 AI 產品排行榜

```
Product Hunt 每天都有新的 AI 產品
- 觀察哪些類型的產品上榜
- 找出規律
- 預測趨勢
```

#### 方法 3：親自使用對手產品

```
不要只下載來看看
要當成真的用戶：
1. 註冊帳號
2. 完成一個完整任務
3. 記錄每一步的感受
4. 找出讓你挫折的地方
5. 那些就是機會！
```

#### 方法 4：利用 AI 輔助分析

```python
# 讓 AI 幫你分析評論
prompt = """
我正在做競品分析，請幫我分析以下用戶評論：

評論內容：[貼上 50 條評論]

請幫我：
1. 歸納出用戶最常抱怨的 5 個問題
2. 歸納出用戶最想要的 5 個功能
3. 給我一個產品改進建議清單
"""
```

## 5️⃣ 實際案例：從 0 到做出差異化產品

### 案例：AI 摘要工具

#### 傳統做法
1. 市場調研 1 個月
2. 寫詳細 PRD
3. 找工程師開發
4. 6 個月後上線
5. 發現市場已經飽和

#### AI 時代做法

**Day 1**：
- 列出 10 個現有 AI 摘要產品
- 每個產品用 30 分鐘徹底試用
- 寫下每個產品的優缺點
- 找出還沒被滿足的需求

**Day 2**：
- 發現「中文長文摘要」是空白市場
- 用 AI 生成一個簡單的 prototype
- 給 5 個目標用戶測試

**Day 3-7**：
- 根據回饋快速迭代
- 持續收集用戶意見

**關鍵洞察**：
- **不是「做大一點」，而是「做得不一樣」**
- **用 AI 快速驗證，比傳統方式快 10 倍**
