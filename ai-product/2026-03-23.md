> 在 AI 時代，開發產品的速度與方式正在發生翻天覆地的變化。這堂課會帶你掌握 AI 時代的產品開發心法。

## 1️⃣ 基礎概念：什麼是 Vibe Coding？

### 傳統開發 vs AI 時代開發

| 面向 | 傳統開發 | AI 時代（Vibe Coding） |
|------|---------|----------------------|
| **開發速度** | 數週～數月 | 數小時～數天 |
| **技術門檻** | 需要會寫程式 | 懂得描述需求即可 |
| **迭代方式** | 瀑布式 / 敏捷 | 持續快速迭代 |
| **MVP 定義** | 幾週做出基本功能 | 幾小時做出可展示原型 |
| **學習曲線** | 陡峭（需學多種技術）| 平緩（會說話就能做）|

### 什麼是 Vibe Coding？

**Vibe Coding** = "用感覺寫程式"

想像傳統程式碼是這樣寫的：
```
1. 學會 Python 語法
2. 學會 SQL
3. 學會 React
4. 學會部署
5. 整合所有技術
6. 終於做出產品
```

Vibe Coding 是這樣的：
```
1. 告訴 AI：「我要做一個能記錄待辦事項的 App」
2. AI 生成程式碼
3. 調整一下细节
4. 產品完成！
```

**你不是在「寫程式」，你是在「創造產品」**

## 2️⃣ AI 時代產品開發的三大階段

### Stage 1：Idea → Prototype（想法到原型）

在傳統時代，做出原型可能需要：
- UI 設計師畫 Wireframe（1-2 週）
- 工程師刻畫面（2-4 週）
- 前後端串接（2-4 週）

**AI 時代：幾小時搞定！**

```python
# 例子：用 Claude Code 快速建立專案
# 你只需要告訴 AI：
"""
我要做一個「個人理財助手」App
功能：
1. 記錄收入與支出
2. 自動分類消費類型
3. 顯示本月財務報表
4. 支援多幣別

請用 Streamlit 做一個 prototype
"""
```

### Stage 2：Prototype → MVP（原型到最小可行性產品）

| 傳統做法 | AI 時代做法 |
|---------|------------|
| 找工程師估時程 | 用 AI 快速生成 |
| 做詳細 PRD | 邊做邊改 |
| 分好幾個 sprint | 每天都能出新版 |

### Stage 3：MVP → Scale（產品到規模化）

AI 時代的獨特挑戰：
- **別人也能用 AI 做出類似產品**
- **護城河更重要了**
- **要快速找到 Product-Market Fit**

## 3️⃣ AI 時代產品開發的核心思維

### 思維 1：快速驗證 > 完美規劃

**傳統思維**：「先規劃好再做」
**AI 時代**：「先做出來再說」

```
為什麼？
- AI 讓修改成本大幅降低
- 市場變化太快，規劃趕不上變化
- 只有做出來才能得到真實用戶回饋
```

### 思維 2：問題導向 > 技術導向

**傳統思維**：「我會這個技術，所以做這個產品」
**AI 時代**：「這個問題需要被解決，用 AI 來解決」

### 思維 3：最小功能 > 功能齊全

**例子**：
- 傳統：做出 30 個功能才上線
- AI 時代：先上 3 個核心功能，快速收集回饋

## 4️⃣ 實戰：如何用 AI 快速開發一個產品？

### Step 1：定義清楚問題（10 分鐘）

```markdown
## 產品點子驗證表

1. **痛點是什麼？**
   - 誰會痛？
   - 多痛？

2. **現有解決方案？**
   - 他們哪裡不好？

3. **我的不一樣？**
   - AI 怎麼讓它更好？

4. **誰會是第一個用戶？**
   - 具體是什麼人？

5. **怎麼驗證？**
   - 用什麼指標證明有人要？
```

### Step 2：用 AI 生成基礎框架（30 分鐘）

你可以這樣問 AI：

```
我要做一個 [產品名稱]
目標用戶：[誰]
解決問題：[什麼痛點]

請幫我：
1. 產生基本的技術架構
2. 寫出第一版程式碼
3. 建議部署方式
```

### Step 3：快速迭代（持續）

| 天數 | 任務 |
|------|------|
| Day 1 | 做出可運行的 prototype |
| Day 2-3 | 加入核心功能 |
| Day 4-5 | 優化用戶體驗 |
| Day 6-7 | 找用戶測試 |

## 5️⃣ AI 時代產品經理的新技能

| 傳統 PM 技能 | AI 時代 PM 技能 |
|-------------|----------------|
| 寫 PRD | 寫清晰的 AI Prompt |
| 管理工程師進度 | 管理 AI 生成品質 |
| 協調團隊溝通 | 協調人類 + AI 協作 |
| 規劃功能優先級 | 規劃 AI 能力範圍 |

### AI Prompt 工程變成核心能力

```markdown
# 好的 Prompt 要包含：
- 角色：你希望 AI 扮演什麼角色
- 背景：產品的背景資訊
- 任務：具體要 AI 做什麼
- 限制：有什麼不能做的
- 範例：給幾個例子參考
```

## 6️⃣ 今日小任務

### 任務 1：體驗 Vibe Coding

1. 打開 Claude Code 或 Cursor

4. 記錄過程中的問題與發現

### 任務 2：寫出你的產品點子
