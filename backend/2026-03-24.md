## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是 FastAPI？

想象一下：

- **傳統函式** = 你打電話給一家餐廳訂餐，對方接聽後你直接說要什麼，對方記下來
- **Web API** = 餐廳提供了一個「菜單網站」，你可以上網瀏覽、下單、查看狀態
- **FastAPI** = 這個網站的「點餐系統」，它規定了一套標準流程，讓客人可以清楚地下單、餐廳可以清楚地處理

### 🔍 什麼是 Route（路由）？

想象你在一家大型圖書館：

- **Route** = 圖書館的「指引標示」
  - `GET /books` = 「書籍區」的標示
  - `GET /books/001` = 「第 001 號書籍」的標示
  - `POST /borrow` = 「借書服務」的標示

每個 Route 就像圖書館的一個服務櫃台，告訴 API「這個網址應該做什麼事」

### ❓ 什麼是 Query Parameters（查詢參數）？

想象你在 Amazon 網站上搜尋商品：

- `https://amazon.com/search?keyword=python&sort=price_asc`
- `?` 後面的部分就是 **Query Parameters**（查詢參數）
- `keyword=python` = 搜尋關鍵字是「python」
- `sort=price_asc` = 按價格「從低到高」排序

這就像你在對服務員說：「我想找 **python** 相關的書，**價格從低到高** 排序」

### 📊 HTTP 方法簡介

| 方法 | 比喻 | 用途 |
|------|------|------|
| GET | 看菜單 | 取得資料（查詢） |
| POST | 點新菜 | 提交新資料（新增） |
| PUT | 換菜 | 更新整筆資料 |
| DELETE | 退菜 | 刪除資料 |
| PATCH | 修改部分菜 | 部分更新 |

### 🎬 第一個 FastAPI 程式

```python
# 導入 FastAPI 類別
# 就像從工具箱裡拿出 FastAPI 這個工具
from fastapi import FastAPI

# 建立 FastAPI 實例（建立一個新的餐廳點餐系統）
# app 就像是我們創建的一個「餐廳實例」
app = FastAPI()

# @app.get() 是裝飾器就像是給這個函式貼上一個標籤
# 這個標籤說：「當有人訪問 / 這個網址時，用 GET 方法」
@app.get("/")
def read_root():
    # 這個函式就是「處理這個請求的廚師」
    # 會回傳一個簡單的歡迎訊息
    return {"message": "歡迎來到我的 API 餐廳！"}

# 啟動指令（在終端機執行）：
# uvicorn main:app --reload
# uvicorn = 伺服器引擎
# main = 檔案名稱（main.py）
# app = 我們定義的 FastAPI 實例
# --reload = 自動重新載入（開發時使用）
```

### 📝 建立帶參數的 Route

```python
# 導入 FastAPI 和查詢參數類型
from fastapi import FastAPI
from typing import Optional

# 建立 FastAPI 實例
app = FastAPI()

# 📖 GET 請求：取得所有使用者
# 這就像「查看所有會員名單」
@app.get("/users")
def get_users():
    # 回傳一個使用者列表
    return [
        {"id": 1, "name": "小明", "age": 25},
        {"id": 2, "name": "小華", "age": 30}
    ]

# 🔍 GET 請求：根據 ID 取得單一使用者
# 路徑參數用 {} 包裹
# 這就像「查看第 5 號會員的資料」
@app.get("/users/{user_id}")
def get_user(user_id: int):
    # user_id 會自動從網址中提取出來
    return {"id": user_id, "name": "小明", "age": 25}

# 🎯 GET 請求：搜尋使用者（使用 Query Parameters）
# 這就像「搜尋符合條件的會員」
# query 参数：可選的搜尋關鍵字
# limit 参数：最多回傳幾筆結果
@app.get("/search")
def search_users(
    query: str = "",           # 可選參數，預設為空字串
    limit: int = 10            # 可選參數，預設為 10
):
    # 模擬搜尋結果
    results = [
        {"id": 1, "name": "小明", "score": 95},
        {"id": 2, "name": "小華", "score": 87}
    ]
    
    # 如果有搜尋關鍵字，就過濾結果
    if query:
        results = [r for r in results if query.lower() in r["name"].lower()]
    
    # 限制回傳數量
    return results[:limit]

# 📝 POST 請求：建立新使用者
# 這就像「填寫入會申請表」
from pydantic import BaseModel

# 定義使用者資料的格式（先簡單介绍）
class UserCreate(BaseModel):
    name: str      # 使用者名稱（必填）
    age: int       # 年齡（必填）
    email: str     # 電子郵件（必填）

@app.post("/users")
def create_user(user: UserCreate):
    # user 會自動從請求 body 中解析出來
    # 這裡模擬建立使用者的邏輯
    return {
        "message": "建立成功！",
        "user": user.dict()  # 轉換為字典格式
    }
```

### 🔧 更完整的範例：使用者管理系統

```python
# 完整範例：簡易使用者管理 API
from fastapi import FastAPI, Query, Path, HTTPException
from pydantic import BaseModel
from typing import Optional, List

# ========================================
# 1️⃣ 建立 FastAPI 實例
# ========================================
# 這就像開一家新的餐廳，需要一個名稱和描述
app = FastAPI(
    title="使用者管理系統",      # API 的標題
    description="這是一個示範的 API 系統",  # API 的描述
    version="1.0.0"            # API 版本
)

# ========================================
# 2️⃣ 建立資料模型
# ========================================
# 就像制定「會員卡」的格式，規定需要哪些欄位
class User(BaseModel):
    """使用者資料模型"""
    id: int                  # 使用者編號
    name: str                # 使用者名稱
    email: str               # 電子郵件
    age: Optional[int] = None  # 年齡（可選欄位，預設 None）

# 模擬資料庫（用列表存放）
# 就像一個放在櫃檯的紙本名冊
users_db = [
    {"id": 1, "name": "小明", "email": "ming@example.com", "age": 25},
    {"id": 2, "name": "小華", "email": "hua@example.com", "age": 30},
]

# ========================================
# 3️⃣ 建立 GET Routes（讀取資料）
# ========================================

# 📋 取得所有使用者
# GET /users
@app.get("/users", response_model=List[User])
def get_users(
    # ----------------------------
    # Query Parameters（查詢參數）
    # ----------------------------
    # 就像櫃台小姐問：「您要找什麼類型的會員？」
    # age_min: 最小年齡篩選
    # age_max: 最大年齡篩選
    # name: 名字關鍵字搜尋
    age_min: Optional[int] = Query(None, description="最小年齡"),
    age_max: Optional[int] = Query(None, description="最大年齡"),
    name: Optional[str] = Query(None, description="名字關鍵字")
):
    """取得所有使用者（可篩選條件）"""
    results = users_db  # 先取得所有資料
    
    # 如果有指定年齡範圍，就過濾
    if age_min is not None:
        results = [u for u in results if u.get("age", 0) >= age_min]
    if age_max is not None:
        results = [u for u in results if u.get("age", 0) <= age_max]
    
    # 如果有指定名字關鍵字，就過濾
    if name:
        results = [u for u in results if name.lower() in u["name"].lower()]
    
    return results  # 回傳結果

# 🔍 取得單一使用者（路徑參數）
# GET /users/{user_id}
@app.get("/users/{user_id}", response_model=User)
def get_user(
    # ----------------------------
    # Path Parameters（路徑參數）
    # ----------------------------
    # 就像說「我要找 5 號會員」
    user_id: int = Path(..., description="使用者 ID")
):
    """根據 ID 取得單一使用者"""
    # 在資料庫中尋找
    for user in users_db:
        if user["id"] == user_id:
            return user
    
    # 如果找不到，拋出 404 錯誤
    # 就像告訴客人：「對不起，沒有這個會員編號」
    raise HTTPException(status_code=404, detail="找不到該使用者")

# 🎲 取得隨機使用者
# GET /users/random
@app.get("/users/random", response_model=User)
def get_random_user():
    """取得一位隨機使用者"""
    import random  # 導入隨機模組
    if not users_db:
        raise HTTPException(status_code=404, detail="沒有使用者資料")
    return random.choice(users_db)

# ========================================
# 4️⃣ 建立 POST Routes（新增資料）
# ========================================

# ➕ 建立新使用者
# POST /users
class UserCreate(BaseModel):
    """建立使用者時的資料格式"""
    name: str                          # 必填：名字
    email: str                         # 必填：電子郵件
    age: Optional[int] = None         # 選填：年齡

@app.post("/users", response_model=User, status_code=201)
def create_user(user: UserCreate):
    """建立新使用者"""
    # 產生新的 ID（取目前最大 ID + 1）
    new_id = max([u["id"] for u in users_db]) + 1 if users_db else 1
    
    # 建立新使用者物件
    new_user = {
        "id": new_id,
        "name": user.name,
        "email": user.email,
        "age": user.age
    }
    
    # 存入資料庫（添加到列表）
    users_db.append(new_user)
    
    # 回傳建立的使用者
    return new_user

# ========================================
# 5️⃣ 啟動伺服器的方式
# ========================================

# 啟動方式 1：直接執行（開發用）
# uvicorn main:app --reload --host 0.0.0.0 --port 8000

# 啟動方式 2：使用 Python 執行
# if __name__ == "__main__":
#     import uvicorn
#     uvicorn.run(app, host="0.0.0.0", port=8000)

# ========================================
# 6️⃣ 測試方式
# ========================================

# 打開瀏覽器訪問：
# http://localhost:8000/users          -> 查看所有使用者
# http://localhost:8000/users?age_min=20 -> 篩選 20 歲以上的使用者
# http://localhost:8000/users/1         -> 查看 ID 為 1 的使用者
# http://localhost:8000/docs            -> 自動產生的 API 文件

# 使用 curl 測試：
# curl http://localhost:8000/users
# curl -X POST http://localhost:8000/users \
#      -H "Content-Type: application/json" \
#      -d '{"name": "小杰", "email": "jie@example.com", "age": 22}'
```

```python
from fastapi import FastAPI, Query
from typing import Optional

app = FastAPI(title="天氣查詢 API")

# 模擬天氣資料（實際開發會從 API 取得）
weather_data = {
    "台北": {"temp": 25, "weather": "晴天", "humidity": 60},
    "台中": {"temp": 28, "weather": "多雲", "humidity": 55},
    "高雄": {"temp": 30, "weather": "晴天", "humidity": 70},
    "花蓮": {"temp": 24, "weather": "雨天", "humidity": 80},
}

# GET /weather - 查詢天氣
@app.get("/weather")
def get_weather(
    city: str = Query(..., description="城市名稱")  # 必填參數
):
    """
    查詢指定城市的天氣
    
    使用方式：
    GET /weather?city=台北

    weather = weather_data.get(city)
    
    # 如果找不到城市
    if weather is None:
        return {
            "success": False,
            "message": f"對不起，沒有{city}的天氣資料",
            "available_cities": list(weather_data.keys())
        }
    
    # 回傳天氣資料
    return {
        "success": True,
        "city": city,
        "temperature": weather["temp"],
        "weather": weather["weather"],
        "humidity": weather["humidity"]
    }

# GET /weather/all - 取得所有城市天氣
@app.get("/weather/all")
def get_all_weather():
    """取得所有城市的天氣"""
    return {
        "cities": weather_data
    }
```

### 🛠️ 實作練習 3：建立簡單的 TODO API

```python
# 檔案：todo.py
# 簡易待辦事項 API

from fastapi import FastAPI, HTTPException, Query
from pydantic import BaseModel
from typing import Optional, List
from datetime import datetime

app = FastAPI(title="待辦事項 API", version="1.0.0")

# ========================================
# 資料模型定義
# ========================================

class TodoItem(BaseModel):
    """待辦事項的資料格式"""
    id: int                          # 事項 ID
    title: str                       # 事項標題
    completed: bool = False          # 是否完成（預設為 False）
    created_at: str                  # 建立時間

class TodoCreate(BaseModel):
    """建立新事項時的資料格式"""
    title: str                       # 事項標題（必填）

# ========================================
# 模擬資料庫
# ========================================
# 用列表模擬資料庫
todos_db: List[TodoItem] = [
    TodoItem(id=1, title="學 Python", completed=True, created_at="2024-01-01"),
    TodoItem(id=2, title="學 FastAPI", completed=False, created_at="2024-01-02"),
]

# ========================================
# API Routes
# ========================================

# 📋 GET /todos - 取得所有待辦事項
@app.get("/todos", response_model=List[TodoItem])
def get_todos(
    completed: Optional[bool] = Query(None, description="篩選完成狀態")
):
    """
    取得所有待辦事項
    
    - 不帶參數：回傳所有事項
    - ?completed=true：只回傳已完成的事項
    - ?completed=false：只回傳未完成的事項
    """
    if completed is None:
        return todos_db
    return [t for t in todos_db if t.completed == completed]

# 🔍 GET /todos/{todo_id} - 取得單一事項
@app.get("/todos/{todo_id}", response_model=TodoItem)
def get_todo(todo_id: int):
    """根據 ID 取得待辦事項"""
    for todo in todos_db:
        if todo.id == todo_id:
            return todo
    raise HTTPException(status_code=404, detail="找不到該事項")

# ➕ POST /todos - 新增待辦事項
@app.post("/todos", response_model=TodoItem, status_code=201)
def create_todo(todo: TodoCreate):
    """建立新的待辦事項"""
    # 產生新 ID
    new_id = max([t.id for t in todos_db]) + 1 if todos_db else 1
    
    # 建立新事項
    new_todo = TodoItem(
        id=new_id,
        title=todo.title,
        completed=False,
        created_at=datetime.now().strftime("%Y-%m-%d")
    )
    
    # 加入資料庫
    todos_db.append(new_todo)
    
    return new_todo

# ✅ PUT /todos/{todo_id} - 標記完成/未完成
@app.put("/todos/{todo_id}", response_model=TodoItem)
def update_todo(todo_id: int, completed: bool):
    """更新事項的完成狀態"""
    for todo in todos_db:
        if todo.id == todo_id:
            todo.completed = completed
            return todo
    raise HTTPException(status_code=404, detail="找不到該事項")

# 🗑️ DELETE /todos/{todo_id} - 刪除事項
@app.delete("/todos/{todo_id}")
def delete_todo(todo_id: int):
    """刪除待辦事項"""
    for i, todo in enumerate(todos_db):
        if todo.id == todo_id:
            todos_db.pop(i)
            return {"message": "刪除成功", "id": todo_id}
    raise HTTPException(status_code=404, detail="找不到該事項")
```

```bash
# 啟動測試
uvicorn todo:app --reload

# 測試各種操作：
# GET    http://localhost:8000/todos
# GET    http://localhost:8000/todos?completed=false
# POST   http://localhost:8000/todos (body: {"title": "新事項"})
# PUT    http://localhost:8000/todos/1?completed=true
# DELETE http://localhost:8000/todos/1
```

## 4️⃣ 常見錯誤提醒

### ❌ 錯誤 1：忘記 import 需要的模組

```python
# ❌ 錯誤示範
from fastapi import FastAPI
# 漏了 Query

app = FastAPI()

# 會出錯：Query 未定義
@app.get("/search")
def search(q: str = Query(...)):
    return {"query": q}
```

```python
# ✅ 正確寫法
from fastapi import FastAPI, Query  # 記得導入 Query
```

### ❌ 錯誤 2：路徑參數與 Query 參數搞混

```python
# ❌ 錯誤示範
@app.get("/users/{id}")
def get_user(id: int):  # 這是路徑參數
    ...

# ❌ 寫成 Query 參數的樣子
@app.get("/users")
def get_user(id: int = Query(...)):  # 這是 Query 參數
    ...
```

```python
# ✅ 區分方式：
# /users/{id} -> Path Parameters（路徑的一部分）
# /users?id=1 -> Query Parameters（? 後面的參數）
```

### ❌ 錯誤 3：返回的資料格式錯誤

```python
# ❌ 錯誤示範
@app.get("/user")
def get_user():
    return "小明"  # 應該回傳 JSON 物件或列表
```

```python
# ✅ 正確寫法
@app.get("/user")
def get_user():
    return {"name": "小明"}  # 回傳字典（會轉為 JSON）
```

### ❌ 錯誤 4：路徑重複定義

```python
# ❌ 錯誤示範
@app.get("/users")
def get_users():
    ...

@app.get("/users")  # 同一個路徑重複定義！
def get_users2():
    ...
```

### 💡 提示

```python
# 1. 先定義資料模型
class Book(BaseModel):
    id: int
    title: str
    author: str
    year: int

class BookCreate(BaseModel):
    title: str
    author: str
    year: int

# 2. 用列表當作模擬資料庫
books_db = [...]

# 3. 實作各個 Route
#    - 注意回傳類型標註
#    - 使用 HTTPException 處理錯誤
#    - Query Parameters 要用 Query()
```
