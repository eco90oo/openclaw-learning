# 📚 Day 9：FastAPI POST 請求 — Request Body 與 Pydantic 模型

## 1️⃣ 基礎概念（用生活比喻）

### 🎯 什麼是 POST 請求？

**比喻**：GET 就像是「看菜單」（只讀取），POST 就像是「填寫訂單」（寫入資料）。

讓我們用餐廳來比喻：

| HTTP 方法 | 餐廳比喻 | 實際用途 |
|----------|---------|---------|
| GET | 看菜單 | 獲取資料（查詢） |
| POST | 填寫訂單 | **新增資料** |
| PUT | 修改訂單 | 更新資料 |
| DELETE | 取消訂單 | 刪除資料 |

**舉例**：
- `GET /users` → 取得用戶列表（就像看餐廳有什麼菜）
- `POST /users` → **新增一個用戶**（就像填寫訂單單）
- `GET /users/1` → 取得 ID 為 1 的用戶
- `PUT /users/1` → 修改 ID 為 1 的用戶
- `DELETE /users/1` → 刪除 ID 為 1 的用戶

### 🎯 什麼是 Request Body（請求主體）？

**比喻**：Request Body 就像是「訂單的內容」。

當你填寫餐廳訂單時：
- **URL** = 餐廳地址（例如 `/orders`）
- **HTTP 方法** = 動作類型（POST = 提交訂單）
- **Request Body（請求主體）** = 訂單內容：
  - 想要的前菜：沙拉
  - 想要的主餐：牛排
  - 想要的水：可樂
  - 數量：1 份

```json
// 就像填寫這張訂單
{
  "appetizer": "沙拉",
  "main_course": "牛排",
  "drink": "可樂",
  "quantity": 1
}
```

### 🎯 什麼是 Pydantic？

**比喻**：Pydantic 就像是「訂單檢查員」。

想像餐廳有個檢查員會檢查：
- ☑️ 桌號是否填寫？（必要欄位）
- ☑️ 數量是不是數字？（類型正確）
- ☑️ 數量能不能是負數？（數值範圍）
- ☑️ 價格有沒有超過預算？（最大值限制）

如果訂單不合規定，檢查員會說：
> 「抱歉！數量不能是負數喔！」

Pydantic 就是這個「檢查員」，它會：
1. **自動驗證**資料格式
2. **自動轉換**資料類型
3. **自動產生錯誤訊息**

### 📌 完整的 CRUD 範例

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field
from typing import Optional
from datetime import datetime

app = FastAPI()

# ========== 定義 Pydantic 模型 ==========

# 用戶模型（POST / PUT 使用）
class UserCreate(BaseModel):
    name: str = Field(..., min_length=1, max_length=50)  # 名字長度限制
    email: str = Field(...)  # 必填
    age: Optional[int] = Field(None, ge=0, le=150)  # 年齡 0-150
    password: str = Field(..., min_length=6)  # 密碼至少 6 個字

# 用戶回應模型（GET 使用，不包含密碼）
class UserResponse(BaseModel):
    id: int
    name: str
    email: str
    age: Optional[int]
    created_at: str

# 模擬資料庫
users_db = {}
next_id = 1

# ========== API 端點 ==========

# POST：建立新用戶
@app.post("/users", response_model=UserResponse)
def create_user(user: UserCreate):
    """建立新用戶"""
    global next_id
    
    # 建立用戶資料
    new_user = {
        "id": next_id,
        "name": user.name,
        "email": user.email,
        "age": user.age,
        "password": user.password,  # 實際開發請加密！
        "created_at": datetime.now().isoformat()
    }
    
    users_db[next_id] = new_user
    next_id += 1
    
    # 回應時不包含密碼（只回應 UserResponse）
    return UserResponse(**new_user)

# GET：取得所有用戶
@app.get("/users", response_model=list[UserResponse])
def get_all_users():
    """取得所有用戶"""
    return [UserResponse(**u) for u in users_db.values()]

# GET：取得單一用戶
@app.get("/users/{user_id}", response_model=UserResponse)
def get_user(user_id: int):
    """根據 ID 取得用戶"""
    if user_id not in users_db:
        raise HTTPException(status_code=404, detail="用戶不存在")
    return UserResponse(**users_db[user_id])

# PUT：更新用戶
@app.put("/users/{user_id}", response_model=UserResponse)
def update_user(user_id: int, user_update: UserCreate):
    """更新用戶資料"""
    if user_id not in users_db:
        raise HTTPException(status_code=404, detail="用戶不存在")
    
    # 更新資料
    existing_user = users_db[user_id]
    existing_user["name"] = user_update.name
    existing_user["email"] = user_update.email
    existing_user["age"] = user_update.age
    existing_user["password"] = user_update.password
    
    return UserResponse(**existing_user)

# DELETE：刪除用戶
@app.delete("/users/{user_id}")
def delete_user(user_id: int):
    """刪除用戶"""
    if user_id not in users_db:
        raise HTTPException(status_code=404, detail="用戶不存在")
    
    del users_db[user_id]
    return {"message": "用戶已刪除", "user_id": user_id}
```

### 📌 Pydantic 驗證裝飾器（Field）

```python
from pydantic import BaseModel, Field, EmailStr
from typing import Optional

class User(BaseModel):
    # 字串驗證
    name: str = Field(..., min_length=1, max_length=50)  # 長度限制
    nickname: Optional[str] = Field(None, max_length=30)  # 可選，有則限制長度
    
    # 數字驗證
    age: int = Field(..., ge=0, le=150)  # ge = greater or equal, le = less or equal
    price: float = Field(..., ge=0)      # 必須 >= 0
    quantity: int = Field(1, ge=1, le=100)  # 預設 1，範圍 1-100
    
    # 特殊格式
    email: EmailStr = Field(...)  # 自動驗證 email 格式！
    website: Optional[str] = None
    
    # 布林值
    is_active: bool = True  # 預設為 True
    is_admin: bool = False

# 使用 EmailStr 需要額外安裝：
pip install email-validator
```

### 📌 巢狀物件（Nested Models）

```python
from pydantic import BaseModel
from typing import List, Optional

# 地址模型
class Address(BaseModel):
    city: str
    district: str
    street: str
    zip_code: str

# 用戶模型（包含地址）
class User(BaseModel):
    name: str
    age: int
    # 巢狀物件
    address: Address
    # 列表
    hobbies: List[str] = []
    # 可選的巢狀物件
    emergency_contact: Optional[Address] = None

# 使用範例
user_data = {
    "name": "小明",
    "age": 25,
    "address": {
        "city": "台北市",
        "district": "信義區",
        "street": "忠孝東路一段",
        "zip_code": "110"
    },
    "hobbies": ["籃球", "看電影"],
    "emergency_contact": None
}

user = User(**user_data)
print(user.name)  # 小明
print(user.address.city)  # 台北市
print(user.hobbies)  # ['籃球', '看電影']
```

```python
# day9_post/restaurant_api.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field, EmailStr
from typing import List, Optional
from datetime import datetime

app = FastAPI(title="餐廳訂餐系統", description="學習 POST 請求的好幫手")

# ========== 定義資料模型 ==========

class MenuItem(BaseModel):
    """菜單項目"""
    id: int
    name: str
    price: float = Field(..., gt=0)  # 價格必須大於 0
    description: str = ""

class OrderItem(BaseModel):
    """訂單中的項目"""
    menu_item_id: int  # 菜單項目 ID
    quantity: int = Field(1, ge=1, le=10)  # 數量 1-10

class OrderCreate(BaseModel):
    """建立訂單的請求"""
    customer_name: str = Field(..., min_length=1)
    customer_email: EmailStr
    items: List[OrderItem] = Field(..., min_length=1)  # 至少點一道菜

class OrderResponse(BaseModel):
    """訂單回應"""
    order_id: int
    customer_name: str
    customer_email: str
    items: List[dict]
    total_price: float
    status: str
    created_at: str

# ========== 模擬資料 ==========

# 菜單
menu = {
    1: {"id": 1, "name": "招牌牛排", "price": 350, "description": "美國頂級牛排"},
    2: {"id": 2, "name": "海鮮義大利麵", "price": 280, "description": "新鮮海產"},
    3: {"id": 3, "name": "凱撒沙拉", "price": 150, "description": "清脆爽口"},
    4: {"id": 4, "name": "蘑菇濃湯", "price": 100, "description": "香濃可口"},
    5: {"id": 5, "name": "提拉米蘇", "price": 120, "description": "經典甜點"},
}

# 訂單資料庫
orders_db = {}
next_order_id = 1

# ========== API 端點 ==========

# GET：取得菜單
@app.get("/menu")
def get_menu():
    """取得所有菜單項目"""
    return {"menu": list(menu.values())}

# GET：取得單一菜單項目
@app.get("/menu/{item_id}")
def get_menu_item(item_id: int):
    """根據 ID 取得菜單項目"""
    if item_id not in menu:
        raise HTTPException(status_code=404, detail="菜單項目不存在")
    return menu[item_id]

# POST：建立訂單
@app.post("/orders", response_model=OrderResponse)
def create_order(order: OrderCreate):
    """建立新訂單"""
    global next_order_id
    
    # 計算總金額
    order_items = []
    total_price = 0
    
    for item in order.items:
        if item.menu_item_id not in menu:
            raise HTTPException(
                status_code=400, 
                detail=f"菜單項目 ID {item.menu_item_id} 不存在"
            )
        
        menu_item = menu[item.menu_item_id]
        item_total = menu_item["price"] * item.quantity
        total_price += item_total
        
        order_items.append({
            "name": menu_item["name"],
            "price": menu_item["price"],
            "quantity": item.quantity,
            "subtotal": item_total
        })
    
    # 建立訂單資料
    new_order = {
        "order_id": next_order_id,
        "customer_name": order.customer_name,
        "customer_email": order.customer_email,
        "items": order_items,
        "total_price": total_price,
        "status": "確認中",
        "created_at": datetime.now().isoformat()
    }
    
    orders_db[next_order_id] = new_order
    next_order_id += 1
    
    return OrderResponse(**new_order)

# GET：取得所有訂單
@app.get("/orders")
def get_orders():
    """取得所有訂單"""
    return {"orders": list(orders_db.values())}

# GET：取得單一訂單
@app.get("/orders/{order_id}")
def get_order(order_id: int):
    """根據 ID 取得訂單"""
    if order_id not in orders_db:
        raise HTTPException(status_code=404, detail="訂單不存在")
    return orders_db[order_id]
```

## 4️⃣ 常見錯誤提醒

### ⚠️ 錯誤 1：忘記 import BaseModel

```python
# 錯！
class User:
    name: str

# 對！必須繼承 BaseModel
from pydantic import BaseModel

class User(BaseModel):
    name: str
```

### ⚠️ 錯誤 2：Field 和模型混用

```python
# 錯！Field 必須 import
name: str = Field(...)  # NameError: name 'Field' is not defined

# 對！
from pydantic import BaseModel, Field

class User(BaseModel):
    name: str = Field(...)
```

### ⚠️ 錯誤 3：EmailStr 需要額外安裝

```python
# 錯！沒有安裝 email-validator
class User(BaseModel):
    email: EmailStr
# ValidationError

# 對！先安裝
pip install email-validator
```

### ⚠️ 錯誤 4：response_model 和回傳類型不匹配

```python
# 錯！回傳 dict，但 response_model 期望 UserResponse
@app.post("/users", response_model=UserResponse)
def create_user(user: UserCreate):
    return {"name": user.name}  # 缺少欄位！

# 對！回傳符合模型的資料
@app.post("/users", response_model=UserResponse)
def create_user(user: UserCreate):
    new_user = {"id": 1, "name": user.name, "email": user.email, "age": user.age}
    return UserResponse(**new_user)
```

### ⚠️ 錯誤 5：在請求模型中暴露敏感資料

```python
# 錯！密碼會被回傳給客戶端
class User(BaseModel):
    name: str
    password: str  # 危險！

# 對！建立兩個模型
class UserCreate(BaseModel):
    name: str
    password: str  # 輸入時需要

class UserResponse(BaseModel):
    name: str
    # 沒有密碼！
```

## 📚 今日重點回顧

| 概念 | 說明 |
|------|------|
| POST 請求 | 用於「新增」資料，就像填寫訂單 |
| Request Body | POST 請求中的資料主體（JSON 格式） |
| Pydantic | Python 的資料驗證庫，就像「檢查員」 |
| BaseModel | Pydantic 的基礎類別，用來定義資料模型 |
| Field | Pydantic 的欄位驗證裝飾器 |
| EmailStr | 自動驗證 email 格式 |
| response_model | 確保 API 回傳的格式正確 |
