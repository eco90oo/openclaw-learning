## 💡 為什麼重要

想像一下：
- 你用手機叫 Uber，Uber 伺服器怎麼知道司機在哪裡？
- 你在網站上購物，訂單如何傳到商家的系統？
- 不同 App 之間的資料如何同步？

**答案就是 API** —— 它是程式與程式之間的「翻譯機」和「橋樑」。

不會 API = 你的程式只能自己玩，無法與世界連接。

## 🧠 核心概念

### 🔌 API 是什麼？

**API（Application Programming Interface）** = 應用程式介面

簡單說：**一套規則，告訴別人如何跟你的程式要求資料或做事**

生活比喻：
- 餐廳的菜單就是 API —— 廚房（伺服器）公布可以做的菜（功能），客人（客戶端）按照菜單點餐
- 菜單上寫：「牛肉麵 $150，可外送」—— 這就是 API 文件

### 🌐 RESTful 是什麼？

**REST（Representational State Transfer）** 是一種設計 API 的風格/原則

符合 REST 原則的 API 稱為 **RESTful API**

核心原則：
1. **每個資源都有獨特的 URL**（像地址一樣）
2. **使用標準的 HTTP 方法**（GET/POST/PUT/DELETE）
3. **伺服器不記得客戶端的狀態**（ stateless）

### 📡 HTTP 方法

| 方法 | 用途 | 生活中的比喻 |
|------|------|-------------|
| **GET** | 取得資料 | 讀菜單、看商品 |
| **POST** | 創建新資料 | 點餐、下訂單 |
| **PUT** | 更新資料（完整） | 修改訂單內容 |
| **PATCH** | 更新資料（部分） | 只改數量 |
| **DELETE** | 刪除資料 | 取消訂單 |

### 📍 URL 設計範例

```
# 取得所有文章
GET    /api/articles

# 取得 ID 為 5 的文章
GET    /api/articles/5

# 發表新文章
POST   /api/articles

# 更新文章
PUT    /api/articles/5

# 刪除文章
DELETE /api/articles/5
```

1. **手機 App 與伺服器溝通** —— 天氣 App 透過 API 取得天氣資料

## 📝 範例教學

### 用 curl 測試一個簡單的 API

```bash
# 取得公開的待辦事項 API
curl https://jsonplaceholder.typicode.com/todos/1

# 回應會像這樣：
{
  "userId": 1,
  "id": 1,
  "title": "delectus aut autem",
  "completed": false
}
```

### 用 JavaScript 呼叫 API

```javascript
// 取得資料
fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('錯誤:', error));

// 發送新資料（POST）
fetch('https://jsonplaceholder.typicode.com/posts', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    title: '我的第一篇文章',
    body: '這是內容',
    userId: 1
  })
})
  .then(response => response.json())
  .then(data => console.log('成功:', data));
```
